自动化构建工具：开发环境 --> 生产环境

  • 开发环境（development）注重开发效率和源码的阅读。
  • 生产环境（production）注重代码的执行效率（压缩）。

gulp / grunt 属于任务流工具（Task Runner）

webpack 是模块化打包工具（bundler）

webpack 是一个模块打包器（bundler），根据模块的依赖关系进行静态分析，生成对应的静态资源。对 webpack 来说，前端所有资源文件（js/json/css/less/img/…）都会作为模块处理。

核心概念

  • Entry 入口起点，指示 webpack 应该使用哪个模块，来作为构建其内部依赖图的开始。
  • Output 告诉 webpack 在哪里输出它所创建的 bundles，以及如何命名这些文件。
  • Loader 让 webpack 能够处理那些非 JS 文件。
  • Plugins 插件，用于执行范围更广的任务，比如打包优化，压缩。
  • Mode 模式，有生产模式（production）和开发模式（development）。

1、开始#

（1）初始化

npm init

（2）安装

# 全局安装
npm i webpack webpack-cli -g

# 局部安装（建议）
npm i webpack webpack-cli -D

2、基本使用#

demo
 ├── src
 │   ├── js
 │   │   ├── app.js
 │   │   └── mod.js
 │   └── index.html
 ├── build
 ├── package.json
 └── webpack.config.js

（1）打包 JS 文件

能够编译打包 js 和 json 文件，并且能将 es6 的模块化语法转换成浏览器能识别的语法。

将 src/js/app.js 打包成开发环境文件 build/js/app.js

npx webpack src/js/app.js -o build/js/app.js --mode=develoment
#                         -output

打包到生产环境（比生产环境多了一个代码压缩）

npx webpack src/js/app.js -o build/js/app.js --mode=production

缺点就是不能编译打包 CSS img 等文件，不能将 ES6 基本语法转化成 ES5 以下语法。

可以通过配置文件改善。

# 对指定文件进行打包
npx webpack xxx.js --output xxx.js

# 打包多个文件
npx webpack x1.js x2.js

（2）默认配置文件 webpack.config.js △

const { resolve } = require("path");

module.exports = {
  // 指定打包文件的入口
  // （单入口写法）
  // entry: ".src/js/app.js",
  // （多入口写法）
  entry: {
    app: "./src/js/app.js",
    mod: "./src/js/mod.js",
  },

  // output
  output: {
    // 指定打包之后的名字
    //               ↓ entry 的属性名
    filename: "js/[name].index.js",
    // 指定打包的绝对路径
    path: resolve(__dirname, "./build"),
  },

  // 指定打包模式
  // - development 开发模式
  // - production  生产模式
  mode: "development",
};

运行

# 这里是局部安装的命令，全局安装直接 webpack 即可
npx webpack

（3）清空上次打包的内容

npm install clean-webpack-plugin -D

// package.json
"scripts": {
    "build": "npx webpack"
  },

// bash
npm run build

3、打包 CSS/LESS/SASS 资源#

（1）打包 CSS

npm install style-loader css-loader -D
// 入口文件 app.js
import "../css/app.css";

// 配置文件 webpack.config.js
module.exports = {
  // ...
  module: {
    rules: [
      {
        test: /\.css$/,
        loader: ["style-loader", "css-loader"],
      },
    ],
  },
};

index.html 把 build/js/app.js 引入之后就能加载样式了。

（2）打包 LESS

npm install less-loader -D
// 入口文件 app.js
import "../css/style.less";

// 配置文件 webpack.config.js
module.exports = {
  // ...
  module: {
    rules: [
      {
        test: /\.css$/,
        loader: ["style-loader", "css-loader"],
      },
      {
        test: /\.less$/,
        use: ["style-loader", "css-loader", "less-loader"],
      },
    ],
  },
};

（3）打包 SASS

npm install sass-loader -D

4、JS eslint#

npm install eslint-loader eslint -D
// webpack.config.js

module.exports = {
    // ...
    module:{
        rules:[...,{
                // 只检测 js 文件
                test:/\.js$/,
                // 排除的目录
                exclude: /node_modules/,
                // 要检查的目录
                include: resolve(__dirname, "./src"),
                // 提前加载使用
                enforce: "pre",
                use:{
                    loader:"eslint-loader"
        }]
    }
}
// .eslintrc.js（项目根目录）

module.exports = {
    "parserOptions":{
        "ecmaVersion": 6,  // 支持 ES6
        "sourceType": "module"  // 使用 ES6 模块化
    },
    "env":{                // 设置环境
        "browser": true,   // 支持浏览器环境
        "node": true       // 支持服务器环境
    },

    // 如果这个地方定义了但是没有用，会过不去
    // Unexpected top-level property "global"
    "global":{             // 声明使用的全局变量
        "$":"readonly",    // 不允许重写 "writable" 可修改
	},
    "rules":{              // 0/off忽略 1/warn警告 2/error错误
        "eqeqeq": 0,       // 必须使用全等
        "no-alert": 0      // 禁止使用 alert confirm prompt
        "no-console": 0,   // 禁止使用 console
        "no-var": 0,       // 禁用 var
        "indent": ["error", 4],  // 缩进风格
    },
    // 使用 eslint 推荐的默认规则
    "extends": "eslint:recommended"
}

5、JS 语法转换#

@babel/preset-env 将新语法转成 ES5。

npm install babel-loader @babel/core @babel/preset-env -D
# @babel/core  babel 的核心库
# @babel/preset-env  babel 预设工具包，默认可以将所有最新的语法转为为 ES5
// webpack.config.js

module.exports = {
    //...
    module:{
        rules:[...,{
            test:/\.js$/,
            exclude: /node_modules/,
            use:{
                loader:"babel-loader",
                options:{
                    presets: ['@babel/preset-env']
                }
            }
        }
    ]
}

6、JS 兼容性处理#

npm install @babel/polyfill
// 入口文件 app.js
import "@babel/polyfill";

解决 babel 只能转换语法的问题，引入 polyfill 可以转换高级语法（如 Promise ）。

7、打包样式文件中的图片资源#

npm install file-loader url-loader -D
// webpack.config.js

module.exports = {
    //...
    module:{
        rules:[...,
            {
                test: /\.(png|jpg|gif)$/,
                use:{
                    loader: "url-loader",
                    options:{
                        // 8KB以下的base64处理
                        limit: 8192,
                        // 文件本地输出路径
                        outputPath: "images",
                        // 图片URL路径
                        publicPath: "../build/images",
                        // 修改文件名称和后缀
                        name: "[hash:8]".[ext],
                    }
                }
            }
        ]
    }
}

8、打包 HTML 文件#

npm install html-webpack-plugin -D
// webpack.config.js

const HtmlWebpackPlugin = require("html-webpack-plugin");
module.exports = {
  //...
  plugins: [
    new HtmlWebpackPlugin({
      // 设置要编译的 HTML 源文件路径
      template: "./src/index.html",
      // 设置打包之后的文件名，默认 index.html
      filename: "home.html",
      title: "我是打包之后的文件",

      // 传入参数
      // -- <%=user%>
      // -- <%if(user)%>
      arr: [1111, 222],
      // 加载的JS增加一个后缀
      // JS内容变化，后缀会重新生成
      hash: true,
      // 指定要引入的JS文件
      // app 是entry对象的属性名不是文件名
      chunk: ["app"],
      excludeChunk: ["app"], // 除了...之外
      // 不引入 JS
      // - "head", 放到head里
      // - "body"
      inject: false,
      minify: {
        // 移除注释
        removeComments: true,
        // 移除属性中的引号
        removeAttributeQuotes: true,
        // 将代码进行折叠，去除空格
        collapseWhitespace: true,
      },
    }),
  ],
};

11、自动编译打包运行#

npm install webpack-dev-server -D
// webpack.config.js

module.exports = {
  // ...
  output: {
    // ...
    // 添加 devServer 服务后需要调整输出的路径
    publicPath: "/",
  },
  module: {
    // ...
    // 删除 publicPath 配置
  },
  // 配置服务
  devServer: {
    // 自动打开浏览器
    open: true,
    // 启动 gzip 压缩
    compress: true,
    // 端口号
    port: 3000,
    // 开启热模替换功能
    hot: true,
  },
  // ...
};
# 启动服务
npx webpack-dev-server

热模替换

HMR，Hot Module Replacement，热模替换功能会在应用程序运行过程中替换。添加或删除模块，而无需重新加载整个界面。

// webpack.config.js

module.exports = {
  // index.html 不能自动刷新的问题
  entry: {
    main: ["./src/js/app.js", "./src/index.html"],
  },
};

12、devtool#

devtools 是 webpack 中的一个配置，可以将压缩/编译文件中的代码映射回源文件中的原始位置，便于调试代码。

// webpack.config.js

module.exports = {
  // ...
  // 设置 devtool 策略（生产环境需改成 none）
  devtool: "cheap-module-eval-sourse-map",
};

14、提取 CSS 成单独文件#

npm install mini-css-exrtract-plugin -D
// webpack.prod.js

const MiniCssExtractPlugin = require("mini-css-extract-plugin")
module.exports= {
    // ...
    module:{
        rules:[...,
              // 修改
              {
               		test: /\.less$\,
               		use:[
               			MiniCssExtractPlugin.loader,
               			'css-loader',
               			'less-loader'
              ]}
        ]
    }
    plugins:[
        new MiniCssExtractPlugin({
            filename: "css.[hash:8].css",
        })
    ]
}

16、压缩 CSS#

npm install optimize-css-assets-webpack-plugin -D
// webpack.prod.js

const OptimizeCssAssetsPlugin = require('optimize-css-assets-webpack-plugin');

module.exports = {
    // ...
    plugins:[...,
        new OptimizeCssAssetsPlugin({
             cssProcessorPluginOptions:{
             	// 移除所有注释
             	preset: ['default', {disardComments: {removeAll:true}}],
             },
             // 解决 无 sourse map 问题
             cssProcessorOptions:{
                 map: true
             }
        })
    ]
}
