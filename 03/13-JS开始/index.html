一、基本概念#

JavaScript 最初的目的是为了“赋予网页生命”。

这种编程语言我们称之为 脚本。它们可以写在 HTML 中，在页面加载的时候会自动执行。脚本作为纯文本存在和执行。它们不需要特殊的准备或编译即可运行。

  • 1995 JavaScript、JScript诞生
  • 1997 ECMA给JS制定了标准ECMA-262，命名为ECMAScript，即ES1.0。ES是JS的规范标准，JS的ES的实现。
  • 1999 发布ES3，成为JS通用标准
  • 2009 发布ES5
  • 2013 ES6草案发布（其实就是之前不被接受的ES4激进方案）
  • 2015 发布ES6，即ES2015，从此JS新版本按ES+年份形式发布。ES6是一个历史名词，也是一个泛指，代表ES6以后的新一代标准。

1、特点#

弱类型 数据类型不固定，可以随时改变。

解释型 解释型是一行一行解释运行，编译型是需要整体编译。

脚本语言 脚本是需要载体运行的。

基于原型 只有对象，没有类，对象继承对象，而不是类继承类。“原型对象”是基于原型语言的核心概念。原型对象是新对象的模板，它将自身的属性共享给新对象。一个对象不但可以享有自己创建时和运行时定义的属性，而且可以享有原型对象的属性。

2、组成#

  • ECMA Script 核心语法，简称ES。
  • DOM Document Object Model 文档对象模型。
  • BOM Browser Object Model 浏览器对象模型。

5、ES5 / ES6#

ECMAScript 是JS的语言规范标准。

ECMA-262 1997年制定的规范，包含了大部分深入的、详细的、规范化的关于 JavaScript 的信息。这份规范明确地定义了这门语言。

ES5 2009年。

ES6 ES6既是一个历史名词，也是一个泛指，含义是 5.1 版以后的 JavaScript 的下一代标准，涵盖了 ES2015、ES2016、ES2017 等等，而 ES2015 则是正式名称，特指该年发布的正式版本的语言标准。

二、基本结构和语法#

1、语句、注释、分号、空白#

  • 语句是执行行为的语法结构和命令。分为 声明语句、调试语句debugger等。
  • 注释的语法和C语言一样。
  • 建议每个语句结束都加分号，否则可能会出现奇奇怪怪的BUG。程序会忽略空白，合适的空白与缩进可以保持良好的代码结构。

2、输出#

  • 系统对话框（弹窗）: alert()、confirm()、prompt()
  • 控制台日志: console.log()、console.dir()
  • 直接显示在文档上: document.write

3、变量#

变量 是一种用来储存数据的可变容器，是数据的命名储存，通常情况下由标识符和值构成。

（1）变量命名/标识符命名规则#

  • 由数字，字母，下划线，美元符组成。
  • 不允许数字开头。
  • 不能和关键字、保留字同名。
  • 严格区分大小写。
  • 变量的起名要有意义。避免使用拼音等。

（2）变量声明和变量赋值#

优先使用const | let ，不要用 var 。

// 声明一个变量
let a;


// 给变量赋值（第一次赋值可以称为初始化）
a = 10;

// 同时声明和初始化
let b = 10;

// 一次定义和赋值多个变量
let c = 1, d = 2;

// 注意这种坑
let e = f = 1; //  f 没有声明

// const 用于定义常量，常量通常是全大写
const PI = 3.1415926;

4、严格模式#

ES6 也称为现代模式，通过在全局第一行或者是函数第一行添加"use strict" 语句开启严格模式。（一旦打开，无法关闭）

特点：

  • 给未定义的元素赋值会抛出异常
  • 删除不可删除的属性会抛出异常

4、null#

空对象指针。

常见环境：

  • 作为函数的参数，表示该函数的参数不是对象。
  • 作为对象原型链的终点。
  • 如果定义的变量准备在将来用于保存对象，那么最好将该变量初始化为null而不是其他值。
  • 让变量指向的对象成为垃圾对象。

5、undefined#

变量声明了但是未初始化的类型。

一般是代表错误类型。

常见环境：

  • 变量被声明了，但没有赋值时。
  • 调用函数时，应该提供的参数没有提供，该参数为undefined。
  • 对象没有赋值的属性。
  • 函数没有返回值时，默认返回值。

1、算术运算符#

加法#

// 如果其中一个操作数是字符串，则会把另外一个操作数转换成字符串。  
1 + 'str'   // 1str 
'str' + 1 // str1

自增、自减#

++i 表示，i自增1后再参与其它运算（先加后用）；而 i++ 则是i参与运算后，i的值再自增1。自增自减运算可能会改变原有变量类型。

var a = 100
1 + a++ // 101

++a; // a = 2

var i = 1

4、逻辑运算符#

&& 与

  • 第一步：计算第一个操作数
  • 第二步：如果第一个操作数转换成false，那么就会结束运算，直接第一个操作数
  • 第三步：如果第一个操作数返回true，则计算第二个操作数的值
  • 第四步：第二个操作数如果返回true 则逻辑与返回第二个操作数，否则返回第一个操作数

|| 或

  • 第一步：计算第一个操作数的值。
  • 第二步：检测第一个操作数的值。如果左侧的表达式的值可以转换为true，那么就会结束运算。直接返回第一个操作数的值
  • 第三步：如果第一个操作数可以转换为false，则计算第二个操作数的值。并返回第二个操作数的值

! 非

1 && 2 // 返回2
'' && 1 // 返回''
"Cat" && "Dog" // 返回”Dog"
''    && false     // f && f 返回 ""
false && ''        // f && f 返回 false

1 || 2 // 1
0 || 2 // 2
0 || null // null
''    || false     // f || f 返回 false
false || ''        // f || f 返回 ""

短路原理#

在逻辑运算中，如果第一个操作数就能决定结果，那么第二个操作数就不会运行了。（就算运行了会报错也能顺利通过）

var a = false;
fasle && qwq();

六、错误与异常处理#

ECMA-262 规范了7种错误类型。其中Error是基类，其他6种错误类型是子类，都继承了基类。基类的主要作用是自定义错误对象。

1、错误类型#

类型 说明
Error 基类
name 可以读写异常类型（TypeError）
message 可以读写详细的错误信息。
EvalError 不正确使用eval()方法时抛出
SyntaxError 语法错误
ReferenceError 读取不存在的变量时抛出
TypeError 值的类型发生错误的时候抛出
RangeError 数字超出合法范围
URIError URI编码和解码错误

2、try…catch#

try{
    // 调试代码块
} catch (e) {
    // 捕获异常，并进行异常处理代码块
} finally {
    
}

一般情况下 try...catch 就够用了。

按顺序执行 try 子句中的代码，如果没有异常发生，将会忽略 catch 跳转到 finally 子句继续执行。

无法捕获逻辑错误。

3、throw#

try {
    let num = 1 / 0;
    if (num = Infinity) {
        // 抛出错误
        throw new Error("除数不能为0");
    }
} catch (exception) {
    console.log(exception.message);
}

抛出错误之后，后续代码不会执行。
