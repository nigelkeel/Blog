一、基本结构#

<!-- 版本声明 -->
<!doctype html>
<!-- 根文档 -->
<html lang="en">
    <!-- 头部 -->
    <head>
    		<meta charset="utf-8">
    		<title>标题</title>
    </head>
    <!-- 主体 -->
    <body>
	</body>
</html>

1、注释#

用<!-- 和-->包裹起来的内容。

2、版本声明#

给浏览器声明，告诉浏览器应该按照HTML5的规范解析当前页面。 如果不写，浏览器就是按照自己默认的模式解析。

版本声明必须写在第一行。我们不把他看做一个html标签，而是把它当做一个指令，并且这个指令对大小写不敏感。

3、html 、head、body#

html 是一个双标签，所有网页内容全部书写在html标签中。lang就是html标签的属性，en就是lang属性的值，属性是对当前标签的信息的扩展。

head 标签代表网页的头部，不会显示在网页中。主要是包含了标题，网页的描述，字符编码等等一些信息。

body 标签是网页显示的主体内容。

4、meta 标签#

控制网页的元信息（信息的信息），根据属性的不同，有很多的作用。

5、解决网页乱码问题 △#

<meta charset="utf-8">

charset即字符编码。utf-8称作为万国码。

二、HTML 标签#

1、标签的语义化 △#

  • 语义化标签更具有 可读性 ，便于团队开发和维护。
  • 在没有加载CSS的情况下，网页也能很好呈现出内容结构和代码结构。
  • 对于SEO来说，搜索引擎更能理解网页中各部分之间的关系，更准确更快速搜索信息。

2、标签的关系#

  • 并列：兄弟关系
  • 嵌套：父子关系（使用TAB缩进）

3、标签的分类#

3.1 块标签#

特性

  • 独占一行
  • 可设宽高、内外边距和边框
  • 可嵌套块元素和行元素（p、h标签除外）

将非块级元素转换成块级元素的方法

  • display: block
  • float: left/right
  • position: absolute/fixed
  • 父级元素使用display:flex

3.2 行标签#

特性

  • 空间足够时可以其他行标签共用一行
  • 设置宽高无效，内外边距和边框的上下设置无效
  • 宽高用内容撑开
  • 可嵌套行元素
  • 受父元素的 line-height 以及自身的 vertical-align影响。

3.3 行内块（inline-block）#

特性

  • 一行可放多个。
  • 宽高、内外边距、边框无法设置。
  • 受父元素的 line-height 以及自身的 vertical-align影响。

4、div#

分块原则：先行，后列。

5、h + p + br#

h，标题

尽量避免跳过某一级标题，依次使用。

同一个网页中最多使用一次h1元素，h1经常用在网页的标题或者是logo上。

p，段落。

br，单纯的换行，被称为软换行。

6、列表#

（1）无序列表和有序列表

ol，有序列表，ul，无序列表。li，列表的每一项。
ul/ol-li，ol/ul只能嵌套li。

（2）自定义列表

dl-dt-dd，组合标签。

<dl>
    <dt></dt>
    <dd></dd>
</dl>

7、表格#

table、tr、td、th

  • 表格的最外层是table元素
  • 表格的标题是caption元素 一般在表格的最上边
  • tr就是表格的行
  • td是普通单元格，th是表头单元格（默认加粗居中）
<table border="1">
       <caption>学员就业信息表</caption>
       <tr>
           <th>序号</th>
           <th>姓名</th>
           <th>年龄</th>
           <th>性别</th>
           <th>备注</th>
       </tr>
       <tr>
           <td>01</td>
           <td>老李</td>
           <td>18</td>
           <td>男</td>
           <td></td>
       </tr>

table的属性

  • border:边框（已经废弃，以后使用css的方法）
  • 合并表格的边框，对table设置：border-collapse: collapse;
  • cellpadding 控制内容和边框之间的距离
  • cellspacing 控制两个单元格之间的距离

跨行和跨列

  • rowspan:控制元素跨行 谁跨行 设置给谁
  • colspan:控制元素跨列 谁跨列 设置给谁

其他

  • 表格布局（过时了，了解即可）
  • thead：表格的头部
  • tbody：表格的内容，可以划分多个
  • tfoot：表格的底部
  • 三个标签table划分为多个模块，优化表格的加载，一般数据表格量大的时候可以使用
  • thead、tfoot、tbody：三个元素，无关书写顺序，一定会按照thead–tbody–tfoot顺序加载

8、header、footer、nav、aside H5#

header 用于定义文档的页眉（头部区域）。

footer 用于定义文档的页脚（底部区域）。

nav 在文档中定义导航。

aside 通常用于定义侧边栏。

9、section 、artcle H5#

section 在文档中对独立的区域进行分块，一般会有独立的标题。

article 在文档中，对可以独立的完整的被外部引用的内容。

？ 比较 div、section、article

10、figure、figcaption#

figure、figcaption

H5新标签，用于引用图片、插画、表格、代码段。

11、span#

span标签在不需要语义的情况下，增加额外的结构，方便设置样式等操作。

12、锚链接#

（1）路径

  • 网络路径：一般有三种协议，一种是http、一种是file、一种是ftp。需要引入网络路径的时候，需要书写http://。
  • 相对路径
  • 绝对路径

（2）标签属性

  • href：路径。
  • target：跳转方式，_self 当前标签页跳转，_blank新标签页跳转。
  • title：鼠标悬浮时的弹窗解释。

（3）锚链接

href="#(id)" → id="(id)"

  • 标签href为空时，点击刷新加载。当设置为##（两个#及以上）或者javascript:;时，点击之后不会有动作
  • 如果a标签的href值只有一个 # 那么可以实现回到顶部的功能。
  • 如果a标签的href属性书写的链接可以使用浏览器打开（比如图片），则浏览器跳转到这个位置并打开，如果a标签的href属性书写的链接不可以使用浏览器打开，则浏览器直接下载。
  • a标签的下载功能：download属性 书写下载内容，可以直接实现下载，必须拥有href属性，否则下载功能失效。

a标签的伪类 △#

伪类 功能
link 当有链接属性时
visited 链接被访问过
hover 鼠标悬浮在链接上面时
active 链接激活时

①“love-hate 爱恨法则” —— 顺序不能互换 ：四个选择器优先级一样，顺序改变可能会覆盖前边的，比如visited写在最后，那么当连接被访问过后，访问过后的颜色就会覆盖hover和active。
②hover适用于多数其他元素，并且伪元素也可以使用。

.a1:link{
	color: red;
}
.a2:visited{
	color: yellowgreen;
}
.a3:hover{
	color: pink;
	font-size: 30px;
}
.a4:active{
	color: blueviolet;
	font-weight: bold;
}

13、b、i、u、em、strong、var#

  • b i u 没有太多的含义，可以增加额外的结构，但是使用率较低。

  • i 标签常用在引入小图标

  • em 标签默认倾斜，起到强调作用，主要是强调关键词的。

  • strong 标签默认加粗，起到更强的强调，主要强调的是一段话。

  • var 标签 没有任何的强调作用，有自己的含义，主要包含变量。

14、mark#

标记，突出显示上下文的关联性，比如搜索页面的关键词。是行内元素。

15、time、datalist、proress#

time

当时间是标准写法的时候，可以放在该标签中。

datalist

包含一组option元素，通常和input一起配套使用。

proress

通常用于进度条。（很少用）

16、img 标签#

属性 描述
src 图片路径
alt 加载失败时显示的替代文本
title 鼠标悬浮时显示的弹窗提示
width、height 宽高

解决底部留白：

  • display: block
  • font-size： 0

消除行内块间距的方式

  • 移除标签间的空格
  • 取消标签闭合
  • 给父元素设置font-size:0，Chrome 默认有最小字体大小限制，可以font-size: 0; -webkit-text-size-adjust:none;
  • 设置 margin 为负值

三、表单#

1、form#

<form action="" method="POST">

</form>

action：表单提交接口。如果省略，则设置为当前页面。

method：表单提交的方式。一般为 POST 或者 GET 。

  • GET 通过URL提交
  • POST 通过请求报文

2、input 标签#

0 属性#

属性 描述
placeholder 占位符。控制元素为空时的提示信息，只要输入内容就会消失。
  • name 当前表单元素键名。
  • value 当前表单元素输入内容，默认内容。
  • type 表单元素的类型。
  • autocomplete 是否提示用户输入历史输入值，默认为 ON。
  • autofocus 自动获取焦点。
  • required 必填。提交表单时必须填写此项。
  • disabled 禁用表单元素。不会被提交。
  • readonly 只读。会被提交。
  • checked 预先选中单选框或者多选框。
  • form 关联表单。如果input存在form属性，表示该input属于某个表单。通过input的form属性为form表单的id，进行关联。 进行关联以后，无论input书写在哪里，都能随着表单提交数据。
  • selected 预先选择一个选项。
  • multiple 实现多选。

2、伪元素#

伪元素 会在内容元素的前后插入额外的元素，但是并不会在文档中生成，在文档的源码中也找不到。虽然在结构上是虚假元素，但是在表现上和普通元素没有什么区别，能为它们添加任何样式，比如改变文字颜色等等。

伪元素必须拥有content属性才能生效，伪元素默认是一个行内元素，伪元素对其他属性基本都是支持的。

伪元素 作用 备注
after 在当前元素的最后边插入一个伪元素
before 在当前元素的最前面插入一个伪元素
first-letter 第一个字符 只针对块元素生效
first-line 第一行 只针对行元素生效
