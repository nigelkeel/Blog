一、概念#

1、并发、并行#

涉及面试题：并发与并行的区别？

并发（Concurrency）是宏观概念，我分别有任务 A 和任务 B，在一段时间内通过任务间的切换完成了这两个任务，这种情况就可以称之为并发。

并行（Parallelism）是微观概念，假设 CPU 中存在两个核心，那么我就可以同时完成任务 A、B。同时完成多个任务的情况就可以称之为并行。

2、回调函数#

涉及面试题：什么是回调函数？回调函数有什么缺点？如何解决回调地狱问题？

回调函数（Callback）分为同步回调（数组遍历）和异步回调。

使用回调函数容易导致回调地狱问题。回调地狱的根本问题就是：

  1. 嵌套函数存在耦合性，一旦有所改动，就会牵一发而动全身
  2. 嵌套函数一多，就很难处理错误。

当然，回调函数还存在着别的几个缺点，比如不能使用 try catch 捕获错误，不能直接 return。

for ( var i=0; i<3; i++){
    setTimeout(function(){console.log(i);},0);
}
// 输出 3 3 3

for ( var i=0; i<3; i++){
    (function(i){
       setTimeout(function(){console.log(i);},0);
    })(i);
}
// 输出 0 1 2

二、Promise △#

ES6 新规范，进行异步编程的新解决方案。以同步的形式把异步代码表达出来，避免回调函数层层嵌套的回调地狱。

1、状态#

状态 描述
pedding 初始化状态
fulfilled 成功状态，完成状态
reject 失败状态

状态只能被改变一次。

2、基本使用#

const p = new Promise((resolve, reject) => {
    // excutor 函数（同步的）
    resolve(1);
    // reject(2);
})

// 写法一
// reslove
p.then((num) => {
    console.log(num);
    // reject
}, (num) => {
    console.log(num);
})
.then((num) => {
    console.log(num);
    // reject
}, (num) => {
    console.log(num);
})


// 写法二
p.then(value =>{
    //
}).catch(reason =>{
    //
})

3、Promise 链#

Promise 实现了链式调用，也就是说每次调用 then 之后返回的都是一个 Promise，并且是一个全新的 Promise，原因也是因为状态不可变。如果你在 then 中 使用了 return，那么 return 的值会被 Promise.resolve() 包装。

4、Promise 方法#

（1）Promise.prototype.then

返回一个Promise 对象。

（2）Promise.prototype.catch

返回一个Promise 对象。

（3）Promise.resolve

返回的Promise对象状态为fulfilled。

（4）Promise.reject

返回一个状态为失败的Promise对象，并将给定的失败信息传递给对应的处理方法。

（5）Promise.all

  • 接收一个数组，数组的每一个元素都的 Promise
  • 返回的是一个 Promise
  • 数组中所有成功，则返回成功，如果有一个失败，则返回失败状态

这个方法返回一个新的promise对象，该promise对象在iterable参数对象里所有的promise对象都成功的时候才会触发成功，一旦有任何一个iterable里面的promise对象失败则立即触发该promise对象的失败。

（6）Promise.race

当iterable参数里的任意一个子promise被成功或失败后，父promise马上也会用子promise的成功返回值或失败详情作为参数调用父promise绑定的相应句柄，并返回该promise对象。

（7）Promise.any

接收一个Promise对象的集合，当其中的一个promise 成功，就返回那个成功的promise的值。

（8）Promise.allSettled

等到所有promises都完成（每个promise返回成功或失败）。
返回一个promise，该promise在所有promise完成后完成。并带有一个对象数组，每个对象对应每个promise的结果。

// 封装 ajax
function ajax(url) {
    return new Promise((resolve, reject)=>{
        const xhr = new XMLHttpRequest();
        xhr.onreadystatechange = () => {
            if(xhr.readyState === 4){
                if(xhr.status >= 200 && xhr.status < 300){
                    resolve(xhr.responseText);
                } else {
                    reject(xhr.responseText);
                }
            }
        }
        xhr.open("GET", url);
        xhr.send();
    })
}

// 使用
ajax("xxx").then(value=>{
    // ....
}, result=>{
    // .....
})

6、手写 Promise △△#

（1）简单版

const PENDING = "pending";
const RESOLVED = "fulfilled";
const REJECTED = "rejected";

function Promise(exec){
    const that = this;
    that.state = PENDING;
    that.value = null;
    that.resolveedCallbacks = [];
    that.rejectedCallbacks = [];
    
    function resolved(value){
        if( this.state !== PENDING) return;
        that.state = RESOLVED;
        that.value = value;
        that.resolvedCallbacks.map(v=>v(that.value));
    }
    function rejected(value){
        if( this.state !== PENDING) return;
        that.state = REJECTED;
        this.value = value;
        that.rejectedCallbacks.map(v=>v(that.value));
    }
    try{
        exec(resolved, rejected);
    } catch (err){
        rejected(err);
    }
}


Promise.prototype.then = function(onResolved, onRejected){
    const that = this;
    onResolved = typeof onResolved === "function" ? onResolved: v=>v;
    onRejected = typeof onRejected === "function" ? onRejected: err => throw err;
  	switch(that.state){
        case PENDING:
            that.resolvedCallbacks.push(onResolved);
            that.rejectedCallbacks.push(onRejected);
            break;
        case RESOLVED:
            onResolved(that.value);
            break;
        case REJECTED:
            onRejected(that.value);
            break;
    }
}

（2）教学版

const _PENDING = "pending";
const _RESOLVED = "fulfilled";
const _REJECTED = "rejected";

function Promise(executor) {
    this.status = _PENDING;
    this.value = undefined;
    this.CallBackList = [];
    function _resolved(value) {
        //   1
        if (this.status !== _PENDING) return;
        
        this.status = _RESOLVED;
        this.value = value;
	    //   2
        if(this.CallBackList.length > 0){
            setTimeout(()=>{
                this.CallBackList.forEach(v => {
                    v.onResolved.call(this);
                });
            })
        }
    };

    function _rejected(reason) {
        if (this.status !== _PENDING) return;
        this.status = _REJECTED;
        this.value = reason;

        if(this.CallBackList.length > 0){
            setTimeout(()=>{
                this.CallBackList.forEach(v => {
                    v.onRejected.call(this);
                });
            })
        }
    }
    try {
        //                  3
        executor(_resolved.bind(this), _rejected.bind(this));
    } catch (err) {
        _rejected.call(this, err);
    }
}

Promise.prototype.then = function (onResolved, onRejected) {
    //     4
    return new Promise((resolve, reject) => {
        //   5
        onResolved = typeof onResolved === "function" ? onResolved : value => value;
        onRejected = typeof onRejected === "function" ? onRejected : error => {
            throw error
        };
        
        function _callback(on) {
            //   6
            setTimeout(() => {
                try {
                    let result = on(this.value);
                    // 7
                    if (result instanceof Promise) {
                        result.then(resolve, reject);
                    } else {
                        resolve(result)
                    }
                } catch (err) {
                    reject(err);
                }
            })
        }
        switch (this.status) {
            case _RESOLVED: {
                _callback.call(this, onResolved);
                break;
            }
            case _REJECTED: {
                _callback.call(this, onRejected);
                break;
            }
            //  8
            case _PENDING: {
                CallBackList.push({
                    onResolved() {
                        _callback.call(this, onResolved);
                    },
                    onRejected() {
                        _callback.call(this, onRejected);
                    }
                })
                break;
            }
        }
    })
};

三、Generstor#

状态机，异步生成器。

基本语法

function* gen(){
    yield ...
    yield ...
}
    
gen.next();
  • Generstor 是可迭代的（for.. of）。

例子

// 使用计时器模拟依次请求 a数据 b数据
function* gen(){
    yield setTimeout(() => {
        console.log("a数据请求成功")
        it.next();
    }, 1000);
    yield setTimeout(() => {
        console.log("b数据请求成功")
    }, 1000);
}

// 开始请求
const it = gen()
it.next();

1、async 函数#

async function fn(){

2、await 表达式#

await 表达式必须写在 async 函数中。

await 右侧的表达式一般为 Promise 对象，但也可以是其他值。

如果表达式是 Promise 对象，返回的是 Promise 成功的值。如果失败的，则会抛出异常，需要通过 try... catch 捕捉。

如果是其他值，则直接将此值作为 await 的返回值。

// 例子1
async function fn() {
    return 1;
}
const result = fn();
console.log(result); // Promise { 1 }
result.then(value => {
    console.log(value); // 1
})


// 例子2
async function fn() {
    throw "异常"
}
const result = fn(); 
console.log(result); // Promise { <rejected> '异常' }
result.then(value => {
    console.log(value);
}).catch(reason => {
    console.error(reason) // 异常
})


// 例子3
async function fn(){
    return new Promise((resolve,reject)=>{
        reject("error");
    })
}
const result = fn();// 失败的Promise
result.catch(reason=>{
    console.log(reason) // error
})

await 会阻塞函数的运行，需要等后边的异步代码执行完毕才会向下执行。
