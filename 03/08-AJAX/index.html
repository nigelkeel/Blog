一、ajax#

AJAX 即 Asynchronous Javascript And XML，异步 JavaScript 和 XML。是指一种无需重新加载整个网页的情况下，能够更新部分网页的技术。通过Ajax可以在浏览器中向服务器发起异步请求。

XML 称为可扩展标记语言，被设计用来传输和存储数据，和 HTML 类似，不同的是 HTML 日常使用大部分都是预定义标签，XML 全部都是自定义标签，用来表示一些数据。

1、工作原理#

ajax 的工作原理相当于在用户和服务器之间加了一个中间层（Ajax引擎），使用户操作与服务器响应异步化。

2、特点#

优点

  • 无需刷新页面来和服务器通信。
  • 允许你根据用户事件来更新部分页面内容。

缺点

  • 没有浏览历史，无法回退。
  • 存在跨域问题。
  • 对SEO不友好。

3、XHR - XMLHttpRequest#

Ajax 的所有操作都是通过 XMLHttpRequest 对象进行的。

4、使用步骤#

（1）创建核心对象#

创建 XMLHttpRequest 核心对象。

const xhr = new XMLHttpRequest();

（2）打开接口，设置数据类型#

通过 open(method, URL) 打开接口。

  • method 请求方法，POST 或者 GET。
  • URL 接口地址。

！这里可以设置数据格式，即请求头，媒体类型有

  • application/json
  • application/x-www-form-urlencoded
  • formdata

xhr.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded')

xhr.open("POST", "http://127.0.0.1/weibo");

（3）发送请求#

发送请求 send(body)

xhr.send( JSON.stringifly( content ));  // 只能传字符串

GET 请求不需要参数， POST 请求需要。

（4）接收数据#

xhr.responseXML 接收XML格式的响应数据。

xhr.responseText 接收文本格式的响应数据。

a. 通过 onreadystatechange 事件接收数据，xhr 对象有 readyState 表示当前的请求状态，状态改变就会触发这个事件。

xhr.onreadystatechange = function (){
	if(xhr.readyState == 4 && xhr.status >= 200 && xhr.status < 300){
		let text = xhr.responseText;
		console.log(text);
	}
}

b. 通过 onload 事件

xhr.onload = function(){
	...xhr.responseText;
}

5、XHR API △#

（01）XMLHttpRequest() 创建XHR对象的构造函数

（02）readyState 当前的请求状态

const xhr = XMLHttpRequest();
// 0
xhr.onreadystatechange = function(){
    // 3
    // 4
}
xhr.open("get", "http://...");
// 1
xhr.send();
// 2
状态 常量 描述
0 UNSENT 表示 XMLHttpRequest 实例已经生成，但是 open() 方法还没有被调用。
1 OPENED 表示 send()方法还没有被调用，仍然可以使用setRequestHeader()，设定HTTP请求的头信息。
2 HEADERS_RECEIVED 表示 send() 方法已经执行，并且头信息和状态码已经收到。
3 LOADING 表示正在接收服务器传来的body部分的数据，如果responseType属性是text或者空字符串，responseText就会包含已经收到的部分信息。
4 DONE 表示服务器数据已经接收完成，或者本次接收已经失败了。

（03）status 响应状态码

（04）statusText 响应状态文本

（05）onreadystatechange 当readyState发生变化就会触发这个事件

（06）responseType 指定响应数据类型，如果是 json，得到响应后自动解析响应体数据

（07）response 响应体数据，类型取决于 responseType 的指定

（08）timeout 指定请求超时时间（毫秒），默认是0，代表没有限制

（09）ontimeout 超时之后触发的事件

（10）onerror 请求网络错误触发的事件

（11）open(method, URL[, async]) 初始化一个请求

（12）send(data) 发送请求

（13）abort() 中断请求

（14）getResponseHeader(name) 获取指定名称的响应头值

（15）getAllResponseHeaders() 获取所有响应头组成的字符串

（16）setRequestHeader(name, value) 设置请求头

（17）xhr.responseXML 接收XML格式的响应数据

（18）xhr.responseText 接收文本格式的响应数据

二、fetch#

fetch 是新型 ajax 请求方式，返回的是一个 Promise 对象。

xhr --> fetch

（1）get

fetch("http://127.0.0.1/sword/1")
	.then(response => response.json())
	.then(res => {
          console.log(res)
     })

（2）post

// application/json
fetch("http://127.0.0.1/sword", {
    method: "post", // 默认是 get
    headers:{
        "content-type": "application/json"
    },
    body: JSON.stringify({
        name:"绒布球之剑",
        attr:{
            atk: 999,
            parry: 999
        } 
    })
})
    .then(response=>response.json())
    .then(res=>{
        console.log(res)
    })


// application/x-www-form-urlencoded
fetch("http://127.0.0.1/sword", {
    method: "post", 
    headers:{
        "content-type": "application/x-www-form-urlencoded"
    },
    body: "name=大绒布球之剑&text=巫师的绒布球之剑"
})
    .then(response=>response.json())
    .then(res=>{
        console.log(res)
    })

（3）delete

fetch("http://127.0.0.1/sword/9", {
    method: "delete"
})
    .then(response=>response.json())
    .then(res=>{
        console.log(res)
    })

（4）put

fetch("http://127.0.0.1/sword/9", {
    method: "put", 
    headers:{
        "content-type": "application/json"
    },
    body: JSON.stringify({
        name:"绒布球之大剑",
        attr:{
            atk: -1,
            parry: -999
        } 
    })
})
    .then(response=>response.json())
    .then(res=>{
        console.log(res)
    })

（5）patch

fetch("http://127.0.0.1/sword/9", {
    method: "patch", 
    headers:{
        "content-type": "application/json"
    },
    body: JSON.stringify({
        name:"绒布球快乐剑",
    })
})
    .then(response=>response.json())
    .then(res=>{
        console.log(res)
    })

三、Axios △#

Axios 是目前最受欢迎的 ajax 封装库， 基于 Promise。

文档

1、开始#

npm i axios
<script src="https://unpkg.com/axios/dist/axios.min.js"></script>

2、基本使用#

axios 相当于 axios.request

返回的是一个 Promise 对象

（1）GET - 默认

axios.get('http://127.0.0.1/sword/9');
axios('http://127.0.0.1/sword/9');

（2）POST

// application/json
axios.post("http://127.0.0.1/sword", {
    id: 1,
    name: "破剑"
});

// application/x-www-form-urlencoded
axios.post("http://127.0.0.1/sword", "name=破晓快乐剑");

axios({
    method: 'post',
    url: 'http://127.0.0.1/sword',
    data:{
        name: "破铁剑"
    }
});

（3）DELETE

axios.delete("http://127.0.0.1/sword/9")
axios({
    method: "delete",
    url: "http://127.0.0.1/sword/9"
})

（4）PUT

axios.put("http://127.0.0.1/sword/9", {
    name: "七色破晓剑"
})
axios({
    method: "put",
    url: "http://127.0.0.1/sword/9",
    data:{
        name: "七色破晓剑"
    }
})

（5）PATCH

axios.patch("http://127.0.0.1/sword/9", {
    name: "轩辕剑"
})
axios({
    method: "patch",
    url: "http://127.0.0.1/sword/9",
    data:{
        name: "轩辕剑"
    }
})

（6）连缀写法

axios.get("http://127.0.0.1/sword/9")
.then(data=>{
    
})

（7）await写法

async function(){
    const data = await axios.get("http://127.0.0.1/sword/9");
}

3、拦截器#

// 请求拦截
axios.interceptors.request.use( config=>{
    config.url = "url" + config.url;
    return config;
})

// 响应拦截
axios.intercertors.response.use( res=>{
    return res.data;
})
