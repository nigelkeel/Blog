HTML 标签#

一、概述#

标签，又称为元素。

1、标签的关系#

兄弟关系
父子关系

2、标签的分类#

（1）块标签 block

  • 代表标签: div
  • 默认的宽度是占据父标签全部内容，即 width: 100%
  • 默认的高度由内容决定
  • 设宽高、边距、边框没有限制
  • 独占一行
  • 允许嵌套块元素和行元素（p/h标签除外）

（2）行内标签 inline

  • 代表标签: span
  • 默认的宽高由内容决定。
  • 内容过多时允许换行并断开（一般都是用于处理文字样式）。
  • 设置宽高无效。上下边距、边框无法设置。
  • 共用一行：在一行中如有剩余空间，允许同其他行内/行内块标签共用。
  • 某些动画样式不能添加和使用。

（3）行内块标签 inline-block

  • 代表标签: img、button、input
  • 默认的宽高由内容决定。
  • 内容过多时不会断开。
  • 可以设置宽高。
  • 在一行中如有剩余空间，允许同其他 行内/行内块 标签公用一行。
  • 行内块和内行标签会有行高的概念。
  • 行内块受父元素的 line-height 以及自身的 vertical-align 影响。

将非块级元素转换成块级元素的方法

  • display: block
  • float: left/right
  • position: absolute/fixed
  • 父级元素使用 display:flex

3、语义化标签#

优点：

  • 语义化标签更具有可读性，便于团队的开发和维护
  • 没有 CSS 的情况下，网页也能很好的呈现出内容结构和代码结构
  • 关于SEO，搜索引擎更能理解到网页中各部分之间的关系，更准确更快速搜索信息

布局的三个发展阶段：

  • 开荒时期：前端的代码主要是后台写的，使用 table 来布局
  • 美工时期：有专门的人来写前端代码了，主要布局方式是 DIV+CSS
  • 现在：使用具有语义的标签，可以表达正确的内容，也利用开发和维护。

二、常用标签#

1、div 分块#

用于分块。
分块原则：先行后列。

2、标题、段落、换行#

（1）标题：h1 h2 h3 …
最多允许6级标题。
依次使用，避免跳过某级标题使用。
同一个网页最多使用一次 h1 标签。

（2）段落：p
（3）换行：br
单纯的换行，被称为软换行。

3、列表#

（1）无序列表：ul-li
常用于重复结构的大块内容布局。
注：
1、ol/ul只能嵌套 li。
2、不建议在ul和li嵌套的中间，书写其他元素。

（2）有序列表：ol-li
用的少。

（3）自定义列表
dl-dt-dd，组合标签。

<dl>
  <dt></dt>
  <dd></dd>
</dl>
