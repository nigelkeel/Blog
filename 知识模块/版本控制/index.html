一、Git#

Git 是一款开源免费的分布式的版本控制系统。相对来说，SVN 是集中式版本控制系统。

Git 可以备份代码、版本回退、协作开发、权限控制。

1、安装和配置#

（1）下载并安装 Git

（2）打开命令行工具检查是否安装成功

（3）配置用户名和邮箱

git config -global user.name "xxx"
git config -global user.email "xxx"

（4）配置完成后查看

git config -l

（5）新建仓库

git init

2、版本库#

版本库分为三个区域

  • 工作区
  • 暂存区 staged
  • 仓库区

3、常用操作命令#

（1）查看当前状态

git status
  • nothing to commit, working tree clean 工作区和仓库一样，暂存区空
  • untracked files 工作区未跟踪的文件
  • changes to be commited 暂存区文件
  • no staged for commit 以下文件还没添加到暂存区

（2）将文件加入暂存区

git add 文件名

# 添加所有修改的文件到暂存区
git add . 
git add *

（3）提交暂存区的文件到仓库

git commit -m '注释内容'

注释不能为空

注释规范

  • feat 新增功能
  • fix 修复bug
  • cors 对核心功能进行修改
  • docs 修改文档

（4）查看工作区和暂存区的差异

# 查看工作区和暂存区的修改
# 不会显示删除或者新增文件
git diff

# 查看暂存区和仓库区的差异
git diff -cached

（5）恢复改动

# 恢复工作区改动的内容
git restore

# 取消暂存区的文件
git restore --staged 文件名

# 回退版本
git reset --hard <七位哈希>

（6）查看日志

# 查看历史版本
git log

# 查看简易历史版本
git log --oneline

# 查看所有日志信息
git reflog

（7）分支

# 创建分支
git branch 分支名

# 查看分支
git branch

# 切换分支
git checkout 分支名

# 删除分支
git branch -d 分支名

# 切换并创建分支
git checkout -b 分支名

（8）冲突和合并

# 冲突 CONFLICT
# 合并分支
git merge 分支名

4、.gitgnore#

配置忽略文件。

# 忽略所有的 .idea 文件
.idea

# 忽略所有以 .test 结尾的文件
*.test

# 忽略 node_modules 文件和文件夹
/node_modules

二、Github#

这边以远程仓库Github为例。

fork 拷贝别人代码到自己仓库

1、常用#

  • 新建本地仓库
  • 创建github仓库 oringin
  • 关联 git remote add origin 远程仓库地址
  • git push -u origin 推送到云端仓库
    • `git push origin dev
  • git pull 下载到本地仓库
  • git clone 远程仓库地址 克隆远程仓库到本地
  • git fetch origin

git fetch是将远程主机的最新内容拉到本地，用户在检查了以后决定是否合并到工作本机分支中。

而git pull 则是将远程主机的最新内容拉下来后直接合并，即：git pull = git fetch + git merge，这样可能会产生冲突，需要手动解决。

2、配置 SSH 免密登录#

  1. 创建非对称加密对

    ssh-keygen //在任意位置输入命令即可

  2. 文件默认存储在家目录（c:/用户/用户名/.ssh）的 .ssh 文件夹中。

    • id_rsa 私钥
    • id_rsa.pub 公钥

  3. 将公钥（.pub）文件内容配置到账号的秘钥中

    • 首页 -> 右上角头像-> settings -> SSH and GPG keys -> new SSH Key

  4. 克隆代码时，选择 ssh 模式进行克隆 （地址 在仓库首页 绿色 克隆的位置 选择 use ssh）

    git clone git@github.com/xiaohigh/team-repo-1.git
