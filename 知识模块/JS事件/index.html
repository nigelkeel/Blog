一、概念#

1、事件模型#

DOM0 基本事件模型。

DOM2 DOM事件模型，标准的事件处理模型。

2、事件流#

事件流就是多个节点对象对同一个事件响应的先后顺序。

冒泡型 从目标元素向 document 进行响应 （IE标准）

捕获型 从 document 向 目标元素 进行响应 （网景标准）

混合型 W3C事件流，先捕获，执行目标事件，最后冒泡。

3、事件绑定#

（1）静态绑定

（2）动态绑定（DOM0级绑定）

元素 . on+事件类型 = 函数

  • 绑定事件简单，把事件函数赋值给DOM对象的事件属性上
  • 执行的事件流是冒泡
  • 对一个元素绑定同一个事件多次，前面的会被后面覆盖
  • 注销事件只需要给事件属性赋值为 null

（3）事件处理函数

元素 . addEventListener(type function Boolean)

  • 可以绑定同一个元素同一个事件多次。
  • 可以控制冒泡或者捕获。
  • DOM2级事件（DOMContentLoader事件），只能使用这种方法绑定。

（4） IE事件模型绑定方法（了解）

元素 . attachEvent( onType function )

事件的this指向事件源对象。

二、事件类型（DOM0级）#

1、失去和获取焦点事件#

类型 描述
focus 获取焦点事件
blur 失去焦点事件

2、点击事件、鼠标事件#

类型 描述
click 鼠标点击（通常左键）[1]
contextmenu 鼠标点击（右键）
dblclick 鼠标双击（左键）[2]
mousedown 鼠标按下
mouseup 鼠标抬起
mousemove[3] 鼠标移动
mouseover 鼠标移入-可以触发事件冒泡[4]
mouseenter 鼠标移入-不会触发事件冒泡
mouseout 鼠标移出-可以触发事件冒泡
mouseleave 鼠标移出-不会触发事件冒泡

3、键盘事件#

类型 描述
keydown 键盘按下
keyup 键盘抬起（常用）

如果是表单输入，则对表单绑定键盘事件。如果不是表单，则对document绑定键盘事件，控制其他元素。

keyCode 对应键盘中对应键位的键值

shiftKey、ctrlKey、altKey 是否按下对应按键 返回布尔值

document.onkeydown = function( e){
	var e = e || window.event;
    console.log(e.keyCode); // 输出按键的ascii码值
}

4、表单事件#

事件 描述
input 当表单内容改变时触发
change 当表单内容改变并且失去焦点时候触发

5、滚动条事件#

scroll 元素滚动条在滚动时触发。

window.onscroll

6、加载完成事件#

load事件 在资源加载完成后执行。

window.onload 等待html所有的DOM节点，以及所有资源全部加载完成后执行。一个页面只能出现一次，因为DOM0级事件会覆盖。

7、其他#

window.onresize 重置窗口大小时触发。

// 获取视口大小封装函数
var getScreen = function () {
    var w = document.documentElement.clientWidth||document.body.clientWidth;
    var h = document.documentElement.clientHeight||document.body.clientHeight;;
    return {
        width:w,
        height:h
   }
}

selectstart 目标对象被开始选中时触发（选中动作刚开始，并未实质开始。）可用于屏蔽页面复制黏贴等。

1、注册事件 addEventListener#

五、定时器 △△#

JS是单线程语言，但它允许通过设置超时值和间歇时间值来调度代码在特定的时刻执行。**超时调用是在指定的时间过后执行代码（一次），而间歇调用**则是每隔指定的时间就执行一次代码。

1、超时调用 setTimeout#

超时调用需要使用 window 对象的 setTimeout() 方法，能够在指定的时间段后执行特定代码。

var TimerID = setTimeout( code, delay [,arg])
// 参数
// code  -- 表示延迟执行的字符串型代码（或者函数）
//          如果包含多个语句，应该使用分号间隔
//          函数不用调用，只是书写函数
// delay -- 延迟时间，单位毫秒
// arg   -- 如果code是一个函数，则之后是任意多个参数

// 返回值
// TimeID 指向延迟执行的代码控制句柄。（唯一的）

// 写法1
var TimerID1 = setTimeout('var a=1;alert(a);',2000);

// 写法2
var TimerID2 = setTimeout(function(){
    var a = 2;
    alert(a);
},2000);

// 写法3
function fn(){
    var a = 2;
    alert(a);
}
var TimerID3 = setTimeout(fn,2000);

// 传参方法1
function f(a,b){
    alert(a+b)
}
var a = window.setTimeout("f(1,2)",1000)

// 传参方法2
var b = window.setTimeout(f,1000,1,2);

取消超时调用（特定条件）：

clearTimeout(timerID);

2、间歇调用 setInterval#

方法是setInterval()，使用方法和超时调用基本相同。

取消方法是clearInterval(timerID)。

注意清除计时器。

// 计时器的this指向

setTimeout(function (){
    console.log(this);// 一般都是指向 window
},1000);

四、事件其他#

1、DOMContentLoaded 事件#

DOM2级事件，需要使用事件函数绑定。当DOM节点加载完成后即可执行，不需要等待加载资源。

而 window.onload 是等待html所有的DOM节点，以及所有资源加载完成后才会执行。

2、event 事件对象#

由事件自动创建，记录当前事件的状态。

3、鼠标的 event 事件#

描述
clientX clientY 浏览器窗口定位
原点是浏览器窗口左上角
offsetX offsetY 当前事件目标对象定位
screenX screenY 计算机屏幕定位
pageX pageY 文档定位

4、阻止传播#

stopPropagation() 现代浏览器

cancelBubble = true IE低版本

5、阻止默认事件#

取消事件的默认操作，阻止默认事件。

（1）通用 preventDefault()

（2）IE低版本 returnValue = false

（3）return false

6、事件委托#

事件委托（delegate） 也称为事件托管、事件代理，就是把目标节点的事件绑定到祖先节点上。

原理：冒泡事件流。

优点：

  • 优化代码，提高效率
  • 给未来的元素添加绑定

7、函数防抖、节流#

防抖（debounce），短时间触发多次事件，只执行最后一次，或者是只执行最开始一次，中间的不执行。

节流（throttle），连续触发事件，但是在一定的时间内只执行一次函数。

8、异步函数#

for ( var i=0; i<3; i++){
    setTimeout(function(){console.log(i);},0);
}
// 输出 3 3 3

for ( var i=0; i<3; i++){
    (function(i){
       setTimeout(function(){console.log(i);},0);
    })(i);
}
// 输出 0 1 2

9、补充内容#

禁止选择：onselectstart="return false"

使用场景：实现元素内文本不被选中

禁止拖放：ondragstart="return false"

使用场景：禁止鼠标在网页上拖动

禁止拷贝：oncopy="return false"

使用场景：很多网站上的页面内容是不允许复制的，这样可以防止用户或者程序恶意的去抓取页面数据。

禁止粘贴：onpaste="return false"

使用场景：网银转账时，输入对方卡号，需要输入两次，通常第二次输入的输入框是不允许粘贴的，这样就在一定程度上保证了卡号的准确性。

禁止剪贴：oncut = "return false"

禁止鼠标右键：oncontextmenu="return false"
