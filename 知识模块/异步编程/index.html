一、概念#

1、并发、并行#

涉及面试题：并发与并行的区别？

并发（Concurrency）是宏观概念，我分别有任务 A 和任务 B，在一段时间内通过任务间的切换完成了这两个任务，这种情况就可以称之为并发。

并行（Parallelism）是微观概念，假设 CPU 中存在两个核心，那么我就可以同时完成任务 A、B。同时完成多个任务的情况就可以称之为并行。

2、回调函数#

涉及面试题：什么是回调函数？回调函数有什么缺点？如何解决回调地狱问题？

回调函数（Callback）分为同步回调（数组遍历）和异步回调。使用回调函数容易导致回调地狱问题。

回调地狱的根本问题就是：

  1. 嵌套函数存在耦合性，一旦有所改动，就会牵一发而动全身
  2. 嵌套函数一多，就很难处理错误。

当然，回调函数还存在着别的几个缺点，比如不能使用 try catch 捕获错误，不能直接 return。

二、Promise △#

ES6 规范，进行异步编程的新解决方案。以同步的形式把异步代码表达出来，避免回调函数层层嵌套的回调地狱。

1、状态#

状态 描述
pedding 初始化状态
fulfilled 成功状态，完成状态
reject 失败状态

状态只能被改变一次。

2、基本语法#


const p = new Promise((resolve, reject) => {
    // excutor 函数（同步的）
    resolve(1);
    // reject(2);
})

// 写法一
// reslove
p.then((num) => {
    console.log(num);
    // reject
}, (num) => {
    console.log(num);
})
.then((num) => {
    console.log(num);
    // reject
}, (num) => {
    console.log(num);
})


// 写法二
p.then(value =>{
    //
}).catch(reason =>{
    //
})

// 如果两种写法一起写，按写法一

Promise 实现了链式调用，也就是说每次调用 then 之后返回的都是一个 Promise，并且是一个全新的 Promise，原因也是因为状态不可变。如果你在 then 中 使用了 return，那么 return 的值会被 Promise.resolve() 包装。

3、返回一个新对象#

不管是成功还是失败（语法错误），返回的都是一个Promise对象。默认是resolve状态，失败和语法错误是 reject 状态。

// 封装 ajax
function ajax(url) {
    return new Promise((resolve, reject)=>{
        const xhr = new XMLHttpRequest();
        xhr.onreadystatechange = () => {
            if(xhr.readyState === 4){
                if(xhr.status >= 200 && xhr.status < 300){
                    resolve(xhr.responseText);
                } else {
                    reject(xhr.responseText);
                }
            }
        }
        xhr.open("GET", url);
        xhr.send();
    })
}

// 使用
ajax("xxx").then(value=>{
    // ....
}, result=>{
    // .....
})

4、思考#

（1）如何改变 Promise 的状态

  • resolve()
  • reject()
  • 抛出异常

（2）一个 Promise 指定多个成功、失败回调函数，都会调用吗？

  • 当Promise改变为对应的状态时都会调用。

（3）先更改状态还是先执行回调？

5、手写 Promise △△#

（1）

function Promise(executor) {
    this.status = "pending";
    function _resolve() {
        this.status = "fulfilled";
    }
    function _reject() {
        this.status = "rejected";
    }
    try {
        executor(_resolve.bind(this), _reject.bind(this));
    } catch (err) {
        _reject.call(this, err);
    }
}

三、Generstor#

状态机。

// 使用计时器模拟依次请求 a数据 b数据
function* gen(){
    yield setTimeout(() => {
        console.log("a数据请求成功")
        it.next();
    }, 1000);
    yield setTimeout(() => {
        console.log("b数据请求成功")
    }, 1000);
}

// 开始请求
const it = gen()
it.next();

四、async、await#

async function foo(){
    await ...
    await ...
}

await 会阻塞函数的运行，需要等后边的异步代码执行完毕才会向下执行。

await 后面必须跟promise对象。

  • padding 一直阻塞
  • resolve 继续
  • reject 直接退出
