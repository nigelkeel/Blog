开发环境（development）注重开发效率和源码的阅读。

生产环境（production）注重代码的执行效率（压缩）。

一、gulp#

1、开始#

（1）安装

# 全局安装
npm install gulp -g

# 仅安装到生产环境
npm install gulp -D

（2）配置 gulpfile.js

const gulp = require("gulp");

// task 
gulp.task("default", async ()=>{
    // 执行任务
    cb();
    // 返回一个可读流
    return fs.createReadStresm(...);
});

（3）执行 gulp | npx gulp（局部安装）

2、插件#

gulp-jshint 语法检查

gulp-babel 语法转换

二、Webpack △#

Webpack 是一个模块打包器（bundler），根据模块的依赖关系进行静态分析，生成对应的静态资源。在 Webpack 看来，前端所有资源文件（js/json/css/less/img/…）都会作为模块处理。

核心概念

Entry 入口起点，指示 Webpack 应该使用哪个模块，来作为构建其内部依赖图的开始。

Output 告诉 Webpack 在哪里输出它所创建的 bundles，以及如何命名这些文件。

Loader 让 Webpack 能够处理那些非JS文件。

Plugins 插件，用于执行范围更广的任务，比如打包优化，压缩。

Mode 模式，有生产模式（production）和开发模式（development ）。

1、开始#

依赖于 NodeJS 环境安装

# 全局安装
npm install webpack webpack-cli -g

# 局部安装（建议）
npm install webpack webpack-cli -D

零配置

不需要配置，会默认将 src/index.js 进行打包，默认打包到 dist 下，生成的文件是 main.js

模式（mode）

npx webpack --mode developent

2、基本使用#

对指定文件进行打包

npx webpack xxx.js  --output xxx.js

打包多个文件

npx webpack x1.js x2.js

入口文件

统一配置管理

// 多入口，单出口
module.exports = {
    // 指定打包模式
    mode: "development",
    // 指定打包文件的入口
    entry： "./src/js/app.js",
    output:{
        // 指定打包之后的名字
    	filename:"bundke.js",
        // 指定打包路径（需绝对路径）
    	path: path.resolve(__dirname, "./build")
	}
};

// 多入口，多出口
module.exports = {
    // 指定打包模式
    mode: "development",
    // 指定打包文件的入口
    entry：{
    	app: "./src/js/app.js",
    	mo: "./src/js/mo.js"
	},
    output:{
        // 指定打包之后的名字
        //               ↓ entry 的属性名
    	filename:"abc/[name].bundke.js",
        // 指定打包路径（须绝对路径）
    	path: path.resolve(__dirname, "./build")
	}
};

清空上次打包的内容

npm install clean-webpack-plugin -D
const {cleanWebpackPlugin} = require("clean-webpack-plugin");

module.exports = {
    ...
    plugins:{
        new CleanWebpackPlugin();
    }
}

3、打包CSS/LESS/SASS资源#

npm install style-loader css-leader -D
// 入口文件
import "../css/app.css";

// 配置文件
module.exports = {
    // 运行相对应的loader
    module:{
        rules:{
            test:/\.css$/,
            loader:["style-loader", "css-loader"]
        }
    }
}
npm install less-loader -D
npm install sass-loader -D

4、JS语法检查#

npm install eslint-loader eslint -D
// webpack.config.js

module.exports = {
    ...
    // 运行相对应的loader
    module:{
        rules:{
            // 只检测 js 文件
            test:/\.js$/,
            // 排除
            exclude: /node_modules/,
            // 提前加载使用
            enforce: "pre",
            use:{
                loader:"eslint-loader"
            }
        }
    }
}
// .eslint.js
module.export = {
    "parserOption":{
        "ecmaVersion": 6,  // 支持 ES6
        "sourceType": "module"  // 使用 ES6 模块化
    },
    "env":{                // 设置环境
        "browser": true,   // 支持浏览器环境
        "node": true       // 支持服务器环境
    },
    "global":{             // 声明使用的全局变量
        "$":"readonly",    // 不允许重写
        "$$":"writable"    // 可修改
	},
    "rules":{              // 0 忽略 1 警告 2 错误
        "no-console": 0,   // 不检查 console
        "eqeqeq": 0,       // 用 == 而不用 ===
        "no-alert": 0      // 不能使用 alert
    },
    // 使用 eslint 推荐的默认规则
    "extends": "eslint:recommended" 
}

5、JS语法转换#

@babel/preset-ent 将新语法转成 ES5。

npm install babel-loader @bable/core @babel/preset-ent -D
// webpack.config.js

module.exports = {
    ...
    // 运行相对应的loader
    module:{
        rules:{
            // 只检测 js 文件
            test:/\.js$/,
            // 排除
            exclude: /node_modules/,
            // 提前加载使用
            enforce: "pre",
            use:{
                loader:"babel-loader",
                options:{
                    presets: ['@babel/preset-env']
                }
            }
        }
    }
}

JS 兼容性处理

npm install @babel/polyfill
// 入口文件 app.js
import "@babel/polyfill";

解决 babel 只能转换语法的问题，引入 polyfill 可以转换高级语法（如 Promise ）。
