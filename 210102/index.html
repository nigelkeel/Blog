兄弟关系
父子关系

2、标签的分类#

（1）块标签 block

  • 代表标签: div
  • 默认的宽度是占据父标签全部内容，即 width: 100%
  • 默认的高度由内容决定
  • 设宽高、边距、边框没有限制
  • 独占一行
  • 允许嵌套块元素和行元素（p/h标签除外）

（2）行内标签 inline

  • 代表标签: span
  • 默认的宽高由内容决定。
  • 内容过多时允许换行并断开（一般都是用于处理文字样式）。
  • 设置宽高无效。上下边距、边框无法设置。
  • 共用一行：在一行中如有剩余空间，允许同其他行内/行内块标签共用。
  • 某些动画样式不能添加和使用。

（3）行内块标签 inline-block

  • 代表标签: img、button、input
  • 默认的宽高由内容决定。
  • 内容过多时不会断开。
  • 可以设置宽高。
  • 在一行中如有剩余空间，允许同其他 行内/行内块 标签公用一行。
  • 行内块和内行标签会有行高的概念。
  • 行内块受父元素的 line-height 以及自身的 vertical-align 影响。

将非块级元素转换成块级元素的方法

  • display: block
  • float: left/right
  • position: absolute/fixed
  • 父级元素使用 display:flex

3、语义化标签#

优点：

  • 语义化标签更具有可读性，便于团队的开发和维护
  • 没有 CSS 的情况下，网页也能很好的呈现出内容结构和代码结构
  • 关于SEO，搜索引擎更能理解到网页中各部分之间的关系，更准确更快速搜索信息

布局的三个发展阶段：

  • 开荒时期：前端的代码主要是后台写的，使用 table 来布局
  • 美工时期：有专门的人来写前端代码了，主要布局方式是 DIV+CSS
  • 现在：使用具有语义的标签，可以表达正确的内容，也利用开发和维护。

二、常用标签#

1、div 分块#

用于分块。
分块原则：先行后列。

2、span 额外结构#

在不需要语义的情况下，增加额外的结构，方便设置样式等操作。

3、标题、段落、换行#

（1）标题：h1 h2 h3 …
最多允许6级标题。
依次使用，避免跳过某级标题使用。
同一个网页最多使用一次 h1 标签。
（2）段落：p
（3）换行：br
单纯的换行，被称为软换行。

4、列表#

（1）无序列表：ul-li
常用于重复结构的大块内容布局。
注：
1、ol/ul只能嵌套 li。
2、不建议在ul和li嵌套的中间，书写其他元素。

（2）有序列表：ol-li
用的少。

（3）自定义列表
dl-dt-dd，组合标签。

5、表格#

table、tr、td、th

  • 表格的最外层是 table 元素
  • 表格的标题是 caption 元素 一般在表格的最上边
  • tr 就是表格的行
  • td 是普通单元格，th 是表头单元格（默认加粗居中）

table 的属性

  • border:边框（已经废弃，以后使用 css 的方法）
  • 合并表格的边框，对 table 设置：border-collapse: collapse;
  • cellpadding 控制内容和边框之间的距离
  • cellspacing 控制两个单元格之间的距离

跨行和跨列

  • rowspan:控制元素跨行 谁跨行 设置给谁
  • colspan:控制元素跨列 谁跨列 设置给谁

其他

  • 表格布局（过时了，了解即可）
  • thead：表格的头部
  • tbody：表格的内容，可以划分多个
  • tfoot：表格的底部
  • 三个标签 table 划分为多个模块，优化表格的加载，一般数据表格量大的时候可以使用
  • thead、tfoot、tbody：三个元素，无关书写顺序，一定会按照 thead–tbody–tfoot 顺序加载

6、锚链接#

（1）路径

  • 网络路径：一般有三种协议，一种是 http、一种是 file、一种是 ftp。需要引入网络路径的时候，需要书写http://。
  • 相对路径
  • 绝对路径

（2）标签属性

  • href：路径。
  • target：跳转方式，_self 当前标签页跳转，_blank新标签页跳转。
  • title：鼠标悬浮时的弹窗解释。

（3）锚链接

href="#(id)" → id="(id)"

  • 标签 href 为空时，点击刷新加载。当设置为##（两个#及以上）或者javascript:;时，点击之后不会有动作
  • 如果 a 标签的 href 值只有一个 # 那么可以实现回到顶部的功能。
  • 如果 a 标签的 href 属性书写的链接可以使用浏览器打开（比如图片），则浏览器跳转到这个位置并打开，如果 a 标签的 href 属性书写的链接不可以使用浏览器打开，则浏览器直接下载。
  • a 标签的下载功能：download 属性 书写下载内容，可以直接实现下载，必须拥有 href 属性，否则下载功能失效。

a 标签的伪类 △#

伪类 功能
link 当有链接属性时
visited 链接被访问过
hover 鼠标悬浮在链接上面时
active 链接激活时

①“love-hate 爱恨法则” —— 顺序不能互换 ：四个选择器优先级一样，顺序改变可能会覆盖前边的，比如 visited 写在最后，那么当连接被访问过后，访问过后的颜色就会覆盖 hover 和 active。
②hover 适用于多数其他元素，并且伪元素也可以使用。

7、图片#

属性 描述
src 图片路径
alt 加载失败时显示的替代文本
title 鼠标悬浮时显示的弹窗提示
width、height 宽高

解决底部留白：

  • display: block
  • font-size： 0

消除行内块间距的方式

  • 移除标签间的空格
  • 取消标签闭合
  • 给父元素设置font-size:0，Chrome 默认有最小字体大小限制，可以font-size: 0; -webkit-text-size-adjust:none;
  • 设置 margin 为负值

三、H5 标签#

8、header、footer、nav、aside H5#

header 用于定义文档的页眉（头部区域）。

footer 用于定义文档的页脚（底部区域）。

nav 在文档中定义导航。

aside 通常用于定义侧边栏。

9、section 、artcle H5#

section 在文档中对独立的区域进行分块，一般会有独立的标题。

article 在文档中，对可以独立的完整的被外部引用的内容。

？ 比较 div、section、article

10、figure#

figure

H5 新标签，用于引用图片、插画、表格、代码段。

14、mark#

标记，突出显示上下文的关联性，比如搜索页面的关键词。是行内元素。

15、time、datalist、proress#

time

当时间是标准写法的时候，可以放在该标签中。

datalist

包含一组 option 元素，通常和 input 一起配套使用。

proress

通常用于进度条。（很少用）

四、其他标签#

  • b i u 没有太多的含义，可以增加额外的结构，但是使用率较低。
  • i 标签常用在引入小图标
  • em 标签默认倾斜，起到强调作用，主要是强调关键词的。
  • strong 标签默认加粗，起到更强的强调，主要强调的是一段话。
  • var 标签 没有任何的强调作用，有自己的含义，主要包含变量。
