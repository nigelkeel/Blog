JS 开始

发表于 2020-07-01 更新于 2020-08-27 分类于 技术栈基

一、基本概念#

JavaScript 最初的目的是为了“赋予网页生命”。

这种编程语言我们称之为 脚本。它们可以写在 HTML 中，在页面加载的时候会自动执行。脚本作为纯文本存在和执行。它们不需要特殊的准备或编译即可运行。

  • 1995 JavaScript、JScript诞生
  • 1997 ECMA给JS制定了标准ECMA-262，命名为ECMAScript，即ES1.0。ES是JS的规范标准，JS的ES的实现。
  • 1999 发布ES3，成为JS通用标准
  • 2009 发布ES5
  • 2013 ES6草案发布（其实就是之前不被接受的ES4激进方案）
  • 2015 发布ES6，即ES2015，从此JS新版本按ES+年份形式发布。ES6是一个历史名词，也是一个泛指，代表ES6以后的新一代标准。

1、特点#

弱类型 数据类型不固定，可以随时改变。

解释型 解释型是一行一行解释运行，编译型是需要整体编译。

脚本语言 脚本是需要载体运行的。

基于原型 只有对象，没有类，对象继承对象，而不是类继承类。“原型对象”是基于原型语言的核心概念。原型对象是新对象的模板，它将自身的属性共享给新对象。一个对象不但可以享有自己创建时和运行时定义的属性，而且可以享有原型对象的属性。

2、组成#

  • ECMA Script 核心语法，简称ES。
  • DOM Document Object Model 文档对象模型。
  • BOM Browser Object Model 浏览器对象模型。

3、网页中使用 JS 的方法#

页面嵌入（常用）

行内写法（几乎不用，局限性大，代码分离性差，可读性不强）

外部文件引入（主流方式）

属性:

  • src 指定外部脚本的URI，如果设置了，标签内容会忽略
  • type 指定脚本的语言，默认为JS。
  • async 规定一旦脚本可用，则会异步执行。
  • defer 是否对脚本执行进行延迟。

5、ES5 / ES6#

ECMAScript 是JS的语言规范标准。

ECMA-262 1997年制定的规范，包含了大部分深入的、详细的、规范化的关于 JavaScript 的信息。这份规范明确地定义了这门语言。

ES5 2009年。

ES6 ES6既是一个历史名词，也是一个泛指，含义是 5.1 版以后的 JavaScript 的下一代标准，涵盖了 ES2015、ES2016、ES2017 等等，而 ES2015 则是正式名称，特指该年发布的正式版本的语言标准。

二、基本结构和语法#

1、语句、注释、分号、空白#

  • 语句是执行行为的语法结构和命令。分为 声明语句、调试语句debugger等。
  • 注释的语法和C语言一样。
  • 建议每个语句结束都加分号，否则可能会出现奇奇怪怪的BUG。程序会忽略空白，合适的空白与缩进可以保持良好的代码结构。

2、输出#

  • 系统对话框（弹窗）: alert()、confirm()、prompt()
  • 控制台日志: console.log()、console.dir()
  • 直接显示在文档上: document.write

3、变量#

变量 是一种用来储存数据的可变容器，是数据的命名储存，通常情况下由标识符和值构成。

（1）变量命名/标识符命名规则#

  • 由数字，字母，下划线，美元符组成。
  • 不允许数字开头。
  • 不能和关键字、保留字同名。
  • 严格区分大小写。
  • 变量的起名要有意义。避免使用拼音等。

（2）变量声明和变量赋值#

优先使用const | let ，不要用 var 。

（3）块级作用域#

特点 var let const
声明提升 会 不会 不会
块作用域{} 没有 有 有
同一作用域多次声明 可以 不能 不能
基本数据 可变 可变 不可变
引用数据 可变 可变 地址不可变

for循环声明的let，只在内部生效

块级作用域和函数作用域类似，可以有作用链

for循环的声明语句是父作用域，语句块是子作用域

4、严格模式#

ES6推出，也称为现代模式，通过在全局第一行或者是函数第一行添加"use strict" 语句开启严格模式。（一旦打开，无法关闭）

特点：

  • 给未定义的元素赋值会抛出异常
  • 删除不可删除的属性会抛出异常

3、break 、continue#

break

跳出switch语句，跳出一层循环。

continue

结束本次循环，进行下次循环。

label

  • 在JS中，label语句可以为一行语句添加标签，以方便在复杂的结构中设置跳转目标。
  • 语法如下：label:语句
  • label可以是任意合法的标识符，然后使用冒号分割标签名与标签语句。
  • label和break语句配合使用，主要应用在循环结构、多分枝结构中，以便与跳出内层嵌套体。

禁止break 和continue 写在三元运算符?右边。

异常处理

ECMA-262 规范了7种错误类型。其中 Error 是基类，其他6种错误类型是子类，都继承了基类。Error类型的主要作用是自定义错误对象。

类型 说明
Error 普通异常。与 thorw 语句和 try/catch 语句一起使用，属性可以读写异常类型，message 属性可以读写详细的错误信息。
EvalError 不正确使用eval()方法时抛出。
SyntaxError 语法错误
ReferenceError 读取不存在的变量时抛出
TypeError 值的类型发生错误的时候抛出
RangeError 数字超出合法范围之抛出
URIError URI编码和解码错误时抛出

基本语法#

  • 正常情况下，JS按顺序执行try子句中的代码，如果没有异常发生，将会忽略catch跳转到finally子句继续执行。

  • 如果在try子句运行错误，或者使用throw语句主动抛出异常，则执行catch子句中的代码，同时传入一个参数，引用Error对象
