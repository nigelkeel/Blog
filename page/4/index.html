数组和字符串

一、数组的基本使用

数组（Array） 是一种引用类型，是==有序==的数据集合。它的数组下标（index）从0开始。数组中的每个成员被称为元素，不同元素的值可以为不同类型，数组的长度是弹性的，可读写的。

扩展运算符（spread）是三个点（...）。它好比 rest 参数的逆运算，将一个数组转为用逗号分隔的参数序列，目前也可以用来展开数组。

7. 其他

类数组 也称为伪数组，就是类似数组结构的对象。这个对象的属性名为非负整数，并且从0开始，有序递增，同时包含 length 属性，确保其值与有序下标属性个数保持一致，方便进行迭代操作。

空位数组 就是数组中包含空元素。所谓空元素，就是在语法上数组中两个逗号之间没有任何值。空元素可以读写，length属性不排斥空位，如果使用for语句和length属性遍历数组，空元素都可以被读取，空元素返回值为undefined。出现空位数组的情况如下:

  • 直接量定义
  • 构造函数定义
  • delete删除

关联数组 如果数组的下标值超出范围，如负数、浮点数、布尔值、对象或其他值，JS会自动把它转换为一个字符串，并定义为关联数组。关联数组就是与数组关联的对象，简单地说就是数组对象，字符串下标就是数组对象的属性。

二、数组方法

1、push / pop / unshift / shift

栈是一种先进后出的数据结构，队列是一种先进先出的数据结构。

.push() 接收任意数量参数，并逐个添加到数组末尾，返回修改后数组的长度。

.pop() 无参数，移除数组最后一项并返回该项。

.unshift() 接收任意数量参数，并逐个添加到数组头部，返回修改后数组的长度。

.shift() 无参数，移除数组第一项并返回该项。

这些方法都==会修改原数组==。

  • 如果有一项返回false，则停止执行，every返回false
  • 如果所有的项全部返回true 那么 整个every方法就返回true
  • every方法的回调函数拥有3个参数：1、item 数组的每一项 2、index 数组的这一项 对应的下标 3、array 原数组
  • every方法还有第二个可选的参数，可在回调函数中为其引用this关键字的对象。如果省略，则undefined是this值

（5）some 检测是否存在符合

  • 使用some方法可以确定数组的元素是否有满足指定的测试的项
  • 只要一项返回true 那么 some 方法就返回 true（一真即真）

三、字符串

模版字符串

包装对象 ==需更新==

在JS中，提供了三个特殊的构造函数Number、String、Boolean，可以使用new来操作，会生成对应类型的对象。（实际开发中不用）

执行过程：

  1. 声明了一个字符串放入了str中。
  2. 当发现你把str当做对象调用时，js内部会自动调用new String('abc')创建了一个对象（字符串对象，包装对象）；
  3. 然后使用包装对象的属性、方法。当使用完成之后，又会将包装对象转换成字符串。

number、boolean、string 都会经过上面的过程。

普通对象和包装对象的区别：主要针对对象的生命周期。

  • 使用 new 操作符创建的引用类型的实例，在执行流离开之前的作用域中将会一直存在。
  • 自动创建的基本包装类型对象，只存在于那一行代码的执行的一瞬间，然后立即销毁了。

一、面向对象基本概念

面向对象和面向过程是两种截然不同的编程思想和方法。面向过程是以过程为中心，就像流水线一样设计程序的思想。面向对象是以对象为中心的编程思想。更符合人类的思维习惯。

在面向对象的概念中，**类[^1]**是一类相似事物的统称，对象是类的实例化，类是对象的模版。

面向对象的三个基本特性: ==封装、继承、多态==。

对象（Object） 可以是一个变量，一个函数，一种数据结构。在JS中，==一切皆为对象==。JS允许自定义对象，从狭义的概念来分析，对象是最基本的数据类型，是复合型的结构、引用型的数据，它是无序数据集合[^2]。对象中每个成员被称为属性。

[^1]: JS中没有具体类的概念，ES6 引入的类实际上是函数。
[^2]: 对象是无序的键值对，数组是有序的键值对，函数是一种可执行的特殊对象。

二、对象的声明和使用

1. 字面量方式声明

字面量方式声明的对象由多个键值对组成，每个键值对之间用逗号分隔。每个键值对就是一个属性，包括属性名和属性值。

属性名是字符串并且通常情况下不加引号，如果是其他类型也会转换成字符串。如果键名不符合变量命名规则必须加上引号。

属性值可以是任意东西，包括对象。

2、构造函数方式声明（new Object）

new 是操作符，调用后面的 构造函数 创建一个对象[^3]。

构造函数可以使用return语句，如果return的是基本类型数据，那么值会被忽略。如果返回的是其他对象，那么就返回这个对象。

函数既是对象也是可以执行的代码块：

  • 在对象方面来说它也有属性和方法。
  • 从代码块方面来说，既可以执行特定的代码块（普通函数），又可以创建特定类型的对象（函数的构造调用）
    • 普通函数，没有return返回的是undefined，return返回的是指定的值。
    • 函数的构造调用，没有return，返回的是新生成的对象。有的话看是基本数据类型还是对象。

[^3]: 字面量方式创建对象也会在内部调用new Object。

三、原型 △△

1. 函数的 prototype 属性（显式原型）

每个函数在创建的时候都会创建一个 prototype 属性（指针），指向函数的原型对象。该属性中存储的是一个对象的引用（默认是一个空对象）。

Foo.prototype === Foo 的原型对象

在原型对象中也有一个属性叫 constructor ，指向的是拥有这个原型对象的构造函数。

Foo.prototype.constructor === Foo

prototype 属性只有在函数被当做构造函数调用的时候（函数的构造调用）才会生效。==由同一个构造函数生成的各个实例对象之间共享属性和方法。==

2. 实例对象的 __proto__ 属性（隐式原型）

当调用构造函数创建一个新实例之后，该实例内部将包含一个内部属性 **__proto__**，指向构造函数的原型对象。==实例化之后才能用==。

构造函数中的 prototype 属性（显示原型）和构造函数实例化出来的对象 __proto__属性（隐式原型）指向同一个对象。

先在本对象中查找对应的属性，找不到去 __proto__ 里面找。

在原型中查找值的过程是搜索。对原型对象所做的任何修改都能立即从实例上反映出来。

原型分为 显式原型（构造函数的prototype属性） 和 隐式原型（调用构造函数实例化出来的对象的__proto__属性）。（ES5只允许操作 prototype，ES6都可以）

3. 原型链 ==必考==

原型链是实现继承的主要方法。

构造函数、原型和实例的关系 每个构造函数都有一个原型对象，原型对象都包含一个指向构造函数的指针，而实例都包含一个指向原型对象的内部指针。那么假如我们让原型对象等于另一个类型的实例。显然，此时的原型对象将包含一个指向另一个原型的指针，相应地，另一个原型中也包含着一个指向另一个构造函数的指针。假如另一个原型又是另一个类型的实例，那么上述关系依然成立。如此层层递进，就构成了实例与原型的链条。这就是所谓的原型链的基本概念。==原型链的终点是 null==。

一、是什么

Browser Object Model 浏览器对象模型。关于客户端浏览器的管理。

Document Object Model 文档对象模型。是W3C制订的一套技术规范，用来描述JS脚本如何与HTML或XML文档进行交互的Web标准。加载HTML页面时，Web 浏览器生成一个树型结构，用来表示页面内部结构。DOM将这种树型结构理解为由节点组成的DOM树。

DOM规定了一系列标准接口（函数），允许开发人员通过标准方式访问文档结构、操作网页内容、控制样式和行为等。

1、DOM 版本

DOM0和DOM2。

2、document 对象

document 对象 是window[^1]对象的一个属性，因此可以将document对象作为一个全局对象来访问。当浏览器载入html文档, 它就会成为document对象。

[^1]: JS 顶层对象是global，window是global的实现。

二、节点访问

在网页中所有对象和内容都被称为节点（Node）。如文档、元素、文本、属性、注释等。节点是DOM最基本的单元，并派生出不同类型的节点，它们共同构成了文档的树形结构模型。

1、节点

常用节点种类

  • document 文档节点
  • documentFragment 文档片段节点
  • Element 元素节点
  • attr 属性节点
  • text 文本节点
  • comment 注释节点

节点关系

DOM把文档视为树形结构，称为节点树。节点之间的关系包括：上下父子关系，相邻兄弟关系。

  • 在节点树中，最顶端节点为根节点。除了根节点之外，每个节点都有一个父节点。
  • 节点可以包含任何数量的子节点。
  • 叶子是没有子节点的节点。
  • 同级节点是拥有相同父节点的节点。

childNodes

childNodes 属性获取元素所有的子节点，保存到一个NodeList对象中。

三、节点操作

节点关系都是只读的，DOM提供了一些操作节点的方法。

创建节点 createElement()

使用document对象的createElement方法能够根据参数指定的标签名创建一个新的元素。并返回对新元素的引用。

使用creatElement方法创建的新元素不会被自动添加到文档里，需要使用appendChild等方法。

创建文本节点 createTextNode()

使用document对象的createTextNode方法可创建文本节点。创建的文本节点需要使用appendChild等方法才能插入到元素节点中，当然也可以使用 innerHTML 方法给元素节点添加内容。

插入节点

都是插入元素的父元素调用的方法。

（1）appendChild()

appendChild()方法可以向当前节点的字节点列表的==末尾==添加新的节点。如果文档树中已经存在参数节点，则将从文档树中删除，然后重新插入新的位置。

（2）insertBefore( )

使用 insertBefore(newChild,oldChild) 方法可以在已有的子节点前插入一个新的子节点。

newChild表示新插入的节点，oldChild用于指定插入节点的后边的相邻位置。插入成功以后，该方法返回新插入的节点。insertBefore可以把指定元素及其所包含的所有子节点都一起插入到指定位置中。同时会先删除移动的元素，再重新插入。

appendChild 和 insertBefore 可以操作已经存在的元素。

复制节点 cloneNode()

cloneNode() 方法可以创建一个节点的副本[^2]。复制后的新节点，也不会被自动插入到文档，需要用到之前的方法去插入。

[^2]: 注意改id 复制的节点会包含原节点的所有特性，如果原节点中包含id属性，就会出现id属性值重叠的情况。为了避免潜在的冲突，应修改其中某个节点的id属性值。

删除节点 removeChild()

removeChild() 方法可以从子节点列表中删除某个节点，该节点所包含的所有子节点将同时被删除。

替换节点 replaceChild()

replaceChild(new, old) 方法可以将某个子节点替换为另一个。替换节点替换的是所有子节点以及包含的所有内容。

4、元素内容操作

innerHTML、innerText、outerHTML、outerText

（1）获取标签

innerHTML 和 innerText 前者可以解析并输出标签，后者是纯文本。

innerHTML 和 outerHTML 前者不包括自身，后者包括自身。

（2）设置标签（直接覆盖原来的内容）

innerText 和 outerText 在获取的时候是相同效果，但在设置时，innerText仅设置标签所包含的文本，而 outerText 设置包含包括标签自身在内的文本。

textContent

textContent 设置或者返回固定节点的文本内容。

（1）textContent和innerHTML的区别

  • textContent 会获取 style= “display:none” 中的文本，而innerText不会。
  • innerText 不会理会html格式，直接输出不换行的文本，textContent会根据标签里面的元素独立一行。
  • innerText 对IE的兼容性较好 ，textContent虽然作为标准方法但是只支持IE8+以上的浏览器 。

6、文档片段节点

DocumentFragment 是一个==虚拟的==节点类型，仅仅存在于内存中，没有添加到文档树中，看不到渲染效果。当文档片段设计完善后，再使用JS一次性添加到文档树中显示出来，提高效率，==减少页面重绘的次数==。避免浏览器渲染和资源占用，解决大量添加节点时候的性能问题。

createDocumentFragment() 创建虚拟节点，使用 appendChild 等方法插入。

八、样式重置

样式重置（CSS reset），将html的默认样式全部去掉，需要的时候我们自行添加。统一页面风格。

为什么要用样式重置

  • 多数元素拥有自己的默认样式，并且我们很多都不需要。
  • 默认样式在不同浏览器中呈现的也不一定相同，就会导致浏览器展示页面不相同的现象。
  • 整个页面中固定的风格代码，可以在样式重置中直接书写，不用每一个都设置。

为什么不用通配符重置

  • 重置简单，不够完整，一些demo可以使用，真正项目不推荐使用，请使用CSS reset
  • 通配符，匹配所有的标签，影响很大，效率很低。

九、文档流

脱离文档流，指的是元素脱离正常元素的布局排版规则，其他处于文档流中的盒子在计算布局排版时，会自动无视已脱离文档流的元素来进行定位。

浮动与绝对定位之间的区别就是：

  • 浮动会使元素脱离文档流，但是不会脱离文本流，在于其他盒子的文本内容计算布局的时候，还是占位置的
  • 绝对定位会使元素脱离文档流，同时也会脱离文本流，在于其他盒子的文本内容计算布局的时候，不占位置。

4、网页的组成

网页由三部分组成：

  • 结构 html
  • 表现 CSS
  • 行为 JavaScript

二、HTML 基本结构

1、注释

用<!-- 和-->包裹起来的内容。

2、版本声明

给浏览器声明，告诉浏览器应该按照HTML5的规范解析当前页面。 如果不写，浏览器就是按照自己默认的模式解析。

版本声明必须写在第一行。我们不把他看做一个html标签，而是把它当做一个指令，并且这个指令对大小写不敏感。

3、html、head、body 标签

html 是一个双标签，所有网页内容全部书写在html标签中。lang就是html标签的属性，en就是lang属性的值，属性是对当前标签的信息的扩展。

head 标签代表网页的头部，不会显示在网页中。主要是包含了标题，网页的描述，字符编码等等一些信息。

body 标签是网页显示的主体内容。

4、meta 标签

控制网页的==元信息==（信息的信息），根据属性的不同，有很多的作用。

5、解决网页乱码问题 △

charset即字符编码。utf-8称作为万国码。

三、HTML 标签

1、标签的语义化 △

  • 语义化标签更具有 可读性 ，便于团队开发和维护。
  • 在没有加载CSS的情况下，网页也能很好呈现出内容结构和代码结构。
  • 对于SEO来说，搜索引擎更能理解网页中各部分之间的关系，更准确更快速搜索信息。

2、标签的关系

  • 并列：兄弟关系
  • 嵌套：父子关系（使用TAB缩进）

3、标签的分类

3.1 块标签

特性

  • 独占一行
  • 可设宽高、内外边距和边框
  • 可嵌套块元素和行元素（p、h标签除外）

将非块级元素转换成块级元素的方法

  • display: block
  • float: left/right
  • position: absolute/fixed
  • 父级元素使用display:flex

3.2 行标签

特性

  • 空间足够时可以其他行标签共用一行
  • 设置宽高无效，内外边距和边框的上下设置无效
  • 宽高用内容撑开
  • 可嵌套行元素
  • 受父元素的 line-height 以及自身的 vertical-align影响。

3.3 行内块

特性

  • 一行可放多个。
  • 宽高、内外边距、边框无法设置。
  • 受父元素的 line-height 以及自身的 vertical-align影响。

（3）锚链接

href="#(id)" → id="(id)"

  • 标签href为空时，点击刷新加载。当设置为##（两个#及以上）或者javascript:;时，点击之后不会有动作
  • 如果a标签的href值只有一个 # 那么可以实现回到顶部的功能。

  • a标签的下载功能：download属性 书写下载内容，可以直接实现下载，必须拥有href属性，否则下载功能失效。

a标签的伪类 ==△==

伪类 功能
link 当有链接属性时
visited 链接被访问过

本地化存储

一、Cookie

由于HTTP协议是无状态的，而很多服务器业务是必须要用状态。Cookie诞生的最初目的是为了存储网页中的状态信息，以方便服务器端使用。比如判断用户是否是第一次访问网站。

1、流程

  • 服务器向客户端发送cookie
  • 浏览器保存cookie
  • 之后每次http请求浏览器都会将cookie发送给服务器端

2、服务器处理cookie

服务器端像客户端发送Cookie是通过HTTP响应报文实现的，在Set-Cookie中设置需要向客户端发送的cookie。

cookie 是一个 键值对 的存储结构。

  • expires 设置失效时间
  • max-age 有效时间，单位是秒（推荐）
  • httpOnly

如果不设置时间，则代表是临时会话存储，浏览器关闭后就会消失。

3、客户端处理cookie

客户端获取 document.cookie

客户端设置 document.cookie = 'name=rose;max-age=3600'

4、缺点

  • 数量和长度有限制
  • 潜在的安全风险：cookie可能被截取篡改，如果cookie被拦截，就可能会获取到所有的Session信息
  • 用户配置为禁用，有的用户禁用了浏览器或者客户端设备接受cookie的能力，因此限制了这一功能
  • 每次都会随着服务器端发送，具有一定的带宽压力

5、将cookie解析成对象 ==通用==

二、Session

session 是存储于服务器的对象，是另一种记录客户状态的机制，cookie保存在客户端，session保存在服务器。session运行在服务器，客户端第一次访问服务器时，就可以将客户的登录信息保存。当客户访问其他界面时，可以判断客户的登录状态，做出提示，相当于登录拦截。

1、特点

  • 存储于服务器上
  • 大小和数量理论上没有限制
  • 传输流量小（只有一个cookie）
  • 安全性较高

2、生成随机id的方法 ==通用==

3、流程

  • 客户端：发起登录请求，在请求报文体或者URL中携带登录信息
  • 服务器：接受请求，创建一个session对象，将用户信息保存在session对象中，并创建一个session-id（唯一的）
  • 服务器：返回响应，cookie携带session-id
  • 客户端：接受session-id，保存在本地
  • 客户端：当发起第二次请求的时候，会携带具有session-id的cookie
  • 服务器：通过解析cookie，得到session-id，去session对象中查找当前id是否存在，从而识别用户是不是第一次访问，如果识别用户成功，直接响应通过，否则需要再次查询数据库等操作
