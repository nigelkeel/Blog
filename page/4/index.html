重绘重排#

重绘 当render tree（渲染树）中更新的属性只会影响元素的外观、风格，不会影响元素的布局的时候，浏览器需要重新绘制当前元素的样式，被称作为重绘。

重排（回流） 当render tree中的一部分或者是全部，因为元素的尺寸、布局、隐藏等等改变引起页面的重新渲染，这个过程称作为重排，完成重排以后，浏览器会重绘。

  • 重排的情况:
    ①添加或者删除可见的DOM元素
    ②元素位置、尺寸、内容改变（例如：一个文本被另一个不同尺寸的图片替代）
    ③页面渲染初始化（无法避免）
    ④浏览器窗口尺寸改变
  • 重绘不会引起重排，但重排一定会引起重绘，一个元素的重排通常会带来一系列的反应，甚至触发整个文档的重排和重绘，性能代价是高昂的。重排和重绘是DOM编程中耗能的主要原因之一。

jQuery 是John Resig创建的一个JS开源类库，它通过封装原生的JavaScript 函数得到一整套定义好的方法。

JS对象 即DOM对象，即是我们用传统的方法获得的对象。

jQuery对象 即是用jQuery类库的选择器获得的类数组对象，就是通过jQuery包装DOM对象后产生的对象，它是jQuery独有的。如果一个对象是jQuery对象，那么就可以使用jQuery里的方法。

二、常规操作#

1、类数组操作#

  • children() 获取选择器元素的子元素节点集合，如果指定了参数，那么该参数也是筛选表达式
  • parent() 获取选择器元素的父元素节点集合（只有一个），，如果指定了参数，那么该参数也是筛选表达式
  • siblings() 获取选择器元素的兄弟元素节点集合，如果指定了参数，那么该参数也是筛选表达式
  • find() 寻找后代元素，参数直接写选择器
  • end 返回上一次破坏性操作之前的伪数组
  • 对于类数组的常用方法还有很多，比如 prev()、next()等等。

2、选择器操作#

直接参数传递选择器。

  • 基础选择器 #id、.class
  • 层次选择器 A B、A+B、A~B
  • 过滤选择器 :first、first-child、not()、even、odd
    • :first / :last 选取第一个元素/最后一个元素
    • :first-child / :last-child 选取第一个子元素/最后一个子元素。
    • :only-child 选取只有一个子元素的节点。
    • nth-child(index/even/odd/equation) 选取序列特定的元素
    • :even / :odd 选取索引是单数、双数的元素
    • :eq(index) 选取指定index值的元素
    • :gt(index) / :lt(index) 选取索引大于/小于指定的元素
    • :not(seletor) 排除选定的元素
    • :contains(text) 选取包含指定文本text的元素。
    • :empty 选取空元素
    • :has(seletor) 选取含有指定选择器的元素。
    • :parent 选取含有子元素或文本的元素。
  • 属性过滤选择器 [attr]
  • 表单选择器 :input

（1）基础使用

animate( params, speed, callback)

  • 第1个参数 包含样式属性和值的映射速度
  • 第2个参数 可选
  • 第3个参数 在动画完成时执行的回调函数

（2）动画队列

如果要按照顺序执行动画，只需要按照代码顺序就可以。（只有动画会放在队列中，其他方法会立即执行 ）

5. 停止动画#

（1） 停止方法

stop( clearQueue, gotoEnd )

  • 第1个参数 可选 布尔值 是否需要清空未执行完的动画队列
  • 第2个参数 可选 布尔值 是否直接将正在执行的动画跳转到末状态

这个方法会结束当前正在进行的动画，并立即执行队列中的下一个动画。

（2）判断元素是否处于动画状态

当用户在某个元素上执行 animate() 动画时，就会出现动画积累，

使用 is(":animated") 来判断是否处于动画状态，如果不处于动画状态，才会给元素添加新动画。

6. 延迟动画#

delay()

参数是延迟的毫秒数。

2. 事件对象#

对于原生的事件对象，由于浏览器差异，开发者总是需要做兼容性处理。jQuery 解决了原生事件对象的兼容性问题，还提供了很多新的属性和方法。

.stopPropagation() 阻止事件冒泡

.preventDefault() 阻止默认行为

3. jQuery 绑定事件#

bind() 方法无法动态绑定尚未存在的元素。此外还有 live() delegate 等已经被废弃，底层实际使用 on()。

on( events, selector, data, fn) 绑定事件jQuery 1.7+

  • events 事件类型
  • selector 触发事件选择器元素的后代
  • data 当一个事件被触发要传递 event.data 给事件处理函数
  • fn 该事件被触发时的执行函数

还可以绑定多个事件绑定同一个函数、多个事件绑定不同函数。

unbind() 解除绑定事件。

off( events, selector, fn) 解除绑定事件。

one( type, data, fn ) 绑定元素执行完毕后自动移除事件，方法仅触发一次。

  • type 添加到元素的一个或者多个事件。
  • data 将要传递给事件处理函数的数据映射。
  • fn 每当事件触发时执行的函数。

一、响应式布局#

响应式布局是在2010年提出的一个概念，简而言之，就是一个网站能够兼容多个终端，而不是为每个终端做一个特定的版本。用户会使用不同的终端，由于屏幕分辨率不同，想要用户体验更好，就有必要让你的网页兼容多个终端。响应式布局设计（HTML + CSS1 + CSS2 + CSS3）可以让一个网站适配多种设备和多个屏幕，可以让网站的布局和功能随用户的使用环境（屏幕大小，输入方式，设备/浏览器能力）而变化。
