2、包装对象 +#

在JS中，提供了三个特殊的构造函数Number、String、Boolean，可以使用new来操作，会生成对应类型的对象。（实际开发中不用）

var str = new String("abc");
// 这确实是一个对象
// 被当成对象
// __proto__指向原型对象。原型方法有大量的方法

执行过程：

  1. 声明了一个字符串放入了str中。
  2. 当发现你把str当做对象调用时，js内部会自动调用new String('abc')创建了一个对象（字符串对象，包装对象）；
  3. 然后使用包装对象的属性、方法。当使用完成之后，又会将包装对象转换成字符串。

number、boolean、string 都会经过上面的过程。

普通对象和包装对象的区别：主要针对对象的生命周期。

  • 使用 new 操作符创建的引用类型的实例，在执行流离开之前的作用域中将会一直存在。
  • 自动创建的基本包装类型对象，只存在于那一行代码的执行的一瞬间，然后立即销毁了。

二、字符串方法#

1、字符串查找#

indexOf、lastIndexOf#

indexOf() 正序查找 lastIndexOf() 逆序查找 根据参数字符串，返回指定子字符串的下标位置，如果没找到，返回-1

  • 第1个参数 查找的目标
  • 第2个参数 可选 开始查找的位置

search#

search() 功能和indexOf()相同，定义匹配模式，只有一个参数，没有反向检索和全局模式。

macth#

macth() 找出所有匹配的子字符串，并以数组的形式返回，如果匹配模式没有指定全局，则进行单次匹配，如果没找到返回null。

charAt、charCodeAt#

charAt(n) 返回字符串中的第n个字符，如果n超出字符串范围，返回空串。

charCodeAt(n) 返回字符串中第n个字符的字符编码。

includes、startsWith、endsWith#

includes(string,n) ES6 返回布尔值，表示是否找到参数字符串

startsWith() endsWith() 判断字符串开头、结尾 中间 有没有包含某个字符串

2、字符串截取、切割#

substr#

substr 根据指定长度截取字符串（不推荐使用该方法）

  • 第1个参数 起始下标
  • 第2个参数 截取的长度（如果省略，则代表截取到末尾）

slice、substring#

slice 、substring 根据指定的起止下标截取字符串

  • 第1个 起始下标
  • 第2个 结束下标（不包含），如果省略，代表剩下的全部截取
  • 起始和结束位置相比较的大小无法确定的时候可以使用substring
  • 如果为负值，slice能把负值当作从右向左。而substring视其无效

三、数组的基本使用#

数组（Array） 是一种引用类型，是有序的数据集合。它的**数组下标（index）**从0开始。数组中的每个成员被称为元素，不同元素的值可以为不同类型，数组的长度是弹性的，可读写的。

1. 创建数组#

（1）构造函数方式

var color = new Array('red','blue','green');
//  1. Array 如果仅一个元素且是数值，将创建数值个数元素的数组。
//  2. new可以省略。

（2）字面量方式（常用）

var color = ['red','blue','green'];

2. 数组的长度#

.length 属性保存数组的元素个数（长度）。可读可写，是一个动态属性，会随元素的变化而自动更新。

3. 数组基本操作#

var arr = [1,2,3,4,5,6];

// 1. 查看元素
arr[1]  // 查看第二个元素

// 2. 修改元素
arr[1] = 3

// 3. 在尾部加入一个元素（push方法替代）
arr[arr.length] = 7

四、数组方法#

1、栈和队列操作 △#

栈是一种先进后出的数据结构，队列是一种先进先出的数据结构。

.push() 接收任意数量参数，并逐个添加到数组末尾，返回修改后数组的长度。

.pop() 无参数，移除数组最后一项并返回该项。

.unshift() 接收任意数量参数，并逐个添加到数组头部，返回修改后数组的长度。

.shift() 无参数，移除数组第一项并返回该项。

这些方法都会修改原数组。

2、检索 △#

indexOf、lastIndexOf#

indexOf 返回的是下标，第二个参数是起始位置

indexOf 是全等查找，找不到NaN

includes △#

includes 返回一个布尔值，表示某个数组是否包含给定的值。

find#

find 找出符合条件的数组成员。

findIndex#

let result = arr.findIndex(fn)

传入一个测试函数，返回符合条件（测试函数返回ture时）数组的第一个元素位置，

如果没有符合条件则返回-1。

如果是空数组，测试函数是不会执行的，这个方法不会改变原先数组的值。

3、排序#

① 对当前数组进行排序，会修改当前数组。

② 参数可选，不传参数按默认unicode码进行排序，如果要进行数字的升序或者降序排序，需要传入一个比较函数作为参数。

// 升序
function fn(a, b){return a-b;}
// 降序
function fn(a, b){return b-a;}
// 按照数组对象中的某个属性进行排序
function fn(prop){
    return function(a, b){
        var v1 = a[prop];
        var v2 = b[prop];
        return v1-v2;
    }
}
// 冒泡排序法
for( var i=0; i<a.length-1; i++){
    for( var j=0; j<a.length-1-i; j++){
        // 正序
        if( a[j]>a[j+1]){
            t = a[j+1];
            a[j+1] = a[j];
            a[j] = t;
            
        }
    }
}

没有参数的情况下，该方法会调用每个数组元素的 toString 方法，然后比较的是字符串。在接收一个比较函数作为参数的情况下，来确定排序。

// v1
var res = values.sort(function (value1,value2)){
    if ( value1 < value2 ){
  			return -1;
		} else if (value1 > value2){
        return 1;
    } else {
      return 0;
    }
};


// v2 正序
var res = values.sort( function (value1,value2) ){
     return value1 - value2;
};

// 倒序
var res = values.sort( function (value1,value2) ){
     return value2 - value1;

splice#

splice 可以对数组进行删除、替换、插入。

会修改原数组。

// （1）删除
//  第一个参数指定开始删除的index
//  第二个参数指定要删除的项数
arr.splice(1,2); 

//  (2) 替换
//  第一个参数指定开始index
//  第二个参数指定要删除的项数
//  第三个参数（或者更多的参数）指定要插入的对象
arr.splice(1,2,1,2);

//  (3) 插入
//  在（2）中，把第二个参数设置为0就可以插入内容了。

//  通过delete删除数组
let arr = [1,2,3,4];

delete arr[1];

slice#

slice 基于当前数组中的一个或者多个项目创建一个新数组，该方法不影响原始数组（实际上就是切割数组），一个参数就是从该位置切割到数组尾，两个参数就是切割起始位置和结束位置（不包括结束位置）。

var arr = ['a','b','c','d'];

var s1 = arr.slice(1);  // [ 'b', 'c', 'd' ]
var s2 = arr.slice(1,2); // ['b']
var s3 = arr.slice(1,3); // ['b','c']
var s4 = arr.slice();   // 和arr相同

（2）map 映射数组

map 被称为映射数组。参数是一个回调函数。该方法可以对数组的每一个元素调用指定的回调函数，并返回包含结果的数组。==返回一个新数组==，其中每一个元素均为关联的原始数组元素的回调函数返回值。不会为缺失的元素调用回调函数。

Arr. map( function( item, index){
    // 回调函数体
}, thisArg );

//   - item 项
//   - index 下标
// 返回值： 返回一个新数组

    return item + 1;

（3）filter 过滤数组

    return item % 2 == 0

    return item % 2 == 0;
}

var obj = {max:80,min:14};
var age = [12,11,34,56,90,92,65,65,34,31];

    return item >= this.min && item <= this.max;

（4）every 检测是否全部符合

  • every方法可以测试的所有元素是否都满足指定的测试
  • every中是函数，每一项数组的值，依次进入函数运行，函数最终要要求返回布尔值。
  • 如果有一项返回false，则停止执行，every返回false
  • 如果所有的项全部返回true 那么 整个every方法就返回true
  • every方法的回调函数拥有3个参数：1、item 数组的每一项 2、index 数组的这一项 对应的下标 3、array 原数组
  • every方法还有第二个可选的参数，可在回调函数中为其引用this关键字的对象。如果省略，则undefined是this值

（5）some 检测是否存在符合

  • 使用some方法可以确定数组的元素是否有满足指定的测试的项
  • 只要一项返回true 那么 some 方法就返回 true（一真即真）
  • 其余和every类似

一、基本概念#

JavaScript 最初的目的是为了“赋予网页生命”。

这种编程语言我们称之为 脚本。它们可以写在 HTML 中，在页面加载的时候会自动执行。脚本作为纯文本存在和执行。它们不需要特殊的准备或编译即可运行。

  • 1995 JavaScript、JScript诞生
  • 1997 ECMA给JS制定了标准ECMA-262，命名为ECMAScript，即ES1.0。ES是JS的规范标准，JS的ES的实现。
  • 1999 发布ES3，成为JS通用标准
  • 2009 发布ES5
  • 2013 ES6草案发布（其实就是之前不被接受的ES4激进方案）
  • 2015 发布ES6，即ES2015，从此JS新版本按ES+年份形式发布。ES6是一个历史名词，也是一个泛指，代表ES6以后的新一代标准。

BFC，块级格式化上下文。

1、创建BFC#

  • 根元素
  • 浮动（float 不为 none）
  • 绝对定位 或者 固定定位
  • 表格的标题和单元格，匿名表格单元格元素
  • 行内块元素
  • overflow的值不为visible

2、BFC的范围#

A block formatting context contains everything inside of the element creating it, that is not also inside a descendant element that creates a new block formatting context.

直译过来就是, BFC包含创建它的元素的所有子元素, 但不包括创建了新BFC的子元素的内部元素。简单来说，子元素如果又创建了一个新的 BFC，那么它里面的内容就不属于上一个 BFC 了，这体现了 BFC隔离 的思想。
也就是所说，一个元素不能同时存在于两个 BFC 中。

3、BFC的特性#

BFC 除了会创建一个隔离的空间外，还具有以下特性：

  • BFC 内部的块级盒会在垂直方向上一个接一个排列。
  • 同一BFC下的相邻块级元素可能发生外边距折叠，创建新的BFC可以避免外边距折叠。（解决塌陷）
  • 浮动盒的区域不会和BFC重叠
  • 计算BFC的高度时，浮动元素也会参与计算

4、BFC的应用#

  • 自适应多栏布局
    中间栏创建 BFC，左右栏宽度固定后浮动。由于盒子的 margin box 的左边和包含块 border box 的左边相接触，同时浮动盒的区域不会和 BFC 重叠，所以中间栏的宽度会自适应。

  • 防止外边距折叠
    创建新的 BFC ，让相邻的块级盒位于不同 BFC 下可以防止外边距折叠

  • 清除浮动
    BFC 内部的浮动元素也会参与高度计算，可以清除 BFC 内部的浮动

一、LESS#

less 是一种动态样式语言，属于CSS预处理器，扩展了CSS语言，增加了变量、混合（Mixin）、函数等特性，使CSS更易维护和扩展。LESS既可以在客户端浏览器上运行，也能借助NODE.JS在服务器端运行。

CSS 自诞生以来，基本语法和核心机制一直没有本质上的变化，它的发展几乎全是表现力层面上的提升。最开始 CSS 在网页中的作用只是辅助性的装饰，轻便易学是最大的需求；然而如今网站的复杂度已经不可同日而语，原生 CSS 已经让开发者力不从心。当一门语言的能力不足而用户的运行环境又不支持其它选择的时候，这门语言就会沦为 “编译目标” 语言。开发者将选择另一门更高级的语言来进行开发，然后编译到底层语言以便实际运行。于是，在前端领域，CSS 预处理器应运而生。而 CSS 这门古老的语言以另一种方式 “重新适应” 了网页开发的需求。

5、margin 层#

margin的作用是撑开当前元素和其他元素的距离。margin有四个方向，可以分别设置margin-left margin-right margin-top margin-bottom，margin的left和top是让当前元素向右向下移动，margin的right和bottom是让元素的兄弟元素向右向下移动，不影响自身。

5.1 合写#

  • 4个值：上 右 下 左（从上开始顺时针）
  • 3个值：上 左右 下
  • 2个值：上下 左右
  • 1个值：上下左右

5.2 水平方向上的auto#

因为块标签拥有满行属性，但是设置宽度以后，剩余的空间就是默认的margin，设置左右的margin值为auto，那么剩余的margin就会平分到左右两边。

5.3 竖直方向的auto#

相对于水平方向来说，块元素在垂直方向上并没有满屏的属性，margin默认在上下的值都是0，并没有剩余空间去平分。

5.4 塌陷和边框合并问题#

  • 父级塌陷：父级的第一个子元素的上margin和最后一个子元素的下margin会进行塌陷给父级，这样的设计是为了方便父级和他的兄弟进行垂直margin的合并，避免缝隙过大。
  • 解决塌陷：给父级开启BFC，给父级设置边框。
  • 兄弟的margin垂直方向会进行折叠，取最大值 。

5.5 margin 负值#

  • margin-top为负值：元素向上走，不会影响上边的元素的位置（会出现重叠），并且下边的兄弟元素会跟着一起向上走。
  • margin-left为负值：元素向左走，不会影响前边的元素位置(会出现重叠)，右边兄弟元素会跟随移动。
  • margin-right为负值：元素不会移动，形态大小也不会发生改变，但是所占用的位置会减少（负margin的值），所以，后边的兄弟元素会自动向左走。
  • margin-bottom为负值：元素不会移动，形态大小也不会发生改变，但是所占用的位置会减少（负margin的值），所以，后边的兄弟元素会自动向上走。

5.6 支持性#

margin 完美支持块元素和行内块元素，不支持行元素的垂直方向。

八、样式重置#

样式重置（CSS reset），将html的默认样式全部去掉，需要的时候我们自行添加。统一页面风格。

为什么要用样式重置

  • 多数元素拥有自己的默认样式，并且我们很多都不需要。

2、脱离文档流#

脱离文档流，指的是元素脱离正常元素的布局排版规则，其他处于文档流中的盒子在计算布局排版时，会自动无视已脱离文档流的元素来进行定位。

浮动与绝对定位之间的区别就是

  • 浮动会使元素脱离文档流，但是不会脱离文本流，在于其他盒子的文本内容计算布局的时候，还是占位置的
  • 绝对定位会使元素脱离文档流，同时也会脱离文本流，在于其他盒子的文本内容计算布局的时候，不占位置。

二、浮动#

浮动（float）可以让多个块元素横向排列，共行。

属性 说明
none 默认，不浮动
left 左浮动
right 右浮动

1、概念#

正常页面流/正常文档流 是行元素从左至右，块元素从上到下布局。浮动 是破坏流（脱离文档流）的一种方式。在目前开发中，很多的元素都是一个个float堆叠起来，但是这种布局很容易崩溃。只要宽度和高度发生一点变化，页面就会发生错乱。

2、浮动的使用#

左浮动 就是让容器沿着容器左侧排序。

右浮动 让容器沿着容器右侧排序。

3、高度塌陷#

浮动元素以后，脱离父级内容区域，父级没有内容撑开自身的高度，而且父级的兄弟元素是按照父级的位置进行布局的，所以页面会乱。

4、清除浮动 △#

清除浮动不是不让元素浮动，是清除浮动对父级高度塌陷的影响。

清除浮动的方法

  • 给浮动元素的父级设置高度。
  • 给浮动元素的父级设置浮动，以浮制浮。
  • overflow设置不为visible的属性。
  • 在最后一个浮动元素的后边添加一个空的块级元素，添加一个块级元素，添加属性和值为 clear: both。
  • （推荐）after伪元素清浮动，给所有浮动元素父级一个clearFix的类名。
.clearFix:after{
    content: "/200B";
    display: block;
    height: 0;
    clear: both;  
}

三、定位#

定位（position） 允许你定义元素框相对于其正常位置应该出现的位置，或者相对于父元素，以及浏览器窗口本身的位置。

属性 描述
static 默认，没有定位
relative 相对定位
absolute 绝对定位
fixed 固定定位

布局一般用在大布局的小范围中。

1、相对定位#

相对定位（relative），让元素相对定位，元素先放置在未添加定位时候的区域，然后在不改变页面布局的情况下（其他元素没有受到任何影响；自身原来的位置也保留；和浮动可以一起使用）进行移动。不会脱离文档流。

2、绝对定位#

绝对定位（absolute），相对于它的 包含块 定位。绝对定位是相对于它的包含块定位的（和自身原有位置无关）。如果没有任何一个父级元素是非static属性，则会相对于文档（html）定位。如果它的父级元素和爷爷级元素都是非position:static 属性，则它会选择距离最近的父元素。脱离正常文档流，原有位置不保留。定位的原点以包含块的padding-box的左上角为原点。right和bottom是距离包含块右下角的距离。

2.1 包含块#

包含块 是视觉格式化模型的一个概念，简单来说就是一个定位参考框。

如何确定一个元素的包含块，完全取决于它自身的 position 属性：

  • 如果一个元素自身的 position 属性是 static 或者是 relative ，它的包含块就是离他最近的祖先元素或者是格式化上下文。
  • 如果一个元素自身的 position 属性是 absolute ，它的包含块就是离他最近的拥有定位属性（值不为 static ）的元素。
  • 如果一个元素自身的 position 属性是 fixed，它的包含块就是 viewport。

补充：
如果一个元素的 position 属性是 absolute 或者是 fixed 在下边几种情况下，包含块会发生改变。

  • 当祖先元素拥有 transform 或 perspective 属性并且值不为none的时候，它也是被当做包含块。
  • 当祖先元素 拥有 filter 属性的时候（值不为none） 它也可以被当做包含块

注：如果由内向外找不到包含块条件的元素，那么html（根元素）被称作为初始包含块。

2.2 “子绝父相”#

父元素相对定位并不设置位移，子元素绝对定位，设置位移。

3、固定定位#

固定定位（fixed），相对于浏览器窗口定位。

四、层叠#

1、z-index#

z-index 指定了一个定位属性的元素及其后代的层叠顺序，只有定位元素（非static）拥有该属性。z-index 的值没有单位，理论上来说 z-index 的值大的元素，会覆盖小的元素。定位元素默认的z-index 的值是 auto。如果一个拥有 z-index 属性的定位元素中，子元素也设置了 z-index 。那么子元素会重新创建一个层叠上下文，子元素的z-index只能在当前的层叠上下文中对比排列。

2、元素层叠顺序#

z-index为负< background< border< 块级元素 < 浮动元素 < 内联元素 < 没有设置z-index的定位元素 < z-index为正
