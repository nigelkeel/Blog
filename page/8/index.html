三、脚本化CSS#

1、获取尺寸#

offsetWidth和offsetHeight 可以获取元素的border-box尺寸。

clientWidth和clientHeight 可以获取元素的padding-box尺寸。

scrollWidth和scrollHeight 可以获取元素完全内容尺寸和padding。（几乎不用）

innerWidth和innerHeight 可以获取浏览器窗口的尺寸。

outerWidth和outerHeight 可以获取取加上工具条与滚动条窗口的尺寸。

style.width/height 只能获取行内样式的尺寸（字符串，“10px”）。

常见应用: 大盒子里面小盒子，大盒子用clientX，小盒子用offsetX。

2、获取位置#

getBoundingClientRect() 返回一个对象，对象中包含元素距离视口的距离信息。

getBoundingClientRect。

获取某个元素相对于视口的位置集合

  • top 元素上边到视窗上的距离
  • right 元素右边到视窗左的距离
  • bottom 元素下边到视窗上的距离
  • left 元素左边到视窗左的距离
  • width
  • height

offsetParent

==offsetLeft / Top==

返回当前元素的相对包含块的偏移位置。

JS 默认把它当成绝对定位属性来寻找包含块，不管是什么属性，都是寻找最近定位父级，如果没找到就是 HTML。

clientLeft / Top

clientLeft 等于 border-left。

==scrollLeft / Top==

==设置或者获取==当前元素滚动条已经滚动过的距离。（值增大，内容左滚）

一、概念#

1、事件模型#

DOM0 - 基本事件模型。

DOM2 - DOM事件模型，标准的事件处理模型。

2、事件流#

事件流就是多个节点对象对同一个事件响应的先后顺序。

冒泡型 从目标元素向 document 进行响应 （IE标准）

捕获型 从 document 向 目标元素 进行响应 （网景标准）

混合型 W3C事件流，==先捕获，执行目标事件，最后冒泡==。

3、事件绑定#

（1）静态绑定

（2）动态绑定（DOM0级绑定）

元素 . on+事件类型 = 函数

  • 绑定事件简单，把事件函数赋值给DOM对象的事件属性上
  • 执行的事件流是冒泡
  • 对一个元素绑定同一个事件多次，前面的会被后面覆盖
  • 注销事件只需要给事件属性赋值为 null

（3）事件处理函数

元素 . addEventListener(type function Boolean)

  • 可以绑定同一个元素同一个事件多次。
  • 可以控制冒泡或者捕获。
  • DOM2级事件（DOMContentLoader事件），只能使用这种方法绑定。

（4） IE事件模型绑定方法（了解）

元素 . attachEvent( onType function )

==事件的this指向事件源对象。==

6、事件委托 ==常考==#

事件委托（delegate） 也称为事件托管、事件代理，就是把目标节点的事件绑定到祖先节点上。

原理：冒泡事件流。

优点：

  • 优化代码，提高效率
  • 给未来的元素添加绑定

7、函数防抖、节流#

==防抖==（debounce），短时间触发多次事件，只执行最后一次，或者是只执行最开始一次，中间的不执行。

==节流==（throttle），连续触发事件，但是在一定的时间内只执行一次函数。

五、线程和事件机制#

1、进程和线程#

进程 是程序的一次执行，占用的一片独有的内存空间。

线程 是进程内的一个独立单元，CPU基本调度单位，是程序执行的一个完整流程。

进程和线程的关系

  • 一个进程至少有一个运行的线程（主线程），如果一个进程有多个进程，则称进程是多线程运行的。
  • 一个进程中的数据可以供其中多个线程直接共享，进程之间的数据是不能直接共享的。

多线程

  • 优点 有效提高CPU的利用率。
  • 缺点 增加开销（创建、切换），死锁与状态同步问题。

单线程

  • 优点 顺序编程简单易懂。
  • 缺点 效率低。

==JS是单线程的==[5]，但主流浏览器是多线程的。

2、事件循环模型 ==常考==#

代码分为 初始化代码（同步代码，绑定dom事件监听，设置定时器，发送ajax请求代码 ） 和 回调代码（异步代码）。

3、异步函数#

一、Web Workert H5#

提供了JS分线程的实现。

相关API

  • Worker: 构造函数, 加载分线程执行的JS文件
  • Worker.prototype.onmessage: 用于接收另一个线程的回调函数
  • Worker.prototype.postMessage: 向另一个线程发送消息

不足

  • worker内代码不能操作DOM（更新UI）
  • 不能跨域加载JS
  • 兼容性问题: 不是每个浏览器都支持这个新特性

DOM#

[TOC]

一、是什么#

Browser Object Model 浏览器对象模型。关于客户端浏览器的管理。

Document Object Model 文档对象模型。是W3C制订的一套技术规范，用来描述JS脚本如何与HTML或XML文档进行交互的Web标准。加载HTML页面时，Web 浏览器生成一个树型结构，用来表示页面内部结构。DOM将这种树型结构理解为由节点组成的DOM树。

DOM规定了一系列标准接口（函数），允许开发人员通过标准方式访问文档结构、操作网页内容、控制样式和行为等。

1、DOM 版本#

DOM0和DOM2。

2、document 对象#

document 对象 是window[1]对象的一个属性，因此可以将document对象作为一个全局对象来访问。当浏览器载入html文档, 它就会成为document对象。

二、节点访问#

在网页中所有对象和内容都被称为节点（Node）。如文档、元素、文本、属性、注释等。节点是DOM最基本的单元，并派生出不同类型的节点，它们共同构成了文档的树形结构模型。

1、节点#

常用节点种类#

  • document 文档节点

节点关系#

DOM把文档视为树形结构，称为节点树。节点之间的关系包括：上下父子关系，相邻兄弟关系。

  • 在节点树中，最顶端节点为根节点。除了根节点之外，每个节点都有一个父节点。
  • 节点可以包含任何数量的子节点。
  • 叶子是没有子节点的节点。
  • 同级节点是拥有相同父节点的节点。

childNodes#

childNodes 属性获取元素所有的子节点，保存到一个NodeList对象中。

方法比较#

getElementsByXX 获取的元素集合是 动态 的，也就是DOM节点改变时，当前获取的集合内容也会改变。（选出来的元素会随着文档的改变而改变）

selector API 是 静态 的集合。（只要取出来，就和页面有没有任何关系；未来元素改变，不影响获取的结果。）

NodeList是静态的，HTMLCollection是动态的

三、节点操作#

节点关系都是只读的，DOM提供了一些操作节点的方法。

创建节点 createElement()#

使用document对象的createElement方法能够根据参数指定的标签名创建一个新的元素。并返回对新元素的引用。

使用creatElement方法创建的新元素不会被自动添加到文档里，需要使用appendChild等方法。

创建文本节点 createTextNode()#

使用document对象的createTextNode方法可创建文本节点。创建的文本节点需要使用appendChild等方法才能插入到元素节点中，当然也可以使用 innerHTML 方法给元素节点添加内容。

插入节点#

都是插入元素的父元素调用的方法。

（1）appendChild()

appendChild()方法可以向当前节点的字节点列表的==末尾==添加新的节点。如果文档树中已经存在参数节点，则将从文档树中删除，然后重新插入新的位置。

（2）insertBefore( )

使用 insertBefore(newChild,oldChild) 方法可以在已有的子节点前插入一个新的子节点。

newChild表示新插入的节点，oldChild用于指定插入节点的后边的相邻位置。插入成功以后，该方法返回新插入的节点。insertBefore可以把指定元素及其所包含的所有子节点都一起插入到指定位置中。同时会先删除移动的元素，再重新插入。

appendChild 和 insertBefore 可以操作已经存在的元素。

复制节点 cloneNode()#

cloneNode() 方法可以创建一个节点的副本[4]。复制后的新节点，也不会被自动插入到文档，需要用到之前的方法去插入。

删除节点 removeChild()#

removeChild() 方法可以从子节点列表中删除某个节点，该节点所包含的所有子节点将同时被删除。

替换节点 replaceChild()#

replaceChild(new, old) 方法可以将某个子节点替换为另一个。替换节点替换的是所有子节点以及包含的所有内容。

4、元素内容操作#

innerHTML、innerText、outerHTML、outerText#

（1）获取标签

innerHTML 和 innerText 前者可以解析并输出标签，后者是纯文本。

innerHTML 和 outerHTML 前者不包括自身，后者包括自身。

（2）设置标签（直接覆盖原来的内容）

innerText 和 outerText 在获取的时候是相同效果，但在设置时，innerText仅设置标签所包含的文本，而 outerText 设置包含包括标签自身在内的文本。

textContent#

textContent 设置或者返回固定节点的文本内容。

（1）textContent和innerHTML的区别

  • textContent 会获取 style= “display:none” 中的文本，而innerText不会。
  • innerText 不会理会html格式，直接输出不换行的文本，textContent会根据标签里面的元素独立一行。
  • innerText 对IE的兼容性较好 ，textContent虽然作为标准方法但是只支持IE8+以上的浏览器 。

5、属性节点操作#

create | get | set | remove

创建属性节点#

createAttribute 方法可以创建属性节点。

  • 使用属性节点的name属性，可以得到属性的名称。
  • 使用属性节点的value属性，可以设置和属性的值。
  • 创建的属性可以通过setAttributeNode来插入到元素中。

读取属性值#

（1） 常用点语法通过元素直接访问HTML属性，如src、href等[5][6]。

（2） getAttribute 方法可以读取指定属性的值（非自有属性）。

设置属性值#

setAttribute(name,value) 方法可以设置元素的属性值。

参数name和value分别代表属性名称和属性值，并且必须是字符串，如果存在相同属性，则值被刷新，如果不存在则创建属性并添加。

通过 ele.value 获取或设置表单的值（string）。

删除属性#

removeAttribute() 使用元素的方法可以删除指定的属性

参数是要删除的属性名。

自定义属性 H5#

.dateset 设置和获取自定义属性

6、文档片段节点#

DocumentFragment 是一个==虚拟的==节点类型，仅仅存在于内存中，没有添加到文档树中，看不到渲染效果。当文档片段设计完善后，再使用JS一次性添加到文档树中显示出来，提高效率，==减少页面重绘的次数==。避免浏览器渲染和资源占用，解决大量添加节点时候的性能问题。

createDocumentFragment() 创建虚拟节点，使用 appendChild 等方法插入。

三、脚本化CSS#

1、获取尺寸#

offsetWidth和offsetHeight 可以获取元素的border-box尺寸。

clientWidth和clientHeight 可以获取元素的padding-box尺寸。

scrollWidth和scrollHeight 可以获取元素完全内容尺寸和padding。（几乎不用）

innerWidth和innerHeight 可以获取浏览器窗口的尺寸。

outerWidth和outerHeight 可以获取取加上工具条与滚动条窗口的尺寸。

style.width/height 只能获取行内样式的尺寸（字符串，“10px”）。

常见应用: 大盒子里面小盒子，大盒子用clientX，小盒子用offsetX。

2、获取位置#

getBoundingClientRect() 返回一个对象，对象中包含元素距离视口的距离信息。

getBoundingClientRect。

获取某个元素相对于视口的位置集合

  • top 元素上边到视窗上的距离
  • right 元素右边到视窗左的距离
  • bottom 元素下边到视窗上的距离
  • left 元素左边到视窗左的距离
  • width
  • height

offsetParent

JS 默认把它当成绝对定位属性来寻找包含块，不管是什么属性，都是寻找最近定位父级，如果没找到就是 HTML。

clientLeft / Top
