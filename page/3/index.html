sqlite#

with Python

1、引用

import sqlite3

2、连接或者创建数据库，如果数据库不存在将在工作目录创建数据库

conn = sqlite3.connect('数据库名称.db')

3、获取游标

c = conn.cursor()

4、新建表

c.execute('''
       CREATE TABLE 表名(
       ID INT PRIMARY KEY NOT NULL,
       NAME TEXT NOT NULL,
       AGE INT NOT NULL,
       ADDRESS CHAR(50),
       SALARY REAL
       );
''')

5、插入操作

c.execute("INSERT INTO 表名 (key1,key2) VALUES(value1,value2);")

6、查询操作

7、更新操作

8、删除操作

9、提交事务

Python 的 SQLite 数据库 API 默认是开启了事务的，因此需提交事务；否则程序对数据库所做的修改（包括插入数据、修改数据、删除数据）不会生效。

conn.commit()

10、关闭游标和连接

Python Random#

随机数库

1、引入

import random

2、生成一个指定范围内的随机浮点数

random.uniform(n1,n2)

3、生成一个指定范围内的整数

random.randint(n1,n2)

4、从指定范围内，按指定的基数递增的集合中获取一个随机数

random.randrange([start],stop[,step])

5、从序列中获取一个随机元素

random.choice(序列)

6、打乱列表元素

random.shuffle(x[,random])

7、从序列中随机获取指定长度的片段

random.sample(n1,n2)

Anaconda#

一、安装和配置环境变量#

1、参考：Anaconda详细安装及使用教程
2、查看版本 conda --version
3、升级工具包 conda upgrade --all

二、管理虚拟环境#

1、创建虚拟环境#

创建一个名称为test的虚拟环境并指定 Python 版本为3.7。

conda  create -n test python=3.7

2、切换环境#

切换到base环境

activate

切换到名叫test的环境。

activate test

查看所有环境

conda env list

3、卸载环境#

卸载名叫test的环境。

conda remove --name test --all

4、安装和卸载第三方包#

安装第三方名叫test的包。

conda install test

或者

pip install test

卸载第三方名叫test的包。

conda remove test

或者

pip uninstall test

查看所有包列表

conda list

三、问题#

1、解决Win10 PowerShell无法激活Anaconda环境的问题

2、更改PowerShell执行策略

1. 以管理员身份运行PowerShell;
2. 执行：get-ExecutionPolicy，显示Restricted，表示状态是禁止的;
3. 执行：set-ExecutionPolicy RemoteSigned;
4. 这时再执行get-ExecutionPolicy，就显示RemoteSigned;

Express#

Express 是一个保持最小规模的灵活的 Node.js Web 应用程序开发框架，为 Web 和移动应用程序提供一组强大的功能。

1、安装#

安装模块

npm install express --save

创建一个 express 项目

express 项目名字

安装 express 的中间件 express-static

npm install express-static --save

安装 express 项目生成器

npm install express-generator -g

2、开启服务器#

const express = require("express");
const app = express();
        //port
app.listen(80, function(){
    ...
});

http 默认 80 端口，https 默认 443 端口。

3、设置静态资源目录#

app.use(express.static(path.resolve(__dirname, "../xxx")));

！明确到一个文件夹。

！可以指定多个。

4、设置路由#

app.all
app.get( URL, (req, res)=>{})
app.post( URL, (req, res)=>{})

如果指定多个，当遇到访问方法和地址相同时，遇到第一个，默认不会向下执行 – 只会匹配第一个

只能响应一次。

也可以用 next() 继续向下查找是否有满足条件的路由。

第二次响应没有意义。

// 允许所有站点跨域
app.all("*", (req,res)=>{
    res.header("Access-Control-Allow-Origin"),"*");
    next();
})

5、接收数据#

body-parser#

接收 post 方式传来的数据。

const bodyParser = require("body-parser");
app.user(bodyParser.json());

const { context } = req.body;

query / params#

接收 get 方式传来的数据。

// "?age=123"
const age = req.query.age / 1;

RESTful API#

REST，Representational State Transfer，表现层状态转移的缩写，它是由罗伊·菲尔丁（Roy Fielding）提出的，是用来描述创建 HTTP API 的标准方法的，他发现这四种常用的行为（查看（view），创建（create），编辑（edit）和删除（delete））都可以直接映射到 HTTP 中已实现的 GET，POST，PUT 和 DELETE 方法。

操作#

查询 - get

（1）get books

../sword

（2）get id 为 1

../sword/1
../sword?id=1

（3）单条件查找

../sword?name=铁剑

（4）多条件查找

../sword?name=铁剑&atk=10

（5）全局模糊搜索

../sword?q=剑

（6）分页

../sword?_limit=1&_page=1

（7）排序（asc/desc）

../sword?sort=id&_order=desc

添加数据 - post

postman 中测试，设置主体 x-www-form-urlencoded

删除数据 - delete

postman 中测试

../sword/6

整体更新数据 - put

postman 中测试，设置主体 x-www-form-urlencoded

局部更新数据 - fatch

postman 中测试，设置主体 x-www-form-urlencoded

静态资源服务器#

// 引入相关模块
const http = require("http");
const fs = require("fs");
const path = require("path");

// 手动封装 Promise，解决回调问题
function readFile(filePath) {
  return new Promise((resolve, reject) => {
    fs.readFile(filePath, (err, data) => {
      if (err) {
        reject(err);
        return;
      }
      const fileContent = data.toString();
      resolve(fileContent);
    });
  });
}

// 创建一个服务器
const server = http.createServer(async (req, res) => {
  try {
    // 获取读取文件的路径
    const filePath = path.resolve(__dirname, "./server.js");
    const data = await readFile(filePath);
    // 设置响应头类型
    res.setHeader("Content-Type", "application/javascript;charset=utf-8");
    // 设置响应状态码
    res.statusCode = 200;
    // 响应内容
    res.end(data);
  } catch (err) {
    console.log("读取文件失败", err);
    res.setHeader("Content-Type", "application/javascript;charset=utf-8");
    res.statusCode = 404;
    res.end("文件资源加载失败");
  }
});

// 配置服务器
const port = "8080";
const host = "127.0.0.1";

// 启动服务器
server.listen(port, host, (err) => {
  if (err) {
    console.log("服务器创建失败" + err);
    return;
  }
  const url = `http://${host}:${port}`;
  console.log(`服务器已启动，请访问${url}`);
});
