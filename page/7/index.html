浮动#

浮动（float）可以让多个块元素横向排列，共行。

属性 说明
none 默认，不浮动
left 左浮动
right 右浮动

1、概念#

正常页面流/正常文档流 是行元素从左至右，块元素从上到下布局。浮动 是破坏流（脱离文档流）的一种方式。在目前开发中，很多的元素都是一个个 float 堆叠起来，但是这种布局很容易崩溃。只要宽度和高度发生一点变化，页面就会发生错乱。

2、浮动的使用#

左浮动 就是让容器沿着容器左侧排序。

右浮动 让容器沿着容器右侧排序。

3、高度塌陷#

浮动元素以后，脱离父级内容区域，父级没有内容撑开自身的高度，而且父级的兄弟元素是按照父级的位置进行布局的，所以页面会乱。

4、清除浮动 常用#

清除浮动不是不让元素浮动，是清除浮动对父级高度塌陷的影响。

清除浮动的方法

  • 给浮动元素的父级设置高度。
  • 给浮动元素的父级设置浮动，以浮制浮。
  • overflow 设置不为 visible 的属性。
  • 在最后一个浮动元素的后边添加一个空的块级元素，添加一个块级元素，添加属性和值为 clear: both。
  • after 伪元素清浮动，给所有浮动元素父级一个 clearFix 的类名。推荐
.clearFix:after {
  content: "/200B";
  display: block;
  height: 0;
  clear: both;
}

CSS BFC#

BFC，块级格式化上下文。 是一个独立的渲染区域，让处于 BFC 内部的元素与外部的元素相互隔离，使内外元素的定位不会相互影响。

1、创建 BFC#

  • 根元素
  • 浮动 float: right / left
  • 定位（仅 absolute 和 fixed）
  • 行内块元素和表格元素 display: inline-block / table
  • overflow 的值不为 visible

2、范围#

A block formatting context contains everything inside of the element creating it, that is not also inside a descendant element that creates a new block formatting context.

直译过来就是, BFC 包含创建它的元素的所有子元素, 但不包括创建了新 BFC 的子元素的内部元素。简单来说，子元素如果又创建了一个新的 BFC，那么它里面的内容就不属于上一个 BFC 了，这体现了 BFC 隔离 的思想。
也就是所说，一个元素不能同时存在于两个 BFC 中。

3、规则#

BFC 除了会创建一个隔离的空间外，还具有以下特性：

  • BFC 内部的块级盒会在垂直方向上一个接一个排列。
  • 同一 BFC 下的相邻块级元素可能发生外边距折叠，创建新的 BFC 可以避免外边距折叠。（解决塌陷）
  • 浮动盒的区域不会和 BFC 重叠 （BFC 的区域不会与 float 的元素区域重叠）
  • 计算 BFC 的高度时，浮动元素也会参与计算
  • 文字层不会被浮动层覆盖，环绕于周围

4、应用#

  • 自适应多栏布局
    中间栏创建 BFC，左右栏宽度固定后浮动。由于盒子的 margin box 的左边和包含块 border box 的左边相接触，同时浮动盒的区域不会和 BFC 重叠，所以中间栏的宽度会自适应。

  • 防止外边距折叠
    创建新的 BFC ，让相邻的块级盒位于不同 BFC 下可以防止外边距折叠

  • 清除浮动
    BFC 内部的浮动元素也会参与高度计算，可以清除 BFC 内部的浮动

    清除浮动的原理是两个div都位于同一个 BFC 区域之中

一、显示#

1、display - 控制元素的显示模式 常#

控制元素的显示状态。

display 描述说明
inline 让元素以行属性显示
block 让元素以块属性显示
inline-block 让元素以行内块属性显示
none 带着所有子元素隐藏，直接消失
flex 开启弹性布局

display:none 隐藏的元素，不可以被 js 获取 DOM 节点及相关的样式属性。让隐藏的元素显示，只需要修改 display 属性的值不为 none 即可。

2、visibility - 控制元素显示或者隐藏#

visible: 元素显示
hidden: 元素隐藏
元素依然占用空间，和隐身一样，带着子元素一起隐身了。
子元素的隐藏是因为继承了父级的 hidden 值，所以单独给子元素设置，覆盖 hidden，即可显示。
使用 visiblity 隐藏的元素是可以用 js 获取的 DOM 节点及相应的宽高等属性。

visbility:hidden 和 display:none 隐藏的区别

①visibility 可以继承，子元素是因为继承了才隐藏（可覆盖）。display 不能被继承，直接带着所有内置元素直接隐藏。
②visbility 隐藏，保留原先占用空间。display 隐藏使原来的空间被占用。
③js 可以获取到 visibility 隐藏元素的可视化宽高，而获取不到 display 隐藏元素的可视化宽高。

3、overflow - 溢出设置 常#

内容溢出设置。

overflow 描述
visible 默认值
超出正常显示，但是不会影响其他元素的位置
hidden 超出部分隐藏
scroll 无论是否超出都生成滚动条
auto 自动生成滚动条，当内容溢出生成滚动条，否则不生成

overflow-x 和 overflow-y 可以单独控制一个方向的内容溢出设置。

隐藏横向滚动条：overflow-x: hidden

4、opacity - 透明度设置#

控制元素不透明度
值是 1-0
1 是完全不透明
0 是完全透明
opacity 直接控制所有的内部元素的不透明度，无论是什么元素都会改变

08、line-height - 行高控制 常用#

1、样式操作#

样式重置（CSS reset），将 html 的默认样式全部去掉，需要的时候我们自行添加。统一页面风格。

为什么要用样式重置

  • 多数元素拥有自己的默认样式，并且我们很多都不需要。
  • 默认样式在不同浏览器中呈现的也不一定相同，就会导致浏览器展示页面不相同的现象。
  • 整个页面中固定的风格代码，可以在样式重置中直接书写，不用每一个都设置。

为什么不用通配符重置

  • 重置简单，不够完整，一些 demo 可以使用，真正项目不推荐使用，请使用 CSS reset
  • 通配符，匹配所有的标签，影响很大，效率很低。

CSS 盒模型#

在 HTML 中把每个元素当成一个盒子，拥有盒子的平面外形和空间。

1、盒模型标准#

通过 box-sizing 设置盒子模型的标准，分为

  • content-box 默认，W3C 标准盒模型
  • border-box IE 盒模型，也称为怪异盒模型

注，理论上还有 padding-box 和 margin-box ，但是现在 W3C 和 MDN 规范均不支持。

怪异盒模型的 width / height 是 border 的 尺寸。

2、内容层 content#

width/height#

width 设置的坑

  • 如果设置宽度为 100%，并且有 border margin padding 等属性值的时候，宽度可能超出。因为 width 内容区域已经和父级一样了，所以可以不设置宽度或者设置宽度为 auto 那么内容的宽度就是可以充分利用可使用空间。

height 设置的坑

  • 如果高度设置百分比，必须父级有一个显性的设置一个 height 属性，并且不能是 auto。

min/max-width/height#

正常 最小 最大
宽度 width min-width max-width
高度 height min-height max-height

当 min-width 和 max-width 冲突的时候，min-width 是优先的。当 width 和 min 或 max 冲突以后，min 或者 max 直接无视 width 的存在（无论是否添加 ！important）。

3、内边距层 padding#

padding 的写法和 margin 一致

padding 主要撑开内容和边框之间的距离

padding 不支持负值

padding 对所有属性的元素都支持，但是对于行内元素撑不开上下的距离

padding 可以用于扩大点击范围

4、边框层 border#

边框属性#

border

  • border-width 线的宽度
  • border-style 线的类型
    • solid 实线
    • dashed 虚线
    • dotted 点状线
  • border-color 线的颜色
  • border:none 不设置边框

边框还分为 4 个方向 要么直接设置 4 个方向，要么一个个方向单独设置

练习：边框画三角形

.box {
  width: 0;
  height: 0;
  border: 100px solid #000;
  border-left-color: transparent;
  border-right-color: transparent;
  border-bottom-color: transparent;
}

50% 八个值一样

5、外边距层 margin#

margin 的作用是撑开当前元素和其他元素的距离。margin 有四个方向，可以分别设置margin-left margin-right margin-top margin-bottom，margin 的 left 和 top 是让当前元素向右向下移动，margin 的 right 和 bottom 是让元素的兄弟元素向右向下移动，不影响自身。

合写#

  • 4 个值：上 右 下 左（从上开始顺时针）
  • 3 个值：上 左右 下
  • 2 个值：上下 左右
  • 1 个值：上下左右

水平方向上的 auto#

因为块标签拥有满行属性，但是设置宽度以后，剩余的空间就是默认的 margin，设置左右的 margin 值为 auto，那么剩余的 margin 就会平分到左右两边。

竖直方向的 auto#

相对于水平方向来说，块元素在垂直方向上并没有满屏的属性，margin 默认在上下的值都是 0，并没有剩余空间去平分。

塌陷和边框合并问题#

  • 父级塌陷：父级的第一个子元素的上 margin 和最后一个子元素的下 margin 会进行塌陷给父级，这样的设计是为了方便父级和他的兄弟进行垂直 margin 的合并，避免缝隙过大。
  • 解决塌陷：给父级开启 BFC，给父级设置边框。
  • 兄弟的 margin 垂直方向会进行折叠，取最大值 。

margin 负值#

  • margin-top 为负值：元素向上走，不会影响上边的元素的位置（会出现重叠），并且下边的兄弟元素会跟着一起向上走。
  • margin-left 为负值：元素向左走，不会影响前边的元素位置(会出现重叠)，右边兄弟元素会跟随移动。
  • margin-right 为负值：元素不会移动，形态大小也不会发生改变，但是所占用的位置会减少（负 margin 的值），所以，后边的兄弟元素会自动向左走。
  • margin-bottom 为负值：元素不会移动，形态大小也不会发生改变，但是所占用的位置会减少（负 margin 的值），所以，后边的兄弟元素会自动向上走。

支持性#

margin 完美支持块元素和行内块元素，不支持行元素的垂直方向。

CSS 开始#

CSS，即 Cascading Style Sheets，层叠样式表。

具有：

  • 像素级精确控制。
  • 将内容和表现形式分离。
  • 层叠特性。

选择器        属性值  可以写多个

二、使用方式#

1. 内联式#

一般不推荐使用。

特点：

  • 优先级高
  • 臃肿，代码冗余（不能复用）
  • 结构和样式没有分离，不方便维护

常用。

  ...
  <style>
    div {
      width: 10px;
      height: 10px;
    }

特点：

  • 方便维护和修改
  • 样式和结构分离

3. 外部引入#

（1）link 方式

<head>
  ...
  <link rel="stylesheet" type="text/css" href="index.css" />
</head>

（2）@import 方式

不推荐。

（3）@import 和 link 方式引入的区别：

  • link 功能较多，比如可以定义 RSS，而 @import 只能用于加载 CSS
  • 当解析到 link 时，页面会同步加载所引用的 CSS，而 @import 所引用的 CSS 会等页面加载完才被加载
  • link 可以用 JS 动态加载 CSS，而 @import 不允许
  • @import 在 IE5 以上才能使用

Hexo 内嵌脑图

发表于 2021-01-28 更新于 2021-02-01 分类于 1818：Hexo

Hexo 内嵌脑图#

工具:

  • MindManager 2020
  • Hexo

第一步: MindManager 导出，选择 HTML5 交互式导图，选择局部导出
第二步: 在 Hexo 博客目录的 source 目录下新建一个 map 文件夹（也可以是其他名字）
第三步: 编辑 Hexo 博客目录下的 _config.yml ，搜索 skip_render，在下面添加 "map/**"，保存。
第四步: hexo clean
第五步: 在你想插入导图的页面:

<iframe src="../map/1.html" width="1000" height="1000" frameborder="0"></iframe>

Hexo#

https://hexo.io/

一、开始#

1、安装#

npm install hexo-cli -g

2、新建博客#

hexo init 博客名称
cd 博客名称
npm i

3、常用命令#

# 启动博客（本地服务器）打开 localhost:4000 查看
hexo s

# 清除缓存
hexo clean

# 构建
hexo g

# 把代码推送到远程仓库
hexo d

4、博客配置#

博客根目录下的 _config 为 博客配置文件
主题目录下的 _config 为 主题配置文件

5、主题配置#

主题库

这边推荐 NEXT 主题。

安装NEXT主题#

git clone https://github.com/theme-next/hexo-theme-next themes/next
cd themes/next
npm i

修改 博客根目录下 _config 文件 里面的主题为 next

6、Github Page / Gitee Page#

npm install hexo-deployer-git --save
# _config.yml

# git 配置
deploy:
  type: git
  repo: https// .... .git
  branch: master

推上去

hexo d

二、写作#

1、YAML Front Matter#

---
title: 这是标题

## 这是创建时间、更新时间
date: 2020-10-01
updated: 2020-12-21

## 这是标签
tags: 标签
tags: [标签一, 标签二]


---

三、插件#

插件库

目录生成#

npm install hexo-toc --save
# _config.yml

toc:
  # 目录深度，最多生成三级目录
  maxDepth: 3

搜索#

npm install hexo-generator-search --save

代码高亮#

npm i -S hexo-prism-plugin

然后，修改 Hexo 根目录下 _config.yml 文件中 highlight.enable 的值为 false，并新增 prism 插件相关的配置，主要配置如下：

prism_plugin:
  mode: 'preprocess'    # realtime/preprocess
  theme: 'tomorrow'     #主题选项
  line_number: false    # default false
  custom_css:

可用的主题有

coy,dark,funky,okaidia,solarizedlight,tomorrow,twilight,a11y-dark,atom-dark,base16-ateliersulphurpool.light,cb,darcula,dracula,duotone-dark,duotone-earth,duotone-forest,duotone-light,duotone-sea,duotone-space,ghcolors,hopscotch,material-dark,material-light,material-oceanic,pojoaque,shades-of-purple,synthwave84,vs,xonokai,default,
