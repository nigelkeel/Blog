一、基本结构#

<!-- 版本声明 -->
<!doctype html>
<!-- 根文档 -->
<html lang="en">
    <!-- 头部 -->
    <head>
    		<meta charset="utf-8">
    		<title>标题</title>
    </head>
    <!-- 主体 -->
    <body>
	</body>
</html>

1、注释#

用<!-- 和-->包裹起来的内容。

2、版本声明#

给浏览器声明，告诉浏览器应该按照HTML5的规范解析当前页面。 如果不写，浏览器就是按照自己默认的模式解析。

版本声明必须写在第一行。我们不把他看做一个html标签，而是把它当做一个指令，并且这个指令对大小写不敏感。

3、html 、head、body#

html 是一个双标签，所有网页内容全部书写在html标签中。lang就是html标签的属性，en就是lang属性的值，属性是对当前标签的信息的扩展。

head 标签代表网页的头部，不会显示在网页中。主要是包含了标题，网页的描述，字符编码等等一些信息。

body 标签是网页显示的主体内容。

4、meta 标签#

控制网页的元信息（信息的信息），根据属性的不同，有很多的作用。

5、解决网页乱码问题 △#

<meta charset="utf-8">

charset即字符编码。utf-8称作为万国码。

二、HTML 标签#

1、标签的语义化 △#

  • 语义化标签更具有 可读性 ，便于团队开发和维护。
  • 在没有加载CSS的情况下，网页也能很好呈现出内容结构和代码结构。
  • 对于SEO来说，搜索引擎更能理解网页中各部分之间的关系，更准确更快速搜索信息。

2、标签的关系#

  • 并列：兄弟关系
  • 嵌套：父子关系（使用TAB缩进）

3、标签的分类#

3.1 块标签#

特性

  • 独占一行
  • 可设宽高、内外边距和边框
  • 可嵌套块元素和行元素（p、h标签除外）

将非块级元素转换成块级元素的方法

  • display: block
  • float: left/right
  • position: absolute/fixed
  • 父级元素使用display:flex

3.2 行标签#

特性

  • 空间足够时可以其他行标签共用一行
  • 设置宽高无效，内外边距和边框的上下设置无效
  • 宽高用内容撑开
  • 可嵌套行元素
  • 受父元素的 line-height 以及自身的 vertical-align影响。

3.3 行内块（inline-block）#

特性

  • 一行可放多个。
  • 宽高、内外边距、边框无法设置。
  • 受父元素的 line-height 以及自身的 vertical-align影响。

12、锚链接#

（1）路径

  • 网络路径：一般有三种协议，一种是http、一种是file、一种是ftp。需要引入网络路径的时候，需要书写http://。
  • 相对路径
  • 绝对路径

（2）标签属性

  • href：路径。
  • target：跳转方式，_self 当前标签页跳转，_blank新标签页跳转。
  • title：鼠标悬浮时的弹窗解释。

（3）锚链接

href="#(id)" → id="(id)"

  • 标签href为空时，点击刷新加载。当设置为##（两个#及以上）或者javascript:;时，点击之后不会有动作
  • 如果a标签的href值只有一个 # 那么可以实现回到顶部的功能。
  • 如果a标签的href属性书写的链接可以使用浏览器打开（比如图片），则浏览器跳转到这个位置并打开，如果a标签的href属性书写的链接不可以使用浏览器打开，则浏览器直接下载。
  • a标签的下载功能：download属性 书写下载内容，可以直接实现下载，必须拥有href属性，否则下载功能失效。

a标签的伪类 △#

伪类 功能
link 当有链接属性时
visited 链接被访问过
hover 鼠标悬浮在链接上面时
active 链接激活时

①“love-hate 爱恨法则” —— 顺序不能互换 ：四个选择器优先级一样，顺序改变可能会覆盖前边的，比如visited写在最后，那么当连接被访问过后，访问过后的颜色就会覆盖hover和active。
②hover适用于多数其他元素，并且伪元素也可以使用。

.a1:link{
	color: red;
}
.a2:visited{
	color: yellowgreen;
}
.a3:hover{
	color: pink;
	font-size: 30px;
}
.a4:active{
	color: blueviolet;
	font-weight: bold;
}

2、伪元素#

伪元素 会在内容元素的前后插入额外的元素，但是并不会在文档中生成，在文档的源码中也找不到。虽然在结构上是虚假元素，但是在表现上和普通元素没有什么区别，能为它们添加任何样式，比如改变文字颜色等等。

一、CSS Sprites#

CSS Sprites，精灵图，是网页图片的常用处理方式。

允许将一个页面中很多零星的小图片包含到一张大图中去，当访问页面的时候，就不会一张张的去请求显示图片。对网络来说，小于200k的文件加载速度是一样的。通过 background-position 属性，将图片定位到需要的位置即可。

优点：

  • 减少图片大小
  • 减少服务器请求次数

定位辅助工具

二、CSS hack#

CSS hack是什么

由于不同的厂商的浏览器，或者是同一个浏览器不同的版本（IE），对css的解析和认识不完全一样,可能会导致不同浏览器显示的效果不相同,那么我们需要针对某个浏览器，去写不同的样式，让代码能够兼容所有的浏览器。

为什么要用 CSS hack

第一种理解：让我们CSS的代码兼容不同的浏览器

第二种理解：我们可以为不同的浏览器定制不同的样式

function Tab(btn, content) {
    //把变量或者参数赋值给实例化对象，这样在构造函数和原型对象的方法中都能够访问这个变量
    this.btns = btn;
    this.contents = content;
    //在其他函数中想要使用指向实例化对象的this比较困难，所以用一个that变量保存一个this,方便以后在其他地方使用
    var that = this;
    //对所有的按钮绑定点击事件
    for (var i = 0; i < this.btns.length; i++) {
        //给所有的元素扩展一个下标index属性
        this.btns[i].index = i;
        this.btns[i].onclick = function () {
            //因为所有的Tab切换的核心方法都是一样的，所以我们把方法写在了原型对象上
            //当点击按钮的时候，去执行原型对象上的方法
            //这样所有的Tab切换的执行切换的方法都是同一个，节省效率
            //把当前的this（指向点击的对象）传参传递给方法，方便使用
            that.click(this);
        }
    }
}
Tab.prototype.click = function (that) {
    //使用that来接收参数 参数代表的是点击事件的事件对象
    //先清空所有的内容

        this.btns[i].classList.remove("active");
        this.contents[i].classList.remove("active");
    }
    //给当前点击的元素及对应的面板添加类名
    that.classList.add("active");
    this.contents[that.index].classList.add("active");
}

BOM，Browser Object Model，浏览器对象模型。

一、window对象#

window 是BOM的核心对象。表示浏览器的一个实例，是客户端浏览器对象模型的基类。在浏览器中，该对象具有双重角色，既是通过JS访问浏览器窗口的一个接口，也是ES规定中的 Global 对象。这意味着在网页中定义的任何一个对象、变量和函数，都以 window 作为其 Global 对象。所有在全局作用域中声明的变量、函数都会变成该对象的属性和方法。

1、客户端对象#

可以访问客户端其他对象，这种关系构成浏览器对象模型。

对象 描述
document 包含整个HTML文档，可以用来访问文档内容及其所有页面元素。
navigator 包含客户端有关浏览器的信息。
screen 包含客户端屏幕的信息
history 包含浏览历史记录信息
location 包含当前网页文档的URL信息

2、系统对话框#

浏览器通过 alert()、confirm() 和 prompt() 方法可以调用系统对话框向用户显示消息。

系统对话框与在浏览器中显示的网页没有关系，也不包含 HTML。它们的外观由操作系统及（或）浏览器设置决定，而不是由 CSS 决定（无法设置）。通过这几个方法打开的对话框都是同步和模态的。也就是说，显示这些对话框的时候代码会停止执行，而关掉这些对话框后代码又会恢复执行。

confirm和prompt有返回值。生产环境下不用，调试使用。

3、打开和关闭窗口#

window.open() 方法既可以导航到一个特定的 URL，也可以打开一个新的浏览器窗口。

参数

  • 要加载的URL
  • 新窗口打开方式：_self _blank（默认新标签页打开）
  • 窗口大小、位置。

返回值 就是新建的窗口名称。

窗口名.close()可以关闭窗口，如果关闭自身那就使用 window.close()。

// 1. 打开窗口
var newWin = window.open("http://www.jd.com","_blank","width=300,height=300,left=200,top=100")

// 2. 关闭窗口
newWin.close();

close() 方法仅适用于通过 window.open() 打开的弹出窗口。

4、浏览器宽高#

window对象有innerWidth和innerHeight属性，可以获取浏览器窗口的内部宽度和高度。内部宽高是指除去菜单栏、工具栏、边框等占位元素后，用于显示网页的净宽高。

对应的，还有一个outerWidth和outerHeight属性，可以获取浏览器窗口的整个宽高。

二、navigator对象#

navigator 对象储存了浏览器信息。通过 navigator 可以引用该对象，并利用它的属性来读取客户端基本信息。

常见属性：

  • onLine 表示浏览器是否连接到因特网
  • platform 浏览器所在的操作系统
  • userAgent 浏览器识别码
  • appName 浏览器名称
  • labguage 浏览器设置的语言

1、浏览器检测方法#

检测浏览器类型的方法有多种，常用的方法包括两种:特征检测法和字符串检测法，这两种方法都存在各自的优点与缺点，用户可以根据需要进行选择。

（1）特征检测法

  • 特征检测法就是根据浏览器是否支持特定功能来决定相应操作的方式。这是一种非精确判断法。但却是最安全的检测方法
  • 因为准确检测浏览器的类型和型号是一件很困难的事情,而且很容易存在误差，如果不关心浏览器的身份，仅仅在意浏览器的执行能力，那么使用特征检测法就完全可以满足需要。
  • 当使用一个对象、方法或属性时，先判断它是否存在，如果存在，则说明浏览器支持该对象、方法属性，那么就可以放心使用

（2）字符串检测法

  • 客户端浏览器每次发送HTTP请求时，都会附带有一个user-agent(用户代理)字符串，对于Web开发人员来说，可以使用用户代理字符串检测浏览器类型。
  • userAgent字符串包含了web浏览器的大量信息，如浏览器的名称和版本。

2、操作系统检测方法#

navigator.userAgent返回值一般都会包含操作系统的基本信息，不过这些信息比较散乱，没有统一的规则。

用户可以检测一些更为通用的信息，如检测是否为 Windows系统，或者为 Macintosh系统，而不去分辨操作系统的版本号。

例如，如果仅检测通用信息，那么所有Windows版本的操作系统都会包含"Win”字符串，所有Macintosh版本的操作系统都包含有"Mac”字符串，所有Umix版本的操作系统都包含有"X11”，而 Linux操作系统会同时包含"X11”和" Linux。

var isWin = (navigator.userAgent.indexOf("Win") != - 1);
    // 如果是Windows系统，则返回true
var isMac = (navigator.userAgent.indexOf("Mac") != - 1); 
    // 如果是Macintosh系统，则返回true 
var isUnix = (navigator.userAgent.indexOf("X11") != - 1); 
    // 如果是UNIX系统，则返回true
var isLinux = (navigator.userAgent.indexOf("Linux") != - 1); 
    // 如果是Linux系统，则返回true

！请注意，navigator的信息可以很容易地被用户修改，所以JavaScript读取的值不一定是正确的。但这样既可能判断不准确，也很难维护代码。正确的方法是充分利用JavaScript对不存在属性返回undefined的特性，直接用短路运算符||计算：

var width = window.innerWidth || document.body.clientWidth;

一、概念#

Document Object Model， DOM，文档对象模型。是W3C制订的一套技术规范，用来描述JS脚本如何与HTML或XML文档进行交互的Web标准。加载HTML页面时，Web 浏览器生成一个树型结构，用来表示页面内部结构。DOM将这种树型结构理解为由节点组成的DOM树。

DOM规定了一系列标准接口（函数），允许开发人员通过标准方式访问文档结构、操作网页内容、控制样式和行为等。

1、DOM 版本#

DOM0和DOM2。

2、document 对象#

document 对象 是window对象的一个属性，因此可以将document对象作为一个全局对象来访问。当浏览器载入html文档, 它就会成为document对象。

JS 顶层对象是global，window是global的实现。

二、节点访问#

在网页中所有对象和内容都被称为节点（Node）。如文档、元素、文本、属性、注释等。节点是DOM最基本的单元，并派生出不同类型的节点，它们共同构成了文档的树形结构模型。

1、节点#

常用节点种类#

  • document 文档节点
  • documentFragment 文档片段节点
  • Element 元素节点
  • attr 属性节点
  • text 文本节点
  • comment 注释节点

节点关系#

DOM把文档视为树形结构，称为节点树。节点之间的关系包括：上下父子关系，相邻兄弟关系。

  • 在节点树中，最顶端节点为根节点。除了根节点之外，每个节点都有一个父节点。
  • 节点可以包含任何数量的子节点。
  • 叶子是没有子节点的节点。
  • 同级节点是拥有相同父节点的节点。

childNodes#

childNodes 属性获取元素所有的子节点，保存到一个NodeList对象中。

节点类型、名称、值#

2.2 selectors API#

直接使用CSS选择器作为参数传递。

核心方法 querySelector 和 querySelectorAll。后者获取的是一个NodeList对象。

// 获取单个节点（如果选择多个，默认选第一个）

// 获取多个节点（NodeList），使用的时候注意加下标
var oRed2 = document.querySelectorAll("#box>.red");

oRed2.forEach(item, index, array){
  	item.style.background = "red";
}

2.3 方法比较#

getElementsByXX 获取的元素集合是 动态 的，也就是DOM节点改变时，当前获取的集合内容也会改变。（选出来的元素会随着文档的改变而改变）

selector API 是 静态 的集合。（只要取出来，就和页面有没有任何关系；未来元素改变，不影响获取的结果。）

NodeList是静态的，HTMLCollection是动态的

三、节点操作#

节点关系都是只读的，DOM提供了一些操作节点的方法。

1、创建节点#

（1）createElement

document.createElement(tag)
/*
	参数：标签名字
	返回值：新元素的引用
*/

使用document对象的createElement方法能够根据参数指定的标签名创建一个新的元素。并返回对新元素的引用。

使用creatElement方法创建的新元素不会被自动添加到文档里，需要使用appendChild等方法。

（2）创建文本节点 createTextNode

document.createTextNode(String)
//                    参数是字符串

使用document对象的createTextNode方法可创建文本节点。创建的文本节点需要使用appendChild等方法才能插入到元素节点中，当然也可以使用 innerHTML 方法给元素节点添加内容。

插入节点#

都是插入元素的父元素调用的方法。

（1）appendChild()

appendChild()

appendChild()方法可以向当前节点的字节点列表的==末尾==添加新的节点。如果文档树中已经存在参数节点，则将从文档树中删除，然后重新插入新的位置。

（2）insertBefore( )

使用 insertBefore(newChild,oldChild) 方法可以在已有的子节点前插入一个新的子节点。

newChild表示新插入的节点，oldChild用于指定插入节点的后边的相邻位置。插入成功以后，该方法返回新插入的节点。insertBefore可以把指定元素及其所包含的所有子节点都一起插入到指定位置中。同时会先删除移动的元素，再重新插入。

appendChild 和 insertBefore 可以操作已经存在的元素。

复制节点 cloneNode()#

cloneNode()
//      参数 true 深复制 复制整个节点和里面的内容
//           false 浅复制 只复制节点（不要里面的内容）

cloneNode() 方法可以创建一个节点的副本[3]。复制后的新节点，也不会被自动插入到文档，需要用到之前的方法去插入。

删除节点 removeChild()#

removeChild()
// 返回值：
// - 删除成功 ==> 返回被删除的节点
// - 删除失败 ==> 返回 null

removeChild() 方法可以从子节点列表中删除某个节点，该节点所包含的所有子节点将同时被删除。

替换节点 replaceChild()#

replaceChild(newNode,oldNode)
// 参数
// - newNode 指定的新节点
// - oldNode 被替换的节点
// 返回值
// - 替换成功 ==> 返回替换的节点
// - 替换失败 ==>  null

replaceChild(new, old) 方法可以将某个子节点替换为另一个。替换节点替换的是所有子节点以及包含的所有内容。

4、元素内容操作#

innerHTML、innerText、outerHTML、outerText#

（1）获取标签

<p id="abc"><span>破晓</span>是猪</p>
<script>
oT = document.getElementById("abc");
console.log(oT.innerHTML) //<span>破晓</span>是猪
console.log(oT.innerText) //破晓是猪
console.log(oT.outerText) //破晓是猪
console.log(oT.outerHTML) //<p id="abc"><span>破晓</span>是猪</p>
</script>

innerHTML 和 innerText 前者可以解析并输出标签，后者是纯文本。

innerHTML 和 outerHTML 前者不包括自身，后者包括自身。

（2）设置标签（直接覆盖原来的内容）

innerText 和 outerText 在获取的时候是相同效果，但在设置时，innerText仅设置标签所包含的文本，而 outerText 设置包含包括标签自身在内的文本。

oTest.innerHTML = "这是内容"
oTest.innerHTML = "<a href = "##"> 这是内容 </a>"

textContent#

textContent 设置或者返回固定节点的文本内容。

document.getElementById("myList").textContent;
document.getElementById("myList").textContent = "你好"；

（1）textContent和innerHTML的区别

  • textContent 会获取 style= “display:none” 中的文本，而innerText不会。
  • innerText 不会理会html格式，直接输出不换行的文本，textContent会根据标签里面的元素独立一行。
  • innerText 对IE的兼容性较好 ，textContent虽然作为标准方法但是只支持IE8+以上的浏览器 。

（2）兼容性处理

function setOrGetContent(node,content) {
    if(arguments.length === 1){
        //代表当前是读取操作
        if(node.textContent){
            //只要能拿到这个dom对象的textContent属性值，代表当前用户是高级浏览器
            return node.textContent;
        }else {
            //代表拿不到  那就是低级浏览器
            return node.innerText;
        }
    }else if(arguments.length === 2){
        //代表写入操作
        if(node.textContent){
            //代表高级
            node.textContent = content;
        }else {
            //代表低级
            node.innerText = content;
        }
    }
}

6、文档片段节点#

DocumentFragment 是一个==虚拟的==节点类型，仅仅存在于内存中，没有添加到文档树中，看不到渲染效果。当文档片段设计完善后，再使用JS一次性添加到文档树中显示出来，提高效率，==减少页面重绘的次数==。避免浏览器渲染和资源占用，解决大量添加节点时候的性能问题。

createDocumentFragment() 创建虚拟节点，使用 appendChild 等方法插入。

三、脚本化CSS#

1、获取尺寸#

offsetWidth和offsetHeight 可以获取元素的border-box尺寸。

clientWidth和clientHeight 可以获取元素的padding-box尺寸。

scrollWidth和scrollHeight 可以获取元素完全内容尺寸和padding。（几乎不用）

常见应用: 大盒子里面小盒子，大盒子用clientX，小盒子用offsetX。

getBoundingClientRect。

获取某个元素相对于视口的位置集合

// 获取系统滚动条
function getDocScroll() {
        var h = document.documentElement;
        var b = document.body;
        var y = self.pageYOffset||
            h.scrollTop||
            b.scrollTop;
        var x = self.pageXOffset||
            h.scrollLeft||
            b.scrollLeft;

            y:y
        }
}

// 设置系统滚动条
function setDocScroll(x,y) {
    if(arguments.length === 1 && typeof x === "number"){
        window.scrollTo(x,0);
        document.documentElement.scrollLeft = x;
        document.body.scrollLeft = x;
    }
    if(arguments.length === 2 && typeof x === "number" && typeof y === "number"){
        window.scrollTo(x,y);
        document.documentElement.scrollLeft = x;
        document.documentElement.scrollTop = y;
        document.body.scrollLeft = x;
        document.body.scrollTop = y;
       }
}
