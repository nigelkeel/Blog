HTML 标签#

一、概述#

标签，又称为元素。

1、标签的关系#

兄弟关系
父子关系

2、标签的分类#

（1）块标签 block

  • 代表标签: div
  • 默认的宽度是占据父标签全部内容，即 width: 100%
  • 默认的高度由内容决定
  • 设宽高、边距、边框没有限制
  • 独占一行
  • 允许嵌套块元素和行元素（p/h标签除外）

（2）行内标签 inline

  • 代表标签: span
  • 默认的宽高由内容决定。
  • 内容过多时允许换行并断开（一般都是用于处理文字样式）。
  • 设置宽高无效。上下边距、边框无法设置。
  • 共用一行：在一行中如有剩余空间，允许同其他行内/行内块标签共用。
  • 某些动画样式不能添加和使用。

（3）行内块标签 inline-block

  • 代表标签: img、button、input
  • 默认的宽高由内容决定。
  • 内容过多时不会断开。
  • 可以设置宽高。
  • 在一行中如有剩余空间，允许同其他 行内/行内块 标签公用一行。
  • 行内块和内行标签会有行高的概念。
  • 行内块受父元素的 line-height 以及自身的 vertical-align 影响。

将非块级元素转换成块级元素的方法

  • display: block
  • float: left/right
  • position: absolute/fixed
  • 父级元素使用 display:flex

3、语义化标签#

优点：

  • 语义化标签更具有可读性，便于团队的开发和维护
  • 没有 CSS 的情况下，网页也能很好的呈现出内容结构和代码结构
  • 关于SEO，搜索引擎更能理解到网页中各部分之间的关系，更准确更快速搜索信息

布局的三个发展阶段：

  • 开荒时期：前端的代码主要是后台写的，使用 table 来布局
  • 美工时期：有专门的人来写前端代码了，主要布局方式是 DIV+CSS
  • 现在：使用具有语义的标签，可以表达正确的内容，也利用开发和维护。

1、div 分块#

用于分块。

1、ol/ul只能嵌套 li。
2、不建议在ul和li嵌套的中间，书写其他元素。

网页基础#

一、基本结构 基础#

<!-- 版本声明 -->
<!DOCTYPE html>
<!-- 根文档 -->
<html lang="en">
  <!-- 头部 -->
  <head>
    <meta charset="utf-8" />
    <title>标题</title>
  </head>
  <!-- 主体 -->
  <body></body>
</html>

1、注释#

用<!-- 和-->包裹起来。

2、文档版本声明#

<!DOCTYPE html>

给浏览器声明，告诉浏览器应该按照 HTML5 的规范解析当前页面。 如果不写，浏览器就是按照自己默认的模式解析。

版本声明必须写在第一行。我们不把他看做一个 html 标签，而是把它当做一个指令，并且这个指令对大小写不敏感。

<!-- html 4 的 版本说明(了解) -->
<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

3、html 、head、body 标签#

html 是一个双标签，所有网页内容全部书写在 html 标签中。lang就是 html 标签的属性，en就是lang属性的值，属性是对当前标签的信息的扩展。

head 标签代表网页的头部，不会显示在网页中。主要是包含了标题，网页的描述，字符编码等等一些信息。

body 标签是网页显示的主体内容。

4、meta 标签#

控制网页的元信息（信息的信息），根据属性的不同，有很多的作用。

5、解决网页乱码问题#

<meta charset="utf-8" />

charset即字符编码。utf-8称作为万国码。

Hexo 内嵌脑图#

工具:

  • MindManager 2020
  • Hexo

第一步: MindManager 导出，选择 HTML5 交互式导图，选择局部导出
第二步: 在 Hexo 博客目录的 source 目录下新建一个 map 文件夹（也可以是其他名字）
第三步: 编辑 Hexo 博客目录下的 _config.yml ，搜索 skip_render，在下面添加 "map/**"，保存。

第五步: 在你想插入导图的页面:

<iframe src="../map/1.html" width="1000" height="1000" frameborder="0"></iframe>

181003：Markdown 使用心得

Markdown 使用心得#

1、链接的文件名字带空格不识别怎么办？

解决: 在链接中用%20替代空格的书写

2、永久插入图片

首先将图片通过在线转码工具转换成base64的编码，并用以下格式嵌入即可，格式如下：

![image] (base64)

但是由于这base64编码往往都很长，很占篇幅，因此我们可以给图片编号，并将所有的图片base64编码放在文档的最后即可，格式如下：

  • 在插入图片的地方使用：![image] [图片编号]
  • 在文档最后使用：[图片编号]:base64编码

这样即可完成图片的永久插入！
