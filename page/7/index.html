函数柯里化#

柯里化（Currying），是一种函数的转换，它是指将一个函数从可调用的 f(a, b, c) 转换为可调用的 f(a)(b)(c)。柯里化不会调用函数。它只是对函数进行转换。

回调函数#

回调函数（Callback）分为同步回调函数，异步回调函数。

什么函数才是回调函数?

  • 你定义的
  • 你没有直接调用
  • 但最终它执行了（在特定条件或时刻）

常见的回调函数:

  • DOM 事件函数
  • 定时器函数
  • AJAX 回调函数
  • 生命周期回调函数

回调函数的特性导致其层层嵌套，形成回调地狱。

涉及面试题：什么是回调函数？回调函数有什么缺点？如何解决回调地狱问题？

回调函数（Callback）分为同步回调（数组遍历）和异步回调。

使用回调函数容易导致回调地狱问题。回调地狱的根本问题就是：

  1. 嵌套函数存在耦合性，一旦有所改动，就会牵一发而动全身
  2. 嵌套函数一多，就很难处理错误。

当然，回调函数还存在着别的几个缺点，比如不能使用 try catch 捕获错误，不能直接 return。

for (var i = 0; i < 3; i++) {
  setTimeout(function () {
    console.log(i);
  }, 0);
}
// 输出 3 3 3

for (var i = 0; i < 3; i++) {
  (function (i) {
    setTimeout(function () {
      console.log(i);
    }, 0);
  })(i);
}
// 输出 0 1 2

面试题 简述同步和异步的区别，如何避免回调地狱？

同步：方法调用一旦开始，调用者必须等到方法调用返回后，才能继续后续的行为

异步：方法调用一旦开始，方法调用就会立刻返回，调用者就可以继续后续的操作，而异步方法通常会在另外一个线程中，整个过程畅通无阻。

避免回调地狱：

  • Promise
  • async / await
  • generator
  • 事件发布/监听模式

ES this#

JS 的作用域是静态的，但是函数的调用却是动态的，由于函数可以在不同的运行环境内执行，因此 JS 在函数体内定义了this关键字来获取当前的运行环境。this 最终代表的是函数的执行者，使用者，调用者。它是一个指针型变量。

1、如何判断 this 指向#

  • 函数是否在new中调用，如果是的话，this绑定的是新创建的对象
  • 函数是否通过call、apply 显示绑定，如果是，则this绑定的是执行的对象
  • 函数是否在某个上下文中调用，如果有，则this绑定在这个上下文对象上
  • 如果都不是，默认绑定window

1.1 默认绑定#

this 默认指向 window，在无法应用其他规则的时候使用

function foo() {
  console.log(this);
}
foo(); //window

var obj = {
  do: function () {
    foo();
  },
};
// foo是直接使用不带任何修饰的函数引用进行调用，因此只能使用默认绑定规则

obj.do(); // window

1.2 隐式绑定#

当函数引用有上下文对象的时候（obj），隐式绑定规则会把函数中的 this 绑定到这个上下文对象上。

function foo() {
  console.log(this.a);
}
var obj = { a: 2, foo: foo };

obj.foo(); // 2
// 当foo调用的时候，它的落脚点确实是指向的obj对象，当函数引用有上下文对象的时候（obj），隐式绑定规则会把函数中的this绑定到这个上下文对象上。

隐式绑定可能会出现隐式丢失问题。被隐式绑定的函数，丢失了绑定对象

function foo() {
  console.log(this.a);
}
var obj = {
  a: 2,
  foo: foo,
};
var fn1 = obj.foo;
var a = "hello";
fn1(); // hello
// 虽然fn1是obj.foo的一个引用，但是实际上它的引用是foo函数本身，因此fn1其实是一个不带任何修饰的函数调用，属于默认绑定

每个函数对象都有 call 和 apply 方法，本质是设置函数体内 this 对象的值。

2、call、apply、bind#

call、apply、bind 是 Function 对象自带的方法。

这三个函数的存在意义是改变函数执行时的上下文，再具体一点就是改变函数运行时的 this 指向。

2.1 call#

call() 方法在使用一个指定的 this 值和若干个指定的参数值的前提下调用某个函数或方法。当调用一个函数时，可以赋值一个不同的 this 对象。this 引用当前对象，即 call 方法的第一个参数。

通过 call 方法，你可以在一个对象上借用另一个对象上的方法

// 语法
fun.call( thisArg[, arg1..])
// 参数列表
// - thisArg 指定函数运行时的 this值
//   - 不传，或者传 null，undefined， 函数中的this指向window对象
//   - 传递另一个函数的函数名，函数中的this指向这个函数的引用，并不一定是该函数执行时真正的this值
//   - 值为原始值(数字，字符串，布尔值)的this会指向该原始值的自动包装对象，如 String Number Boolean
//   - 传递一个对象，函数中的this指向这个对象
// - arg 函数所需要的参数，逗号间隔
function a() {
  //输出函数a中的this对象
  console.log(this);
}
//定义函数b
function b() {}

var obj = { name: "我是obj" }; //定义对象obj
a.call(); //window
a.call(null); //window
a.call(undefined); //window
a.call(1); //Number
a.call(""); //String
a.call(true); //Boolean
a.call(b); // function b(){}
a.call(obj); //Object

2.2 apply#

apply() 语法与 call() 方法的语法几乎完全相同，唯一的区别在于，apply 的第二个参数必须是一个包含多个参数的数组（或类数组对象）。

2.3 bind#

bind 函数会创建一个新函数（称为绑定函数）

  • 传参和 call 或 apply 类似
  • 不会执行对应的函数，call 或 apply 会自动执行对应的函数
  • 返回对函数的引用
  • 语法 fun.bind(thisArg[, arg1[, arg2[, ...]]])。

2.4 call 的实现#

  // 1. 把调用call方法的对象this指向call方法的第一个参数
  // 2. 调用call方法的自己的this

闭包#

1、什么是闭包#

闭包是一个包含被引用局部变量的 closure 对象，存在嵌套的内部函数中。

2、怎么产生闭包 & 生命周期#

必须满足三个条件

  • 函数嵌套函数
  • 内部函数调用外部函数的局部变量（引入全局变量是不会产生闭包的）
  • 调用外部函数

生命周期

  • 内部函数定义执行完成就产生
  • 内部函数成为垃圾对象就销毁

3、闭包的作用与缺点#

作用

  • 延长了局部变量的存活时间
  • 让函数外部操作函数内部的数据

缺点

  • 用太多可能会导致内存泄露（因为占用内部空间不释放）

4、闭包的应用#

  • 迭代器
  • React 组件高阶函数（为了复用函数，给内部函数传参）
  • Vue 原理中存在（双向数据绑定原理）

ES 面向对象#

面向对象和面向过程是两种不同的编程思想和方法。
面向过程是以过程为中心，就像流水线一样设计程序的思想。面向对象是以对象为中心的编程思想。更符合人类的思维习惯。

面向对象的三个基本特性: 封装、继承、多态。
对象（Object） 可以是一个变量，一个函数，一种数据结构。在 JS 中，一切皆为对象。JS 允许自定义对象，从狭义的概念来分析，对象是最基本的数据类型，是复合型的结构、引用型的数据，它是无序数据集合。对象中每个成员被称为属性。对象是无序的键值对，数组是有序的键值对，函数是一种可执行的特殊对象。

函数#

函数是可以反复使用的并且命名的代码块。本质上也是一种属于对象类型的数据。与其他数据类型不同的是，函数的值是可执行的代码。分为系统函数和自定义函数。

函数是数据，函数名是这个数据的标识。需要访问函数的数据去掉圆括号就好。一个函数可以有多个标识。

1、函数的作用#

  • 解决代码的冗余问题，实现代码的复用。
  • 封装代码，让函数内部的代码对外部不可见。
  • 可以让整个代码项目通过函数模块化。

2、新建函数#

函数名定义的规则遵循标识符的规则。函数的声明函数名可以重复，一旦重名，前面的覆盖后面的。

2.1 声明函数 常#

function name(args) {
  // 函数体
  return statement; // 返回语句
}

// name -- 函数名
// args -- 参数列表（可选）

var 和 function 都是声明语句，它们声明的变量和函数都在 JS 预编译时被解析，也被称为变量提升和函数提升。

ES5 中，不允许函数书写在非函数的代码块中（比如 if 语句中）。

2.2 函数表达式 常#

// 赋值调用（被称为函数表达式，这种方法只能提升函数名，不能提升函数体）
// 调用需在创建之后
let fun = function () {
  // 函数体
};

函数可以不书写名字，这种被称为匿名函数。

执行方法：

// 自调用
(function () {
  // 函数体
})();

!(function () {
  // 函数体
})();

2.3 构造函数式 了解#

类如 const x = new XXXX() 的形式，被称为构造函数。

const fun = new Function("执行字符串");
// 这个定义函数的方法几乎不用（会执行两次）

2.4 箭头函数 ES6 常#

  • 内部没有this ，箭头函数与包围它的代码共享一个this。
  • 不能用于构造函数（不能被new 实例化）。
  • 内部没有没有argument对象，可用rest参数代替。
  • 箭头函数不能使用yield命令，即不能用作Generator函数。
// 1、函数有参数，函数体只有一句返回值
function fn1(n) {
  return n + 1;
}

const fn1 = (n) => n + 1;

// 2、函数没有参数，函数体只有一句返回值
function fn2() {
  return "hello world";
}
const fn2 = () => "hello world";

// 3、函数参数随意，函数体不仅仅是一句返回值
function fn3() {}
const fn3 = () => {};

// 用箭头函数和reduce数组求和
const arrSum = (arr) => arr.reduce((p, n) => p + n);

3、使用函数#

函数需要调用才会执行。

4、参数#

JS 数据传递只有值传递，没有引用传递。

4.1 形参与实参#

形参 形式上的参数，声明函数时候提供的参数。

实参 在调用时，实际使用的参数。

4.2 默认参数#

// ES6之前
function f(name, age) {
  age ? age : (age = 18);
}
// ES6
function f(name, age = 18) {}

4.3 剩余参数#

// 可以把不确定的参数表示成数组
function f(a, ...b) {
  return b;
}
f(1, 2, 3, 4); // [2,3,4]

4.4 argument 对象#

二者在传值的时候是一一对应的。二者的个数可以不相等，形参多的时候，多出来的自动被赋值 undefined，实参多的时候，多出来的放在**arguments**对象中。该对象在函数内部才有，实际上是一个类数组 array-like 的对象，可以访问这个对象获得传递给函数的每一个实参。函数中的形参可写可不写，即使不写形参也可以使用 arguments 对象获得，使用形参比较方便。arguments 对象的 length 属性用来确定传递进来多少个实参。可以通过 length 属性的值的不同（根据传入的参数个数的不同），进行不同的操作。arguments对象中保存着一个叫callee的属性，这个属性是一个指针，它指向拥有这个arguments对象的函数。

  • 使用 arguments 对象的 length 属性可以获取函数的实参个数。
  • argument 对象只能在函数内可见,因此 arguments.legth 也只能在函数体内使用。
  • 使用函数对象的 length 属性可以获取函数的形参个数,该属性为只读属性,在函数体内,体外都可以使用

callee 是 arguments 对象的属性，它引用当前 argument 对象所在的函数，使用该属性可以在函数体内调用自身。

// 检测函数的形参和实参是否一致
arguments.length != arguments.callee.length;

5、返回值#

return 没有指定值时函数返回的是undefined，函数在执行完该语句以后立即停止执行并退出函数。

返回值是一个函数的函数称为高阶函数。

错误与异常处理#

ECMA-262 规范了7种错误类型。其中Error是基类，其他6种错误类型是子类，都继承了基类。基类的主要作用是自定义错误对象。

1、错误类型#

类型 说明
Error 基类
name 可以读写异常类型（TypeError）
message 可以读写详细的错误信息。
EvalError 不正确使用eval()方法时抛出
SyntaxError 语法错误
ReferenceError 读取不存在的变量时抛出
TypeError 值的类型发生错误的时候抛出
RangeError 数字超出合法范围
URIError URI编码和解码错误

2、try…catch#

try{
    // 调试代码块
} catch (e) {
    // 捕获异常，并进行异常处理代码块
} finally {
    
}

一般情况下 try...catch 就够用了。

按顺序执行 try 子句中的代码，如果没有异常发生，将会忽略 catch 跳转到 finally 子句继续执行。

无法捕获逻辑错误。

3、throw#

抛出异常。

try {
    let num = 1 / 0;
    if (num = Infinity) {
        // 抛出错误
        throw new Error("除数不能为0");
    }
} catch (exception) {
    console.log(exception.message);
}

抛出错误之后，后续代码不会执行。

ES 运算符与类型转换#

一、算术运算符#

1、加法#

// 如果其中一个操作数是字符串，则会把另外一个操作数转换成字符串。
1 + "str"; // 1str
"str" + 1; // str1
1 + 1 + "str"; // 2str
1 + (1 + str); // 11str

// 如果都有是字符串，会拼接在一起
"str" + "is"; // stris

// 其他类型自动调用Number()进行转换
true + true; // 2
null + null; // 0
NaN + 1; // NaN
1 +
  undefined + // NaN
  // 放在非数值前面，则会自动调用Number()进行转换（不会影响本身）
  "123"; // 123

2、减法、乘法、除法#

// 如果有操作数是非字符，则自动调用Number()转换。
"99" - 0; // 99
"99" * 1; // 99
"99" / 1; // 99

// 特殊
1 / 0; // Infinity
Infinity * -1; // - Infinity
0 / 0; // NaN

3、取余#

3 % 5; // 3
5 % 0; // NaN

4、自增、自减#

++i 表示，i 自增 1 后再参与其它运算（先加后用）；而 i++ 则是 i 参与运算后，i 的值再自增 1。自增自减运算可能会改变原有变量类型。

var a = 100
1 + a++ // 101
1 + ++a // 102

// 会改变原变量类型
var a = true;
++a; // a = 2

var i = 1
i = i++ // i = 1

二、关系运算符#

// 如果两个操作数都是字符串，比较对应的字符编码
"234" > "123455" // true
// == 和 === 的区别
undefined == null // true
undefined === null // false
// 坑
null == 0 // false 相等比较不进行类型转换
null >= 0 // true

三、赋值运算符#

a += b; // 等价于 a = a + b

四、逻辑运算符#

&& 与

  • 第一步：计算第一个操作数
  • 第二步：如果第一个操作数转换成 false，那么就会结束运算，直接第一个操作数
  • 第三步：如果第一个操作数返回 true，则计算第二个操作数的值
  • 第四步：第二个操作数如果返回 true 则逻辑与返回第二个操作数，否则返回第一个操作数

|| 或

  • 第一步：计算第一个操作数的值。
  • 第二步：检测第一个操作数的值。如果左侧的表达式的值可以转换为 true，那么就会结束运算。直接返回第一个操作数的值
  • 第三步：如果第一个操作数可以转换为 false，则计算第二个操作数的值。并返回第二个操作数的值

! 非

1 && 2; // 返回2
"" && 1; // 返回''
"Cat" && "Dog"; // 返回”Dog"
"" && false; // f && f 返回 ""
false && ""; // f && f 返回 false

1 || 2; // 1
0 || 2; // 2
0 || null; // null
"" || false; // f || f 返回 false
false || ""; // f || f 返回 ""

短路原理#

在逻辑运算中，如果第一个操作数就能决定结果，那么第二个操作数就不会运行了。（就算运行了会报错也能顺利通过）

var a = false;

ES 开始#

JavaScript 最初的目的是为了赋予网页生命。
这种编程语言我们称之为脚本。它们可以写在 HTML 中，在页面加载的时候会自动执行。脚本作为纯文本存在和执行。它们不需要特殊的准备或编译即可运行。

  • 1995 JavaScript、JScript 诞生
  • 1997 ECMA 给 JS 制定了标准ECMA-262，命名为ECMAScript，即 ES1.0。ES 是 JS 的规范标准，JS 是 ES 的实现。
  • 1999 发布 ES3，成为 JS 通用标准。
  • 2009 发布 ES5。
  • 2013 ES6 草案发布（其实就是之前不被接受的 ES4 激进方案）
  • 2015 发布ES6，即 ES2015，从此 JS 新版本按 ES+年份形式发布。ES6 是一个历史名词，也是一个泛指，代表 ES6 以后的新一代标准。

一、特点#

弱类型 数据类型不固定，可以随时改变。
解释型 解释型是一行一行解释运行，编译型是需要整体编译。
脚本语言 脚本是需要载体运行的。
基于对象/原型 只有对象，没有类，对象继承对象，而不是类继承类。原型对象是基于原型语言的核心概念。原型对象是新对象的模板，它将自身的属性共享给新对象。一个对象不但可以享有自己创建时和运行时定义的属性，而且可以享有原型对象的属性。

二、组成#

JavaScript 由 ECMAScript、BOM、DOM三部分组成，其中 ECMAScript （简称ES）是JS的核心语法规范。

  • ECMAScript 核心语法，简称 ES。
  • DOM Document Object Model 文档对象模型。
  • BOM Browser Object Model 浏览器对象模型。

三、网页中使用 JS 的方法#

（1）页面嵌入（常用）

<script>
// 这里写JS代码
</script>

（2）行内写法（几乎不用，局限性大，代码分离性差，可读性不强）
（3）外部文件引入（常用）

script 标签属性

  • src 指定外部脚本的 URI，如果设置了，标签内容会忽
  • type 指定脚本的语言，默认为 JS。
  • async 规定一旦脚本可用，则会异步执行。
  • defer 是否对脚本执行进行延迟。
<script src="链接"></script>

四、在网页中调试方法#

  • 控制台日志: console.log()、console.dir()
  • 系统对话框（弹窗）: alert()、confirm()、prompt()
  • 直接显示在文档上: document.write

CSS 动画#

transition: 过渡动画

transition-property: 属性
transition-duration: 间隔
transition-timing-function: 曲线
transition-delay: 延迟

1、过渡动画 transition#

允许 CSS 的属性在一定的时间区间内平滑过渡。在鼠标点击，鼠标滑过或对元素任何改变中触发，并平滑地以动画改变 CSS 的属性值。

transition

  • property 参与过渡的属性
  • duration 持续时间
  • timing-function 动画类型 linear 线性匀速
  • delay 延迟时间
transition: height 3s linear 2s;

animation / keyframes

animation-name: 动画名称，对应@keyframes
animation-duration: 间隔
animation-timing-function: 曲线
animation-delay: 延迟
animation-iteration-count: 次数

infinite: 循环动画

animation-direction: 方向

alternate: 反向播放

animation-fill-mode: 静止模式

forwards: 停止时，保留最后一帧
backwards: 停止时，回到第一帧
both: 同时运用 forwards / backwards

动画分为 过渡动画 和 关键帧动画。

2、关键帧动画 animation#

关键帧动画是控制关键位置，而不是所有位置。

1、@keyframes 定义新关键帧#

@keyframes <name> {
  from/0% {
    // ...
  }
  to/100% {
    // ...
  }
}

2、animation#

  • name 动画属性名
  • duration 关键帧持续时间
  • timing-function 动画执行方式（针对于每一个关键帧之间的动画）
  • delay 动画延迟时间
  • iteration-count 动画循环次数，次数或者 infinite（无穷次）
  • direction 控制动画循环反向播放，默认正常播放，alternate，轮流反向播放

3、动画的暂停和播放#

  • animation-play-state
  • paused

Electron 环境配置#

Windows 10#

  • electron打包踩过的坑总结

MacOS 打包环境配置#

推荐系统版本：MacOS 10.14，经过测试，10.15 会出现兼容性问题

  1. 通过 SVN 工具 checkout 代码到本地

    推荐用 Cornerstone

  2. 在官网下载并安装 Node.js

    推荐版本：8.12

    也可以安装最新稳定版本，然后通过 npm 版本切换工具 n 来切换版本

    sudo npm i n -g

    sudo n 8.12

    sudo n 选择相应的版本回车即可

  3. 全局安装 electron-builder

    sudo npm i electron-builder@20.19.2 -g

    另外如果 20.19.2 不行可以换 20.28.4 试试

  4. 终端进入要打包的版本目录，安装依赖

    cd 你的目录

    sudo npm i -unsafe-perm

    注：如果没有科学上网的话，下载 electron 会非常慢

  5. 安装完依赖执行打包命令

    sudo npm run build:mac

    注：这里需要科学上网，不然可能卡在 downloading 地方

    如果失败，请进入资源库/Cache/Electron 删除全部内容，重新尝试打包

    科学上网只要第一次就可以了，下次打包会有缓存

  6. 下次打包之前运行

    sudo npm run build:clean

  7. 如果出现问题，可以尝试清 npm 缓存

    sudo npm cache verify

    然后删除版本目录下面的 node_modules ，重新 sudo npm i --unsafe-perm 安装依赖
