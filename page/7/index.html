开始使用 Electron#

一、开始#

1、创建我的第一个应用#

01、需要先安装 Node.js
本质是一种 Node.js 应用，所以应用程序的起点是 package.json

app
├── package.json
├── main.js
└── index.html

02、创建项目并安装 Electron

mkdir my-electron-app
cd my-electron-app
npm init -y
npm i --save-dev electron  # 安装到开发依赖中

03、创建主脚本文件 main.js

const { app, BrowserWindow } = require("electron");

// 创建一个新的窗口
function createWindow() {
  const win = new BrowserWindow({
    width: 800,
    height: 600,
    webPreferences: {
      nodeIntegration: true,
    },
  });
  win.loadFile("index.html");
}

app.whenReady().then(createWindow);

// 当程序不再有任何打开窗口时试图退出
app.on("windows-all-closed", () => {
  if (process.platform !== "darwin") {
    app.quit();
  }
});

// 只有当应用程序激活后没有可见窗口才能创建新的窗口
app.on('activate', ()=>{
  if(BrowserWindow.getAllWindows().length === 0){
    createWindow()
  }
})

04、创建页面 index.html

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Hello World!</title>
    <meta http-equiv="Content-Security-Policy" content="script-src 'self' 'unsafe-inline';" />
</head>
<body style="background: white;">
    <h1>Hello World!</h1>
    <p>
        We are using node <script>document.write(process.versions.node)</script>,
        Chrome <script>document.write(process.versions.chrome)</script>,
        and Electron <script>document.write(process.versions.electron)</script>.
    </p>
</body>
</html>

05、修改 package.json

{
  "name": "electron-demo",
  "version": "1.0.0",
  "description": "",
  "main": "main.js",
  "scripts": {
    "start": "electron ."
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "devDependencies": {
    "electron": "^11.1.0"
  }
}

06、运行：npm start

2、打包并分发程序#

导入 Electron Forge 到你的应用文件夹

npx @electron-forge/cli import

创建一个分发版本

npm run make

（:bug: 如果打包出错，大概率是因为你的 package.json 文件里面的 author 和 description 为空）

3、应用程序结构#

Electron 由三个主要部分组成：

  • Chromium
  • Node.js
  • 自定义 APIs

进程有两个：主进程和渲染进程。
进程之间的通信可以通过 ipcMain 和 ipcRenderer 来进行
APIs 分为 Electron API、Node.js API

MacOS 打包环境配置#

推荐系统版本：MacOS 10.14，经过测试，10.15 会出现兼容性问题

  1. 通过 SVN 工具 checkout 代码到本地

    推荐用 Cornerstone

  2. 在官网下载并安装 Node.js

    推荐版本：8.12

    也可以安装最新稳定版本，然后通过 npm 版本切换工具 n 来切换版本

    sudo npm i n -g

    sudo n 8.12

    sudo n 选择相应的版本回车即可

  3. 全局安装 electron-builder

    sudo npm i electron-builder@20.19.2 -g

    另外如果 20.19.2 不行可以换 20.28.4 试试

  4. 终端进入要打包的版本目录，安装依赖

    cd 你的目录

    sudo npm i -unsafe-perm

    注：如果没有科学上网的话，下载 electron 会非常慢

  5. 安装完依赖执行打包命令

    sudo npm run build:mac

    注：这里需要科学上网，不然可能卡在 downloading 地方

    如果失败，请进入资源库/Cache/Electron 删除全部内容，重新尝试打包

    科学上网只要第一次就可以了，下次打包会有缓存

  6. 下次打包之前运行

    sudo npm run build:clean

  7. 如果出现问题，可以尝试清 npm 缓存

    sudo npm cache verify

    然后删除版本目录下面的 node_modules ，重新 sudo npm i --unsafe-perm 安装依赖

270101：Electron 使用心得

发表于 2021-01-31 分类于 2701：Electron

Electron#

一、项目构建和打包#

项目搭建：
electron-vue

项目打包：
electron-builder

  • 可以支持 Windows/Mac/Linux 平台
  • 支持自动更新，打包更轻量
  • 可以打包出不暴露源码的安装程序
    electron-packager
  • 配置 nsis 自定义安装目录以及安装个性化

二、Electron 知识点#

1、主进程和渲染进程#

主进程：package.json 中的 main.js
Electron 内核是 Chromium
在主进程中通过 new BrowserWindow 创建页面，每个 BrowserWindow 就是渲染进程，当某个页面被销毁后，对应的渲染进程就会终止。

2、优化窗口#

（1）无边框窗口

mainWindow = new BrowserWindow({width: 800, height: 600, frame: false, resizable: false});

（2）窗口拖动
现在窗口已经没有标题栏和菜单栏了，所以窗口没办法拖动了，给拖动区域增加 -webkit-app-region: drag 的 CSS 代码。
<body style="-webkit-app-region: drag"></body>
需要注意的是，如果在拖动区域里有按钮等需要点击的元素，需要增加 -webkit-app-region: no-drag 设置，否则会无法点击。

3、进程间通信#

因为安全限制，页面不能直接访问原生 GUI 资源，在 Electron 中也是一样。
渲染进程要进行原生的 GUI 操作，比如和主进程进行通信，请求相应的操作，在 Electron 中提供了几种通信方式：

（1）ipcMain 和 ipcRenderer
在渲染进程中使用 ipcRenderer 模块来向主进程发送消息，主进程中使用 ipcMain 模块接受消息，进行操作。如果还需要进行反馈，则通知渲染进程，渲染进程根据接收的内容执行对应的操作。

// 渲染进程中
const { ipcRenderer } = require("electron");
ipcRender.send("somemsg", data);
ipcRender.on("replaymsg", (evt, otherData) => {
  console.log(otherData);
});
// 主进程中
const { ipcMain } = require("electron");
ipcMain.on("somemsg", (evt, data) => {
  console.log(data);
  evt.sender.send("replymsg", otherData);
});

（2）remote
remote 模块为渲染进程和主进程通信提供了一种简单方法

（3）webContents

// 主进程中
win.webContents.on("did-finish-load", () => {
  win.webContents.send("ping", "23333333333");
});
// 渲染进程中
require("electron").ipcRenderer.on("ping", (event, message) => {
  console.log(message);
});

（4）渲染进程之间通信
使用 Vuex 和 router
但如果对数据实时性要求高，那就要结合上面的方法来实现：

// 主进程中，将两个窗口的 id 分别发送到对方的渲染进程里
mainWindow.webContents.on("did-finish-load", () => {
  mainWindow.webContents.send("getChild", {
    childId: childWindow.id,
  });
});
childWindow.webContents.on("did-finish-load", () => {
  childWindow.webContents.send("getMain", {
    mainId: mainWindow.id,
  });
});

// 渲染进程 mainWindow，通过 id 得到 childWindow 并发送一条消息
ipcRenderer.on("getChild", (event, arg) => {
  remote.BrowserWindow.fromId(arg.childId).webContents.send(
    "say",
    "Hello World!"
  );
});
// 渲染进程 childWindow，接收 mainWindow 的消息
ipcRenderer.on("say", (event, arg) => {
  console.log(arg); // 输出：Hello World!
});
// 同理 childWindow 也可以通过 id 给 mainWindow 发送消息

4、踩坑#

electron-vue 里使用 elementUI 等，需要在 webpack.renderer.config.js 的白名单里配置：let whiteListedModules = ['vue', 'element-ui']

5、难点#

多窗口的管理，每个窗口使用 remote.getCurrentWindow().id 命名，在主进程中采用单个实例 app.makeSingleInstance 管理

robot.txt#

robots.txt 是网站和搜索引擎的协议的纯文本文件。
当一个搜索引擎蜘蛛来访问站点时，它首先爬行来检查该站点根目录下是否存在 robots.txt，如果存在，根据文件内容来确定访问范围，如果没有，蜘蛛就沿着链接抓取。这个文件放在项目的根目录下。

1、允许所有搜索引擎访问网站的所有内容#

User-agent: *
Disallow:

或者

User-agent: *
Allow: /

注意第一个字母要大写，冒号是英文冒号。

2、禁止所有搜索引擎访问网站的所有内容#

User-agent: *
Disallow: /

3、禁止爬虫访问某个目录#

User-agent: *
Disallow: /某个目录1/
Disallow: /某个目录2/

注意如果结尾不加/会视为关键字进行匹配。

参考#

  • robots.txt 禁止收录协议的写法

静态资源服务器#

// 引入相关模块
const http = require("http");
const fs = require("fs");
const path = require("path");

// 手动封装 Promise，解决回调问题
function readFile(filePath) {
  return new Promise((resolve, reject) => {
    fs.readFile(filePath, (err, data) => {
      if (err) {
        reject(err);
        return;
      }
      const fileContent = data.toString();
      resolve(fileContent);
    });
  });
}

// 创建一个服务器
const server = http.createServer(async (req, res) => {
  try {
    // 获取读取文件的路径
    const filePath = path.resolve(__dirname, "./server.js");
    const data = await readFile(filePath);
    // 设置响应头类型
    res.setHeader("Content-Type", "application/javascript;charset=utf-8");
    // 设置响应状态码
    res.statusCode = 200;
    // 响应内容
    res.end(data);
  } catch (err) {
    console.log("读取文件失败", err);
    res.setHeader("Content-Type", "application/javascript;charset=utf-8");
    res.statusCode = 404;
    res.end("文件资源加载失败");
  }
});

// 配置服务器
const port = "8080";
const host = "127.0.0.1";

// 启动服务器
server.listen(port, host, (err) => {
  if (err) {
    console.log("服务器创建失败" + err);
    return;
  }
  const url = `http://${host}:${port}`;
  console.log(`服务器已启动，请访问${url}`);
});

RESTful API#

REST，Representational State Transfer，表现层状态转移的缩写，它是由罗伊·菲尔丁（Roy Fielding）提出的，是用来描述创建 HTTP API 的标准方法的，他发现这四种常用的行为（查看（view），创建（create），编辑（edit）和删除（delete））都可以直接映射到 HTTP 中已实现的 GET，POST，PUT 和 DELETE 方法。

操作#

查询 - get

（1）get books

../sword

（2）get id 为 1

../sword/1
../sword?id=1

（3）单条件查找

../sword?name=铁剑

（4）多条件查找

../sword?name=铁剑&atk=10

（5）全局模糊搜索

../sword?q=剑

（6）分页

../sword?_limit=1&_page=1

（7）排序（asc/desc）

../sword?sort=id&_order=desc

添加数据 - post

postman 中测试，设置主体 x-www-form-urlencoded

删除数据 - delete

postman 中测试

整体更新数据 - put

postman 中测试，设置主体 x-www-form-urlencoded

局部更新数据 - fatch
