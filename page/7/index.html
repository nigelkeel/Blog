一、CSS 未归类内容#

1、CSS Sprites#

CSS Sprites 精灵图，是网页图片的常用处理方式。允许将一个页面中很多零星的小图片包含到一张大图中去，当访问页面的时候，就不会一张张的去请求显示图片。对于当前的网络来说，小于200k的文件加载速度是一样的。通过 background-position 属性，将图片定位到需要的位置即可。

优点：①减少图片大小 ②减少服务器请求次数

定位辅助工具

2、CSS hack#

CSS hack是什么

由于不同的厂商的浏览器，或者是同一个浏览器不同的版本（IE），对css的解析和认识不完全一样,可能会导致不同浏览器显示的效果不相同,那么我们需要针对某个浏览器，去写不同的样式，让代码能够兼容所有的浏览器。

为什么要用 CSS hack

第一种理解：让我们CSS的代码兼容不同的浏览器

第二种理解：我们可以为不同的浏览器定制不同的样式

CSS属性前缀法

BOM对象#

BOM，Browser Object Model 浏览器对象模型。

[TOC]

一、window 对象#

window 是BOM的核心对象。表示浏览器的一个实例，是客户端浏览器对象模型的基类。在浏览器中，该对象具有双重角色，既是通过JS访问浏览器窗口的一个接口，也是ES规定中的 Global 对象。这意味着在网页中定义的任何一个对象、变量和函数，都以 window 作为其 Global 对象。所有在全局作用域中声明的变量、函数都会变成该对象的属性和方法。

1、客户端对象#

可以访问客户端其他对象，这种关系构成浏览器对象模型。

对象 描述
document 包含整个HTML文档，可以用来访问文档内容及其所有页面元素。
navigator 包含客户端有关浏览器的信息。
screen 包含客户端屏幕的信息
history 包含浏览历史记录信息
location 包含当前网页文档的URL信息

2、系统对话框#

浏览器通过 alert()、confirm() 和 prompt() 方法可以调用系统对话框向用户显示消息。

系统对话框与在浏览器中显示的网页没有关系，也不包含 HTML。它们的外观由操作系统及（或）浏览器设置决定，而不是由 CSS 决定（无法设置）。通过这几个方法打开的对话框都是同步和模态的。也就是说，显示这些对话框的时候代码会停止执行，而关掉这些对话框后代码又会恢复执行。

confirm和prompt有返回值。生产环境下不用，调试使用。

3、打开和关闭窗口#

window.open() 方法既可以导航到一个特定的 URL，也可以打开一个新的浏览器窗口。

参数

  • 要加载的URL
  • 新窗口打开方式：_self _blank（默认新标签页打开）
  • 窗口大小、位置。

返回值 就是新建的窗口名称。

窗口名.close()可以关闭窗口，如果关闭自身那就使用 window.close()[1]。

// 1. 打开窗口
var newWin = window.open("http://www.jd.com","_blank","width=300,height=300,left=200,top=100")

// 2. 关闭窗口
newWin.close();

二、navigator 对象#

储存了浏览器信息。通过 navigator 可以引用该对象，并利用它的属性来读取客户端基本信息。

常见属性：

  • onLine 表示浏览器是否连接到因特网
  • platform 浏览器所在的系统平台
  • userAgent 浏览器识别码

1、浏览器检测方法#

检测浏览器类型的方法有多种，常用的方法包括两种:特征检测法和字符串检测法，这两种方法都存在各自的优点与缺点，用户可以根据需要进行选择。

特征检测法

  • 特征检测法就是根据浏览器是否支持特定功能来决定相应操作的方式。这是一种非精确判断法。但却是最安全的检测方法
  • 因为准确检测浏览器的类型和型号是一件很困难的事情,而且很容易存在误差，如果不关心浏览器的身份，仅仅在意浏览器的执行能力，那么使用特征检测法就完全可以满足需要。
  • 当使用一个对象、方法或属性时，先判断它是否存在，如果存在，则说明浏览器支持该对象、方法属性，那么就可以放心使用

字符串检测法

  • 客户端浏览器每次发送HTTP请求时，都会附带有一个user-agent(用户代理)字符串，对于Web开发人员来说，可以使用用户代理字符串检测浏览器类型。
  • userAgent字符串包含了web浏览器的大量信息，如浏览器的名称和版本。

2、操作系统检测方法#

navigator.userAgent返回值一般都会包含操作系统的基本信息，不过这些信息比较散乱，没有统一的规则。

用户可以检测一些更为通用的信息，如检测是否为 Windows系统，或者为 Macintosh系统，而不去分辨操作系统的版本号。

例如，如果仅检测通用信息，那么所有 Windows版本的操作系统都会包含"Win”字符串，所有Macintosh版本的操作系统都包含有"Mac”字符串，所有Umix版本的操作系统都包含有"X11”，而 Linux操作系统会同时包含"X11”和" Linux。

var isWin = (navigator.userAgent.indexOf("Win") != - 1);
    // 如果是Windows系统，则返回true
var isMac = (navigator.userAgent.indexOf("Mac") != - 1); 
    // 如果是Macintosh系统，则返回true 
var isUnix = (navigator.userAgent.indexOf("X11") != - 1); 
    // 如果是UNIX系统，则返回true
var isLinux = (navigator.userAgent.indexOf("Linux") != - 1); 
    // 如果是Linux系统， xc则返回true

三、脚本化CSS#

1、获取尺寸#

offsetWidth和offsetHeight 可以获取元素的border-box尺寸。

clientWidth和clientHeight 可以获取元素的padding-box尺寸。

scrollWidth和scrollHeight 可以获取元素完全内容尺寸和padding。（几乎不用）

innerWidth和innerHeight 可以获取浏览器窗口的尺寸。

outerWidth和outerHeight 可以获取取加上工具条与滚动条窗口的尺寸。

style.width/height 只能获取行内样式的尺寸（字符串，“10px”）。

常见应用: 大盒子里面小盒子，大盒子用clientX，小盒子用offsetX。

2、获取位置#

getBoundingClientRect() 返回一个对象，对象中包含元素距离视口的距离信息。

getBoundingClientRect。

获取某个元素相对于视口的位置集合

  • top 元素上边到视窗上的距离
  • right 元素右边到视窗左的距离
  • bottom 元素下边到视窗上的距离
  • left 元素左边到视窗左的距离
  • width
  • height

offsetParent

==offsetLeft / Top==

返回当前元素的相对包含块的偏移位置。

JS 默认把它当成绝对定位属性来寻找包含块，不管是什么属性，都是寻找最近定位父级，如果没找到就是 HTML。

clientLeft / Top

clientLeft 等于 border-left。

==scrollLeft / Top==

==设置或者获取==当前元素滚动条已经滚动过的距离。（值增大，内容左滚）

// 获取文档大小
function getDocSize(){
       var h = document.documentElement;
       var b = document.body;
       return {
           y:h.offsetHeight || b.offsetHeight,
           x:h.offsetWidth || b.offsetWidth
       }
}

// 获取视口大小
function getWinSize(){
    var h = document.documentElement;
    var b =document.body;
    return {
         y:h.clientHeight || b.clientHeight,
         x:h.clientWidth || b.clientWidth
    }
}

4、获取和设置系统滚动条#

// 获取系统滚动条
function getDocScroll() {
        var h = document.documentElement;
        var b = document.body;
        var y = self.pageYOffset||
            h.scrollTop||
            b.scrollTop;
        var x = self.pageXOffset||
            h.scrollLeft||
            b.scrollLeft;

        return {
            x:x,
            y:y
        }
}

// 设置系统滚动条
function setDocScroll(x,y) {
    if(arguments.length === 1 && typeof x === "number"){
        window.scrollTo(x,0);
        document.documentElement.scrollLeft = x;
        document.body.scrollLeft = x;
    }
    if(arguments.length === 2 && typeof x === "number" && typeof y === "number"){
        window.scrollTo(x,y);
        document.documentElement.scrollLeft = x;
        document.documentElement.scrollTop = y;
        document.body.scrollLeft = x;
        document.body.scrollTop = y;
       }
}

一、概念#

1、事件模型#

DOM0 - 基本事件模型。

DOM2 - DOM事件模型，标准的事件处理模型。

2、事件流#

事件流就是多个节点对象对同一个事件响应的先后顺序。

冒泡型 从目标元素向 document 进行响应 （IE标准）

捕获型 从 document 向 目标元素 进行响应 （网景标准）

混合型 W3C事件流，==先捕获，执行目标事件，最后冒泡==。

3、事件绑定#

（1）静态绑定

（2）动态绑定（DOM0级绑定）

元素 . on+事件类型 = 函数

  • 绑定事件简单，把事件函数赋值给DOM对象的事件属性上
  • 执行的事件流是冒泡
  • 对一个元素绑定同一个事件多次，前面的会被后面覆盖
  • 注销事件只需要给事件属性赋值为 null

（3）事件处理函数

元素 . addEventListener(type function Boolean)

  • 可以绑定同一个元素同一个事件多次。
  • 可以控制冒泡或者捕获。
  • DOM2级事件（DOMContentLoader事件），只能使用这种方法绑定。

（4） IE事件模型绑定方法（了解）

元素 . attachEvent( onType function )

==事件的this指向事件源对象。==

五、定时器 ==核心== ==必考==#

JS是单线程语言，但它允许通过设置超时值和间歇时间值来调度代码在特定的时刻执行。前者是在指定的时间过后执行代码（一次），而后者则是每隔指定的时间就执行一次代码。

1、超时调用 setTimeout#

超时调用需要使用 window 对象的 setTimeout() 方法，能够在指定的时间段后执行特定代码。

var TimerID = setTimeout( code, delay [,arg])
// 参数
// code  -- 表示延迟执行的字符串型代码（或者函数）
//          如果包含多个语句，应该使用分号间隔
//          函数不用调用，只是书写函数
// delay -- 延迟时间，单位毫秒
// arg   -- 如果code是一个函数，则之后是任意多个参数

// 返回值
// TimeID 指向延迟执行的代码控制句柄。（唯一的）

// 写法1
var TimerID1 = setTimeout('var a=1;alert(a);',2000);

// 写法2
var TimerID2 = setTimeout(function(){
    var a = 2;
    alert(a);
},2000);

// 写法3
function fn(){
    var a = 2;
    alert(a);
}
var TimerID3 = setTimeout(fn,2000);

// 传参方法1
function f(a,b){
    alert(a+b)
}
var a = window.setTimeout("f(1,2)",1000)

// 传参方法2
var b = window.setTimeout(f,1000,1,2);

取消超时调用（特定条件）：

clearTimeout(timerID);

==！注意传参方法==

冒泡型 从目标元素向 document 进行响应 （IE标准）

捕获型 从 document 向 目标元素 进行响应 （网景标准）

混合型 W3C事件流，==先捕获，执行目标事件，最后冒泡==。

五、线程和事件机制#

1、进程和线程#

进程 是程序的一次执行，占用的一片独有的内存空间。

线程 是进程内的一个独立单元，CPU基本调度单位，是程序执行的一个完整流程。

进程和线程的关系

  • 一个进程至少有一个运行的线程（主线程），如果一个进程有多个进程，则称进程是多线程运行的。
  • 一个进程中的数据可以供其中多个线程直接共享，进程之间的数据是不能直接共享的。

多线程

  • 优点 有效提高CPU的利用率。
  • 缺点 增加开销（创建、切换），死锁与状态同步问题。

单线程

  • 优点 顺序编程简单易懂。
  • 缺点 效率低。

==JS是单线程的==[5]，但主流浏览器是多线程的。

代码分为 初始化代码（同步代码，绑定dom事件监听，设置定时器，发送ajax请求代码 ） 和 回调代码（异步代码）。

一、是什么#

Browser Object Model 浏览器对象模型。关于客户端浏览器的管理。

Document Object Model 文档对象模型。是W3C制订的一套技术规范，用来描述JS脚本如何与HTML或XML文档进行交互的Web标准。加载HTML页面时，Web 浏览器生成一个树型结构，用来表示页面内部结构。DOM将这种树型结构理解为由节点组成的DOM树。

DOM规定了一系列标准接口（函数），允许开发人员通过标准方式访问文档结构、操作网页内容、控制样式和行为等。

1、DOM 版本#

DOM0和DOM2。

2、document 对象#

document 对象 是window[1]对象的一个属性，因此可以将document对象作为一个全局对象来访问。当浏览器载入html文档, 它就会成为document对象。

二、节点访问#

在网页中所有对象和内容都被称为节点（Node）。如文档、元素、文本、属性、注释等。节点是DOM最基本的单元，并派生出不同类型的节点，它们共同构成了文档的树形结构模型。

1、节点#

常用节点种类#

  • document 文档节点
  • documentFragment 文档片段节点
  • Element 元素节点
  • attr 属性节点
  • text 文本节点
  • comment 注释节点

节点关系#

DOM把文档视为树形结构，称为节点树。节点之间的关系包括：上下父子关系，相邻兄弟关系。

  • 在节点树中，最顶端节点为根节点。除了根节点之外，每个节点都有一个父节点。
  • 节点可以包含任何数量的子节点。
  • 叶子是没有子节点的节点。
  • 同级节点是拥有相同父节点的节点。

childNodes#

childNodes 属性获取元素所有的子节点，保存到一个NodeList对象中。

appendChild 和 insertBefore 可以操作已经存在的元素。

// - 替换成功 ==> 返回替换的节点
// - 替换失败 ==>  null

replaceChild(new, old) 方法可以将某个子节点替换为另一个。替换节点替换的是所有子节点以及包含的所有内容。

4、元素内容操作#

innerHTML、innerText、outerHTML、outerText#

（1）获取标签

<p id="abc"><span>破晓</span>是猪</p>
<script>
oT = document.getElementById("abc");
console.log(oT.innerHTML) //<span>破晓</span>是猪
console.log(oT.innerText) //破晓是猪
console.log(oT.outerText) //破晓是猪
console.log(oT.outerHTML) //<p id="abc"><span>破晓</span>是猪</p>
</script>

innerHTML 和 innerText 前者可以解析并输出标签，后者是纯文本。

innerHTML 和 outerHTML 前者不包括自身，后者包括自身。

（2）设置标签（直接覆盖原来的内容）

innerText 和 outerText 在获取的时候是相同效果，但在设置时，innerText仅设置标签所包含的文本，而 outerText 设置包含包括标签自身在内的文本。

oTest.innerHTML = "这是内容"
oTest.innerHTML = "<a href = "##"> 这是内容 </a>"

textContent#

textContent 设置或者返回固定节点的文本内容。

document.getElementById("myList").textContent;
document.getElementById("myList").textContent = "你好"；

（1）textContent和innerHTML的区别

  • textContent 会获取 style= “display:none” 中的文本，而innerText不会。
  • innerText 不会理会html格式，直接输出不换行的文本，textContent会根据标签里面的元素独立一行。
  • innerText 对IE的兼容性较好 ，textContent虽然作为标准方法但是只支持IE8+以上的浏览器 。

（2）兼容性处理

function setOrGetContent(node,content) {
    if(arguments.length === 1){
        //代表当前是读取操作
        if(node.textContent){
            //只要能拿到这个dom对象的textContent属性值，代表当前用户是高级浏览器
            return node.textContent;
        }else {
            //代表拿不到  那就是低级浏览器
            return node.innerText;
        }
    }else if(arguments.length === 2){
        //代表写入操作
        if(node.textContent){
            //代表高级
            node.textContent = content;
        }else {
            //代表低级
            node.innerText = content;
        }
    }
}

6、文档片段节点#

DocumentFragment 是一个==虚拟的==节点类型，仅仅存在于内存中，没有添加到文档树中，看不到渲染效果。当文档片段设计完善后，再使用JS一次性添加到文档树中显示出来，提高效率，==减少页面重绘的次数==。避免浏览器渲染和资源占用，解决大量添加节点时候的性能问题。

createDocumentFragment() 创建虚拟节点，使用 appendChild 等方法插入。

三、脚本化CSS#

1、获取尺寸#

offsetWidth和offsetHeight 可以获取元素的border-box尺寸。

clientWidth和clientHeight 可以获取元素的padding-box尺寸。

scrollWidth和scrollHeight 可以获取元素完全内容尺寸和padding。（几乎不用）

innerWidth和innerHeight 可以获取浏览器窗口的尺寸。

outerWidth和outerHeight 可以获取取加上工具条与滚动条窗口的尺寸。

style.width/height 只能获取行内样式的尺寸（字符串，“10px”）。

常见应用: 大盒子里面小盒子，大盒子用clientX，小盒子用offsetX。

2、获取位置#

getBoundingClientRect() 返回一个对象，对象中包含元素距离视口的距离信息。

getBoundingClientRect。

获取某个元素相对于视口的位置集合

  • top 元素上边到视窗上的距离
  • right 元素右边到视窗左的距离
  • bottom 元素下边到视窗上的距离
  • left 元素左边到视窗左的距离
  • width
  • height

offsetParent

==offsetLeft / Top==

返回当前元素的相对包含块的偏移位置。

JS 默认把它当成绝对定位属性来寻找包含块，不管是什么属性，都是寻找最近定位父级，如果没找到就是 HTML。

clientLeft / Top

clientLeft 等于 border-left。

==scrollLeft / Top==

==设置或者获取==当前元素滚动条已经滚动过的距离。（值增大，内容左滚）

// 获取文档大小
function getDocSize(){
       var h = document.documentElement;
       var b = document.body;
       return {
           y:h.offsetHeight || b.offsetHeight,
           x:h.offsetWidth || b.offsetWidth
       }
}

// 获取视口大小
function getWinSize(){
    var h = document.documentElement;
    var b =document.body;
    return {
         y:h.clientHeight || b.clientHeight,
         x:h.clientWidth || b.clientWidth
    }
}

4、获取和设置系统滚动条#

// 获取系统滚动条
function getDocScroll() {
        var h = document.documentElement;
        var b = document.body;
        var y = self.pageYOffset||
            h.scrollTop||
            b.scrollTop;
        var x = self.pageXOffset||
            h.scrollLeft||
            b.scrollLeft;

        return {
            x:x,
            y:y
        }
}

// 设置系统滚动条
function setDocScroll(x,y) {
    if(arguments.length === 1 && typeof x === "number"){
        window.scrollTo(x,0);
        document.documentElement.scrollLeft = x;
        document.body.scrollLeft = x;
    }
    if(arguments.length === 2 && typeof x === "number" && typeof y === "number"){
        window.scrollTo(x,y);
        document.documentElement.scrollLeft = x;
        document.documentElement.scrollTop = y;
        document.body.scrollLeft = x;
        document.body.scrollTop = y;
       }
}
