ES 开始#

JavaScript 最初的目的是为了赋予网页生命。
这种编程语言我们称之为脚本。它们可以写在 HTML 中，在页面加载的时候会自动执行。脚本作为纯文本存在和执行。它们不需要特殊的准备或编译即可运行。

  • 1995 JavaScript、JScript 诞生
  • 1997 ECMA 给 JS 制定了标准ECMA-262，命名为ECMAScript，即 ES1.0。ES 是 JS 的规范标准，JS 是 ES 的实现。
  • 1999 发布 ES3，成为 JS 通用标准。
  • 2009 发布 ES5。
  • 2013 ES6 草案发布（其实就是之前不被接受的 ES4 激进方案）
  • 2015 发布ES6，即 ES2015，从此 JS 新版本按 ES+年份形式发布。ES6 是一个历史名词，也是一个泛指，代表 ES6 以后的新一代标准。

一、特点#

弱类型 数据类型不固定，可以随时改变。
解释型 解释型是一行一行解释运行，编译型是需要整体编译。
脚本语言 脚本是需要载体运行的。
基于对象/原型 只有对象，没有类，对象继承对象，而不是类继承类。原型对象是基于原型语言的核心概念。原型对象是新对象的模板，它将自身的属性共享给新对象。一个对象不但可以享有自己创建时和运行时定义的属性，而且可以享有原型对象的属性。

二、组成#

JavaScript 由 ECMAScript、BOM、DOM三部分组成，其中 ECMAScript （简称ES）是JS的核心语法规范。

  • ECMAScript 核心语法，简称 ES。
  • DOM Document Object Model 文档对象模型。
  • BOM Browser Object Model 浏览器对象模型。

三、网页中使用 JS 的方法#

（1）页面嵌入（常用）

（2）行内写法（几乎不用，局限性大，代码分离性差，可读性不强）
（3）外部文件引入（常用）

script 标签属性

  • src 指定外部脚本的 URI，如果设置了，标签内容会忽
  • type 指定脚本的语言，默认为 JS。
  • async 规定一旦脚本可用，则会异步执行。
  • defer 是否对脚本执行进行延迟。

四、在网页中调试方法#

  • 控制台日志: console.log()、console.dir()
  • 系统对话框（弹窗）: alert()、confirm()、prompt()
  • 直接显示在文档上: document.write

防抖与节流#

一、节流#

防抖就是防止把一次事件触发多次，一般用在登录、发短信、调整窗口大小，文本编辑器的实时保存。
假设有一个场景，滚动就会发起网络请求，但是我们不希望用户在滚动过程中一直发请求，而是隔一段时间发一次，对于这种情况，我们可以使用节流。

二、防抖#

节流就是控制事件的发生频率，一般用在滚动事件，播放器进度条计算，实时搜索请求。
假设有一个按钮，用户点击一次就会触发网络请求，但是我们不希望用户一直点这个按钮发请求，而是这个用户点击这个按钮一段时间内没有再去点击才去发请求，对于这种情况，我们可以使用防抖。

建立知识体系#

一、开始#

为什么要建立知识体系？

  • 大脑的工作机制是基于索引
  • 学那么多东西，为什么记不住，用不上？是因为你学到的都是零碎的，不成体系的。
  • 我们给知识建立体系就是给知识建立索引，以及索引间的逻辑关系，当知识体系大厦搭建完毕以后，你就可以随意调用，只要在你知识体系涵盖的范围内，就可以任意使用。

二、如何建立知识体系#

构造知识体系的材料#

（1）元认知（地基）

  • 能指导你思考背后的思考，指导你解释原理背后的原理。
  • 最底层的知识，它为一切现象原理，提供最底层的解释和支撑。

（2）方法（钢筋水泥）

  • 方法，套路。

（3）概念（砖瓦）

  • 我们身处的世界都是由一个个概念构成的，这个时候我们理解事物的基本方式。
  • 可以说我们认知这个世界的方式，就是在认知一个个概念，概念是组成我们思想的基本方式，你了解的概念越多，你认识这个世界的维度就会越丰富。

（4）事实（边料）

  • 浩如烟海。
  • 这些知识只在特定情况有用，大多数情况下是没有用的，所以它的可迁移能力最弱。

知识的可迁移#

  • 知识的可迁移能力，就是我们学习的任何东西，都是为了实现知识的最大可迁移，我们可以从一个领域，迅速用同样的方式，进行跨界。
  • 元认知类知识最有用，迁移能力最强，但学习难度最高，而事实类知识迁移能力最弱，学习难度也是最容易的。

元认知意识 —— 从底层出发#

从问题、现象出发#

注意事项#

  • 无用之用是为大用

    很多人觉得哲学、美学、逻辑学、社会学、心理学、经济学、历史学等各大学科很高大上，很空洞，感觉学了也不能找到工作啥的，事实上恰恰相反，这些东西才是真正承载人类顶级智慧和知识的学科，你所见到的一切都是在这些底层学科之上构建而成的。

  • 让知识间建立关系，切记一定不要孤立

  • 寻找一切现象，填充你的知识框架：为理论找到实例，为实例提供理论。

  • 搭建知识体系的过程一定是需要过程的

Electron 环境配置#

Windows 10#

  • electron打包踩过的坑总结

MacOS 打包环境配置#

推荐系统版本：MacOS 10.14，经过测试，10.15 会出现兼容性问题

  1. 通过 SVN 工具 checkout 代码到本地

    推荐用 Cornerstone

  2. 在官网下载并安装 Node.js

    推荐版本：8.12

    也可以安装最新稳定版本，然后通过 npm 版本切换工具 n 来切换版本

    sudo npm i n -g

    sudo n 8.12

    sudo n 选择相应的版本回车即可

  3. 全局安装 electron-builder

    sudo npm i electron-builder@20.19.2 -g

    另外如果 20.19.2 不行可以换 20.28.4 试试

  4. 终端进入要打包的版本目录，安装依赖

    cd 你的目录

    sudo npm i -unsafe-perm

    注：如果没有科学上网的话，下载 electron 会非常慢

  5. 安装完依赖执行打包命令

    sudo npm run build:mac

    注：这里需要科学上网，不然可能卡在 downloading 地方

    如果失败，请进入资源库/Cache/Electron 删除全部内容，重新尝试打包

    科学上网只要第一次就可以了，下次打包会有缓存

  6. 下次打包之前运行

    sudo npm run build:clean

  7. 如果出现问题，可以尝试清 npm 缓存

    sudo npm cache verify

    然后删除版本目录下面的 node_modules ，重新 sudo npm i --unsafe-perm 安装依赖

一、基本结构#

1、注释#

2、文档版本声明#

给浏览器声明，告诉浏览器应该按照 HTML5 的规范解析当前页面。 如果不写，浏览器就是按照自己默认的模式解析。

版本声明必须写在第一行。我们不把他看做一个 html 标签，而是把它当做一个指令，并且这个指令对大小写不敏感。

3、html 、head、body 标签#

html 是一个双标签，所有网页内容全部书写在 html 标签中。lang就是 html 标签的属性，en就是lang属性的值，属性是对当前标签的信息的扩展。

head 标签代表网页的头部，不会显示在网页中。主要是包含了标题，网页的描述，字符编码等等一些信息。

body 标签是网页显示的主体内容。

4、meta 标签#

控制网页的元信息（信息的信息），根据属性的不同，有很多的作用。

5、解决网页乱码问题#

CSS 选择器#

要使用 CSS 对 HTML 页面元素实现一对一，一对多，多对一的控制，就要用到选择器。

一、基本选择器#

1. 元素选择器#

也叫标签选择器。

通过元素标签名字选择页面中所有这个标签的元素。

2. 类选择器#

class 属性用来给元素分组，是允许多个元素拥有的，且一个元素可以拥有多个class。

通过 . + 元素的class属性进行选择。

3. id 选择器#

id相对class来说，是不允许重复的，且一个元素只能有一个id属性。

id 属性命名规范：

  • 以字母开头
  • 不出现汉字、空格
  • 不推荐带有ad 相关的英文单词，容易被浏览器广告过滤插件屏蔽
  • 见名知意

通过 # + 元素的id属性进行选择。

4. 通配符选择器#

表示选择所有的元素。

几乎不用，影响性能。

二、关系选择器#

1. 群组选择器（并集选择器）#

用 ,就能对多个选择器一起设置样式。

2. 交集选择器#

由两个选择器直接连接构成，其结果是选中二者各自元素范围的交集。其中第一必须是标签选择器，第二个必须是类选择器或者id选择器，两个选择器之间不能有空格，必须连续书写。

如：p#xxx 或者 p.xxx。

3. 子级选择器#

用>隔开。如 A>B，在A元素中寻找子元素是B的。

4. 后代选择器#

用空格隔开。如 A B，意思是在A元素中寻找后代是B的元素。

5. 兄弟选择器#

相邻兄弟选择器 A+B，意思是在A元素下紧贴这A的并且是B才能被选中。

通用兄弟选择器 A~B，在A元素下边的兄弟元素，并且是B就能被选中。

三、属性选择器#

选择器 描述
E[attr] 有属性attr的所有E元素
E[attr='val'] 属性attr的值是val的元素
E[attr^='val'] 属性attr的值以val开头的元素
E[attr$='val'] 属性attr的值以val结尾的元素
E[attr*='val'] 属性attr的值包含val字符串的元素

四、伪类选择器#

1. 动态伪类选择器#

  • 锚点伪类：link、visited
  • 用户行为伪类：hover、active、focus
  • 目标伪类：target（IE9及以上）

2. 状态伪类选择器#

选择器 描述
E:enabled 匹配表单中可用的元素
E:disabled 匹配表单中禁用的元素
E:checked 匹配表单中被选中的radio（单选框）或checkbox（复选框）元素

3. 结构性伪类选择器#

选择器 描述
E:first-child 选择第一个子元素E
E:last-child 选择最后一个子元素E
E:nth-child(n) 选择一个或多个特定的子元素 （第一个编号为1）
E:nth-last-child(n) 选择一个或多个特定的子元素，从最后一个子元素开始算
E:only-child 匹配父元素下仅有的一个子元素
E:empty 匹配一个不包含任何子元素的元素 ( 文本节点也被看作子元素 )
E:not(s) 匹配不符合当前选择器的任何元素( 反选 )
