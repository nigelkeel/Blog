JavaScript Promise

发表于 2020-11-27 更新于 2021-02-02 分类于 2202：ES高级
本文字数： 2.9k

进行异步编程的新解决方案。
以同步的形式把异步代码表达出来，避免回调函数层层嵌套的回调地狱。

1、状态#

状态 描述
pending 初始化状态
fulfilled（resolve） 成功状态，完成状态
reject 失败状态

状态只能被改变一次。

2、基本用法#

3、链式调用#

Promise 实现了链式调用，也就是说每次调用 then 之后返回的都是一个 Promise，并且是一个全新的 Promise，原因也是因为状态不可变。如果你在 then 中 使用了 return，那么 return 的值会被 Promise.resolve() 包装。

4、更多方法#

（1）Promise.prototype.then

返回一个Promise 对象。

（2）Promise.prototype.catch

返回一个Promise 对象。

（3）Promise.resolve

返回的Promise对象状态为fulfilled。

（4）Promise.reject

返回一个状态为失败的Promise对象，并将给定的失败信息传递给对应的处理方法。

（5）Promise.all

  • 接收一个数组，数组的每一个元素都的 Promise
  • 返回的是一个 Promise
  • 数组中所有成功，则返回成功，如果有一个失败，则返回失败状态

这个方法返回一个新的 promise 对象，该 promise 对象在 iterable 参数对象里所有的 promise 对象都成功的时候才会触发成功，一旦有任何一个 iterable 里面的 promise 对象失败则立即触发该 promise 对象的失败。

（6）Promise.race

当 iterable 参数里的任意一个子 promise 被成功或失败后，父 promise 马上也会用子 promise 的成功返回值或失败详情作为参数调用父 promise 绑定的相应句柄，并返回该 promise 对象。

（7）Promise.any

接收一个 Promise 对象的集合，当其中的一个 promise 成功，就返回那个成功的 promise 的值。

（8）Promise.allSettled

等到所有 promises 都完成（每个 promise 返回成功或失败）。
返回一个 promise，该 promise 在所有 promise 完成后完成。并带有一个对象数组，每个对象对应每个 promise 的结果。

in/of#

1、in 用来判断对象是否是数组/对象的元素/属性

2、for...in 对数组或者对象循环（迭代）操作

对于数组来说，遍历出来的是数组元素

对于对象来说，遍历出来的是对象属性（key）

可枚举属性（enumerable ，不包括 symbol），会循环原型链和手动添加的键

某些情况下，循环顺序是随机的

3、for...of 遍历出来的是对象（数组、类数组、字符串）的值

可以使用 break continue return
