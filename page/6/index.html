Generstor#

状态机，异步生成器。

function* gen(){
    yield ...
    yield ...
}
    
gen.next();

Generstor 是可迭代的（for.. of）。

例子：

// 使用计时器模拟依次请求 a数据 b数据
function* gen(){
    yield setTimeout(() => {
        console.log("a数据请求成功")
        it.next();
    }, 1000);
    yield setTimeout(() => {
        console.log("b数据请求成功")
    }, 1000);
}

// 开始请求
const it = gen()
it.next();

JavaScript DOM#

Document Object Model， 文档对象模型。是 W3C 制订的一套技术规范，用来描述 JS 如何与 HTML 或 XML 文档进行交互的 Web 标准。DOM 规定了一系列标准 API，允许开发人员通过标准方式访问文档结构、操作网页内容、控制样式和行为等。

加载 HTML 页面时，Web 浏览器生成一个树型结构，用来表示页面内部结构。DOM 将这种树型结构理解为由节点组成的 DOM 树。

1、DOM 版本#

主要分为 DOM 0级 和 DOM 2级。

2、document 对象#

document 对象 是window对象的一个属性，因此可以将document对象作为一个全局对象来访问。当浏览器载入 html 文档, 它就会成为document对象。

JS 顶层对象是global，window是global在浏览器的实现。

DOM 节点#

在网页中所有的对象和内容称为节点（Node），如文档、元素、文本、属性、注释等，节点是 DOM 最基本的单元，并派生出不同类型的节点，它们共同的构成了文档的树形结构。

1、节点关系#

节点之间的关系包括：

  • 父子关系
  • 兄弟关系。

在节点树中，最顶端节点为根节点。

除了根节点之外，每个节点都有一个父节点。

节点可以包含任何数量的子节点。

叶子是没有子节点的节点。

同级节点是拥有相同父节点的节点。

2、节点属性#

childNodes 属性获取元素所有的子节点，保存到一个NodeList伪数组对象中。

textContent#

设置或者返回固定节点的文本内容。

document.getElementById("myList").textContent;
document.getElementById("myList").textContent = "你好"；

（1）textContent 和 innerHTML 的区别

  • textContent 会获取 style= “display:none” 中的文本，而 innerText 不会。
  • innerText 不会理会 html 格式，直接输出不换行的文本，textContent 会根据标签里面的元素独立一行。
  • innerText 对 IE 的兼容性较好 ，textContent 虽然作为标准方法但是只支持 IE8+以上的浏览器 。

（2）兼容性处理

function setOrGetContent(node, content) {
  if (arguments.length === 1) {
    //代表当前是读取操作
    if (node.textContent) {
      //只要能拿到这个dom对象的textContent属性值，代表当前用户是高级浏览器
      return node.textContent;
    } else {
      //代表拿不到  那就是低级浏览器
      return node.innerText;
    }
  } else if (arguments.length === 2) {
    //代表写入操作
    if (node.textContent) {
      //代表高级
      node.textContent = content;
    } else {
      //代表低级
      node.innerText = content;
    }
  }
}

// 获取系统滚动条
function getDocScroll() {
  var h = document.documentElement;
  var b = document.body;
  var y = self.pageYOffset || h.scrollTop || b.scrollTop;
  var x = self.pageXOffset || h.scrollLeft || b.scrollLeft;

// 设置系统滚动条
function setDocScroll(x, y) {
  if (arguments.length === 1 && typeof x === "number") {
    window.scrollTo(x, 0);
    document.documentElement.scrollLeft = x;
    document.body.scrollLeft = x;
  }
  if (
    arguments.length === 2 &&
    typeof x === "number" &&
    typeof y === "number"
  ) {
    window.scrollTo(x, y);
    document.documentElement.scrollLeft = x;
    document.documentElement.scrollTop = y;
    document.body.scrollLeft = x;
    document.body.scrollTop = y;
  }
}

TCP & UDP#

TCP，Transmission Control Protool，传输控制协议是一种面向连接的、可靠的、基于字节流的传输层通信协议，由 IETF 的 RFC-793 定义。是一种面向广域网的通信协议，目的是在跨越多个网络通信时，为两个通信端点之间提供一条具有下列特点的通信方式：

  • 基于流的方式
  • 面向连接
  • 可靠通信方式
  • 在网络状况不佳的时候尽量降低系统由于重传带来的带宽开销
  • 通信连接维护是面向通信的两个端点的，而不考虑中间网段和节点

首先 UDP 协议是面向无连接的，也就是说不需要在正式传递数据之前先连接起双方。然后 UDP 协议只是数据报文的搬运工，不保证有序且不丢失的传递到对端，并且 UDP 协议也没有任何控制流量的算法，总的来说 UDP 相较于 TCP 更加的轻便。

UDP 相比 TCP 简单的多，不需要建立连接，不需要验证数据报文，不需要流量控制，只会把想发的数据报文一股脑的丢给对端。虽然 UDP 并没有 TCP 传输来的准确，但是也能在很多实时性要求高的地方有所作为。

TCP 建立连接需要三次握手，断开连接需要四次握手。滑动窗口解决了数据的丢包、顺序不对和流量控制问题拥塞窗口实现了对流量的控制，保证在全天候环境下最优的传递数据。

ES this#

JS 的作用域是静态的，但是函数的调用却是动态的，由于函数可以在不同的运行环境内执行，因此 JS 在函数体内定义了this关键字来获取当前的运行环境。this 最终代表的是函数的执行者，使用者，调用者。它是一个指针型变量。

1、如何判断 this 指向#

  • 函数是否在new中调用，如果是的话，this绑定的是新创建的对象
  • 函数是否通过call、apply 显示绑定，如果是，则this绑定的是执行的对象
  • 函数是否在某个上下文中调用，如果有，则this绑定在这个上下文对象上
  • 如果都不是，默认绑定window

1.1 默认绑定#

this 默认指向 window，在无法应用其他规则的时候使用

function foo() {
  console.log(this);
}
foo(); //window

var obj = {
  do: function () {
    foo();
  },
};
// foo是直接使用不带任何修饰的函数引用进行调用，因此只能使用默认绑定规则

obj.do(); // window

1.2 隐式绑定#

当函数引用有上下文对象的时候（obj），隐式绑定规则会把函数中的 this 绑定到这个上下文对象上。

function foo() {
  console.log(this.a);
}
var obj = { a: 2, foo: foo };

obj.foo(); // 2
// 当foo调用的时候，它的落脚点确实是指向的obj对象，当函数引用有上下文对象的时候（obj），隐式绑定规则会把函数中的this绑定到这个上下文对象上。

隐式绑定可能会出现隐式丢失问题。被隐式绑定的函数，丢失了绑定对象

function foo() {
  console.log(this.a);
}
var obj = {
  a: 2,
  foo: foo,
};
var fn1 = obj.foo;
var a = "hello";
fn1(); // hello
// 虽然fn1是obj.foo的一个引用，但是实际上它的引用是foo函数本身，因此fn1其实是一个不带任何修饰的函数调用，属于默认绑定

每个函数对象都有 call 和 apply 方法，本质是设置函数体内 this 对象的值。

2、call、apply、bind#

call、apply、bind 是 Function 对象自带的方法。

这三个函数的存在意义是改变函数执行时的上下文，再具体一点就是改变函数运行时的 this 指向。

2.1 call#

call() 方法在使用一个指定的 this 值和若干个指定的参数值的前提下调用某个函数或方法。当调用一个函数时，可以赋值一个不同的 this 对象。this 引用当前对象，即 call 方法的第一个参数。

  • 语法 fun.bind(thisArg[, arg1[, arg2[, ...]]])。

function readFile(filePath) {
  return new Promise((resolve, reject) => {
    fs.readFile(filePath, (err, data) => {
      if (err) {
        reject(err);
        return;
      }
      const fileContent = data.toString();
      resolve(fileContent);
    });
  });

// 创建一个服务器
const server = http.createServer(async (req, res) => {
  try {
    // 获取读取文件的路径
    const filePath = path.resolve(__dirname, "./server.js");
    const data = await readFile(filePath);
    // 设置响应头类型
    res.setHeader("Content-Type", "application/javascript;charset=utf-8");
    // 设置响应状态码
    res.statusCode = 200;
    // 响应内容
    res.end(data);
  } catch (err) {
    console.log("读取文件失败", err);
    res.setHeader("Content-Type", "application/javascript;charset=utf-8");
    res.statusCode = 404;
    res.end("文件资源加载失败");
  }
});

（2）判断元素是否处于动画状态

当用户在某个元素上执行 animate() 动画时，就会出现动画积累，

使用 is(":animated") 来判断是否处于动画状态，如果不处于动画状态，才会给元素添加新动画。

2、事件对象#

对于原生的事件对象，由于浏览器差异，开发者总是需要做兼容性处理。jQuery 解决了原生事件对象的兼容性问题，还提供了很多新的属性和方法。

stopPropagation() 阻止事件冒泡

preventDefault() 阻止默认行为

3、jQuery 绑定事件#

bind() 方法无法动态绑定尚未存在的元素。此外还有 live() delegate 等已经被废弃，底层实际使用 on()。

on( events, selector, data, fn) 绑定事件 jQuery 1.7+

  • events 事件类型
  • selector 触发事件选择器元素的后代
  • data 当一个事件被触发要传递 event.data 给事件处理函数
  • fn 该事件被触发时的执行函数

还可以绑定多个事件绑定同一个函数、多个事件绑定不同函数。

unbind() 解除绑定事件。

off( events, selector, fn) 解除绑定事件。

one( type, data, fn ) 绑定元素执行完毕后自动移除事件，方法仅触发一次。

  • type 添加到元素的一个或者多个事件。
  • data 将要传递给事件处理函数的数据映射。
  • fn 每当事件触发时执行的函数。
