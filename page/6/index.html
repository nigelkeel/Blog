ES 开始#

JavaScript 最初的目的是为了赋予网页生命。
这种编程语言我们称之为脚本。它们可以写在 HTML 中，在页面加载的时候会自动执行。脚本作为纯文本存在和执行。它们不需要特殊的准备或编译即可运行。

  • 1995 JavaScript、JScript 诞生
  • 1997 ECMA 给 JS 制定了标准ECMA-262，命名为ECMAScript，即 ES1.0。ES 是 JS 的规范标准，JS 是 ES 的实现。
  • 1999 发布 ES3，成为 JS 通用标准。
  • 2009 发布 ES5。
  • 2013 ES6 草案发布（其实就是之前不被接受的 ES4 激进方案）
  • 2015 发布ES6，即 ES2015，从此 JS 新版本按 ES+年份形式发布。ES6 是一个历史名词，也是一个泛指，代表 ES6 以后的新一代标准。

一、特点#

弱类型 数据类型不固定，可以随时改变。
解释型 解释型是一行一行解释运行，编译型是需要整体编译。
脚本语言 脚本是需要载体运行的。
基于对象/原型 只有对象，没有类，对象继承对象，而不是类继承类。原型对象是基于原型语言的核心概念。原型对象是新对象的模板，它将自身的属性共享给新对象。一个对象不但可以享有自己创建时和运行时定义的属性，而且可以享有原型对象的属性。

二、组成#

JavaScript 由 ECMAScript、BOM、DOM三部分组成，其中 ECMAScript （简称ES）是JS的核心语法规范。

  • ECMAScript 核心语法，简称 ES。
  • DOM Document Object Model 文档对象模型。
  • BOM Browser Object Model 浏览器对象模型。

三、网页中使用 JS 的方法#

（1）页面嵌入（常用）

<script>
// 这里写JS代码
</script>

（2）行内写法（几乎不用，局限性大，代码分离性差，可读性不强）
（3）外部文件引入（常用）

script 标签属性

  • src 指定外部脚本的 URI，如果设置了，标签内容会忽
  • type 指定脚本的语言，默认为 JS。
  • async 规定一旦脚本可用，则会异步执行。
  • defer 是否对脚本执行进行延迟。
<script src="链接"></script>

四、在网页中调试方法#

  • 控制台日志: console.log()、console.dir()
  • 系统对话框（弹窗）: alert()、confirm()、prompt()
  • 直接显示在文档上: document.write

CSS 动画#

transition: 过渡动画

transition-property: 属性
transition-duration: 间隔
transition-timing-function: 曲线
transition-delay: 延迟

1、过渡动画 transition#

允许 CSS 的属性在一定的时间区间内平滑过渡。在鼠标点击，鼠标滑过或对元素任何改变中触发，并平滑地以动画改变 CSS 的属性值。

transition

  • property 参与过渡的属性
  • duration 持续时间
  • timing-function 动画类型 linear 线性匀速
  • delay 延迟时间
transition: height 3s linear 2s;

animation / keyframes

animation-name: 动画名称，对应@keyframes
animation-duration: 间隔
animation-timing-function: 曲线
animation-delay: 延迟
animation-iteration-count: 次数

infinite: 循环动画

animation-direction: 方向

alternate: 反向播放

animation-fill-mode: 静止模式

forwards: 停止时，保留最后一帧
backwards: 停止时，回到第一帧
both: 同时运用 forwards / backwards

动画分为 过渡动画 和 关键帧动画。

2、关键帧动画 animation#

关键帧动画是控制关键位置，而不是所有位置。

1、@keyframes 定义新关键帧#

@keyframes <name> {
  from/0% {
    // ...
  }
  to/100% {
    // ...
  }
}

2、animation#

  • name 动画属性名
  • duration 关键帧持续时间
  • timing-function 动画执行方式（针对于每一个关键帧之间的动画）
  • delay 动画延迟时间
  • iteration-count 动画循环次数，次数或者 infinite（无穷次）
  • direction 控制动画循环反向播放，默认正常播放，alternate，轮流反向播放

3、动画的暂停和播放#

  • animation-play-state
  • paused

4、动画的起始和结束状态#

5、逐帧动画#

timing-function: steps()

Windows 10#

MacOS 打包环境配置#

推荐系统版本：MacOS 10.14，经过测试，10.15 会出现兼容性问题

  1. 通过 SVN 工具 checkout 代码到本地

    推荐用 Cornerstone

  2. 在官网下载并安装 Node.js

    推荐版本：8.12

    也可以安装最新稳定版本，然后通过 npm 版本切换工具 n 来切换版本

    sudo npm i n -g

    sudo n 8.12

    sudo n 选择相应的版本回车即可

  3. 全局安装 electron-builder

    sudo npm i electron-builder@20.19.2 -g

    另外如果 20.19.2 不行可以换 20.28.4 试试

  4. 终端进入要打包的版本目录，安装依赖

    cd 你的目录

    sudo npm i -unsafe-perm

    注：如果没有科学上网的话，下载 electron 会非常慢

  5. 安装完依赖执行打包命令

    sudo npm run build:mac

    注：这里需要科学上网，不然可能卡在 downloading 地方

    如果失败，请进入资源库/Cache/Electron 删除全部内容，重新尝试打包

    科学上网只要第一次就可以了，下次打包会有缓存

  6. 下次打包之前运行

    sudo npm run build:clean

  7. 如果出现问题，可以尝试清 npm 缓存

    sudo npm cache verify

    然后删除版本目录下面的 node_modules ，重新 sudo npm i --unsafe-perm 安装依赖

响应式布局#

响应式布局是在 2010 年提出的一个概念，简而言之，就是一个网站能够兼容多个终端，而不是为每个终端做一个特定的版本。用户会使用不同的终端，由于屏幕分辨率不同，想要用户体验更好，就有必要让你的网页兼容多个终端。响应式布局设计可以让一个网站适配多种设备和多个屏幕，可以让网站的布局和功能随用户的使用环境（屏幕大小，输入方式，设备/浏览器能力）而变化。

三个特征

  • 媒体查询：检查设备在渲染我们作品时的物理特性
  • 流动布局
  • 灵活的视觉效果

LESS#

less 是一种动态样式语言，属于 CSS 预处理器，扩展了 CSS 语言，增加了变量、混合、函数等特性，使 CSS 更易维护和扩展。LESS 既可以在客户端浏览器上运行，也能借助 node.js 在服务器端运行。

CSS 自诞生以来，基本语法和核心机制一直没有本质上的变化，它的发展几乎全是表现力层面上的提升。最开始 CSS 在网页中的作用只是辅助性的装饰，轻便易学是最大的需求；然而如今网站的复杂度已经不可同日而语，原生 CSS 已经让开发者力不从心。当一门语言的能力不足而用户的运行环境又不支持其它选择的时候，这门语言就会沦为 “编译目标” 语言。开发者将选择另一门更高级的语言来进行开发，然后编译到底层语言以便实际运行。于是，在前端领域，CSS 预处理器应运而生。而 CSS 这门古老的语言以另一种方式 “重新适应” 了网页开发的需求。

1、安装#

中文 | 英文 | 源码

  1. 安装 Node.JS
  2. 通过命令行程序，node -v 和 npm -v 检查是否安装成功。
  3. 使用npm i -g less 进行全局安装 less, 并输入 lessc -V 检查版本

2、特性#

  • 动态样式 CSS 是非程式式语言，需要书写大量没有逻辑的代码，不方便维护和扩展、复用。
  • 预处理器 生成 CSS 前某一语法的处理程序。

(2) 在命令行编译 less 文件

(3) 通过 VSCode 的Easy LESS插件，在文件夹新建 less 文件，保存，即可在当前文件夹编译出相应的 css 文件。

弹性布局#

flex ，即弹性布局。

1、开启弹性布局#

display: flex;
display: -webkit-flex; /* Safari */

开启 flex 的元素自身不受影响，只会影响子元素，子元素的float、clear和vertical-align属性将失效。对子元素的子元素不会影响。
a
开启flex的盒子称为 容器（container），子元素称为 项目（item）。横向默认是 主轴（main axis）。纵向默认是 侧轴（交叉轴，cross axis）。默认情况下，项目沿着主轴从左到右依次排列。项目的高度撑满容器（侧轴撑满）。

当项目的总宽度超出，并不会换行，而是压缩项目的宽度。

涉及到 12 个 CSS 属性，父容器，子容器各 6 个。

2、常用属性#

flex-direction#

默认情况下，主轴的方向是从左向右的，交叉轴垂直于主轴。

交叉轴是由主轴决定的，主轴又是由 flex-direction 决定的，这个属性设置在父容器上，才可以生效。

flex-direction: row | row-reverse | column | column-reverse;

CSS 定位

发表于 2021-01-31 更新于 2021-02-01 分类于 2103：CSS布局

定位#

定位（position） 允许你定义元素框相对于其正常位置应该出现的位置，或者相对于父元素，以及浏览器窗口本身的位置。

属性 描述
static 默认，没有定位
relative 相对定位
absolute 绝对定位
fixed 固定定位

布局一般用在大布局的小范围中。

1、相对定位 - relative#

相对定位（relative），让元素相对定位，元素先放置在未添加定位时候的区域，然后在不改变页面布局的情况下（其他元素没有受到任何影响；自身原来的位置也保留；和浮动可以一起使用）进行移动。不会脱离文档流。

2、绝对定位 - absolute#

绝对定位（absolute），相对于它的 包含块 定位。绝对定位是相对于它的包含块定位的（和自身原有位置无关）。如果没有任何一个父级元素是非 static 属性，则会相对于文档（html）定位。如果它的父级元素和爷爷级元素都是非 position:static 属性，则它会选择距离最近的父元素。脱离正常文档流，原有位置不保留。定位的原点以包含块的 padding-box 的左上角为原点。right 和 bottom 是距离包含块右下角的距离。

包含块#

包含块 是视觉格式化模型的一个概念，简单来说就是一个定位参考框。

如何确定一个元素的包含块，完全取决于它自身的 position 属性：

  • 如果一个元素自身的 position 属性是 static 或者是 relative ，它的包含块就是离他最近的祖先元素或者是格式化上下文。
  • 如果一个元素自身的 position 属性是 absolute ，它的包含块就是离他最近的拥有定位属性（值不为 static ）的元素。
  • 如果一个元素自身的 position 属性是 fixed，它的包含块就是 viewport。

补充：
如果一个元素的 position 属性是 absolute 或者是 fixed 在下边几种情况下，包含块会发生改变。

  • 当祖先元素拥有 transform 或 perspective 属性并且值不为 none 的时候，它也是被当做包含块。
  • 当祖先元素 拥有 filter 属性的时候（值不为 none） 它也可以被当做包含块

注：如果由内向外找不到包含块条件的元素，那么 html（根元素）被称作为初始包含块。

子绝父相#

父元素相对定位并不设置位移，子元素绝对定位，设置位移。

3、固定定位 - fixed#

固定定位（fixed），相对于浏览器窗口定位。

4、粘性定位 - sticky#

sticky 的意思是“粘性的”，position:sticky 表现也符合这个粘性的表现。基本上，可以看出是position:relative和position:fixed的结合体——当元素在屏幕内，表现为relative，就要滚出显示器屏幕的时候，表现为fixed。

nav {
  position: -webkit-sticky;
  position: sticky;
  top: 0;
}

杀了个回马枪，还是说说 position:sticky 吧

发表于 2021-01-31 更新于 2021-02-01 分类于 2103：CSS布局

浮动#

浮动（float）可以让多个块元素横向排列，共行。

属性 说明
none 默认，不浮动
left 左浮动
right 右浮动

1、概念#

正常页面流/正常文档流 是行元素从左至右，块元素从上到下布局。浮动 是破坏流（脱离文档流）的一种方式。在目前开发中，很多的元素都是一个个 float 堆叠起来，但是这种布局很容易崩溃。只要宽度和高度发生一点变化，页面就会发生错乱。

2、浮动的使用#

左浮动 就是让容器沿着容器左侧排序。

右浮动 让容器沿着容器右侧排序。

3、高度塌陷#

浮动元素以后，脱离父级内容区域，父级没有内容撑开自身的高度，而且父级的兄弟元素是按照父级的位置进行布局的，所以页面会乱。

4、清除浮动 常用#

清除浮动不是不让元素浮动，是清除浮动对父级高度塌陷的影响。

清除浮动的方法

  • 给浮动元素的父级设置高度。
  • 给浮动元素的父级设置浮动，以浮制浮。
  • overflow 设置不为 visible 的属性。
  • 在最后一个浮动元素的后边添加一个空的块级元素，添加一个块级元素，添加属性和值为 clear: both。
  • after 伪元素清浮动，给所有浮动元素父级一个 clearFix 的类名。推荐
.clearFix:after {
  content: "/200B";
  display: block;
  height: 0;
  clear: both;
}

CSS BFC#

BFC，块级格式化上下文。 是一个独立的渲染区域，让处于 BFC 内部的元素与外部的元素相互隔离，使内外元素的定位不会相互影响。

1、创建 BFC#

  • 根元素
  • 浮动 float: right / left
  • 定位（仅 absolute 和 fixed）
  • 行内块元素和表格元素 display: inline-block / table
  • overflow 的值不为 visible

2、范围#

A block formatting context contains everything inside of the element creating it, that is not also inside a descendant element that creates a new block formatting context.

直译过来就是, BFC 包含创建它的元素的所有子元素, 但不包括创建了新 BFC 的子元素的内部元素。简单来说，子元素如果又创建了一个新的 BFC，那么它里面的内容就不属于上一个 BFC 了，这体现了 BFC 隔离 的思想。
也就是所说，一个元素不能同时存在于两个 BFC 中。

3、规则#

BFC 除了会创建一个隔离的空间外，还具有以下特性：

  • BFC 内部的块级盒会在垂直方向上一个接一个排列。
  • 同一 BFC 下的相邻块级元素可能发生外边距折叠，创建新的 BFC 可以避免外边距折叠。（解决塌陷）
  • 浮动盒的区域不会和 BFC 重叠 （BFC 的区域不会与 float 的元素区域重叠）
  • 计算 BFC 的高度时，浮动元素也会参与计算
  • 文字层不会被浮动层覆盖，环绕于周围

4、应用#

  • 自适应多栏布局
    中间栏创建 BFC，左右栏宽度固定后浮动。由于盒子的 margin box 的左边和包含块 border box 的左边相接触，同时浮动盒的区域不会和 BFC 重叠，所以中间栏的宽度会自适应。

  • 防止外边距折叠
    创建新的 BFC ，让相邻的块级盒位于不同 BFC 下可以防止外边距折叠

  • 清除浮动
    BFC 内部的浮动元素也会参与高度计算，可以清除 BFC 内部的浮动

    清除浮动的原理是两个div都位于同一个 BFC 区域之中

CSS 盒模型#

在 HTML 中把每个元素当成一个盒子，拥有盒子的平面外形和空间。

1、盒模型标准#

通过 box-sizing 设置盒子模型的标准，分为

  • content-box 默认，W3C 标准盒模型
  • border-box IE 盒模型，也称为怪异盒模型

注，理论上还有 padding-box 和 margin-box ，但是现在 W3C 和 MDN 规范均不支持。

怪异盒模型的 width / height 是 border 的 尺寸。

2、内容层 content#

width/height#

width 设置的坑

  • 如果设置宽度为 100%，并且有 border margin padding 等属性值的时候，宽度可能超出。因为 width 内容区域已经和父级一样了，所以可以不设置宽度或者设置宽度为 auto 那么内容的宽度就是可以充分利用可使用空间。

height 设置的坑

  • 如果高度设置百分比，必须父级有一个显性的设置一个 height 属性，并且不能是 auto。

min/max-width/height#

正常 最小 最大
宽度 width min-width max-width
高度 height min-height max-height

当 min-width 和 max-width 冲突的时候，min-width 是优先的。当 width 和 min 或 max 冲突以后，min 或者 max 直接无视 width 的存在（无论是否添加 ！important）。

3、内边距层 padding#

padding 的写法和 margin 一致

padding 主要撑开内容和边框之间的距离

padding 不支持负值

padding 对所有属性的元素都支持，但是对于行内元素撑不开上下的距离

padding 可以用于扩大点击范围

4、边框层 border#

边框属性#

border

  • border-width 线的宽度
  • border-style 线的类型
    • solid 实线
    • dashed 虚线
    • dotted 点状线
  • border-color 线的颜色
  • border:none 不设置边框

边框还分为 4 个方向 要么直接设置 4 个方向，要么一个个方向单独设置

练习：边框画三角形

.box {
  width: 0;
  height: 0;
  border: 100px solid #000;
  border-left-color: transparent;
  border-right-color: transparent;
  border-bottom-color: transparent;
}

5、外边距层 margin#

margin 的作用是撑开当前元素和其他元素的距离。margin 有四个方向，可以分别设置margin-left margin-right margin-top margin-bottom，margin 的 left 和 top 是让当前元素向右向下移动，margin 的 right 和 bottom 是让元素的兄弟元素向右向下移动，不影响自身。

合写#

  • 4 个值：上 右 下 左（从上开始顺时针）
  • 3 个值：上 左右 下
  • 2 个值：上下 左右
  • 1 个值：上下左右

水平方向上的 auto#

因为块标签拥有满行属性，但是设置宽度以后，剩余的空间就是默认的 margin，设置左右的 margin 值为 auto，那么剩余的 margin 就会平分到左右两边。

竖直方向的 auto#

相对于水平方向来说，块元素在垂直方向上并没有满屏的属性，margin 默认在上下的值都是 0，并没有剩余空间去平分。

塌陷和边框合并问题#

  • 父级塌陷：父级的第一个子元素的上 margin 和最后一个子元素的下 margin 会进行塌陷给父级，这样的设计是为了方便父级和他的兄弟进行垂直 margin 的合并，避免缝隙过大。
  • 解决塌陷：给父级开启 BFC，给父级设置边框。
  • 兄弟的 margin 垂直方向会进行折叠，取最大值 。

margin 负值#

  • margin-top 为负值：元素向上走，不会影响上边的元素的位置（会出现重叠），并且下边的兄弟元素会跟着一起向上走。
  • margin-left 为负值：元素向左走，不会影响前边的元素位置(会出现重叠)，右边兄弟元素会跟随移动。
  • margin-right 为负值：元素不会移动，形态大小也不会发生改变，但是所占用的位置会减少（负 margin 的值），所以，后边的兄弟元素会自动向左走。
  • margin-bottom 为负值：元素不会移动，形态大小也不会发生改变，但是所占用的位置会减少（负 margin 的值），所以，后边的兄弟元素会自动向上走。

支持性#

margin 完美支持块元素和行内块元素，不支持行元素的垂直方向。

6、轮廓层 outline#

outline 轮廓层。

outline: none;
