ES 函数

发表于 2021-01-30 更新于 2021-02-01 分类于 2201：ES基础

函数#

函数是可以反复使用的并且命名的代码块。本质上也是一种属于对象类型的数据。与其他数据类型不同的是，函数的值是可执行的代码。分为系统函数和自定义函数。

函数是数据，函数名是这个数据的标识。需要访问函数的数据去掉圆括号就好。一个函数可以有多个标识。

1、函数的作用#

  • 解决代码的冗余问题，实现代码的复用。
  • 封装代码，让函数内部的代码对外部不可见。
  • 可以让整个代码项目通过函数模块化。

2、新建函数#

函数名定义的规则遵循标识符的规则。函数的声明函数名可以重复，一旦重名，前面的覆盖后面的。

2.1 声明函数 常#

function name(args) {
  // 函数体
  return statement; // 返回语句
}

// name -- 函数名
// args -- 参数列表（可选）

var 和 function 都是声明语句，它们声明的变量和函数都在 JS 预编译时被解析，也被称为变量提升和函数提升。

ES5 中，不允许函数书写在非函数的代码块中（比如 if 语句中）。

2.2 函数表达式 常#

// 赋值调用（被称为函数表达式，这种方法只能提升函数名，不能提升函数体）
// 调用需在创建之后
let fun = function () {
  // 函数体
};

函数可以不书写名字，这种被称为匿名函数。

执行方法：

// 自调用
(function () {
  // 函数体
})();

!(function () {
  // 函数体
})();

2.3 构造函数式 了解#

类如 const x = new XXXX() 的形式，被称为构造函数。

const fun = new Function("执行字符串");
// 这个定义函数的方法几乎不用（会执行两次）

2.4 箭头函数 ES6 常#

  • 内部没有this ，箭头函数与包围它的代码共享一个this。
  • 不能用于构造函数（不能被new 实例化）。
  • 内部没有没有argument对象，可用rest参数代替。
  • 箭头函数不能使用yield命令，即不能用作Generator函数。
// 1、函数有参数，函数体只有一句返回值
function fn1(n) {
  return n + 1;
}

const fn1 = (n) => n + 1;

// 2、函数没有参数，函数体只有一句返回值
function fn2() {
  return "hello world";
}
const fn2 = () => "hello world";

// 3、函数参数随意，函数体不仅仅是一句返回值
function fn3() {}
const fn3 = () => {};

// 用箭头函数和reduce数组求和
const arrSum = (arr) => arr.reduce((p, n) => p + n);

3、使用函数#

函数需要调用才会执行。

4、参数#

JS 数据传递只有值传递，没有引用传递。

4.1 形参与实参#

形参 形式上的参数，声明函数时候提供的参数。

实参 在调用时，实际使用的参数。

4.2 默认参数#

// ES6之前
function f(name, age) {
  age ? age : (age = 18);
}
// ES6
function f(name, age = 18) {}

4.3 剩余参数#

// 可以把不确定的参数表示成数组
function f(a, ...b) {
  return b;
}
f(1, 2, 3, 4); // [2,3,4]

4.4 argument 对象#

二者在传值的时候是一一对应的。二者的个数可以不相等，形参多的时候，多出来的自动被赋值 undefined，实参多的时候，多出来的放在**arguments**对象中。该对象在函数内部才有，实际上是一个类数组 array-like 的对象，可以访问这个对象获得传递给函数的每一个实参。函数中的形参可写可不写，即使不写形参也可以使用 arguments 对象获得，使用形参比较方便。arguments 对象的 length 属性用来确定传递进来多少个实参。可以通过 length 属性的值的不同（根据传入的参数个数的不同），进行不同的操作。arguments对象中保存着一个叫callee的属性，这个属性是一个指针，它指向拥有这个arguments对象的函数。

  • 使用 arguments 对象的 length 属性可以获取函数的实参个数。
  • argument 对象只能在函数内可见,因此 arguments.legth 也只能在函数体内使用。
  • 使用函数对象的 length 属性可以获取函数的形参个数,该属性为只读属性,在函数体内,体外都可以使用

callee 是 arguments 对象的属性，它引用当前 argument 对象所在的函数，使用该属性可以在函数体内调用自身。

// 检测函数的形参和实参是否一致
arguments.length != arguments.callee.length;

5、返回值#

return 没有指定值时函数返回的是undefined，函数在执行完该语句以后立即停止执行并退出函数。

返回值是一个函数的函数称为高阶函数。

错误与异常处理#

ECMA-262 规范了7种错误类型。其中Error是基类，其他6种错误类型是子类，都继承了基类。基类的主要作用是自定义错误对象。

1、错误类型#

类型 说明
Error 基类
name 可以读写异常类型（TypeError）
message 可以读写详细的错误信息。
EvalError 不正确使用eval()方法时抛出
SyntaxError 语法错误
ReferenceError 读取不存在的变量时抛出
TypeError 值的类型发生错误的时候抛出
RangeError 数字超出合法范围
URIError URI编码和解码错误

2、try…catch#

try{
    // 调试代码块
} catch (e) {
    // 捕获异常，并进行异常处理代码块
} finally {
    
}

一般情况下 try...catch 就够用了。

按顺序执行 try 子句中的代码，如果没有异常发生，将会忽略 catch 跳转到 finally 子句继续执行。

无法捕获逻辑错误。

3、throw#

抛出异常。

try {
    let num = 1 / 0;
    if (num = Infinity) {
        // 抛出错误
        throw new Error("除数不能为0");
    }
} catch (exception) {
    console.log(exception.message);
}

抛出错误之后，后续代码不会执行。

ES 运算符与类型转换#

一、算术运算符#

1、加法#

// 如果其中一个操作数是字符串，则会把另外一个操作数转换成字符串。
1 + "str"; // 1str
"str" + 1; // str1
1 + 1 + "str"; // 2str
1 + (1 + str); // 11str

// 如果都有是字符串，会拼接在一起
"str" + "is"; // stris

// 其他类型自动调用Number()进行转换
true + true; // 2
null + null; // 0
NaN + 1; // NaN
1 +
  undefined + // NaN
  // 放在非数值前面，则会自动调用Number()进行转换（不会影响本身）
  "123"; // 123

2、减法、乘法、除法#

// 如果有操作数是非字符，则自动调用Number()转换。
"99" - 0; // 99
"99" * 1; // 99
"99" / 1; // 99

// 特殊
1 / 0; // Infinity
Infinity * -1; // - Infinity
0 / 0; // NaN

3、取余#

3 % 5; // 3
5 % 0; // NaN

4、自增、自减#

++i 表示，i 自增 1 后再参与其它运算（先加后用）；而 i++ 则是 i 参与运算后，i 的值再自增 1。自增自减运算可能会改变原有变量类型。

var a = 100
1 + a++ // 101
1 + ++a // 102

// 会改变原变量类型
var a = true;
++a; // a = 2

var i = 1
i = i++ // i = 1

二、关系运算符#

// 如果两个操作数都是字符串，比较对应的字符编码
"234" > "123455" // true
// == 和 === 的区别
undefined == null // true
undefined === null // false
// 坑
null == 0 // false 相等比较不进行类型转换
null >= 0 // true

三、赋值运算符#

a += b; // 等价于 a = a + b

四、逻辑运算符#

&& 与

  • 第一步：计算第一个操作数
  • 第二步：如果第一个操作数转换成 false，那么就会结束运算，直接第一个操作数
  • 第三步：如果第一个操作数返回 true，则计算第二个操作数的值
  • 第四步：第二个操作数如果返回 true 则逻辑与返回第二个操作数，否则返回第一个操作数

|| 或

  • 第一步：计算第一个操作数的值。
  • 第二步：检测第一个操作数的值。如果左侧的表达式的值可以转换为 true，那么就会结束运算。直接返回第一个操作数的值
  • 第三步：如果第一个操作数可以转换为 false，则计算第二个操作数的值。并返回第二个操作数的值

! 非

1 && 2; // 返回2
"" && 1; // 返回''
"Cat" && "Dog"; // 返回”Dog"
"" && false; // f && f 返回 ""
false && ""; // f && f 返回 false

1 || 2; // 1
0 || 2; // 2
0 || null; // null
"" || false; // f || f 返回 false
false || ""; // f || f 返回 ""

短路原理#

在逻辑运算中，如果第一个操作数就能决定结果，那么第二个操作数就不会运行了。（就算运行了会报错也能顺利通过）

var a = false;
false && qwq();

ES 开始#

JavaScript 最初的目的是为了赋予网页生命。
这种编程语言我们称之为脚本。它们可以写在 HTML 中，在页面加载的时候会自动执行。脚本作为纯文本存在和执行。它们不需要特殊的准备或编译即可运行。

  • 1995 JavaScript、JScript 诞生
  • 1997 ECMA 给 JS 制定了标准ECMA-262，命名为ECMAScript，即 ES1.0。ES 是 JS 的规范标准，JS 是 ES 的实现。
  • 1999 发布 ES3，成为 JS 通用标准。
  • 2009 发布 ES5。
  • 2013 ES6 草案发布（其实就是之前不被接受的 ES4 激进方案）
  • 2015 发布ES6，即 ES2015，从此 JS 新版本按 ES+年份形式发布。ES6 是一个历史名词，也是一个泛指，代表 ES6 以后的新一代标准。

一、特点#

弱类型 数据类型不固定，可以随时改变。
解释型 解释型是一行一行解释运行，编译型是需要整体编译。
脚本语言 脚本是需要载体运行的。
基于对象/原型 只有对象，没有类，对象继承对象，而不是类继承类。原型对象是基于原型语言的核心概念。原型对象是新对象的模板，它将自身的属性共享给新对象。一个对象不但可以享有自己创建时和运行时定义的属性，而且可以享有原型对象的属性。

二、组成#

JavaScript 由 ECMAScript、BOM、DOM三部分组成，其中 ECMAScript （简称ES）是JS的核心语法规范。

  • ECMAScript 核心语法，简称 ES。
  • DOM Document Object Model 文档对象模型。
  • BOM Browser Object Model 浏览器对象模型。

三、网页中使用 JS 的方法#

（1）页面嵌入（常用）

<script>
// 这里写JS代码
</script>

（2）行内写法（几乎不用，局限性大，代码分离性差，可读性不强）
（3）外部文件引入（常用）

script 标签属性

  • src 指定外部脚本的 URI，如果设置了，标签内容会忽
  • type 指定脚本的语言，默认为 JS。
  • async 规定一旦脚本可用，则会异步执行。
  • defer 是否对脚本执行进行延迟。
<script src="链接"></script>

四、在网页中调试方法#

  • 控制台日志: console.log()、console.dir()
  • 系统对话框（弹窗）: alert()、confirm()、prompt()
  • 直接显示在文档上: document.write

CSS 动画#

transition: 过渡动画

transition-property: 属性
transition-duration: 间隔
transition-timing-function: 曲线
transition-delay: 延迟

1、过渡动画 transition#

允许 CSS 的属性在一定的时间区间内平滑过渡。在鼠标点击，鼠标滑过或对元素任何改变中触发，并平滑地以动画改变 CSS 的属性值。

transition

  • property 参与过渡的属性
  • duration 持续时间
  • timing-function 动画类型 linear 线性匀速
  • delay 延迟时间
transition: height 3s linear 2s;

animation / keyframes

animation-name: 动画名称，对应@keyframes
animation-duration: 间隔
animation-timing-function: 曲线
animation-delay: 延迟
animation-iteration-count: 次数

infinite: 循环动画

animation-direction: 方向

alternate: 反向播放

animation-fill-mode: 静止模式

forwards: 停止时，保留最后一帧
backwards: 停止时，回到第一帧
both: 同时运用 forwards / backwards

动画分为 过渡动画 和 关键帧动画。

2、关键帧动画 animation#

关键帧动画是控制关键位置，而不是所有位置。

1、@keyframes 定义新关键帧#

@keyframes <name> {
  from/0% {
    // ...
  }
  to/100% {
    // ...
  }
}

2、animation#

  • name 动画属性名
  • duration 关键帧持续时间
  • timing-function 动画执行方式（针对于每一个关键帧之间的动画）
  • delay 动画延迟时间
  • iteration-count 动画循环次数，次数或者 infinite（无穷次）
  • direction 控制动画循环反向播放，默认正常播放，alternate，轮流反向播放

3、动画的暂停和播放#

  • animation-play-state
  • paused

4、动画的起始和结束状态#

5、逐帧动画#

timing-function: steps()

响应式布局#

响应式布局是在 2010 年提出的一个概念，简而言之，就是一个网站能够兼容多个终端，而不是为每个终端做一个特定的版本。用户会使用不同的终端，由于屏幕分辨率不同，想要用户体验更好，就有必要让你的网页兼容多个终端。响应式布局设计可以让一个网站适配多种设备和多个屏幕，可以让网站的布局和功能随用户的使用环境（屏幕大小，输入方式，设备/浏览器能力）而变化。

三个特征

  • 媒体查询：检查设备在渲染我们作品时的物理特性
  • 流动布局
  • 灵活的视觉效果

LESS#

less 是一种动态样式语言，属于 CSS 预处理器，扩展了 CSS 语言，增加了变量、混合、函数等特性，使 CSS 更易维护和扩展。LESS 既可以在客户端浏览器上运行，也能借助 node.js 在服务器端运行。

CSS 自诞生以来，基本语法和核心机制一直没有本质上的变化，它的发展几乎全是表现力层面上的提升。最开始 CSS 在网页中的作用只是辅助性的装饰，轻便易学是最大的需求；然而如今网站的复杂度已经不可同日而语，原生 CSS 已经让开发者力不从心。当一门语言的能力不足而用户的运行环境又不支持其它选择的时候，这门语言就会沦为 “编译目标” 语言。开发者将选择另一门更高级的语言来进行开发，然后编译到底层语言以便实际运行。于是，在前端领域，CSS 预处理器应运而生。而 CSS 这门古老的语言以另一种方式 “重新适应” 了网页开发的需求。

1、安装#

中文 | 英文 | 源码

  1. 安装 Node.JS
  2. 通过命令行程序，node -v 和 npm -v 检查是否安装成功。
  3. 使用npm i -g less 进行全局安装 less, 并输入 lessc -V 检查版本

  • 动态样式 CSS 是非程式式语言，需要书写大量没有逻辑的代码，不方便维护和扩展、复用。

弹性布局#

flex ，即弹性布局。

1、开启弹性布局#

display: flex;
display: -webkit-flex; /* Safari */

开启 flex 的元素自身不受影响，只会影响子元素，子元素的float、clear和vertical-align属性将失效。对子元素的子元素不会影响。
a
开启flex的盒子称为 容器（container），子元素称为 项目（item）。横向默认是 主轴（main axis）。纵向默认是 侧轴（交叉轴，cross axis）。默认情况下，项目沿着主轴从左到右依次排列。项目的高度撑满容器（侧轴撑满）。

当项目的总宽度超出，并不会换行，而是压缩项目的宽度。

涉及到 12 个 CSS 属性，父容器，子容器各 6 个。

2、常用属性#

flex-direction#

默认情况下，主轴的方向是从左向右的，交叉轴垂直于主轴。

交叉轴是由主轴决定的，主轴又是由 flex-direction 决定的，这个属性设置在父容器上，才可以生效。

flex-direction: row | row-reverse | column | column-reverse;

CSS 定位

发表于 2021-01-31 更新于 2021-02-01 分类于 2103：CSS布局

定位#

定位（position） 允许你定义元素框相对于其正常位置应该出现的位置，或者相对于父元素，以及浏览器窗口本身的位置。

属性 描述
static 默认，没有定位
relative 相对定位
absolute 绝对定位
fixed 固定定位

布局一般用在大布局的小范围中。

1、相对定位 - relative#

相对定位（relative），让元素相对定位，元素先放置在未添加定位时候的区域，然后在不改变页面布局的情况下（其他元素没有受到任何影响；自身原来的位置也保留；和浮动可以一起使用）进行移动。不会脱离文档流。

2、绝对定位 - absolute#

绝对定位（absolute），相对于它的 包含块 定位。绝对定位是相对于它的包含块定位的（和自身原有位置无关）。如果没有任何一个父级元素是非 static 属性，则会相对于文档（html）定位。如果它的父级元素和爷爷级元素都是非 position:static 属性，则它会选择距离最近的父元素。脱离正常文档流，原有位置不保留。定位的原点以包含块的 padding-box 的左上角为原点。right 和 bottom 是距离包含块右下角的距离。

包含块#

包含块 是视觉格式化模型的一个概念，简单来说就是一个定位参考框。

如何确定一个元素的包含块，完全取决于它自身的 position 属性：

  • 如果一个元素自身的 position 属性是 static 或者是 relative ，它的包含块就是离他最近的祖先元素或者是格式化上下文。
  • 如果一个元素自身的 position 属性是 absolute ，它的包含块就是离他最近的拥有定位属性（值不为 static ）的元素。
  • 如果一个元素自身的 position 属性是 fixed，它的包含块就是 viewport。

补充：
如果一个元素的 position 属性是 absolute 或者是 fixed 在下边几种情况下，包含块会发生改变。

  • 当祖先元素拥有 transform 或 perspective 属性并且值不为 none 的时候，它也是被当做包含块。
  • 当祖先元素 拥有 filter 属性的时候（值不为 none） 它也可以被当做包含块

注：如果由内向外找不到包含块条件的元素，那么 html（根元素）被称作为初始包含块。

子绝父相#

父元素相对定位并不设置位移，子元素绝对定位，设置位移。

3、固定定位 - fixed#

固定定位（fixed），相对于浏览器窗口定位。

4、粘性定位 - sticky#

sticky 的意思是“粘性的”，position:sticky 表现也符合这个粘性的表现。基本上，可以看出是position:relative和position:fixed的结合体——当元素在屏幕内，表现为relative，就要滚出显示器屏幕的时候，表现为fixed。

nav {
  position: -webkit-sticky;
  position: sticky;
  top: 0;
}
