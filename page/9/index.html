ES 面向对象#

面向对象和面向过程是两种不同的编程思想和方法。
面向过程是以过程为中心，就像流水线一样设计程序的思想。面向对象是以对象为中心的编程思想。更符合人类的思维习惯。
在面向对象的概念中，类是一类相似事物的统称，对象是类的实例化，类是对象的模版。（JS 中没有具体类的概念，ES6 引入的类实际上是函数。）
面向对象的三个基本特性: 封装、继承、多态。
对象（Object） 可以是一个变量，一个函数，一种数据结构。在 JS 中，一切皆为对象。JS 允许自定义对象，从狭义的概念来分析，对象是最基本的数据类型，是复合型的结构、引用型的数据，它是无序数据集合。对象中每个成员被称为属性。对象是无序的键值对，数组是有序的键值对，函数是一种可执行的特殊对象。

跨域解决方案#

一、同源策略#

Same-Origin Policy，同源策略 最早由 Netscape 公司提出，是浏览器的一种安全策略。该策略表示协议、域名、端口号必须完全相同。

违背同源策略就是跨域，主要是针对 ajax 而言。

二、JSONP#

JSON with Padding，JSONP 是一个非官方的跨域解决方案，只支持 get 请求，而且需要服务器的配合。

1、工作原理#

在网页有一些标签天生具有跨域能力，比如 img link iframe script。JSONP 就是利用 script 标签的跨域能力来发送请求的。

2、应用示例#

客户端

服务器

三、CORS#

Cross-Origin Resource Sharing，CORS，跨域资源共享是官方的跨域解决方案，它的特点是不需要在客户端做任何特殊的操作，完全在服务器中进行处理，支持 GET 和 POST 请求。是通过设置一个响应头来告诉浏览器，该请求允许跨域，浏览器收到该响应以后就会对响应放行。

四、服务器代理#

服务器和服务器之间没有同源策略限制，所以可以通过代理的方式进行跨域。

LESS#

less 是一种动态样式语言，属于 CSS 预处理器，扩展了 CSS 语言，增加了变量、混合、函数等特性，使 CSS 更易维护和扩展。LESS 既可以在客户端浏览器上运行，也能借助 node.js 在服务器端运行。

CSS 自诞生以来，基本语法和核心机制一直没有本质上的变化，它的发展几乎全是表现力层面上的提升。最开始 CSS 在网页中的作用只是辅助性的装饰，轻便易学是最大的需求；然而如今网站的复杂度已经不可同日而语，原生 CSS 已经让开发者力不从心。当一门语言的能力不足而用户的运行环境又不支持其它选择的时候，这门语言就会沦为 “编译目标” 语言。开发者将选择另一门更高级的语言来进行开发，然后编译到底层语言以便实际运行。于是，在前端领域，CSS 预处理器应运而生。而 CSS 这门古老的语言以另一种方式 “重新适应” 了网页开发的需求。

一、显示#

1、display - 控制元素的显示模式 常#

控制元素的显示状态。

display 描述说明
inline 让元素以行属性显示
block 让元素以块属性显示
inline-block 让元素以行内块属性显示
none 带着所有子元素隐藏，直接消失
flex 开启弹性布局

display:none 隐藏的元素，不可以被 js 获取 DOM 节点及相关的样式属性。让隐藏的元素显示，只需要修改 display 属性的值不为 none 即可。

2、visibility - 控制元素显示或者隐藏#

visible: 元素显示
hidden: 元素隐藏
元素依然占用空间，和隐身一样，带着子元素一起隐身了。
子元素的隐藏是因为继承了父级的 hidden 值，所以单独给子元素设置，覆盖 hidden，即可显示。
使用 visiblity 隐藏的元素是可以用 js 获取的 DOM 节点及相应的宽高等属性。

visbility:hidden 和 display:none 隐藏的区别

①visibility 可以继承，子元素是因为继承了才隐藏（可覆盖）。display 不能被继承，直接带着所有内置元素直接隐藏。
②visbility 隐藏，保留原先占用空间。display 隐藏使原来的空间被占用。
③js 可以获取到 visibility 隐藏元素的可视化宽高，而获取不到 display 隐藏元素的可视化宽高。

3、overflow - 溢出设置 常#

内容溢出设置。

overflow 描述
visible 默认值
超出正常显示，但是不会影响其他元素的位置
hidden 超出部分隐藏
scroll 无论是否超出都生成滚动条
auto 自动生成滚动条，当内容溢出生成滚动条，否则不生成

overflow-x 和 overflow-y 可以单独控制一个方向的内容溢出设置。

隐藏横向滚动条：overflow-x: hidden

4、opacity - 透明度设置#

控制元素不透明度
值是 1-0
1 是完全不透明
0 是完全透明
opacity 直接控制所有的内部元素的不透明度，无论是什么元素都会改变

1、样式操作#

样式重置（CSS reset），将 html 的默认样式全部去掉，需要的时候我们自行添加。统一页面风格。

为什么要用样式重置

  • 多数元素拥有自己的默认样式，并且我们很多都不需要。
  • 默认样式在不同浏览器中呈现的也不一定相同，就会导致浏览器展示页面不相同的现象。
  • 整个页面中固定的风格代码，可以在样式重置中直接书写，不用每一个都设置。

为什么不用通配符重置

  • 重置简单，不够完整，一些 demo 可以使用，真正项目不推荐使用，请使用 CSS reset
  • 通配符，匹配所有的标签，影响很大，效率很低。

CSS 盒模型#

在 HTML 中把每个元素当成一个盒子，拥有盒子的平面外形和空间。

1、盒模型标准#

通过 box-sizing 设置盒子模型的标准，分为

  • content-box 默认，W3C 标准盒模型
  • border-box IE 盒模型，也称为怪异盒模型

注，理论上还有 padding-box 和 margin-box ，但是现在 W3C 和 MDN 规范均不支持。

怪异盒模型的 width / height 是 border 的 尺寸。

2、内容层 content#

width/height#

width 设置的坑

  • 如果设置宽度为 100%，并且有 border margin padding 等属性值的时候，宽度可能超出。因为 width 内容区域已经和父级一样了，所以可以不设置宽度或者设置宽度为 auto 那么内容的宽度就是可以充分利用可使用空间。

height 设置的坑

  • 如果高度设置百分比，必须父级有一个显性的设置一个 height 属性，并且不能是 auto。

min/max-width/height#

正常 最小 最大
宽度 width min-width max-width
高度 height min-height max-height

当 min-width 和 max-width 冲突的时候，min-width 是优先的。当 width 和 min 或 max 冲突以后，min 或者 max 直接无视 width 的存在（无论是否添加 ！important）。

3、内边距层 padding#

padding 的写法和 margin 一致

padding 主要撑开内容和边框之间的距离

padding 不支持负值

padding 对所有属性的元素都支持，但是对于行内元素撑不开上下的距离

padding 可以用于扩大点击范围

4、边框层 border#

边框属性#

border

  • border-width 线的宽度
  • border-style 线的类型
    • solid 实线
    • dashed 虚线
    • dotted 点状线
  • border-color 线的颜色
  • border:none 不设置边框

边框还分为 4 个方向 要么直接设置 4 个方向，要么一个个方向单独设置

练习：边框画三角形

5、外边距层 margin#

margin 的作用是撑开当前元素和其他元素的距离。margin 有四个方向，可以分别设置margin-left margin-right margin-top margin-bottom，margin 的 left 和 top 是让当前元素向右向下移动，margin 的 right 和 bottom 是让元素的兄弟元素向右向下移动，不影响自身。

合写#

  • 4 个值：上 右 下 左（从上开始顺时针）
  • 3 个值：上 左右 下
  • 2 个值：上下 左右
  • 1 个值：上下左右

水平方向上的 auto#

因为块标签拥有满行属性，但是设置宽度以后，剩余的空间就是默认的 margin，设置左右的 margin 值为 auto，那么剩余的 margin 就会平分到左右两边。

竖直方向的 auto#

相对于水平方向来说，块元素在垂直方向上并没有满屏的属性，margin 默认在上下的值都是 0，并没有剩余空间去平分。

塌陷和边框合并问题#

  • 父级塌陷：父级的第一个子元素的上 margin 和最后一个子元素的下 margin 会进行塌陷给父级，这样的设计是为了方便父级和他的兄弟进行垂直 margin 的合并，避免缝隙过大。
  • 解决塌陷：给父级开启 BFC，给父级设置边框。
  • 兄弟的 margin 垂直方向会进行折叠，取最大值 。

margin 负值#

  • margin-top 为负值：元素向上走，不会影响上边的元素的位置（会出现重叠），并且下边的兄弟元素会跟着一起向上走。
  • margin-left 为负值：元素向左走，不会影响前边的元素位置(会出现重叠)，右边兄弟元素会跟随移动。
  • margin-right 为负值：元素不会移动，形态大小也不会发生改变，但是所占用的位置会减少（负 margin 的值），所以，后边的兄弟元素会自动向左走。
  • margin-bottom 为负值：元素不会移动，形态大小也不会发生改变，但是所占用的位置会减少（负 margin 的值），所以，后边的兄弟元素会自动向上走。

支持性#

margin 完美支持块元素和行内块元素，不支持行元素的垂直方向。

2、标签的分类#

（1）块标签 block

  • 代表标签: div
  • 默认的宽度是占据父标签全部内容，即 width: 100%
  • 默认的高度由内容决定
  • 设宽高、边距、边框没有限制
  • 独占一行
  • 允许嵌套块元素和行元素（p/h标签除外）

  • 代表标签: span
  • 默认的宽高由内容决定。

  • 共用一行：在一行中如有剩余空间，允许同其他行内/行内块标签共用。
  • 某些动画样式不能添加和使用。

（3）行内块标签 inline-block

  • 代表标签: img、button、input
  • 默认的宽高由内容决定。
  • 内容过多时不会断开。
  • 可以设置宽高。
  • 在一行中如有剩余空间，允许同其他 行内/行内块 标签公用一行。
  • 行内块和内行标签会有行高的概念。
  • 行内块受父元素的 line-height 以及自身的 vertical-align 影响。

将非块级元素转换成块级元素的方法

ES Map#

ES6+

类似于对象的一种数据结构，是键值对的集合，但是键的范围不限于字符串，各种类型的值（包括对象）都可以当作键。也就是说，Object 结构提供了“字符串—值”的对应，Map 结构提供了“值—值”的对应，是一种更完善的 Hash 结构实现。如果你需要“键值对”的数据结构，Map 比 Object 更合适。

API#

  • size属性返回 Map 结构的成员总数。
  • set方法设置键名key对应的键值为value，然后返回整个 Map 结构。如果key已经有值，则键值会被更新，否则就新生成该键。set方法返回的是当前的Map对象，因此可以采用链式写法。
  • get方法读取key对应的键值，如果找不到key，返回undefined。
  • has方法返回一个布尔值，表示某个键是否在当前 Map 对象之中。
  • delete方法删除某个键，返回true。如果删除失败，返回false。
  • clear方法清除所有成员，没有返回值。
  • keys()：返回键名的遍历器。
  • values()：返回键值的遍历器。
  • entries()：返回所有成员的遍历器。
  • forEach()：遍历 Map 的所有成员。

字符串 API#

字符串是基本类型，所有的方法都无法修改原字符串。

查找 - indexOf / lastIndexOf#

indexOf() 返回指定字符串在字符串首次出现的位置（向后查找，对大小写敏感），如果没找到，返回-1。

lastIndexOf() 同 indexOf() ，区别是逆序的（向前查找）。

查找 - includes#

includes() 查找字符串是否包含指定字符串（对大小写敏感），如果找到返回true ,否则返回false。

查找 - search#

search()功能和indexOf()相同，定义匹配模式，只有一个参数，没有反向检索和全局模式。（正则表达式常用）

查找 - match#

match() 找出所有匹配的子字符串，并以数组的形式返回，如果匹配模式没有指定全局（正则表达式），则进行单次匹配，如果没找到返回null。该方法类似 indexOf() 和 lastIndexOf()，但是它返回指定的值，而不是字符串的位置。

查找 - charAt/charCodeAt#

charAt(n) 返回字符串中的第 n 个字符，如果 n 超出字符串范围，返回空串。

charCodeAt(n) 返回字符串中第 n 个字符的字符编码。

判断头尾 - startsWith/endsWith#

startsWith() endsWith() 判断字符串开头、结尾有没有包含某个字符串。

切成数组 - split#

split() 把一个字符串切割成数组，这个操作和数组join是相反的。

截取 - substr/slice/substring#

substr 根据指定长度截取字符串（不推荐使用该方法）

slice 、substring 根据指定的起止下标截取字符串

起始和结束位置相比较的大小无法确定的时候可以使用substring。

如果为负值，slice 能把负值当作从右向左。而substring视其无效

大小写转换 - toLowerCase / toUpperCase#

toLowerCase() 将字符串转换成小写

toUpperCase() 将字符串转换成大写

拼接 - concat 了解#

concat(s1[, s2, s3])

功能：把多个参数添加到指定字符串的尾部。

参数：要添加到字符串尾部的，参数类型和个数没有限制。

返回：连接以后的字符串。

原来数组的 join 方法替代 concat 方法，速度很快，但是现代浏览器优化加号拼接，所以推荐使用加号拼接。

复制 - repeat#

repeat() 复制字符串。

补全 - padStart / padEnd#

padStart() padEnd() 是 ES2017 引入的字符串补全长度的方法。如果某个字符串不够指定长度，会在头部或尾部补全。

替换 - replace#

如果没有使用正则表达式，只能替换一次。

这是用于搜索和替换的通用方法，是最有用的方法之一。它是搜索和替换字符串的瑞士军刀。

当 replace 的第一个参数是字符串时，它仅替换第一个匹配项。

删首尾空格 - trim / trimStart / trimEnd#

trim() trimStart() trimEnd() 去除字符串的头尾空格。

转换 - valueOf / toString#

valueOf() 返回对象的字符串、数值或者布尔值表示。

toString() 返回对象的字符串表示。

函数#

函数是可以反复使用的并且命名的代码块。本质上也是一种属于对象类型的数据。与其他数据类型不同的是，函数的值是可执行的代码。分为系统函数和自定义函数。

函数是数据，函数名是这个数据的标识。需要访问函数的数据去掉圆括号就好。一个函数可以有多个标识。

1、函数的作用#

  • 解决代码的冗余问题，实现代码的复用。
  • 封装代码，让函数内部的代码对外部不可见。
  • 可以让整个代码项目通过函数模块化。

2、新建函数#

函数名定义的规则遵循标识符的规则。函数的声明函数名可以重复，一旦重名，前面的覆盖后面的。

2.1 声明函数 常#

var 和 function 都是声明语句，它们声明的变量和函数都在 JS 预编译时被解析，也被称为变量提升和函数提升。

ES5 中，不允许函数书写在非函数的代码块中（比如 if 语句中）。

2.4 箭头函数 ES6 常#

  • 内部没有this ，箭头函数与包围它的代码共享一个this。
  • 不能用于构造函数（不能被new 实例化）。
  • 内部没有没有argument对象，可用rest参数代替。
  • 箭头函数不能使用yield命令，即不能用作Generator函数。

4.4 argument 对象#

二者在传值的时候是一一对应的。二者的个数可以不相等，形参多的时候，多出来的自动被赋值 undefined，实参多的时候，多出来的放在**arguments**对象中。该对象在函数内部才有，实际上是一个类数组 array-like 的对象，可以访问这个对象获得传递给函数的每一个实参。函数中的形参可写可不写，即使不写形参也可以使用 arguments 对象获得，使用形参比较方便。arguments 对象的 length 属性用来确定传递进来多少个实参。可以通过 length 属性的值的不同（根据传入的参数个数的不同），进行不同的操作。arguments对象中保存着一个叫callee的属性，这个属性是一个指针，它指向拥有这个arguments对象的函数。

  • 使用 arguments 对象的 length 属性可以获取函数的实参个数。
  • argument 对象只能在函数内可见,因此 arguments.legth 也只能在函数体内使用。
  • 使用函数对象的 length 属性可以获取函数的形参个数,该属性为只读属性,在函数体内,体外都可以使用

callee 是 arguments 对象的属性，它引用当前 argument 对象所在的函数，使用该属性可以在函数体内调用自身。

5、返回值#

return 没有指定值时函数返回的是undefined，函数在执行完该语句以后立即停止执行并退出函数。

返回值是一个函数的函数称为高阶函数。
