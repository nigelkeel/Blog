HTML & 网页基础#

一、基本结构#

<!-- 版本声明 -->
<!DOCTYPE html>
<!-- 根文档 -->
<html lang="en">
  <!-- 头部 -->
  <head>
    <meta charset="utf-8" />
    <title>标题</title>
  </head>
  <!-- 主体 -->
  <body></body>
</html>

1、注释#

用<!-- 和-->包裹起来。

2、文档版本声明#

<!DOCTYPE html>

给浏览器声明，告诉浏览器应该按照 HTML5 的规范解析当前页面。 如果不写，浏览器就是按照自己默认的模式解析。

版本声明必须写在第一行。我们不把他看做一个 html 标签，而是把它当做一个指令，并且这个指令对大小写不敏感。

<!-- html 4 的 版本说明(了解) -->
<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

3、html 、head、body 标签#

html 是一个双标签，所有网页内容全部书写在 html 标签中。lang就是 html 标签的属性，en就是lang属性的值，属性是对当前标签的信息的扩展。

head 标签代表网页的头部，不会显示在网页中。主要是包含了标题，网页的描述，字符编码等等一些信息。

body 标签是网页显示的主体内容。

4、meta 标签#

控制网页的元信息（信息的信息），根据属性的不同，有很多的作用。

5、解决网页乱码问题#

<meta charset="utf-8" />

charset即字符编码。utf-8称作为万国码。

Hexo 内嵌脑图

Hexo 内嵌脑图#

工具:

  • MindManager 2020
  • Hexo

第一步: MindManager 导出，选择 HTML5 交互式导图，选择局部导出
第二步: 在 Hexo 博客目录的 source 目录下新建一个 map 文件夹（也可以是其他名字）
第三步: 编辑 Hexo 博客目录下的 _config.yml ，搜索 skip_render，在下面添加 "map/**"，保存。
第四步: hexo clean
第五步: 在你想插入导图的页面:

<iframe src="../map/1.html" width="1000" height="1000" frameborder="0"></iframe>

Hexo#

https://hexo.io/

一、开始#

1、安装#

npm install hexo-cli -g

2、新建博客#

hexo init 博客名称
cd 博客名称
npm i

3、常用命令#

# 启动博客（本地服务器）打开 localhost:4000 查看
hexo s

# 清除缓存
hexo clean

# 构建
hexo g

# 把代码推送到远程仓库
hexo d

4、博客配置#

博客根目录下的 _config 为 博客配置文件
主题目录下的 _config 为 主题配置文件

5、主题配置#

主题库

这边推荐 NEXT 主题。

安装NEXT主题#

git clone https://github.com/theme-next/hexo-theme-next themes/next
cd themes/next
npm i

修改 博客根目录下 _config 文件 里面的主题为 next

6、Github Page / Gitee Page#

npm install hexo-deployer-git --save
# _config.yml

# git 配置
deploy:
  type: git
  repo: https// .... .git
  branch: master

推上去

hexo d

二、写作#

1、YAML Front Matter#

---
title: 这是标题

## 这是创建时间、更新时间
date: 2020-10-01
updated: 2020-12-21

## 这是标签
tags: 标签
tags: [标签一, 标签二]


---

三、插件#

插件库

目录生成#

npm install hexo-toc --save
# _config.yml

toc:
  # 目录深度，最多生成三级目录
  maxDepth: 3

搜索#

npm install hexo-generator-search --save

代码高亮#

npm i -S hexo-prism-plugin

然后，修改 Hexo 根目录下 _config.yml 文件中 highlight.enable 的值为 false，并新增 prism 插件相关的配置，主要配置如下：

prism_plugin:
  mode: 'preprocess'    # realtime/preprocess
  theme: 'tomorrow'     #主题选项
  line_number: false    # default false
  custom_css:

可用的主题有

coy,dark,funky,okaidia,solarizedlight,tomorrow,twilight,a11y-dark,atom-dark,base16-ateliersulphurpool.light,cb,darcula,dracula,duotone-dark,duotone-earth,duotone-forest,duotone-light,duotone-sea,duotone-space,ghcolors,hopscotch,material-dark,material-light,material-oceanic,pojoaque,shades-of-purple,synthwave84,vs,xonokai,default,

  34. Color Info：提供你在 CSS中使用颜色的相关信息。你只需在颜色上悬停光标，就可以预览色块中色彩模型的（HEX、 RGB、HSL 和 CMYK）相关信息了。
