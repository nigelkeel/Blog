防抖&节流#

节流#

假设有一个场景，滚动就会发起网络请求，但是我们不希望用户在滚动过程中一直发请求，而是隔一段时间发一次，对于这种情况，我们可以使用节流。

防抖#

假设有一个按钮，用户点击一次就会触发网络请求，但是我们不希望用户一直点这个按钮发请求，而是这个用户点击这个按钮一段时间内没有再去点击才去发请求，对于这种情况，我们可以使用防抖。

230504：跨域解决方案

发表于 2021-01-30 分类于 2504：网络请求

跨域解决方案#

一、同源策略#

Same-Origin Policy，同源策略 最早由 Netscape 公司提出，是浏览器的一种安全策略。该策略表示协议、域名、端口号必须完全相同。

违背同源策略就是跨域，主要是针对 ajax 而言。

二、JSONP#

JSON with Padding，JSONP 是一个非官方的跨域解决方案，只支持 get 请求，而且需要服务器的配合。

1、工作原理#

在网页有一些标签天生具有跨域能力，比如 img link iframe script。JSONP 就是利用 script 标签的跨域能力来发送请求的。

2、应用示例#

客户端

<script>
  const script = null;
  // 这是一个事件触发
  ..{
      // 新建一个 script 标签
      script = document.createElement("script");
      // 设置需要跨域的URL
      script.src = "http://.../test?a=1&cb=jp";
      // 插入到页面
      document.body.appendChild(script);
  }
  // 回调函数
  function jp(n){
      // 这边要移除script 标签
      document.body.removeChild(script);
      // 后续操作
  }
</script>

服务器

res.end(cb(xxx));

三、CORS#

Cross-Origin Resource Sharing，CORS，跨域资源共享是官方的跨域解决方案，它的特点是不需要在客户端做任何特殊的操作，完全在服务器中进行处理，支持 GET 和 POST 请求。是通过设置一个响应头来告诉浏览器，该请求允许跨域，浏览器收到该响应以后就会对响应放行。

// 服务端
app.all("*", (req,res)=>{
    // 设置白名单
    const arrOrigin = [
        "http://xxxxx",
        "xxxx"
    ]
    // 如果访问的URL在白名单中
    if(arrOrigin.oncludes(req.header.origin)){
        // 允许跨域
        res.header("Access-Control-Allow-Origin"),req.header.origin);
        // 允许携带内容
        res.header("Access-Control-Allow-Headers", "content-type")
    }
    next();
})

四、服务器代理#

服务器和服务器之间没有同源策略限制，所以可以通过代理的方式进行跨域。

220405：ES Map

发表于 2021-01-30 分类于 2204：ES6

ES Map#

ES6+

类似于对象的一种数据结构，是键值对的集合，但是键的范围不限于字符串，各种类型的值（包括对象）都可以当作键。也就是说，Object 结构提供了“字符串—值”的对应，Map 结构提供了“值—值”的对应，是一种更完善的 Hash 结构实现。如果你需要“键值对”的数据结构，Map 比 Object 更合适。

//为什么会有Map，因为对象属性名称必须是字符串，如果是其他类型则不行
let p1 = { name: "lily" };
let obj1 = {
  id: 1,
  [p1]: "good",
};
console.log(obj1);

// Map也是新增的数据解构  类似于对象
let mp1 = new Map([
  ["a", 1],
  ["b", 2],
]);
console.log(mp1);

let p2 = { name: "lily" };
let mp2 = new Map([
  ["a", 1],
  [p2, 2],
]);
console.log(mp2);

API#

  • size属性返回 Map 结构的成员总数。
  • set方法设置键名key对应的键值为value，然后返回整个 Map 结构。如果key已经有值，则键值会被更新，否则就新生成该键。set方法返回的是当前的Map对象，因此可以采用链式写法。
  • get方法读取key对应的键值，如果找不到key，返回undefined。
  • has方法返回一个布尔值，表示某个键是否在当前 Map 对象之中。
  • delete方法删除某个键，返回true。如果删除失败，返回false。
  • clear方法清除所有成员，没有返回值。
  • keys()：返回键名的遍历器。
  • values()：返回键值的遍历器。
  • entries()：返回所有成员的遍历器。
  • forEach()：遍历 Map 的所有成员。

ES Set#

ES6+

一种数据结构，类似数组，但是成员的值都是唯一的，不能重复。

let set = new Set([1, 2, 2, 3]);
console.log(set); // Set {1, 2, 3}

API#

api 描述
size 返回 Set 的长度
add() 添加一个值，返回 Set 结构本身
delete() 删除一个值，返回一个布尔值
has() 判断是否存在某成员，返回一个布尔值
clear() 清除所有成员
keys() values 遍历键值，同 for..of
entries() 遍历返回键值对
forEach() 使用回调函数遍历每个成员

应用#

去除数组重复成员

[...new Set(array)];

去除字符串里面重复字符

[...new Set(str)], join("");

Symbol#

保证对象的每个属性的名字都是独一无二的，从根本上面防止属性名冲突。

一、使用#

1、不用加 new

let s = Symbol()
typeof s // "symbol"

2、可以加描述符

let s = Symbol('foo') 
s.toString() // 'Symbol(foo)'
``` 

3、`Symbol.prototype.description`
> ES2019新增
```js
const s = Symbol('foo')
s.description // "foo"

4、Symbol.for / Symbol.keyFor
有时，我们希望重新使用同一个Symbol值，Symbol.for()方法可以做到这一点。它接受一个字符串作为参数，然后搜索有没有以该参数作为名称的 Symbol 值。如果有，就返回这个 Symbol 值，否则就新建一个以该字符串为名称的 Symbol 值，并将其注册到全局。

let s = Symbol.for('foo')
let s2 = Symbol.for('foo')
s1 === s2 // true

Symbol.keyFor () 方法返回一个已登记的 Symbol 类型值的 key。

let s = Symbol.for('foo')
Symbol.keyFor(s) // "foo"

let s2 = Symbol('foo')
Symbol.keyFor(s2) // undefined

二、遍历#

无法遍历: for...in、for...of、Object.keys()、Object.getOwnPropertyNames()、JSON.stringify()

可以遍历：
Object.getOwnPropertySymbols()

const obj = {}
let a = Symbol('a')
let b = Symbol('b')

obj[a] = "Hello"
obj[b] = "World"

const objectSymbols = Object.getOwnPropertySymbols(obj)
// [Symbol(a), Symbol(b)]

Reflect.ownKeys()

let obj = {
  [Symbol('key')]: 1,
  enum: 2,
  nonEnum: 3
}
Reflect.ownKeys(obj)
// ["enum", "nonEnum", Symbol(key)]

三、应用#

1、作为对象的属性名

// 第一种写法
let a = {}
a[mySymbol] = "hello"

// 第二种写法
let b = {
  [mySymbol]: "hello"
}

// 第三种写法
let c = {}
Object.defineProperty(c, mySymbol, { value: "hello"})

// 以上写法都得到同样的结果
// 只能用[]不能用.取值
a[mySymbol]

创建一个时间

ES Math#

取整#

// parseInt 保留整数部分
parseInt(3 / 2); // 1
// floor 向下取整（丢弃小数部分）
Math.floor(3 / 2); // 1
// ceil 向上取整
Math.ceil(3 / 2); // 2
// round 四舍五入
Math.round(3 / 2); // 2

获取随机数#

获取 0~1 之间的随机数。

Math.random();

获取指定范围的随机整数（常用）

// 公式（MAX，最大值；MIN，最小值）
const Result = Math.floor(Math.random() * (MAX - MIN + 1) + MIN);
// 简写
const Result = ~~(Math.random() * (MAX - MIN + 1) + MIN);

为什么不用 ceil 和 round 取整而用 floor

  • 用 ceil 的话，第一个数被随机到的概率极小。
  • 用 round 的话，第一个和最后一个的概率比中间各项的概率低 50%。
