正则表达式

正则表达式（regexp） 是搜索和替换字符串一种强大的方法。在 JavaScript 中，正则表达式通过内置的 RegExp 类的对象来实现，并和字符串集成。

一、基本语法#

正则表达式包含 模式 和可选的 修饰符。

1、search#

创建一个正则表达式有两种语法：

// 1st 允许从字符串动态构造模式
reg = new RegExp("pattern","flags");

// 2nd 常用
reg = /pattern/;
reg2 = /pattern/gmi;

如果要在字符串中搜索，可以使用 search 方法。这个方法返回匹配项的索引位置，如果没找到返回-1。

示例：

let str = "I love you!";
let reg = /love/;

console.log( str.search(reg) ); // 2

2、修饰符#

正则表达式的修饰符影响搜索结果，在 JS 中有五个修饰符：

修饰符 描述
i 搜索不区分大小写
g 搜索会查找所有的匹配项，而不是第一个
m 多行模式
u 开启完整的unicode支持
y 粘滞模式

一、NodeJS 模块化#

NodeJS 借鉴了 Common 规范实现了一套模块系统，称为 CommonJS 模块化系统。

1、模块暴露#

默认情况下模块内部代码对于外部来说都是不可见的，可以通过以下方式向外部暴露变量和函数。

（1）通过 module.exports 来向外部暴露变量和函数

module.exports 是导出的对象。

无论添加还是直接覆盖，他永远都是导出的对象。

如果是导出一个对象，可以写

module.exports.obj = obj;
//或者
module.exports = {obj:obj}

如果导出一组对象，可以写

moudle.exports.obj = {obj:obj, obj2:obj2};

如果直接导出

module.exports = fun; 
// 导出的对象是 fun 函数

例子

function add(x, y) {
    return x + y;
}
// 暴露出去
module.exports = add;

（2）将变量和函数设置为 exports 的属性来暴露变量和函数为了方便，Node为每个模块提供一个exports变量，指向module.exports，例如： module.exports.fun = …，相当于exports.fun = ...。

exports 本身不具备导出功能，但是修改它的时候，其实是修改了module.exports 。exports 是指向 moudlue.exports 的对象地址的一个新对象。（exports = xxx 的用法是错误的，这样使其不再绑定。正确的用法是 exports.xxx = xxx ）

不能将一个值赋值给exports，而是使用exports.XXX来暴露。否则这样它将不再绑定到module.exports。如果exports导出的变量类型是引用类型如函数，则会断开与module.exports的地址指向，导致变量导出失败。因为最终还是要靠module.exports来导出变量的。

例子

function add(x, y) {
    return x + y;
}
// 暴露出去
exports.add = add; // { add: [Function: add] }

2、模块导入#

通过 require() 来引入外部的模块

模块分为三种类型。

（1）系统模块

const path = require("path");

（2）第三方模块

（3）自定义模块

可以不书写后缀名：默认按照 js json node 顺序解析

一定要加上./ 或者 ../ 之类的路径 否则可能解析错误

==>

console.log(obj) // { o1: {...}, o2: {...} }

const {f1, f2} = require("./mod/test");

// (4)

二、ES6 模块化#

CommonJS 模块输出的是一个值的拷贝，ES6 模块输出的是值的引用。

ES6的模块化分为导出（export）与导入（import）。

1、导出#

使用关键字 export 导出模块。

// 单个导出
export let a = 1;
export let b = 2;
export funcion fn(){
    console.log("abc");
}

// 批量导出
let c = 3;
let d = 4;
export {c, d as e}; // 这里也能定义别名

2、导入#

使用关键字 import 导出模块。

<script type="module">
	import {a, b, fn} from "./module/mo.js"
</script>

可以少写，但是不能多写。

3、定义别名#

出现重名的情况，会报错，此时可以用关键字 as 定义别名。

通过别名可以解决命名冲突。

import {fn, name as ns} from "./module/mo.js"

4、默认导出导入#

常用。

export default{
    // 这里写对象
    a:1,
    b:2
};

// x 可以得到一个对象
import x from "./module/mo.js"

一个文件只能定义一个 export default 。

混合导出导入 import a, {b} from ... 注意书写顺序

5、通配符导出(* as)#

import * as mo from ...

导出所有的。

AJAX 即 Asynchronous Javascript And XML，异步 JavaScript 和 XML。是指一种无需重新加载整个网页的情况下，能够更新部分网页的技术。通过Ajax可以在浏览器中向服务器发起异步请求。

XML 称为可扩展标记语言，被设计用来传输和存储数据，和 HTML 类似，不同的是 HTML 日常使用大部分都是预定义标签，XML 全部都是自定义标签，用来表示一些数据。

一、工作原理和基本使用#

1、工作原理#

Ajax 的工作原理相当于在用户和服务器之间加了一个中间层（Ajax引擎），使用户操作与服务器响应异步化。

2、特点#

优点

  • 无需刷新页面来和服务器通信。
  • 允许你根据用户事件来更新部分页面内容。

缺点

  • 没有浏览历史，无法回退。
  • 存在跨域问题。
  • 对 SEO 不友好。

3、XMLHttpRequest 核心对象#

Ajax 的所有操作都是通过 XMLHttpRequest 对象进行的。

4、使用步骤#

（1）创建核心对象#

创建 XMLHttpRequest 核心对象。

const xhr = new XMLHttpRequest();

（2）打开接口，设置数据类型#

通过 open(method, URL) 打开接口。

  • method 请求方法，POST 或者 GET。
  • URL 接口地址。

！这里可以设置数据格式，即请求头，媒体类型有

  • application/json
  • application/x-www-form-urlencoded
  • formdata

xhr.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded')

xhr.open("POST", "http://127.0.0.1/weibo");

（3）发送请求#

发送请求 send(body)

xhr.send( JSON.stringifly( content ));  // 只能传字符串

GET 请求不需要参数， POST 请求需要。

（4）接收数据#

xhr.responseXML 接收XML格式的响应数据。

xhr.responseText 接收文本格式的响应数据。

a. 通过 onreadystatechange 事件接收数据，xhr 对象有 readyState 表示当前的请求状态，状态改变就会触发这个事件。

xhr.onreadystatechange = function (){
	if(xhr.readyState == 4 && xhr.status >= 200 && xhr.status < 300){
		let text = xhr.responseText;
		console.log(text);
	}
}

b. 通过 onload 事件

xhr.onload = function(){
	...xhr.responseText;
}

5、请求和响应状态#

（1）xhr.readyState 当前的请求状态

状态 常量 描述
0 UNSENT 表示 XMLHttpRequest 实例已经生成，但是 open() 方法还没有被调用。
1 OPENED 表示 send()方法还没有被调用，仍然可以使用setRequestHeader()，设定HTTP请求的头信息。
2 HEADERS_RECEIVED 表示 send() 方法已经执行，并且头信息和状态码已经收到。
3 LOADING 表示正在接收服务器传来的body部分的数据，如果responseType属性是text或者空字符串，responseText就会包含已经收到的部分信息。
4 DONE 表示服务器数据已经接收完成，或者本次接收已经失败了。

（2）xhr.status 响应状态，同HTTP协议响应状态码。

一、进程与线程#

JS 是单线程执行的，大多数浏览器是多线程的，HTML5的 Web Workers 可以构建多进程。

进程 描述了 CPU 在运行指令及加载和保存上下文所需的时间，放在应用上来说就代表了一个程序。

线程是进程中的更小单位，是进程内的一个独立单元，CPU基本调度单位，是程序执行的一个完整流程。

进程和线程的关系

  • 一个进程至少有一个运行的线程（主线程），如果一个进程有多个进程，则称进程是多线程运行的。
  • 一个进程中的数据可以供其中多个线程直接共享，进程之间的数据是不能直接共享的。

多线程

  • 优点 有效提高CPU的利用率。
  • 缺点 增加开销（创建、切换），死锁与状态同步问题。

单线程

  • 优点 顺序编程简单易懂。
  • 缺点 效率低。

二、Web Workert#

HTML5提供了JS分线程的实现。

相关API

  • `Worker 构造函数, 加载分线程执行的JS文件
  • Worker.prototype.onmessage 用于接收另一个线程的回调函数
  • Worker.prototype.postMessage 向另一个线程发送消息

缺点

  • worker内代码不能操作DOM（更新UI）。
  • 不能跨域加载JS。
  • 兼容性问题，不是每个浏览器都支持这个新特性。

oBtn.onclick = function(){

五、NodeJS 中的事件循环 △#

JS是单线程的，那么NodeJS是如何做到非阻塞的？ 在NodeJS内部使用了第三方库libuv ，NodeJS会把IO、文件读取等异步操作交由它处理，而JS主线程可以去处理其他的事情。libuv 会开启不同的线程去处理这些延迟操作，处理完后，会将异步操作的回调函数放在NodeJS的轮询队列中，NodeJS会在适当的时候处理轮询队列中的回调函数，从而实现非阻塞。所以NodeJS在处理阻塞操作时，并不是单线程的。

分为六个阶段。

NodeJS事件轮询队列分为6个阶段

  1. timers 处理定时器 setTimeout setInterval
  2. pending callbacks 处理操作系统级的回调函数（了解）
  3. idle prepare 处理NodeJS内部操作调用（了解）
  4. poll 处理读写操作，网络请求。（1）当这个阶段的回调函数队列不为空的时候，则处理队列中的回调函数，直到队列为空或者最大回调数的时候，就跳过此阶段，处理下一阶段。（2）当进入poll阶段的时候，如果此阶段的回调队列为空，系统会在此阶段等待新的回调函数入队，再进行处理。在这个阶段会同时进行检测**timers阶段是否已经有回调函数超时**，如果有，则马上跳过poll阶段，进入下一个阶段。如果**check设置了setImmediate**，也会进入下一个阶段。
  5. check 专门处理setImmediate回调函数。
  6. close callbacks 专门执行关闭的回调函数。

process.nextTick Promise 不由libuv 管理，**每次事件的轮询进入到下一个队列的时候，都会检查这两个队列是否为空，不为空则马上处理。（注意因为process.nextTick会在事件轮询每个阶段之间执行， 如果递归调用nextTick就会导致轮询阻塞，所以尽量避免使用process.nextTick， 可以使用setImmediate代替）

五、宏任务、微任务 △#

1、微任务#

  • process.nextTick（优先级最高）
  • Primise.then/catch/fanally （依顺序）
  • queueMicrostask

执行微队列时新增的微任务添加到当前微任务的下一个。（新添加的等下一轮执行）。

2、宏任务#

宿主环境提供的任务，具体看 NodeJS 事件循环机制。

微任务优先级高于宏任务。

3、代码执行顺序#

同步代码 --> 微任务（process.nextTick --> Primise async ） --> 宏任务（setTimeout setInterval --> setImmediate）

一、同源策略#

Same-Origin Policy，同源策略 最早由 Netscape 公司提出，是浏览器的一种安全策略。该策略表示协议、域名、端口号必须完全相同。

违背同源策略就是跨域，主要是针对 AJAX 而言。

二、JSONP 方案#

JSON with Padding，JSONP 是一个非官方的跨域解决方案，只支持get请求，而且需要服务器的配合。

工作原理#

在网页有一些标签天生具有跨域能力，比如 img link iframe script。JSONP就是利用 script 标签的跨域能力来发送请求的。

应用示例#

客户端

<script>
    const script = null;
    // 这是一个事件触发

三、CORS 方案#

Cross-Origin Resource Sharing，CORS，跨域资源共享 是官方的跨域解决方案，它的特点是不需要在客户端做任何特殊的操作，完全在服务器中进行处理，支持 GET 和 POST 请求。是通过设置一个响应头来告诉浏览器，该请求允许跨域，浏览器收到该响应以后就会对响应放行。

        res.header("Access-Control-Allow-Origin"),req.header.origin);
        // 允许携带内容
        res.header("Access-Control-Allow-Headers", "content-type")
    }
    next();
})

一、Cookie#

由于HTTP协议是无状态的，而很多服务器业务是必须要用状态。Cookie诞生的最初目的是为了存储网页中的状态信息，以方便服务器端使用。比如判断用户是否是第一次访问网站。

1、流程#

  • 服务器向客户端发送cookie
  • 浏览器保存cookie
  • 之后每次http请求浏览器都会将cookie发送给服务器端

2、服务器处理cookie#

服务器端像客户端发送Cookie是通过HTTP响应报文实现的。

cookie 是一个键值对的存储结构。

  • expires 设置失效时间
  • max-age 有效时间，单位是秒（推荐）
  • httpOnly 仅服务器可修改和可见
// 创建
res.setHeader("Set-Cookie", "user=jack;max-age=3600;httpOnly=true");

// 获取
req.headers.cookie;

// 删除
res.setHeader("Set-Cookie", "user=;max-age=0");

如果不设置时间，则代表是临时会话存储，浏览器关闭后就会消失。

3、客户端处理cookie#

客户端获取 document.cookie

客户端设置 document.cookie = 'name=rose;max-age=3600'

4、缺点#

  • 数量和长度有限制
  • 潜在的安全风险：可能被截取篡改，如果被拦截，就可能会获取到所有的Session信息
  • 用户配置为禁用，有的用户禁用了浏览器或者客户端设备接受cookie的能力，因此限制了这一功能
  • 每次都会随着服务器端发送，具有一定的带宽压力

5、将cookie解析成对象 △#

     const [key, value] = c.split("=");
     p[key] = value;
     return p;

二、Session#

session 是存储于服务器的对象，是另一种记录客户状态的机制，cookie保存在客户端，session保存在服务器。session运行在服务器，客户端第一次访问服务器时，就可以将客户的登录信息保存。当客户访问其他界面时，可以判断客户的登录状态，做出提示，相当于登录拦截。

1、特点#

  • 存储于服务器上
  • 大小和数量理论上没有限制
  • 传输流量小（只有一个cookie）
  • 安全性较高

2、流程#

  • 客户端：发起登录请求，在请求报文体或者URL中携带登录信息
  • 服务器：接受请求，创建一个session对象，将用户信息保存在session对象中，并创建一个session-id（唯一的）
  • 服务器：返回响应，cookie携带session-id
  • 客户端：接受session-id，保存在本地
  • 客户端：当发起第二次请求的时候，会携带具有session-id的cookie
  • 服务器：通过解析cookie，得到session-id，去session对象中查找当前id是否存在，从而识别用户是不是第一次访问，如果识别用户成功，直接响应通过，否则需要再次查询数据库等操作

三、HTTP + TLS = HTTPS#

HTTPS 还是通过了 HTTP 来传输信息，但是信息通过 TLS 协议进行了加密。TLS 协议位于传输层之上，应用层之下。首次进行 TLS 协议传输需要两个 RTT ，接下来可以通过 Session Resumption 减少到一个 RTT。

在 TLS 中使用了两种加密技术，分别为对称加密和非对称加密。

1、两种加密方式#

对称加密

对称加密就是两边拥有相同的秘钥，两边都知道如何将密文加密解密。

这种加密方式固然很好，但是问题就在于如何让双方知道秘钥。因为传输数据都是走的网络，如果将秘钥通过网络的方式传递的话，一旦秘钥被截获就没有加密的意义的。

非对称加密

有公钥私钥之分，公钥所有人都可以知道，可以将数据用公钥加密，但是将数据解密必须使用私钥解密，私钥只有分发公钥的一方才知道。

这种加密方式就可以完美解决对称加密存在的问题。假设现在两端需要使用对称加密，那么在这之前，可以先使用非对称加密交换秘钥。

简单流程如下：首先服务端将公钥公布出去，那么客户端也就知道公钥了。接下来客户端创建一个秘钥，然后通过公钥加密并发送给服务端，服务端接收到密文以后通过私钥解密出正确的秘钥，这时候两端就都知道秘钥是什么了。

2、TLS 握手#

客户端发送一个随机值以及需要的协议和加密方式。

服务端收到客户端的随机值，自己也产生一个随机值，并根据客户端需求的协议和加密方式来使用对应的方式，并且发送自己的证书（如果需要验证客户端证书需要说明）

客户端收到服务端的证书并验证是否有效，验证通过会再生成一个随机值，通过服务端证书的公钥去加密这个随机值并发送给服务端，如果服务端需要验证客户端证书的话会附带证书

服务端收到加密过的随机值并使用私钥解密获得第三个随机值，这时候两端都拥有了三个随机值，可以通过这三个随机值按照之前约定的加密方式生成密钥，接下来的通信就可以通过该密钥来加密解密

通过以上步骤可知，在 TLS 握手阶段，两端使用非对称加密的方式来通信，但是因为非对称加密损耗的性能比对称加密大，所以在正式传输数据时，两端使用对称加密的方式通信。

以上说明的都是 TLS 1.2 协议的握手情况，在 1.3 协议中，首次建立连接只需要一个 RTT，后面恢复连接不需要 RTT 了。

五、WebSocket#

1、概念#

  • 它是一种网络通信协议，是 HTML5 开始提供的一种在单个 TCP 连接上进行全双工通讯的协议。
  • 因为 HTTP 协议有一个缺陷：通信只能由客户端发起
  • 我们都知道轮询的效率低，非常浪费资源（因为必须不停连接，或者 HTTP 连接始终打开）, 因此websocket应运而生。
  • WebSocket用于在Web浏览器和服务器之间进行任意的双向数据传输的一种技术。WebSocket协议基于TCP协议实现，包含初始的握手过程，以及后续的多次数据帧双向传输过程。其目的是在WebSocket应用和WebSocket服务器进行频繁双向通信时，可以使服务器避免打开多个HTTP连接进行工作来节约资源，提高了工作效率和资源利用率。
  • WebSocket目前支持两种统一资源标志符ws和wss，类似于HTTP和HTTPS
  • 浏览器发出webSocket的连线请求，服务器发出响应，这个过程称为握手,握手的过程只需要一次，就可以实现持久连接。

一、概念#

1、并发、并行#

涉及面试题：并发与并行的区别？

并发（Concurrency）是宏观概念，我分别有任务 A 和任务 B，在一段时间内通过任务间的切换完成了这两个任务，这种情况就可以称之为并发。

并行（Parallelism）是微观概念，假设 CPU 中存在两个核心，那么我就可以同时完成任务 A、B。同时完成多个任务的情况就可以称之为并行。

2、回调函数#

涉及面试题：什么是回调函数？回调函数有什么缺点？如何解决回调地狱问题？

回调函数（Callback）分为同步回调（数组遍历）和异步回调。使用回调函数容易导致回调地狱问题。

回调地狱的根本问题就是：

  1. 嵌套函数存在耦合性，一旦有所改动，就会牵一发而动全身
  2. 嵌套函数一多，就很难处理错误。

当然，回调函数还存在着别的几个缺点，比如不能使用 try catch 捕获错误，不能直接 return。

二、Promise △#

ES6 规范，进行异步编程的新解决方案。以同步的形式把异步代码表达出来，避免回调函数层层嵌套的回调地狱。

1、状态#

状态 描述
pedding 初始化状态
fulfilled 成功状态，完成状态
reject 失败状态

状态只能被改变一次。

2、基本语法#


const p = new Promise((resolve, reject) => {
    // excutor 函数（同步的）
    resolve(1);
    // reject(2);
})

// 写法一
// reslove
p.then((num) => {
    console.log(num);
    // reject
}, (num) => {
    console.log(num);
})
.then((num) => {
    console.log(num);
    // reject
}, (num) => {
    console.log(num);
})


// 写法二
p.then(value =>{
    //
}).catch(reason =>{
    //
})

// 如果两种写法一起写，按写法一

Promise 实现了链式调用，也就是说每次调用 then 之后返回的都是一个 Promise，并且是一个全新的 Promise，原因也是因为状态不可变。如果你在 then 中 使用了 return，那么 return 的值会被 Promise.resolve() 包装。

3、返回一个新对象#

不管是成功还是失败（语法错误），返回的都是一个Promise对象。默认是resolve状态，失败和语法错误是 reject 状态。

// 封装 ajax
function ajax(url) {
    return new Promise((resolve, reject)=>{
        const xhr = new XMLHttpRequest();
        xhr.onreadystatechange = () => {
            if(xhr.readyState === 4){
                if(xhr.status >= 200 && xhr.status < 300){
                    resolve(xhr.responseText);
                } else {
                    reject(xhr.responseText);
                }
            }
        }
        xhr.open("GET", url);
        xhr.send();
    })
}

// 使用
ajax("xxx").then(value=>{
    // ....
}, result=>{
    // .....
})

5、手写 Promise △△#

（1）

function Promise(executor) {
    this.status = "pending";
    function _resolve() {
        this.status = "fulfilled";
    }
    function _reject() {
        this.status = "rejected";
    }
    try {
        executor(_resolve.bind(this), _reject.bind(this));
    } catch (err) {
        _reject.call(this, err);
    }
}
