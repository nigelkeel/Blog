Express#

Express 是一个保持最小规模的灵活的 Node.js Web 应用程序开发框架，为 Web 和移动应用程序提供一组强大的功能。

1、安装#

安装模块

npm install express --save

创建一个 express 项目

express 项目名字

安装 express 的中间件 express-static

npm install express-static --save

安装 express 项目生成器

npm install express-generator -g

2、开启服务器#

const express = require("express");
const app = express();
        //port
app.listen(80, function(){
    ...
});

http 默认 80 端口，https 默认 443 端口。

3、设置静态资源目录#

app.use(express.static(path.resolve(__dirname, "../xxx")));

！明确到一个文件夹。

！可以指定多个。

4、设置路由#

app.all
app.get( URL, (req, res)=>{})
app.post( URL, (req, res)=>{})

如果指定多个，当遇到访问方法和地址相同时，遇到第一个，默认不会向下执行 – 只会匹配第一个

只能响应一次。

也可以用 next() 继续向下查找是否有满足条件的路由。

第二次响应没有意义。

// 允许所有站点跨域
app.all("*", (req,res)=>{
    res.header("Access-Control-Allow-Origin"),"*");
    next();
})

5、接收数据#

body-parser#

接收 post 方式传来的数据。

const bodyParser = require("body-parser");
app.user(bodyParser.json());

const { context } = req.body;

query / params#

接收 get 方式传来的数据。

// "?age=123"
const age = req.query.age / 1;

RESTful API#

REST，Representational State Transfer，表现层状态转移的缩写，它是由罗伊·菲尔丁（Roy Fielding）提出的，是用来描述创建 HTTP API 的标准方法的，他发现这四种常用的行为（查看（view），创建（create），编辑（edit）和删除（delete））都可以直接映射到 HTTP 中已实现的 GET，POST，PUT 和 DELETE 方法。

操作#

查询 - get

（1）get books

../sword

（2）get id 为 1

../sword/1
../sword?id=1

（3）单条件查找

../sword?name=铁剑

（4）多条件查找

../sword?name=铁剑&atk=10

（5）全局模糊搜索

../sword?q=剑

（6）分页

../sword?_limit=1&_page=1

（7）排序（asc/desc）

../sword?sort=id&_order=desc

添加数据 - post

postman 中测试，设置主体 x-www-form-urlencoded

删除数据 - delete

静态资源服务器#

// 引入相关模块
const http = require("http");
const fs = require("fs");
const path = require("path");

// 手动封装 Promise，解决回调问题
function readFile(filePath) {
  return new Promise((resolve, reject) => {
    fs.readFile(filePath, (err, data) => {
      if (err) {
        reject(err);
        return;
      }
      const fileContent = data.toString();
      resolve(fileContent);
    });
  });
}

// 创建一个服务器
const server = http.createServer(async (req, res) => {
  try {
    // 获取读取文件的路径
    const filePath = path.resolve(__dirname, "./server.js");
    const data = await readFile(filePath);
    // 设置响应头类型
    res.setHeader("Content-Type", "application/javascript;charset=utf-8");
    // 设置响应状态码
    res.statusCode = 200;
    // 响应内容
    res.end(data);
  } catch (err) {
    console.log("读取文件失败", err);
    res.setHeader("Content-Type", "application/javascript;charset=utf-8");
    res.statusCode = 404;
    res.end("文件资源加载失败");
  }
});

// 配置服务器
const port = "8080";
const host = "127.0.0.1";

// 启动服务器
server.listen(port, host, (err) => {
  if (err) {
    console.log("服务器创建失败" + err);
    return;
  }
  const url = `http://${host}:${port}`;
  console.log(`服务器已启动，请访问${url}`);
});

Electron#

一、项目构建和打包#

项目搭建：
electron-vue

项目打包：
electron-builder

  • 可以支持 Windows/Mac/Linux 平台
  • 支持自动更新，打包更轻量
  • 可以打包出不暴露源码的安装程序
    electron-packager
  • 配置 nsis 自定义安装目录以及安装个性化

二、Electron 知识点#

1、主进程和渲染进程#

主进程：package.json 中的 main.js
Electron 内核是 Chromium
在主进程中通过 new BrowserWindow 创建页面，每个 BrowserWindow 就是渲染进程，当某个页面被销毁后，对应的渲染进程就会终止。

2、优化窗口#

（1）无边框窗口

mainWindow = new BrowserWindow({width: 800, height: 600, frame: false, resizable: false});

（2）窗口拖动
现在窗口已经没有标题栏和菜单栏了，所以窗口没办法拖动了，给拖动区域增加 -webkit-app-region: drag 的 CSS 代码。
<body style="-webkit-app-region: drag"></body>
需要注意的是，如果在拖动区域里有按钮等需要点击的元素，需要增加 -webkit-app-region: no-drag 设置，否则会无法点击。

3、进程间通信#

因为安全限制，页面不能直接访问原生 GUI 资源，在 Electron 中也是一样。
渲染进程要进行原生的 GUI 操作，比如和主进程进行通信，请求相应的操作，在 Electron 中提供了几种通信方式：

（1）ipcMain 和 ipcRenderer
在渲染进程中使用 ipcRenderer 模块来向主进程发送消息，主进程中使用 ipcMain 模块接受消息，进行操作。如果还需要进行反馈，则通知渲染进程，渲染进程根据接收的内容执行对应的操作。

// 渲染进程中
const { ipcRenderer } = require("electron");
ipcRender.send("somemsg", data);
ipcRender.on("replaymsg", (evt, otherData) => {
  console.log(otherData);
});
// 主进程中
const { ipcMain } = require("electron");
ipcMain.on("somemsg", (evt, data) => {
  console.log(data);
  evt.sender.send("replymsg", otherData);
});

（2）remote
remote 模块为渲染进程和主进程通信提供了一种简单方法

（3）webContents

// 主进程中
win.webContents.on("did-finish-load", () => {
  win.webContents.send("ping", "23333333333");
});
// 渲染进程中
require("electron").ipcRenderer.on("ping", (event, message) => {
  console.log(message);
});

（4）渲染进程之间通信
使用 Vuex 和 router
但如果对数据实时性要求高，那就要结合上面的方法来实现：

// 主进程中，将两个窗口的 id 分别发送到对方的渲染进程里
mainWindow.webContents.on("did-finish-load", () => {
  mainWindow.webContents.send("getChild", {
    childId: childWindow.id,
  });
});
childWindow.webContents.on("did-finish-load", () => {
  childWindow.webContents.send("getMain", {
    mainId: mainWindow.id,
  });
});

// 渲染进程 mainWindow，通过 id 得到 childWindow 并发送一条消息
ipcRenderer.on("getChild", (event, arg) => {
  remote.BrowserWindow.fromId(arg.childId).webContents.send(
    "say",
    "Hello World!"
  );
});
// 渲染进程 childWindow，接收 mainWindow 的消息
ipcRenderer.on("say", (event, arg) => {
  console.log(arg); // 输出：Hello World!
});
// 同理 childWindow 也可以通过 id 给 mainWindow 发送消息

4、踩坑#

electron-vue 里使用 elementUI 等，需要在 webpack.renderer.config.js 的白名单里配置：let whiteListedModules = ['vue', 'element-ui']

5、难点#

多窗口的管理，每个窗口使用 remote.getCurrentWindow().id 命名，在主进程中采用单个实例 app.makeSingleInstance 管理
