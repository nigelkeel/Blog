// 路由跳转
// 如果当前组件路由跳转的时候，地址栏有 params 参数，那么和 query 参数应该一起携带，如果没有，那么携带 query参数即可

  })

  toSearch() {
    const {
      path,
      query
    } = this.$route
    // 判断文本框是否输入内容
    if (this.keyword) {
      // 此时说明文本框中已经有内容
      if (path.indexOf('/search') === 0) {
        // 此时说明点击按钮的时候，文本框中有搜索关键字，而且也点击了分类信息
        // 需要携带 params 和 query 参数
        this.$router.push({
          name: 'search',
          params: {
            keyword: this.keyword
          },
          query
        })
      } else {
        // 此时说明文本框有搜索关键字，但是没有 search 界面中，所以，也不需要携带 query 参数
        this.$router.push({
          name: 'search',
          params: {
            keyword: this.keyword
          }
        })
      }
    } else {
      // 此时文本框中没有搜索关键字
      if (path.indexOf('/search') === 0) {
        // 确实在 search 中，所以要携带 query 参数
        this.$router.push({
          name: 'search',
          query
        })
      } else {
        // 没有搜索关键字，也没有在 /search 中
        this.$router.push({
          name: 'search'
        })
      }
    }
  }
}

      left = offsetX - maskWidth / 2
      top = offsetY - maskWidth / 2
      // 限制
      left: left < 0 ? 0 : left > maskWidth ? maskWidth : left
      top: top < 0 ? 0 : top > maskWidth ? maskWidth : top
      // 设置遮挡层的移动的位置
      maskDiv.style.left = left + 'px'
      maskDiv.style.top = top + 'px'
      // 设置大图的移动
      bigImg.style.left = -left * 2 + 'px'
      bigImg.style.top = -top * 2 + 'px'

import UUID from 'uuidjs'
export function getUUID() {
  // 先从浏览器的缓存中读取用户的临时凭证
  let uuid = localStorage.getItem('UUID_KEY')
  // 判断是否存在
  if (!uuid) {
    // 产生临时用户id
    uuid = UUID.generate()
    // 保存到浏览器
    localStorage.setItem('UUID_KEY', uuid)
  }
  // 返回
  return uuid
}

1、栈内存和堆内存#

内存根据程序运行时候的逻辑分为栈内存和堆内存。在 JS 中，并没有严格意义区分。大部分的数据都保存在堆内存中，但是在某些场景，仍需要用栈（有时候也叫堆栈）的数据结构的思路进行处理，如执行上下文。

栈内存 堆内存
栈结构 链表结构
占内存小，速度快 占内存大，速度慢
自动分配，自动回收 自己申请，手动释放

2、执行上下文#

JS 引擎并不是逐行解析代码，而是逐段去分析和执行，当开始解析一段代码时，会先进行预处理（声明提升和创建作用域链）。生成一个执行上下文。

执行上下文（执行环境）分为两种变量对象（VO）：

  • Global | Window，默认的全局执行环境。
  • Function，调用函数时会进入的函数执行环境。

代码执行时会开辟一个栈空间来说明程序的执行顺序，这个执行上下文堆栈称为执行栈。开始执行时先将全局执行环境压入栈底（压栈），执行到函数时会创建对应的函数执行环境，并将函数执行环境压入栈底。在函数执行完成后将函数执行环境弹出并销毁。直到所有代码执行完成销毁全局执行环境。

可以把执行栈认为是一个存储函数调用的栈结构，遵循先进后出的原则。

当我们使用递归的时候，因为栈可存放的函数是有限制的，一旦存放了过多的函数且没有得到释放的话，就会出现爆栈的问题。

4、作用域#

作用域 scope 是变量在程序中可以访问的有效访问，也被称为变量的可见性。作用域在进入全局执行环境或者函数执行环境时就已经确定好了。在ES5中，作用域分为两种：

  • 全局作用域
  • 函数作用域

同一个函数的多次调用都会生成不同的执行环境，每次调用都会生成。

全局变量 是定义在全局作用域中的变量，一般在函外部定义。在程序的所有地方都可以使用和操作。

局部变量 是在函数内部定义的，在函数执行域中。只能在自己的作用域中起作用，只能在函数内部使用。外部无法使用和操作局部变量。

5、作用域链#

作用域链 是用来查找变量的这样一系列的过程。一个函数在定义时会生成一个属性[[scope]]，这个属性存储的是函数定义时作用域的层级关系。作用域链是函数执行时创建的，是当前函数作用域链 -> [[scope]]属性作用域组成的。函数当前执行的作用域是作用域链的顶端，全局作用域在作用域链的最后。执行环境销毁时，作用域链也会销毁。

// 全局作用域
var a = 1;

function f() {
  // test -> 全局作用域
  var a = 10;
  function f1() {
    // test2 -> test -> 全局作用域
    var a = 100;
    console.log(a); //100
  }
  function f2() {
    console.log(a); //10
  }
  f1();
  f2();
}

console.log(a);

当执行 f1 时，创建函数 f1 的执行环境，并将该对象置于链表开头，然后将函数 f 的调用对象放在第二位，最后是全局对象，作用域链的链表的结构是 f1->f->window 。从链表的开头寻找变量 a，即 f1 函数内部找变量 a，找到了，结果是 20。同样，执行 f2 时，作用域链的链表的结构是 f2->f->window 。从链表的开头寻找变量 a，即 f2 函数内部找变量 a，找不到，于是从 f 内部找变量 a，找到了，结果是 10。最后在最外层打印出变量 a，直接从变量 a 的作用域即全局作用域内寻找，结果为 1。

6、内存中的变量存储#

变量如果是基本数据类型，会在直接在栈内存开辟一段存储空间标识符。然后将基本数据类型的值也存储在栈内存中和标识符对应起来。要求变量名不能重名，重名就会覆盖。

如果是引用数据类型，值储存在堆内存中，存储引用类型的时候标识符是储存在栈内存中的。（JS 不允许直接访问堆内存中的数据，只能通过引用访问，差不多就是在变量对象中存储一个指向对象的句柄，可以理解成是一个地址，要访问这个对象，就要通过这个引用句柄来访问）。

var a = 100;
var b = [1, 2, 3];
var c = b;
c[0] = 1000;

console.log(b); // [1000,2,3]

程序一开始执行，进入全局环境，首先会创建全局环境并且压栈（执行栈），全局代码执行的时候依赖的是全局环境中的东西。全局变量如果是基本类型，那么这个值直接存在栈当中，如果这个变量的值是引用类型，那么引用类型会在堆内存中开辟自己的空间专门存储（数据本身）。然后里面的这块空间的地址存在栈当中对应的变量。

当函数执行碰到函数调用，函数执行时也要有自己的环境去依赖。函数执行也是创建自己的函数环境进行压栈（函数执行环境一定压在全局执行环境的上面）。局部变量，是在函数环境当中存在的，只有函数执行，局部变量才会出现，函数执行完成之后，函数环境要弹出栈（销毁归还内存），局部变量也就不存在了。

当函数调用完成之后，会继续执行全局代码，一直到所有的全局代码执行完成，代表程序执行结束，程序执行结束的时候，全局环境最后出栈（销毁）。

7、声明提升#

也称为预解析。在执行环境的第一步（创建阶段），会将带 var 的变量声明和函数声明，放到作用域中。在执行阶段时，它的相关逻辑还是放在原来的位置进行赋值和其他的逻辑处理。函数表达式和函数构造器不会被提升。函数声明和变量都会提升，但是函数声明会先提升，然后是变量。

变量声明提升 通过 var 声明的变量，在定义语句之前就能访问到，值为 undefined。

函数声明提升 通过 function 声明的函数，在前面就可以直接调用，值为函数定义（对象）。

// 导出的对象是 fun 函数

例子

function add(x, y) {
  return x + y;

（2）将变量和函数设置为 exports 的属性来暴露变量和函数为了方便，Node 为每个模块提供一个 exports 变量，指向 module.exports，例如： module.exports.fun = …，相当于exports.fun = ...。

exports 本身不具备导出功能，但是修改它的时候，其实是修改了module.exports 。exports 是指向 moudlue.exports 的对象地址的一个新对象。（exports = xxx 的用法是错误的，这样使其不再绑定。正确的用法是 exports.xxx = xxx ）

不能将一个值赋值给exports，而是使用exports.XXX来暴露。否则这样它将不再绑定到module.exports。如果 exports 导出的变量类型是引用类型如函数，则会断开与module.exports的地址指向，导致变量导出失败。因为最终还是要靠module.exports来导出变量的。

例子

function add(x, y) {
  return x + y;
}
// 暴露出去
exports.add = add; // { add: [Function: add] }

Markdown#

Markdown 是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式。
Markdown 具有一系列衍生版本，用于扩展 Markdown 的功能（如表格、脚注、内嵌HTML等等），这些功能原初的 Markdown 尚不具备，它们能让 Markdown 转换成更多的格式。这些衍生版本要么基于工具，如Typora；要么基于网站，如GitHub和Wikipedia，在语法上基本兼容，但在一些语法和渲染效果上有改动。

1、宗旨和目标#

Markdown 提倡于注重内容而不是格式。
Markdown 的目标是实现易读易写。

2、HTML兼容性#

Markdown 允许使用HTML语法进行文本格式化。

3、编辑工具#

首选 Typora ，另外 VSCode 也有比较好的编辑体验。需安装这些插件Markdown All in One，Markdown Extended，Markdown Preview Enhance，并重新加载。

Markdown 基础语法

发表于 2021-02-02 分类于 1810：Markdown
本文字数： 1.5k 阅读时长 ≈ 1 分钟

1、换行#

句末加两个空格或者使用HTML的<br/>换行。

2、分隔线#

你可以在一行中用三个以上的*、+、_来建立一个分隔线，行内不能有其他东西。也可以使用HTML的<hr/>画分隔符。

3、标题#

Markdown 标题支持两种形式：
1）Atx式：格式为# + 空格 + 标题内容，级别由#的数量来确定，#是一级标题，######则是六级标题，最高六级标题。
2）Setext式：在标题底下加上任意个=代表一级标题，-代表二级标题。只支持一级标题和二级标题。

注：Setext式标题不支持目录渲染。

4、区块引用#

区块引用（Blockquotes）由>确定。

这是区块引用。
区块引用可以嵌套。
Markdown 也允许你偷懒只在整个段落的第一行最前面加上“>”。只要你在后面换个行。
引用的区块内可以使用其他大多数的 Markdown 语法。

5、列表#

5.1 有序列表#

有序列表则使用数字接着一个英文句点。如：

  1. 苹果
  2. 番茄
  3. 香蕉

5.2 无序列表#

无序列表使用*号、+号或是-号作为列表标记。如：

  • 苹果
  • 番茄
  • 香蕉

6、代码块#

6.1 行内代码#

行内代码区块用一对反引号把它包起来。如：

这是一句话。

6.2 普通代码块#

Markdown 中建立代码块只要简单地缩进4个空格或是1个制表符就可以。也可以用三个反引号包围起来。
在前面后加语言名称可以加强代码的显示。如：

printf("Hello World!")

7、链接#

7.1 自动链接#

Markdown 支持直接黏贴链接地址，Markdown就会自动把它转成链接。如果不能识别则使用尖括号括起来即可。

7.2 行内式链接#

要建立一个行内式的链接，只要在方块括号后面紧接着圆括号并插入网址链接即可，如果你还想要加上链接的 title 文字，只要在网址后面，用双引号把 title 文字包起来即可。如：

这是一个 [链接](http://example.com/ "这是链接") 。

渲染为：
这是一个 链接 。

7.3 参考式链接#

参考式的链接是在链接文字的括号后面再接上另一个方括号，而在第二个方括号里面要填入用以辨识链接的标记。如：

这是一个 [链接][id]。

[id]: http://example.com/  "这是链接"

渲染为：
这是一个 链接。

8、强调#

8.1 加粗#

用两个 * 或 _ 包起来的话，就是加粗。

8.2 斜体#

使用一个星号*和底线_包起来，就是斜体。

8.3 下划线#

Markdown 并无直接出现下划线的语法，可以使用HTML的<u>和</u>实现。

8.4 删除文本#

删除文本使用两个~包围起来实现。

一个惊叹号 !。接着一个方括号，里面放上图片的替代文字。接着一个普通括号，里面放上图片的网址，最后还可以用引号包住并加上选择性的 ‘title’ 文字。如：

![](img/test.jpg)

[pic]:img/test.jpg

10、插入表格#

Markdown 中可以创建表格，如：

|标题|标题|标题|
