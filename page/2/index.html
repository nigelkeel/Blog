一、同源策略#

Same-Origin Policy，同源策略最早由 Netscape 公司提出，是浏览器的一种安全策略。该策略表示协议、域名、端口号必须完全相同。

违背同源策略就是跨域。

二、跨域解决方案#

1、JSONP#

JSON with Padding，JSONP 是一个非官方的跨域解决方案，纯粹凭借程序员的聪明才智开发出来，只支持get请求。

（1）工作原理#

在网页有一些标签天生具有跨域能力，比如 img link iframe script。JSONP就是利用 script 标签的跨域能力来发送请求的。

2、CORS#

Cross-Origin Resource Sharing，CORS，跨域资源共享是官方的跨域解决方案，它的特点是不需要在客户端做任何特殊的操作，完全在服务器中进行处理，支持 GET 和 POST 请求。是通过设置一个响应头来告诉浏览器，该请求允许跨域，浏览器收到该响应以后就会对响应放行。

AJAX

发表于 2020-08-25 更新于 2020-08-27 分类于 技术栈进阶

AJAX 即 Asynchronous Javascript And XML，异步 JavaScript 和 XML。是指一种无需重新加载整个网页的情况下，能够更新部分网页的技术。通过Ajax可以在浏览器中向服务器发起异步请求。

XML 称为可扩展标记语言，被设计用来传输和存储数据，和 HTML 类似，不同的是 HTML 日常使用大部分都是预定义标签，XML 全部都是自定义标签，用来表示一些数据。

一、工作原理和基本使用#

1、工作原理#

Ajax 的工作原理相当于在用户和服务器之间加了一个中间层（Ajax引擎），使用户操作与服务器响应异步化。

2、特点#

优点

  • 无需刷新页面来和服务器通信。
  • 允许你根据用户事件来更新部分页面内容。

缺点

  • 没有浏览历史，无法回退。
  • 存在跨域问题。
  • 对 SEO 不友好。

5、请求和响应状态#

（1）xhr.readyState 当前的请求状态

状态 常量 描述
0 UNSENT 表示 XMLHttpRequest 实例已经生成，但是 open() 方法还没有被调用。
1 OPENED 表示 send()方法还没有被调用，仍然可以使用setRequestHeader()，设定HTTP请求的头信息。
2 HEADERS_RECEIVED 表示 send() 方法已经执行，并且头信息和状态码已经收到。
3 LOADING 表示正在接收服务器传来的body部分的数据，如果responseType属性是text或者空字符串，responseText就会包含已经收到的部分信息。
4 DONE 表示服务器数据已经接收完成，或者本次接收已经失败了。

（2）xhr.status 响应状态，同HTTP协议响应状态码。

一、Cookie#

由于HTTP协议是无状态的，而很多服务器业务是必须要用状态。Cookie诞生的最初目的是为了存储网页中的状态信息，以方便服务器端使用。比如判断用户是否是第一次访问网站。

1、流程#

  • 服务器向客户端发送cookie
  • 浏览器保存cookie
  • 之后每次http请求浏览器都会将cookie发送给服务器端

2、服务器处理cookie#

服务器端像客户端发送Cookie是通过HTTP响应报文实现的。

cookie 是一个键值对的存储结构。

  • expires 设置失效时间
  • max-age 有效时间，单位是秒（推荐）
  • httpOnly

如果不设置时间，则代表是临时会话存储，浏览器关闭后就会消失。

3、客户端处理cookie#

客户端获取 document.cookie

客户端设置 document.cookie = 'name=rose;max-age=3600'

4、缺点#

  • 数量和长度有限制
  • 潜在的安全风险：可能被截取篡改，如果被拦截，就可能会获取到所有的Session信息
  • 用户配置为禁用，有的用户禁用了浏览器或者客户端设备接受cookie的能力，因此限制了这一功能
  • 每次都会随着服务器端发送，具有一定的带宽压力

5、将cookie解析成对象 △#

二、Session#

session 是存储于服务器的对象，是另一种记录客户状态的机制，cookie保存在客户端，session保存在服务器。session运行在服务器，客户端第一次访问服务器时，就可以将客户的登录信息保存。当客户访问其他界面时，可以判断客户的登录状态，做出提示，相当于登录拦截。

1、特点#

  • 存储于服务器上
  • 大小和数量理论上没有限制
  • 传输流量小（只有一个cookie）
  • 安全性较高

2、流程#

  • 客户端：发起登录请求，在请求报文体或者URL中携带登录信息
  • 服务器：接受请求，创建一个session对象，将用户信息保存在session对象中，并创建一个session-id（唯一的）
  • 服务器：返回响应，cookie携带session-id
  • 客户端：接受session-id，保存在本地
  • 客户端：当发起第二次请求的时候，会携带具有session-id的cookie
  • 服务器：通过解析cookie，得到session-id，去session对象中查找当前id是否存在，从而识别用户是不是第一次访问，如果识别用户成功，直接响应通过，否则需要再次查询数据库等操作

3、流式文件操作#

  • fs模块处理文件的缺点：将文件的数据全读到内存中，在把数据写到文件内，会大量占用内存
  • 流（stream）是 Node.js 中处理流式数据的抽象接口，是一组有序的，有起点和终点的字节数据传输手段。可以实现将数据从一个地方流动到另一个地方，其边读取边写入的特点有别于fs模块的文件处理，并且可以做到控制读取文件和写入文件的速度，从而减少内存的占用
  • 流是基于事件的，所有的流对象都用 on(once)绑定事件，并触发
  • 流式文件写入适用于一些比较大的文件，可以分多次向文件中写入内容，有效避免内存溢出的问题。

一、WebSocket#

1、概念#

  • 它是一种网络通信协议，是 HTML5 开始提供的一种在单个 TCP 连接上进行全双工通讯的协议。
  • 因为 HTTP 协议有一个缺陷：通信只能由客户端发起
  • 我们都知道轮询的效率低，非常浪费资源（因为必须不停连接，或者 HTTP 连接始终打开）, 因此websocket应运而生。
  • WebSocket用于在Web浏览器和服务器之间进行任意的双向数据传输的一种技术。WebSocket协议基于TCP协议实现，包含初始的握手过程，以及后续的多次数据帧双向传输过程。其目的是在WebSocket应用和WebSocket服务器进行频繁双向通信时，可以使服务器避免打开多个HTTP连接进行工作来节约资源，提高了工作效率和资源利用率。
  • WebSocket目前支持两种统一资源标志符ws和wss，类似于HTTP和HTTPS
  • 浏览器发出webSocket的连线请求，服务器发出响应，这个过程称为握手,握手的过程只需要一次，就可以实现持久连接。

2、基本使用#

WS 表示一个WebSocket对象。

WS.readyState 检查连接状态

  • 0 未连接
  • WS.OPEN 连接已建立
  • 2 连接正在关闭
  • 3 连接已经关闭

二、NodeJS的事件轮询机制 △△#

JS是单线程的，那么NodeJS是如何做到非阻塞的？ 在NodeJS内部使用了第三方库libuv ，NodeJS会把IO、文件读取等异步操作交由它处理，而JS主线程可以去处理其他的事情。libuv 会开启不同的线程去处理这些延迟操作，处理完后，会将异步操作的回调函数放在NodeJS的轮询队列中，NodeJS会在适当的时候处理轮询队列中的回调函数，从而实现非阻塞。所以NodeJS在处理阻塞操作时，并不是单线程的

1、六阶段#

NodeJS事件轮询队列分为6个阶段

  1. timers 处理定时器 setTimeout setInterval
  2. pending callbacks 处理操作系统级的回调函数（了解）
  3. idle prepare 处理NodeJS内部操作调用（了解）
  4. poll 处理读写操作，网络请求。（1）当这个阶段的回调函数队列不为空的时候，则处理队列中的回调函数，直到队列为空或者最大回调数的时候，就跳过此阶段，处理下一阶段。（2）当进入poll阶段的时候，如果此阶段的回调队列为空，系统会在此阶段等待新的回调函数入队，再进行处理。在这个阶段会同时进行检测**timers阶段是否已经有回调函数超时，如果有，则马上跳过poll阶段，进入下一个阶段。如果check设置了setImmediate**，也会进入下一个阶段。
  5. check 专门处理setImmediate回调函数。
  6. close callbacks 专门执行关闭的回调函数。

process.nextTick Promise 不由libuv 管理，**每次事件的轮询进入到下一个队列的时候，都会检查这两个队列是否为空，不为空则马上处理。（注意因为process.nextTick会在事件轮询每个阶段之间执行， 如果递归调用nextTick就会导致轮询阻塞，所以尽量避免使用process.nextTick， 可以使用setImmediate代替）

2、宏任务、微任务#

微任务

process.nextTick（优先级最高）

Primise.then/catch/fanally （依顺序）

queueMicrostask

执行微队列时新增的微任务添加到当前微任务的下一个。（新添加的等下一轮执行）

宏任务

3、代码执行顺序#

同步代码 –> 微任务（process.nextTick –> Primise async ） –> 宏任务（setTimeout setInterval –> setImmediate）

git fetch是将远程主机的最新内容拉到本地，用户在检查了以后决定是否合并到工作本机分支中。

而git pull 则是将远程主机的最新内容拉下来后直接合并，即：git pull = git fetch + git merge，这样可能会产生冲突，需要手动解决。

二、HTTP 协议 △△#

Hyper Text Transfer Protocol 超文本传输协议（通信协议规范）

客户端和服务器端通信传输的内容称为报文。

请求报文 响应报文

HTTP + SSL = HTTPS

1、响应的内容#

Headers 报文的头部

  • General 通用头部
  • Response Headers 响应头部
  • Request Headers 请求头部

Preview 预览响应的数据

Response 响应的具体数据

Initiator 初始化内容

Timing 时间

Cookies 携带的cookie

2、报文#

HTTP应用程序之间发送的数据块。在客户端、服务器、代理之间流动。分为请求报文和响应报文。

GET 请求报文#

请求报文首行

请求方式 请求地址 请求参数(查询字符串) 协议版本

  • 请求方式 GET
  • 协议版本 HTTP/1.1 兼容性 HTTP/2 最新

请求报文头部

  • 请求的主机地址 Host
  • 连接方式 Connection 保持长连接 keep-alive
  • Upgrade-Insecure-Requests 允许使用https访问
  • User-Agent 用户代理（浏览器信息）
  • Accept 可以接收的类型
  • Sec-Fetch-Size -Mode -User -Dest
  • Accept-Encoding 客户端可以接受的压缩格式
  • Accept-Language 可以支持的语言

请求报文空行 一个回车或者空格

请求报文体

GET 响应报文#

响应报文首行

协议版本 状态码

响应报文头部

  • Context-Type 响应数据类型
  • Date 响应的时间
  • Connection 连接方式
  • Content-Length 响应的字节长度

响应报文空行

响应报文体

POST 请求报文#

请求内容在报文体里

POST 响应报文#

3、请求方式#

GET 查 用于请求指定的页面信息，并返回实体

POST 增 向指定资源提交数据，进行处理 一般用于登陆注册

PUT 改 更改服务器数据

DELETE 删

OPTIONS 跨域请求

4、MIME 类型#

  • text/plain 纯文本类型
  • text/html
  • text/css
  • application/javascript

5、响应状态码 △#

1xx 信息响应

  • 100 请求继续
  • 101 正在切换协议

2xx 请求成功

  • 200 请求成功
  • 204 请求成功 页面主体不发生修改 put delete 一般返回

3xx 重定向

  • 301 永久重定向
  • 302 临时重定向
  • 304 读取缓存（协商缓存）

4xx 客户端错误

  • 400 响应报文有问题需要修改
  • 403 服务器拒绝访问
  • 404 服务器找不到请求的资源，服务端不想响应

5xx 服务端错误

  • 500 服务器执行请求发生错误
  • 503 服务器超负荷或者停机维护中 无法请求处理

6、网络模型#

HTTP - TCP - IP - 以太网

7、从输入URL到网页最终渲染 △#

（1）DNS解析

  • 输入的是域名，解析成IP地址
  • 四个缓存（浏览器DNS缓存，计算机DNS缓存，路由器DNS缓存，网络运营商DNS缓存），一个递归查询

（2）TCP连接（三次握手）

  • 浏览器 我准备好了
  • 服务器 我收到了
  • 浏览器 我收到确认信息了，准备发
  • 确认客户端、服务端双方都有正常的收发能力（为什么要三次握手呢？）
  • 保持长连接说的就是这里。

（3）发送请求 客户端发请求给服务器

（4）返回响应 服务器返回响应

（5）渲染页面

  • 构建DOM树（html - DOM树）
  • 样式计算（样式树）
  • 布局
  • 分层
  • 图层绘制
  • 栅格化
  • 合成和显示

（6）断开连接（四次挥手）

  • 客户端 我请求完了
  • 服务器 我收到了
  • 服务器 我响应完了
  • 客户端 我收到了
