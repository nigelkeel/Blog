LESS#

一、概述#

less 是一种动态样式语言，属于CSS预处理器，扩展了CSS语言，增加了变量、混合（Mixin）、函数等特性，使CSS更易维护和扩展。LESS既可以在客户端浏览器上运行，也能借助NODE.JS在服务器端运行。

CSS 自诞生以来，基本语法和核心机制一直没有本质上的变化，它的发展几乎全是表现力层面上的提升。最开始 CSS 在网页中的作用只是辅助性的装饰，轻便易学是最大的需求；然而如今网站的复杂度已经不可同日而语，原生 CSS 已经让开发者力不从心。当一门语言的能力不足而用户的运行环境又不支持其它选择的时候，这门语言就会沦为 “编译目标” 语言。开发者将选择另一门更高级的语言来进行开发，然后编译到底层语言以便实际运行。于是，在前端领域，CSS 预处理器应运而生。而 CSS 这门古老的语言以另一种方式 “重新适应” 了网页开发的需求。

盒模型

发表于 2020-06-17 更新于 2020-08-26 分类于 技术栈

一、盒子模型#

1、概念#

在 HTML 中把每个元素当成一个盒子，元素拥有盒子的平面外形和空间。

分为 标准盒模型 和 怪异盒模型。

2、content 层#

2.1 width/height#

width设置的坑 如果设置宽度为100%，并且有border margin padding等属性值的时候，宽度可能超出。因为width内容区域已经和父级一样了，所以可以不设置宽度或者设置宽度为auto那么内容的宽度就是可以充分利用可使用空间。

height设置的坑 如果高度设置百分比，必须父级有一个显性的设置一个height属性，并且不能是auto。

2.2 min/max-width/height#

正常 最小 最大
宽度 width min-width max-width
高度 height min-height max-height

当min-width和max-width 冲突的时候，min-width是优先的。当width和 min或max冲突以后，min或者max直接无视width的存在（无论是否添加 ！important）。

3、padding 内边距层#

padding的写法和margin一致

padding主要撑开内容和边框之间的距离

padding不支持负值

padding对所有属性的元素都支持，但是对于行内元素撑不开上下的距离

padding可以用于扩大点击范围

4、border 边框层#

4.1 边框属性#

border

  • border-width 线的宽度
  • border-style 线的类型
    • solid 实线
    • dashed 虚线
    • dotted 点状线
  • border-color 线的颜色
  • border:none 不设置边框

边框还分为4个方向 要么直接设置4个方向，要么一个个方向单独设置

练习：边框画三角形

A B C D

5、margin 层#

margin的作用是撑开当前元素和其他元素的距离。margin有四个方向，可以分别设置margin-left margin-right margin-top margin-bottom，margin的left和top是让当前元素向右向下移动，margin的right和bottom是让元素的兄弟元素向右向下移动，不影响自身。

5.1 合写#

  • 4个值：上 右 下 左（从上开始顺时针）
  • 3个值：上 左右 下
  • 2个值：上下 左右
  • 1个值：上下左右

5.2 水平方向上的auto#

因为块标签拥有满行属性，但是设置宽度以后，剩余的空间就是默认的margin，设置左右的margin值为auto，那么剩余的margin就会平分到左右两边。

5.3 竖直方向的auto#

相对于水平方向来说，块元素在垂直方向上并没有满屏的属性，margin默认在上下的值都是0，并没有剩余空间去平分。

5.4 塌陷和边框合并问题#

  • 父级塌陷：父级的第一个子元素的上margin和最后一个子元素的下margin会进行塌陷给父级，这样的设计是为了方便父级和他的兄弟进行垂直margin的合并，避免缝隙过大。
  • 解决塌陷：给父级开启BFC，给父级设置边框。
  • 兄弟的margin垂直方向会进行折叠，取最大值 。

5.5 margin 负值#

  • margin-top为负值：元素向上走，不会影响上边的元素的位置（会出现重叠），并且下边的兄弟元素会跟着一起向上走。
  • margin-left为负值：元素向左走，不会影响前边的元素位置(会出现重叠)，右边兄弟元素会跟随移动。
  • margin-right为负值：元素不会移动，形态大小也不会发生改变，但是所占用的位置会减少（负margin的值），所以，后边的兄弟元素会自动向左走。
  • margin-bottom为负值：元素不会移动，形态大小也不会发生改变，但是所占用的位置会减少（负margin的值），所以，后边的兄弟元素会自动向上走。

5.6 支持性#

margin 完美支持块元素和行内块元素，不支持行元素的垂直方向。

八、样式重置#

样式重置（CSS reset），将html的默认样式全部去掉，需要的时候我们自行添加。统一页面风格。

为什么要用样式重置

  • 多数元素拥有自己的默认样式，并且我们很多都不需要。
  • 默认样式在不同浏览器中呈现的也不一定相同，就会导致浏览器展示页面不相同的现象。
  • 整个页面中固定的风格代码，可以在样式重置中直接书写，不用每一个都设置。

为什么不用通配符重置

  • 重置简单，不够完整，一些demo可以使用，真正项目不推荐使用，请使用CSS reset
  • 通配符，匹配所有的标签，影响很大，效率很低。

九、文档流#

脱离文档流，指的是元素脱离正常元素的布局排版规则，其他处于文档流中的盒子在计算布局排版时，会自动无视已脱离文档流的元素来进行定位。

浮动与绝对定位之间的区别就是：

  • 浮动会使元素脱离文档流，但是不会脱离文本流，在于其他盒子的文本内容计算布局的时候，还是占位置的
  • 绝对定位会使元素脱离文档流，同时也会脱离文本流，在于其他盒子的文本内容计算布局的时候，不占位置。

三、其他#

1、visibility#

visible: 元素显示
hidden: 元素隐藏
元素依然占用空间，和隐身一样，带着子元素一起隐身了。
子元素的隐藏是因为继承了父级的hidden值，所以单独给子元素设置，覆盖hidden，即可显示。
使用visiblity隐藏的元素是可以用js获取的DOM节点及相应的宽高等属性。

visbility:hidden 和 display:none 隐藏的区别

①visibility可以继承，子元素是因为继承了才隐藏（可覆盖）。display不能被继承，直接带着所有内置元素直接隐藏。

一、浮动#

float 浮动 可以让多个块元素横向排列，共行。

属性 说明
none 默认，不浮动
left 左浮动
right 右浮动

1、概念#

正常页面流/正常文档流 是行元素从左至右，块元素从上到下布局。浮动 是破坏流（脱离文档流）的一种方式。在目前开发中，很多的元素都是一个个float堆叠起来，但是这种布局很容易崩溃。只要宽度和高度发生一点变化，页面就会发生错乱。

2、浮动的使用#

左浮动 就是让容器沿着容器左侧排序。

右浮动 让容器沿着容器右侧排序。

3、高度塌陷#

浮动元素以后，脱离父级内容区域，父级没有内容撑开自身的高度，而且父级的兄弟元素是按照父级的位置进行布局的，所以页面会乱。

4、清除浮动#

清除浮动不是不让元素浮动，是清除浮动对父级高度塌陷的影响。

清除浮动的方法

  • 给浮动元素的父级设置高度。
  • 给浮动元素的父级设置浮动，以浮制浮。
  • overflow设置不为visible的属性。
  • 在最后一个浮动元素的后边添加一个空的块级元素，添加一个块级元素，添加属性和值为 clear: both。
  • （推荐）after伪元素清浮动，给所有浮动元素父级一个clearFix的类名。

二、定位#

定位（position） 允许你定义元素框相对于其正常位置应该出现的位置，或者相对于父元素，以及浏览器窗口本身的位置。

属性 描述
static 默认，没有定位
relative 相对定位
absolute 绝对定位
fixed 固定定位

布局一般用在大布局的小范围中。

1、相对定位 relative#

让元素相对定位，元素先放置在未添加定位时候的区域，然后在不改变页面布局的情况下（其他元素没有受到任何影响；自身原来的位置也保留；和浮动可以一起使用）进行移动。不会脱离文档流。

2、绝对定位 absolute#

绝对定位元素，相对于它的 包含块 定位。绝对定位是相对于它的包含块定位的（和自身原有位置无关）。如果没有任何一个父级元素是非static属性，则会相对于文档（html）定位。如果它的父级元素和爷爷级元素都是非position:static 属性，则它会选择距离最近的父元素。脱离正常文档流，原有位置不保留。定位的原点以包含块的padding-box的左上角为原点。right和bottom是距离包含块右下角的距离。

2.1 包含块#

包含块 是视觉格式化模型的一个概念，简单来说就是一个定位参考框。

如何确定一个元素的包含块，完全取决于它自身的 position 属性：

  • 如果一个元素自身的 position 属性是 static 或者是 relative ，它的包含块就是离他最近的祖先元素或者是格式化上下文。
  • 如果一个元素自身的 position 属性是 absolute ，它的包含块就是离他最近的拥有定位属性（值不为 static ）的元素。
  • 如果一个元素自身的 position 属性是 fixed，它的包含块就是 viewport。

补充：
如果一个元素的 position 属性是 absolute 或者是 fixed 在下边几种情况下，包含块会发生改变。

  • 当祖先元素拥有 transform 或 perspective 属性并且值不为none的时候，它也是被当做包含块。
  • 当祖先元素 拥有 filter 属性的时候（值不为none） 它也可以被当做包含块

注：如果由内向外找不到包含块条件的元素，那么html（根元素）被称作为初始包含块。

2.2 “子绝父相”#

父元素相对定位并不设置位移，子元素绝对定位，设置位移。

3、固定定位 fixed#

相对于浏览器窗口定位。

4、z-index 属性#

z-index 指定了一个定位属性的元素及其后代的层叠顺序，只有定位元素（非static）拥有该属性。z-index 的值没有单位，理论上来说 z-index 的值大的元素，会覆盖小的元素。定位元素默认的z-index 的值是 auto。如果一个拥有 z-index 属性的定位元素中，子元素也设置了 z-index 。那么子元素会重新创建一个 层叠上下文，子元素的z-index只能在当前的层叠上下文中对比排列。

4.1 元素层叠顺序#

z-index为负< background< border< 块级元素 < 浮动元素 < 内联元素 < 没有设置z-index的定位元素 < z-index为正

4.2 层叠上下文 +#

想要一个元素支持 z-index 属性，但是又不想定位影响其他元素或者影响自己，则可以给该元素设置相对定位。

  • WWW，World Wide Web，万维网
  • B/S结构，浏览器-服务器结构
  • C/S结构，客户端-服务器结构
  • HTTP协议，超文本传输协议
  • Domain Name，域名，DNS；Domain Name System，域名解析系统
  • W3C，万维网联盟
  • HTML，Hypertext Markup Language，超文本标记语言。
  • CSS，层叠样式表。

2、开发环境#

  • VSCode
  • Chrome、Firefox、IE
  • Photoshop

4、网页的组成#

网页由三部分组成：

  • 结构 html
  • 表现 CSS
  • 行为 JavaScript

二、HTML 基本结构#

1、注释#

用<!-- 和-->包裹起来的内容。

2、版本声明#

给浏览器声明，告诉浏览器应该按照HTML5的规范解析当前页面。 如果不写，浏览器就是按照自己默认的模式解析。

版本声明必须写在第一行。我们不把他看做一个html标签，而是把它当做一个指令，并且这个指令对大小写不敏感。

3、html、head、body 标签#

html 是一个双标签，所有网页内容全部书写在html标签中。lang就是html标签的属性，en就是lang属性的值，属性是对当前标签的信息的扩展。

head 标签代表网页的头部，不会显示在网页中。主要是包含了标题，网页的描述，字符编码等等一些信息。

body 标签是网页显示的主体内容。

4、meta 标签#

控制网页的元信息（信息的信息），根据属性的不同，有很多的作用。

5、解决网页乱码问题 △#

charset即字符编码。utf-8称作为万国码。

三、HTML 标签#

1、标签的语义化 △#

  • 语义化标签更具有 可读性 ，便于团队开发和维护。
  • 在没有加载CSS的情况下，网页也能很好呈现出内容结构和代码结构。
  • 对于SEO来说，搜索引擎更能理解网页中各部分之间的关系，更准确更快速搜索信息。

2、标签的关系#

  • 并列：兄弟关系
  • 嵌套：父子关系（使用TAB缩进）

3、标签的分类#

3.1 块标签#

特性

  • 独占一行
  • 可设宽高、内外边距和边框
  • 可嵌套块元素和行元素（p、h标签除外）

将非块级元素转换成块级元素的方法

  • display: block
  • float: left/right
  • position: absolute/fixed
  • 父级元素使用display:flex

3.2 行标签#

特性

  • 空间足够时可以其他行标签共用一行
  • 设置宽高无效，内外边距和边框的上下设置无效
  • 宽高用内容撑开
  • 可嵌套行元素
  • 受父元素的 line-height 以及自身的 vertical-align影响。

3.3 行内块#

特性

  • 一行可放多个。
  • 宽高、内外边距、边框无法设置。
  • 受父元素的 line-height 以及自身的 vertical-align影响。

4、div#

分块原则：先行，后列。

5、h + p + br#

h，标题

尽量避免跳过某一级标题，依次使用。

同一个网页中最多使用一次h1元素，h1经常用在网页的标题或者是logo上。

p，段落。

br，单纯的换行，被称为软换行。

6、列表#

（1）无序列表和有序列表

ol，有序列表，ul，无序列表。li，列表的每一项。
ul/ol-li，ol/ul只能嵌套li。

（2）自定义列表

dl-dt-dd，组合标签。

7、表格#

table、tr、td、th

  • 表格的最外层是table元素
  • 表格的标题是caption元素 一般在表格的最上边
  • tr就是表格的行
  • td是普通单元格，th是表头单元格（默认加粗居中）

table的属性

  • border:边框（已经废弃，以后使用css的方法）
  • 合并表格的边框，对table设置：border-collapse: collapse;
  • cellpadding 控制内容和边框之间的距离
  • cellspacing 控制两个单元格之间的距离

跨行和跨列

  • rowspan:控制元素跨行 谁跨行 设置给谁
  • colspan:控制元素跨列 谁跨列 设置给谁

其他

  • 表格布局（过时了，了解即可）
  • thead：表格的头部
  • tbody：表格的内容，可以划分多个
  • tfoot：表格的底部
  • 三个标签table划分为多个模块，优化表格的加载，一般数据表格量大的时候可以使用
  • thead、tfoot、tbody：三个元素，无关书写顺序，一定会按照thead–tbody–tfoot顺序加载

8、header、footer、nav、aside H5#

header 用于定义文档的页眉（头部区域）。

footer 用于定义文档的页脚（底部区域）。

nav 在文档中定义导航。

aside 通常用于定义侧边栏。

9、section 、artcle H5#

section 在文档中对独立的区域进行分块，一般会有独立的标题。

article 在文档中，对可以独立的完整的被外部引用的内容。

？ 比较 div、section、article

10、figure、figcaption#

figure、figcaption

H5新标签，用于引用图片、插画、表格、代码段。

11、span#

span标签在不需要语义的情况下，增加额外的结构，方便设置样式等操作。

12、a标签、锚链接#

（1）路径

  • 网络路径：一般有三种协议，一种是http、一种是file、一种是ftp。需要引入网络路径的时候，需要书写http://。
  • 相对路径
  • 绝对路径

（2）标签属性

  • href：路径。
  • target：跳转方式，_self 当前标签页跳转，_blank新标签页跳转。
  • title：鼠标悬浮时的弹窗解释。

（3）锚链接

href="#(id)" → id="(id)"

  • 标签href为空时，点击刷新加载。当设置为##（两个#及以上）或者javascript:;时，点击之后不会有动作
  • 如果a标签的href值只有一个 # 那么可以实现回到顶部的功能。
  • 如果a标签的href属性书写的链接可以使用浏览器打开（比如图片），则浏览器跳转到这个位置并打开，如果a标签的href属性书写的链接不可以使用浏览器打开，则浏览器直接下载。
  • a标签的下载功能：download属性 书写下载内容，可以直接实现下载，必须拥有href属性，否则下载功能失效。

a标签的伪类 △#

伪类 功能
link 当有链接属性时
visited 链接被访问过
hover 鼠标悬浮在链接上面时
active 链接激活时

①“love-hate 爱恨法则” —— 顺序不能互换 ：四个选择器优先级一样，顺序改变可能会覆盖前边的，比如visited写在最后，那么当连接被访问过后，访问过后的颜色就会覆盖hover和active。
②hover适用于多数其他元素，并且伪元素也可以使用。

13、b、i、u、em、strong、var#

  • b i u 没有太多的含义，可以增加额外的结构，但是使用率较低。

  • i 标签常用在引入小图标

  • em 标签默认倾斜，起到强调作用，主要是强调关键词的。

  • strong 标签默认加粗，起到更强的强调，主要强调的是一段话。

  • var 标签 没有任何的强调作用，有自己的含义，主要包含变量。

14、mark#

标记，突出显示上下文的 ==关联性== ，比如搜索页面的关键词。是行内元素。

15、time、datalist、proress#

time

当时间是标准写法的时候，可以放在该标签中。

datalist

包含一组option元素，通常和input一起配套使用。

proress

通常用于进度条。（很少用）

16、img 标签#

属性 描述
src 图片路径
alt 加载失败时显示的替代文本
title 鼠标悬浮时显示的弹窗提示
width、height 宽高

解决底部留白：

  • display: block
  • font-size： 0

消除行内块间距的方式

  • 移除标签间的空格
  • 取消标签闭合
  • 给父元素设置font-size:0，Chrome 默认有最小字体大小限制，可以font-size: 0; -webkit-text-size-adjust:none;
  • 设置 margin 为负值
