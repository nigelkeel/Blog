NPM#

npm 是 Node 的模块管理器，全称 Node Package Manager。

一、常用命令#

1、安装

npm install <package>

npm install 可以简写为 npm i （？二者有一些微小的差异）

安装指定版本 在包名@指定版本

全局安装 -g 或者 -global

安装到生产和开发环境(dependencies) -D 或者 --save-dev （默认）

npm view <package>

package.json#

NodeJS 的包基本遵循 CommonJS 规范，包将一组相关的模块组合在一起，形成一套完整的工具，主要文件就是 package.json。

package.json 文件其实就是对项目或者模块包的描述，里面包含许多元信息。比如项目名称，项目版本，项目执行入口文件，项目贡献者等等。npm install命令会根据这个文件下载所有依赖模块。

在项目根目录下执行 npm init，然后根据提示一步步输入相应的内容完成后即可自动创建该文件。

包描述文件包含以下字段：

name - 包名，不能有中文、特殊字符等
version - 包的版本号
description - 包的描述
homepage - 包的官网URL
author - 包的作者
contributors - 包的其他贡献者
description - 项目描述，是一个字符串
keywords - 项目关键字，是一个字符串数组
private - 是否私有，设置为 true 时，npm 拒绝发布
license - 软件授权条款，让用户知道他们的使用权利和限制
bugs - bug 提交地址
repository - 项目仓库地址
homepage - 项目包的官网 URL
dependencies - 生产环境下，项目运行所需依赖，运行包时需要使用的依赖

\> 显示的版本号：
\> 1.12.4 --> 必须是 1.12.4 版本
\> ^1.12.4 --> 必须是 1.12.x 版本，x取最新的
\> ~1.12.4 --> 必须是 1.x.x 版本，x取最新的

devDependencies - 开发环境下，项目所需依赖，构建包时需要使用依赖
scripts - 执行 npm 脚本命令简写，比如 “start”: “react-scripts start”, 执行 npm start 就是运行 “react-scripts start”
bin - 内部命令对应的可执行文件的路径。
eslintConfig - EsLint 检查文件配置，自动读取验证
engines - 项目运行的平台
browseslist - 供浏览器使用的版本列表
files - 被项目包含的文件名数组

ES Map#

ES6+

类似于对象的一种数据结构，是键值对的集合，但是键的范围不限于字符串，各种类型的值（包括对象）都可以当作键。也就是说，Object 结构提供了“字符串—值”的对应，Map 结构提供了“值—值”的对应，是一种更完善的 Hash 结构实现。如果你需要“键值对”的数据结构，Map 比 Object 更合适。

//为什么会有Map，因为对象属性名称必须是字符串，如果是其他类型则不行
let p1 = { name: "lily" };
let obj1 = {
  id: 1,
  [p1]: "good",
};
console.log(obj1);

// Map也是新增的数据解构  类似于对象
let mp1 = new Map([
  ["a", 1],
  ["b", 2],
]);
console.log(mp1);

let p2 = { name: "lily" };
let mp2 = new Map([
  ["a", 1],
  [p2, 2],
]);
console.log(mp2);

API#

  • size属性返回 Map 结构的成员总数。
  • set方法设置键名key对应的键值为value，然后返回整个 Map 结构。如果key已经有值，则键值会被更新，否则就新生成该键。set方法返回的是当前的Map对象，因此可以采用链式写法。
  • get方法读取key对应的键值，如果找不到key，返回undefined。
  • has方法返回一个布尔值，表示某个键是否在当前 Map 对象之中。
  • delete方法删除某个键，返回true。如果删除失败，返回false。
  • clear方法清除所有成员，没有返回值。
  • keys()：返回键名的遍历器。
  • values()：返回键值的遍历器。
  • entries()：返回所有成员的遍历器。
  • forEach()：遍历 Map 的所有成员。

ES Set#

ES6+

一种数据结构，类似数组，但是成员的值都是唯一的，不能重复。

let set = new Set([1, 2, 2, 3]);
console.log(set); // Set {1, 2, 3}

API#

api 描述
size 返回 Set 的长度
add() 添加一个值，返回 Set 结构本身
delete() 删除一个值，返回一个布尔值
has() 判断是否存在某成员，返回一个布尔值
clear() 清除所有成员
keys() values 遍历键值，同 for..of
entries() 遍历返回键值对
forEach() 使用回调函数遍历每个成员

应用#

去除数组重复成员

[...new Set(array)];

去除字符串里面重复字符

[...new Set(str)], join("");

Symbol#

保证对象的每个属性的名字都是独一无二的，从根本上面防止属性名冲突。

一、使用#

1、不用加 new

let s = Symbol()
typeof s // "symbol"

2、可以加描述符

let s = Symbol('foo') 
s.toString() // 'Symbol(foo)'
``` 

3、`Symbol.prototype.description`
> ES2019新增
```js
const s = Symbol('foo')
s.description // "foo"

4、Symbol.for / Symbol.keyFor
有时，我们希望重新使用同一个Symbol值，Symbol.for()方法可以做到这一点。它接受一个字符串作为参数，然后搜索有没有以该参数作为名称的 Symbol 值。如果有，就返回这个 Symbol 值，否则就新建一个以该字符串为名称的 Symbol 值，并将其注册到全局。

let s = Symbol.for('foo')
let s2 = Symbol.for('foo')
s1 === s2 // true

Symbol.keyFor () 方法返回一个已登记的 Symbol 类型值的 key。

let s = Symbol.for('foo')
Symbol.keyFor(s) // "foo"

let s2 = Symbol('foo')
Symbol.keyFor(s2) // undefined

二、遍历#

无法遍历: for...in、for...of、Object.keys()、Object.getOwnPropertyNames()、JSON.stringify()

可以遍历：
Object.getOwnPropertySymbols()

const obj = {}
let a = Symbol('a')
let b = Symbol('b')

obj[a] = "Hello"
obj[b] = "World"

const objectSymbols = Object.getOwnPropertySymbols(obj)
// [Symbol(a), Symbol(b)]

Reflect.ownKeys()

let obj = {
  [Symbol('key')]: 1,
  enum: 2,
  nonEnum: 3
}
Reflect.ownKeys(obj)
// ["enum", "nonEnum", Symbol(key)]

三、应用#

1、作为对象的属性名

let mySymbol = Symbol()

// 第一种写法
let a = {}
a[mySymbol] = "hello"

// 第二种写法
let b = {
  [mySymbol]: "hello"
}

// 第三种写法
let c = {}
Object.defineProperty(c, mySymbol, { value: "hello"})

// 以上写法都得到同样的结果
// 只能用[]不能用.取值
a[mySymbol]

ES String API

发表于 2021-01-30 更新于 2021-02-01 分类于 2203：ES API

字符串 API#

字符串是基本类型，所有的方法都无法修改原字符串。

查找 - indexOf / lastIndexOf#

indexOf() 返回指定字符串在字符串首次出现的位置（向后查找，对大小写敏感），如果没找到，返回-1。

STR. indexOf(search [, start])
// 参数
//    search 需检索的字符串
//    start  可选，指定开始检索的位置
//          如果没指定就是从首字符开始搜索
// 返回值
//    目标字符串第一次出现的位置/-1

let str = “poxiaoispig";
str.indexOf("poxiao"); // 0
str.indexOf("pig"); // 8

lastIndexOf() 同 indexOf() ，区别是逆序的（向前查找）。

str.lastIndexOf("pig"); // 8

查找 - includes#

includes() 查找字符串是否包含指定字符串（对大小写敏感），如果找到返回true ,否则返回false。

STR. includes(search, [, start])
// 参数
//    search 需查找的字符串
//    start  可选，指定开始检索的位置
//          如果没指定就是从最后一个字符开始搜索
// 返回值
//    目标字符串第一次出现的位置/-1

let str = “poxiaoispig";
str.includes("poxiao"); // true
str.includes("px");   // false

查找 - search#

search()功能和indexOf()相同，定义匹配模式，只有一个参数，没有反向检索和全局模式。（正则表达式常用）

STR.search(tar);
// 参数
//    tar 要搜索的字符串
// 返回值
//    目标字符串第一次出现的位置/-1

let str = "poxiaoispig";
str.search("pig"); // 8

查找 - match#

match() 找出所有匹配的子字符串，并以数组的形式返回，如果匹配模式没有指定全局（正则表达式），则进行单次匹配，如果没找到返回null。该方法类似 indexOf() 和 lastIndexOf()，但是它返回指定的值，而不是字符串的位置。

STR.match(re);
// 参数
//    re  字符串或者正则表达式
// 返回
//    数组（包含所有匹配的字符串）

let str = "poxiao";
str.match("xiao");
// [ 'xiao', index: 2, input: 'poxiao', groups: undefined ]

查找 - charAt/charCodeAt#

charAt(n) 返回字符串中的第 n 个字符，如果 n 超出字符串范围，返回空串。

charCodeAt(n) 返回字符串中第 n 个字符的字符编码。

判断头尾 - startsWith/endsWith#

startsWith() endsWith() 判断字符串开头、结尾有没有包含某个字符串。

STR.startsWith(key);
STR.endsWith(key);
// 参数
//    key 目标字符串
// 返回
//    true / false

"poxiao".startsWith("po"); // true
"poxiao".endsWith("po"); // false

切成数组 - split#

split() 把一个字符串切割成数组，这个操作和数组join是相反的。

STR. split(rule[, max])
// 参数
//    rule 字符串或者正则表达式
//    max  可选，指定返回数组的的最大长度
// 返回值
//    数组

let str = "poxiao";
str.split(); // ["poxiao", ]
str.split(""); // ["p","o","x","i","a","o"]
str.split("", 2); // ["p","o"]

截取 - substr/slice/substring#

substr 根据指定长度截取字符串（不推荐使用该方法）

STR. substr(start[, length]);
// 参数
//    start  起始下标
//    length 可选，截取的长度
//           如果省略，则截取到末尾
// 返回值
//    截取之后的字符串

slice 、substring 根据指定的起止下标截取字符串

STR. slice(start[, end]);
STR. substring(start[, end]);
// 参数
//    start 起始下标
//    end   可选，结束下标（不包括）
//          如果省略，代表剩下的全部截取

起始和结束位置相比较的大小无法确定的时候可以使用substring。

如果为负值，slice 能把负值当作从右向左。而substring视其无效

大小写转换 - toLowerCase / toUpperCase#

toLowerCase() 将字符串转换成小写

toUpperCase() 将字符串转换成大写

STR.toLowerCase();
STR.toUpperCase();
// 参数无
// 返回
//     转换之后的字符串

let str = "poxiao";
str.toLowerCase(); //"POXIAO"

拼接 - concat 了解#

concat(s1[, s2, s3])

功能：把多个参数添加到指定字符串的尾部。

参数：要添加到字符串尾部的，参数类型和个数没有限制。

返回：连接以后的字符串。

let str = "poxiao";
str.concat("is", "pig"); // "poxiaoispig"

原来数组的 join 方法替代 concat 方法，速度很快，但是现代浏览器优化加号拼接，所以推荐使用加号拼接。

复制 - repeat#

repeat() 复制字符串。

STR.repeat(count);
// 参数
//    count 复制次数
// 返回值
//    复制之后的字符串

let str = "poxiao";
str.repeat(3); // "poxiaopoxiaopoxiao"

补全 - padStart / padEnd#

padStart() padEnd() 是 ES2017 引入的字符串补全长度的方法。如果某个字符串不够指定长度，会在头部或尾部补全。

STR.padSrart(length, pad);
STR.padEnd(length, pad);
// 参数
//    length 指定字符串的最小长度
//    pad    用来补全的字符串
// 返回值
//    补全之后的字符串
//    如果原字符串的长度，等于或大于指定的最小长度，则返回原字符串。

let str = "poxiao";
str.padStart(2, "1"); // "poxiao"
str.padStart(10, "is"); // "isispoxiao"

替换 - replace#

如果没有使用正则表达式，只能替换一次。

这是用于搜索和替换的通用方法，是最有用的方法之一。它是搜索和替换字符串的瑞士军刀。

当 replace 的第一个参数是字符串时，它仅替换第一个匹配项。

删首尾空格 - trim / trimStart / trimEnd#

trim() trimStart() trimEnd() 去除字符串的头尾空格。

STR.trim();
STR.trimStart();
STR.trimEnd();
// 无参数
// 返回处理之后的字符串（不会改变原字符串）

let str = "   poxiao is pig   ";
str.trim(); // "poxiao is pig"
str.trimStart(); // "poxiao is pig   "
str.trimEnd(); // "   poxiao is pig"

进行异步编程的新解决方案。

3、链式调用#

Promise 实现了链式调用，也就是说每次调用 then 之后返回的都是一个 Promise，并且是一个全新的 Promise，原因也是因为状态不可变。如果你在 then 中 使用了 return，那么 return 的值会被 Promise.resolve() 包装。

4、更多方法#

（1）Promise.prototype.then

返回一个Promise 对象。

（2）Promise.prototype.catch

返回一个Promise 对象。

（3）Promise.resolve

返回的Promise对象状态为fulfilled。

（4）Promise.reject

返回一个状态为失败的Promise对象，并将给定的失败信息传递给对应的处理方法。

（5）Promise.all

  • 接收一个数组，数组的每一个元素都的 Promise
  • 返回的是一个 Promise
  • 数组中所有成功，则返回成功，如果有一个失败，则返回失败状态

这个方法返回一个新的 promise 对象，该 promise 对象在 iterable 参数对象里所有的 promise 对象都成功的时候才会触发成功，一旦有任何一个 iterable 里面的 promise 对象失败则立即触发该 promise 对象的失败。

（6）Promise.race

当 iterable 参数里的任意一个子 promise 被成功或失败后，父 promise 马上也会用子 promise 的成功返回值或失败详情作为参数调用父 promise 绑定的相应句柄，并返回该 promise 对象。

（7）Promise.any

接收一个 Promise 对象的集合，当其中的一个 promise 成功，就返回那个成功的 promise 的值。

（8）Promise.allSettled

等到所有 promises 都完成（每个 promise 返回成功或失败）。
返回一个 promise，该 promise 在所有 promise 完成后完成。并带有一个对象数组，每个对象对应每个 promise 的结果。

function ajax(url) {
  return new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();

        if (xhr.status >= 200 && xhr.status < 300) {
          resolve(xhr.responseText);
        } else {
          reject(xhr.responseText);
        }
      }
    };
    xhr.open("GET", url);
    xhr.send();
  });
}

6、手写一个简单的 Promise#

const PENDING = "pending";
const RESOLVED = "fulfilled";
const REJECTED = "rejected";

function Promise(exec) {
  const that = this;
  that.state = PENDING;
  that.value = null;
  that.resolvedCallbacks = [];
  that.rejectedCallbacks = [];

  function resolved(value) {
    if (this.state !== PENDING) return;
    that.state = RESOLVED;
    that.value = value;
    that.resolvedCallbacks.map((v) => v(that.value));
  }
  function rejected(value) {
    if (this.state !== PENDING) return;
    that.state = REJECTED;
    this.value = value;
    that.rejectedCallbacks.map((v) => v(that.value));
  }
  try {
    exec(resolved, rejected);
  } catch (err) {
    rejected(err);
  }
}

Promise.prototype.then = function (onResolved, onRejected) {
  const that = this;
  onResolved = typeof onResolved === "function" ? onResolved : (v) => v;
  onRejected =
    typeof onRejected === "function" ? onRejected : (err) => throw err;
  switch (that.state) {
    case PENDING:
      that.resolvedCallbacks.push(onResolved);
      that.rejectedCallbacks.push(onRejected);
      break;
    case RESOLVED:
      onResolved(that.value);
      break;
    case REJECTED:
      onRejected(that.value);
      break;
  }
};

JavaScript 函数执行过程

发表于 2021-02-02 分类于 2202：ES高级

1、栈内存和堆内存#

内存根据程序运行时候的逻辑分为栈内存和堆内存。在 JS 中，并没有严格意义区分。大部分的数据都保存在堆内存中，但是在某些场景，仍需要用栈（有时候也叫堆栈）的数据结构的思路进行处理，如执行上下文。

栈内存 堆内存
栈结构 链表结构
占内存小，速度快 占内存大，速度慢
自动分配，自动回收 自己申请，手动释放

2、执行上下文#

JS 引擎并不是逐行解析代码，而是逐段去分析和执行，当开始解析一段代码时，会先进行预处理（声明提升和创建作用域链）。生成一个执行上下文。

执行上下文（执行环境）分为两种变量对象（VO）：

  • Global | Window，默认的全局执行环境。
  • Function，调用函数时会进入的函数执行环境。

代码执行时会开辟一个栈空间来说明程序的执行顺序，这个执行上下文堆栈称为执行栈。开始执行时先将全局执行环境压入栈底（压栈），执行到函数时会创建对应的函数执行环境，并将函数执行环境压入栈底。在函数执行完成后将函数执行环境弹出并销毁。直到所有代码执行完成销毁全局执行环境。

可以把执行栈认为是一个存储函数调用的栈结构，遵循先进后出的原则。

当我们使用递归的时候，因为栈可存放的函数是有限制的，一旦存放了过多的函数且没有得到释放的话，就会出现爆栈的问题。

4、作用域#

作用域 scope 是变量在程序中可以访问的有效访问，也被称为变量的可见性。作用域在进入全局执行环境或者函数执行环境时就已经确定好了。在ES5中，作用域分为两种：

  • 全局作用域
  • 函数作用域

同一个函数的多次调用都会生成不同的执行环境，每次调用都会生成。

全局变量 是定义在全局作用域中的变量，一般在函外部定义。在程序的所有地方都可以使用和操作。

局部变量 是在函数内部定义的，在函数执行域中。只能在自己的作用域中起作用，只能在函数内部使用。外部无法使用和操作局部变量。

5、作用域链#

作用域链 是用来查找变量的这样一系列的过程。一个函数在定义时会生成一个属性[[scope]]，这个属性存储的是函数定义时作用域的层级关系。作用域链是函数执行时创建的，是当前函数作用域链 -> [[scope]]属性作用域组成的。函数当前执行的作用域是作用域链的顶端，全局作用域在作用域链的最后。执行环境销毁时，作用域链也会销毁。

// 全局作用域
var a = 1;

function f() {
  // test -> 全局作用域
  var a = 10;
  function f1() {
    // test2 -> test -> 全局作用域
    var a = 100;
    console.log(a); //100
  }
  function f2() {
    console.log(a); //10
  }
  f1();
  f2();
}

f();
console.log(a);

当执行 f1 时，创建函数 f1 的执行环境，并将该对象置于链表开头，然后将函数 f 的调用对象放在第二位，最后是全局对象，作用域链的链表的结构是 f1->f->window 。从链表的开头寻找变量 a，即 f1 函数内部找变量 a，找到了，结果是 20。同样，执行 f2 时，作用域链的链表的结构是 f2->f->window 。从链表的开头寻找变量 a，即 f2 函数内部找变量 a，找不到，于是从 f 内部找变量 a，找到了，结果是 10。最后在最外层打印出变量 a，直接从变量 a 的作用域即全局作用域内寻找，结果为 1。

6、内存中的变量存储#

变量如果是基本数据类型，会在直接在栈内存开辟一段存储空间标识符。然后将基本数据类型的值也存储在栈内存中和标识符对应起来。要求变量名不能重名，重名就会覆盖。

如果是引用数据类型，值储存在堆内存中，存储引用类型的时候标识符是储存在栈内存中的。（JS 不允许直接访问堆内存中的数据，只能通过引用访问，差不多就是在变量对象中存储一个指向对象的句柄，可以理解成是一个地址，要访问这个对象，就要通过这个引用句柄来访问）。

var a = 100;
var b = [1, 2, 3];
var c = b;
c[0] = 1000;

console.log(b); // [1000,2,3]

程序一开始执行，进入全局环境，首先会创建全局环境并且压栈（执行栈），全局代码执行的时候依赖的是全局环境中的东西。全局变量如果是基本类型，那么这个值直接存在栈当中，如果这个变量的值是引用类型，那么引用类型会在堆内存中开辟自己的空间专门存储（数据本身）。然后里面的这块空间的地址存在栈当中对应的变量。

当函数执行碰到函数调用，函数执行时也要有自己的环境去依赖。函数执行也是创建自己的函数环境进行压栈（函数执行环境一定压在全局执行环境的上面）。局部变量，是在函数环境当中存在的，只有函数执行，局部变量才会出现，函数执行完成之后，函数环境要弹出栈（销毁归还内存），局部变量也就不存在了。

当函数调用完成之后，会继续执行全局代码，一直到所有的全局代码执行完成，代表程序执行结束，程序执行结束的时候，全局环境最后出栈（销毁）。

7、声明提升#

也称为预解析。在执行环境的第一步（创建阶段），会将带 var 的变量声明和函数声明，放到作用域中。在执行阶段时，它的相关逻辑还是放在原来的位置进行赋值和其他的逻辑处理。函数表达式和函数构造器不会被提升。函数声明和变量都会提升，但是函数声明会先提升，然后是变量。

变量声明提升 通过 var 声明的变量，在定义语句之前就能访问到，值为 undefined。

函数声明提升 通过 function 声明的函数，在前面就可以直接调用，值为函数定义（对象）。

exports 本身不具备导出功能，但是修改它的时候，其实是修改了module.exports 。exports 是指向 moudlue.exports 的对象地址的一个新对象。（exports = xxx 的用法是错误的，这样使其不再绑定。正确的用法是 exports.xxx = xxx ）

不能将一个值赋值给exports，而是使用exports.XXX来暴露。否则这样它将不再绑定到module.exports。如果 exports 导出的变量类型是引用类型如函数，则会断开与module.exports的地址指向，导致变量导出失败。因为最终还是要靠module.exports来导出变量的。

例子

function add(x, y) {
  return x + y;
}
// 暴露出去
exports.add = add; // { add: [Function: add] }

const obj = {

async / await#

async 是 Promise 和 Generstor 语法糖。

1、async 函数#

async function fn() {}

2、await 表达式#

await 表达式必须写在 async 函数中。

await 右侧的表达式一般为 Promise 对象，但也可以是其他值。

如果表达式是 Promise 对象，返回的是 Promise 成功的值。如果失败的，则会抛出异常，需要通过 try... catch 捕捉。

如果是其他值，则直接将此值作为 await 的返回值。

// 例子1
async function fn() {
  return 1;
}
const result = fn();
console.log(result); // Promise { 1 }
result.then((value) => {
  console.log(value); // 1
});

// 例子2
async function fn() {
  throw "异常";
}
const result = fn();
console.log(result); // Promise { <rejected> '异常' }
result
  .then((value) => {
    console.log(value);
  })
  .catch((reason) => {
    console.error(reason); // 异常
  });

// 例子3
async function fn() {
  return new Promise((resolve, reject) => {
    reject("error");
  });
}
const result = fn(); // 失败的Promise
result.catch((reason) => {
  console.log(reason); // error
});

await 会阻塞函数的运行，需要等后边的异步代码执行完毕才会向下执行。
