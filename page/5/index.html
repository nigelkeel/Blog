函数柯里化#

柯里化（Currying），是一种函数的转换，它是指将一个函数从可调用的 f(a, b, c) 转换为可调用的 f(a)(b)(c)。柯里化不会调用函数。它只是对函数进行转换。

弹性布局#

flex ，即弹性布局。

1、开启弹性布局#

开启 flex 的元素自身不受影响，只会影响子元素，子元素的float、clear和vertical-align属性将失效。对子元素的子元素不会影响。
a
开启flex的盒子称为 容器（container），子元素称为 项目（item）。横向默认是 主轴（main axis）。纵向默认是 侧轴（交叉轴，cross axis）。默认情况下，项目沿着主轴从左到右依次排列。项目的高度撑满容器（侧轴撑满）。

当项目的总宽度超出，并不会换行，而是压缩项目的宽度。

涉及到 12 个 CSS 属性，父容器，子容器各 6 个。

2、常用属性#

flex-direction#

默认情况下，主轴的方向是从左向右的，交叉轴垂直于主轴。

交叉轴是由主轴决定的，主轴又是由 flex-direction 决定的，这个属性设置在父容器上，才可以生效。

定位#

定位（position） 允许你定义元素框相对于其正常位置应该出现的位置，或者相对于父元素，以及浏览器窗口本身的位置。

属性 描述
static 默认，没有定位
relative 相对定位
absolute 绝对定位
fixed 固定定位

布局一般用在大布局的小范围中。

1、相对定位 - relative#

相对定位（relative），让元素相对定位，元素先放置在未添加定位时候的区域，然后在不改变页面布局的情况下（其他元素没有受到任何影响；自身原来的位置也保留；和浮动可以一起使用）进行移动。不会脱离文档流。

2、绝对定位 - absolute#

绝对定位（absolute），相对于它的 包含块 定位。绝对定位是相对于它的包含块定位的（和自身原有位置无关）。如果没有任何一个父级元素是非 static 属性，则会相对于文档（html）定位。如果它的父级元素和爷爷级元素都是非 position:static 属性，则它会选择距离最近的父元素。脱离正常文档流，原有位置不保留。定位的原点以包含块的 padding-box 的左上角为原点。right 和 bottom 是距离包含块右下角的距离。

包含块#

包含块 是视觉格式化模型的一个概念，简单来说就是一个定位参考框。

如何确定一个元素的包含块，完全取决于它自身的 position 属性：

  • 如果一个元素自身的 position 属性是 static 或者是 relative ，它的包含块就是离他最近的祖先元素或者是格式化上下文。
  • 如果一个元素自身的 position 属性是 absolute ，它的包含块就是离他最近的拥有定位属性（值不为 static ）的元素。
  • 如果一个元素自身的 position 属性是 fixed，它的包含块就是 viewport。

补充：
如果一个元素的 position 属性是 absolute 或者是 fixed 在下边几种情况下，包含块会发生改变。

  • 当祖先元素拥有 transform 或 perspective 属性并且值不为 none 的时候，它也是被当做包含块。
  • 当祖先元素 拥有 filter 属性的时候（值不为 none） 它也可以被当做包含块

注：如果由内向外找不到包含块条件的元素，那么 html（根元素）被称作为初始包含块。

子绝父相#

父元素相对定位并不设置位移，子元素绝对定位，设置位移。

3、固定定位 - fixed#

固定定位（fixed），相对于浏览器窗口定位。

4、粘性定位 - sticky#

sticky 的意思是“粘性的”，position:sticky 表现也符合这个粘性的表现。基本上，可以看出是position:relative和position:fixed的结合体——当元素在屏幕内，表现为relative，就要滚出显示器屏幕的时候，表现为fixed。

浮动#

浮动（float）可以让多个块元素横向排列，共行。

属性 说明
none 默认，不浮动
left 左浮动
right 右浮动

1、概念#

正常页面流/正常文档流 是行元素从左至右，块元素从上到下布局。浮动 是破坏流（脱离文档流）的一种方式。在目前开发中，很多的元素都是一个个 float 堆叠起来，但是这种布局很容易崩溃。只要宽度和高度发生一点变化，页面就会发生错乱。

2、浮动的使用#

左浮动 就是让容器沿着容器左侧排序。

右浮动 让容器沿着容器右侧排序。

3、高度塌陷#

浮动元素以后，脱离父级内容区域，父级没有内容撑开自身的高度，而且父级的兄弟元素是按照父级的位置进行布局的，所以页面会乱。

4、清除浮动 常用#

清除浮动不是不让元素浮动，是清除浮动对父级高度塌陷的影响。

清除浮动的方法

  • 给浮动元素的父级设置高度。
  • 给浮动元素的父级设置浮动，以浮制浮。
  • overflow 设置不为 visible 的属性。
  • 在最后一个浮动元素的后边添加一个空的块级元素，添加一个块级元素，添加属性和值为 clear: both。
  • after 伪元素清浮动，给所有浮动元素父级一个 clearFix 的类名。推荐

回调函数#

回调函数（Callback）分为同步回调函数，异步回调函数。

什么函数才是回调函数?

  • 你定义的
  • 你没有直接调用
  • 但最终它执行了（在特定条件或时刻）

常见的回调函数:

  • DOM 事件函数
  • 定时器函数
  • AJAX 回调函数
  • 生命周期回调函数

回调函数的特性导致其层层嵌套，形成回调地狱。

涉及面试题：什么是回调函数？回调函数有什么缺点？如何解决回调地狱问题？

回调函数（Callback）分为同步回调（数组遍历）和异步回调。

使用回调函数容易导致回调地狱问题。回调地狱的根本问题就是：

  1. 嵌套函数存在耦合性，一旦有所改动，就会牵一发而动全身
  2. 嵌套函数一多，就很难处理错误。

当然，回调函数还存在着别的几个缺点，比如不能使用 try catch 捕获错误，不能直接 return。

面试题 简述同步和异步的区别，如何避免回调地狱？

同步：方法调用一旦开始，调用者必须等到方法调用返回后，才能继续后续的行为

异步：方法调用一旦开始，方法调用就会立刻返回，调用者就可以继续后续的操作，而异步方法通常会在另外一个线程中，整个过程畅通无阻。

避免回调地狱：

  • Promise
  • async / await
  • generator
  • 事件发布/监听模式

Ajax#

AJAX 即 Asynchronous Javascript And XML，异步 JavaScript 和 XML。是指一种无需重新加载整个网页的情况下，能够更新部分网页的技术。通过 Ajax 可以在浏览器中向服务器发起异步请求。

XML 称为可扩展标记语言，被设计用来传输和存储数据，和 HTML 类似，不同的是 HTML 日常使用大部分都是预定义标签，XML 全部都是自定义标签，用来表示一些数据。

1、工作原理#

相当于在用户和服务器之间加了一个中间层（Ajax 引擎），使用户操作与服务器响应异步化。

2、特点#

优点

  • 无需刷新页面来和服务器通信。
  • 允许你根据用户事件来更新部分页面内容。

缺点

  • 没有浏览历史，无法回退。
  • 存在跨域问题。
  • 对 SEO 不友好。

3、XHR - XMLHttpRequest#

Ajax 的所有操作都是通过 XMLHttpRequest 对象进行的。

4、使用步骤 考#

（1）创建异步调用对象#

创建 XMLHttpRequest 核心对象。

（2）创建 HTTP 请求#

通过 open(method, URL) 打开接口。

  • method 请求方法，POST 或者 GET。
  • URL 接口地址。

POST 方式要设置请求头

  • application/json
  • application/x-www-form-urlencoded
  • formdata

xhr.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded')

（3）发送请求#

发送请求 send(body)

GET 请求不需要参数， POST 请求需要。

（4）通过指定回调函数接收数据#

xhr.responseXML 接收 XML 格式的响应数据。

xhr.responseText 接收文本格式的响应数据。

a. 通过 onreadystatechange 事件接收数据，xhr 对象有 readyState 表示当前的请求状态，状态改变就会触发这个事件。

b. 通过 onload 事件

5、XHR API △#

（01）XMLHttpRequest() 创建 XHR 对象的构造函数

（02）readyState 当前的请求状态

状态 常量 描述
0 UNSENT 表示 XMLHttpRequest 实例已经生成，但是 open() 方法还没有被调用。
1 OPENED 表示 send()方法还没有被调用，仍然可以使用setRequestHeader()，设定 HTTP 请求的头信息。

3 LOADING 表示正在接收服务器传来的 body 部分的数据，如果 responseType 属性是 text 或者空字符串，responseText 就会包含已经收到的部分信息。
4 DONE 表示服务器数据已经接收完成，或者本次接收已经失败了。

JavaScript BOM#

Browser Object Model，浏览器对象模型。

一、window 对象#

window 是 BOM 的核心对象。表示浏览器的一个实例，是客户端浏览器对象模型的基类。在浏览器中，该对象具有双重角色，既是通过 JS 访问浏览器窗口的一个接口，也是 ES 规定中的 Global 对象。这意味着在网页中定义的任何一个对象、变量和函数，都以 window 作为其 Global 对象。所有在全局作用域中声明的变量、函数都会变成该对象的属性和方法。

1、客户端对象#

可以访问客户端其他对象，这种关系构成浏览器对象模型。

对象 描述
document 包含整个 HTML 文档，可以用来访问文档内容及其所有页面元素。
navigator 包含客户端有关浏览器的信息。
screen 包含客户端屏幕的信息
history 包含浏览历史记录信息
location 包含当前网页文档的 URL 信息

2、系统对话框#

浏览器通过 alert()、confirm() 和 prompt() 方法可以调用系统对话框向用户显示消息。

系统对话框与在浏览器中显示的网页没有关系，也不包含 HTML。它们的外观由操作系统及（或）浏览器设置决定，而不是由 CSS 决定（无法设置）。通过这几个方法打开的对话框都是同步和模态的。也就是说，显示这些对话框的时候代码会停止执行，而关掉这些对话框后代码又会恢复执行。

confirm和prompt有返回值。生产环境下不用，调试使用。

3、打开和关闭窗口#

window.open() 方法既可以导航到一个特定的 URL，也可以打开一个新的浏览器窗口。

参数

  • 要加载的 URL
  • 新窗口打开方式：_self _blank（默认新标签页打开）
  • 窗口大小、位置。

返回值 就是新建的窗口名称。

窗口名.close()可以关闭窗口，如果关闭自身那就使用 window.close()。

close() 方法仅适用于通过 window.open() 打开的弹出窗口。

4、浏览器宽高#

window对象有innerWidth和innerHeight属性，可以获取浏览器窗口的内部宽度和高度。内部宽高是指除去菜单栏、工具栏、边框等占位元素后，用于显示网页的净宽高。

对应的，还有一个outerWidth和outerHeight属性，可以获取浏览器窗口的整个宽高。

二、navigator 对象#

navigator 对象储存了浏览器信息。通过 navigator 可以引用该对象，并利用它的属性来读取客户端基本信息。

常见属性：

  • onLine 表示浏览器是否连接到因特网
  • platform 浏览器所在的操作系统
  • userAgent 浏览器识别码
  • appName 浏览器名称
  • labguage 浏览器设置的语言

1、浏览器检测方法#

检测浏览器类型的方法有多种，常用的方法包括两种:特征检测法和字符串检测法，这两种方法都存在各自的优点与缺点，用户可以根据需要进行选择。

（1）特征检测法

  • 特征检测法就是根据浏览器是否支持特定功能来决定相应操作的方式。这是一种非精确判断法。但却是最安全的检测方法
  • 因为准确检测浏览器的类型和型号是一件很困难的事情,而且很容易存在误差，如果不关心浏览器的身份，仅仅在意浏览器的执行能力，那么使用特征检测法就完全可以满足需要。
  • 当使用一个对象、方法或属性时，先判断它是否存在，如果存在，则说明浏览器支持该对象、方法属性，那么就可以放心使用

（2）字符串检测法

  • 客户端浏览器每次发送 HTTP 请求时，都会附带有一个 user-agent(用户代理)字符串，对于 Web 开发人员来说，可以使用用户代理字符串检测浏览器类型。
  • userAgent 字符串包含了 web 浏览器的大量信息，如浏览器的名称和版本。

2、操作系统检测方法#

navigator.userAgent 返回值一般都会包含操作系统的基本信息，不过这些信息比较散乱，没有统一的规则。

用户可以检测一些更为通用的信息，如检测是否为 Windows 系统，或者为 Macintosh 系统，而不去分辨操作系统的版本号。

例如，如果仅检测通用信息，那么所有 Windows 版本的操作系统都会包含"Win”字符串，所有 Macintosh 版本的操作系统都包含有"Mac”字符串，所有 Umix 版本的操作系统都包含有"X11”，而 Linux 操作系统会同时包含"X11”和" Linux。

四、history 对象 基础#

history对象保存了浏览器的历史记录，JS 可以调用history对象的back()或forward ()，相当于用户点击了浏览器的后退或前进按钮。

这个对象属于历史遗留对象，对于现代 Web 页面来说，由于大量使用 AJAX 和页面交互，简单粗暴地调用history.back()可能会让用户体验非常不好。

transition-property: 属性
transition-duration: 间隔

1、过渡动画 transition#

允许 CSS 的属性在一定的时间区间内平滑过渡。在鼠标点击，鼠标滑过或对元素任何改变中触发，并平滑地以动画改变 CSS 的属性值。

transition

  • property 参与过渡的属性
  • duration 持续时间
  • timing-function 动画类型 linear 线性匀速
  • delay 延迟时间

animation / keyframes

animation-name: 动画名称，对应@keyframes
animation-duration: 间隔
animation-timing-function: 曲线
animation-delay: 延迟
animation-iteration-count: 次数

infinite: 循环动画

animation-direction: 方向

alternate: 反向播放

animation-fill-mode: 静止模式

forwards: 停止时，保留最后一帧
backwards: 停止时，回到第一帧
both: 同时运用 forwards / backwards

动画分为 过渡动画 和 关键帧动画。
