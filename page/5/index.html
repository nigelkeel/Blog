ES String API

发表于 2021-01-30 更新于 2021-02-01 分类于 2203：ES API

字符串 API#

字符串是基本类型，所有的方法都无法修改原字符串。

查找 - indexOf / lastIndexOf#

indexOf() 返回指定字符串在字符串首次出现的位置（向后查找，对大小写敏感），如果没找到，返回-1。

STR. indexOf(search [, start])
// 参数
//    search 需检索的字符串
//    start  可选，指定开始检索的位置
//          如果没指定就是从首字符开始搜索
// 返回值
//    目标字符串第一次出现的位置/-1

let str = “poxiaoispig";
str.indexOf("poxiao"); // 0
str.indexOf("pig"); // 8

lastIndexOf() 同 indexOf() ，区别是逆序的（向前查找）。

str.lastIndexOf("pig"); // 8

查找 - includes#

includes() 查找字符串是否包含指定字符串（对大小写敏感），如果找到返回true ,否则返回false。

STR. includes(search, [, start])
// 参数
//    search 需查找的字符串
//    start  可选，指定开始检索的位置
//          如果没指定就是从最后一个字符开始搜索
// 返回值
//    目标字符串第一次出现的位置/-1

let str = “poxiaoispig";
str.includes("poxiao"); // true
str.includes("px");   // false

查找 - search#

search()功能和indexOf()相同，定义匹配模式，只有一个参数，没有反向检索和全局模式。（正则表达式常用）

STR.search(tar);
// 参数
//    tar 要搜索的字符串
// 返回值
//    目标字符串第一次出现的位置/-1

let str = "poxiaoispig";
str.search("pig"); // 8

查找 - match#

match() 找出所有匹配的子字符串，并以数组的形式返回，如果匹配模式没有指定全局（正则表达式），则进行单次匹配，如果没找到返回null。该方法类似 indexOf() 和 lastIndexOf()，但是它返回指定的值，而不是字符串的位置。

STR.match(re);
// 参数
//    re  字符串或者正则表达式
// 返回
//    数组（包含所有匹配的字符串）

let str = "poxiao";
str.match("xiao");
// [ 'xiao', index: 2, input: 'poxiao', groups: undefined ]

查找 - charAt/charCodeAt#

charAt(n) 返回字符串中的第 n 个字符，如果 n 超出字符串范围，返回空串。

charCodeAt(n) 返回字符串中第 n 个字符的字符编码。

判断头尾 - startsWith/endsWith#

startsWith() endsWith() 判断字符串开头、结尾有没有包含某个字符串。

STR.startsWith(key);
STR.endsWith(key);
// 参数
//    key 目标字符串
// 返回
//    true / false

"poxiao".startsWith("po"); // true
"poxiao".endsWith("po"); // false

切成数组 - split#

split() 把一个字符串切割成数组，这个操作和数组join是相反的。

STR. split(rule[, max])
// 参数
//    rule 字符串或者正则表达式
//    max  可选，指定返回数组的的最大长度
// 返回值
//    数组

let str = "poxiao";
str.split(); // ["poxiao", ]
str.split(""); // ["p","o","x","i","a","o"]
str.split("", 2); // ["p","o"]

截取 - substr/slice/substring#

substr 根据指定长度截取字符串（不推荐使用该方法）

STR. substr(start[, length]);
// 参数
//    start  起始下标
//    length 可选，截取的长度
//           如果省略，则截取到末尾
// 返回值
//    截取之后的字符串

slice 、substring 根据指定的起止下标截取字符串

STR. slice(start[, end]);
STR. substring(start[, end]);
// 参数
//    start 起始下标
//    end   可选，结束下标（不包括）
//          如果省略，代表剩下的全部截取

起始和结束位置相比较的大小无法确定的时候可以使用substring。

如果为负值，slice 能把负值当作从右向左。而substring视其无效

大小写转换 - toLowerCase / toUpperCase#

toLowerCase() 将字符串转换成小写

toUpperCase() 将字符串转换成大写

STR.toLowerCase();
STR.toUpperCase();
// 参数无
// 返回
//     转换之后的字符串

let str = "poxiao";
str.toLowerCase(); //"POXIAO"

拼接 - concat 了解#

concat(s1[, s2, s3])

功能：把多个参数添加到指定字符串的尾部。

参数：要添加到字符串尾部的，参数类型和个数没有限制。

返回：连接以后的字符串。

let str = "poxiao";
str.concat("is", "pig"); // "poxiaoispig"

原来数组的 join 方法替代 concat 方法，速度很快，但是现代浏览器优化加号拼接，所以推荐使用加号拼接。

复制 - repeat#

repeat() 复制字符串。

STR.repeat(count);
// 参数
//    count 复制次数
// 返回值
//    复制之后的字符串

let str = "poxiao";
str.repeat(3); // "poxiaopoxiaopoxiao"

补全 - padStart / padEnd#

padStart() padEnd() 是 ES2017 引入的字符串补全长度的方法。如果某个字符串不够指定长度，会在头部或尾部补全。

STR.padSrart(length, pad);
STR.padEnd(length, pad);
// 参数
//    length 指定字符串的最小长度
//    pad    用来补全的字符串
// 返回值
//    补全之后的字符串
//    如果原字符串的长度，等于或大于指定的最小长度，则返回原字符串。

let str = "poxiao";
str.padStart(2, "1"); // "poxiao"
str.padStart(10, "is"); // "isispoxiao"

替换 - replace#

如果没有使用正则表达式，只能替换一次。

这是用于搜索和替换的通用方法，是最有用的方法之一。它是搜索和替换字符串的瑞士军刀。

当 replace 的第一个参数是字符串时，它仅替换第一个匹配项。

删首尾空格 - trim / trimStart / trimEnd#

trim() trimStart() trimEnd() 去除字符串的头尾空格。

STR.trim();
STR.trimStart();
STR.trimEnd();
// 无参数
// 返回处理之后的字符串（不会改变原字符串）

let str = "   poxiao is pig   ";
str.trim(); // "poxiao is pig"
str.trimStart(); // "poxiao is pig   "
str.trimEnd(); // "   poxiao is pig"

转换 - valueOf / toString#

valueOf() 返回对象的字符串、数值或者布尔值表示。

toString() 返回对象的字符串表示。

async / await#

async 是 Promise 和 Generstor 语法糖。

1、async 函数#

async function fn() {}

2、await 表达式#

await 表达式必须写在 async 函数中。

await 右侧的表达式一般为 Promise 对象，但也可以是其他值。

如果表达式是 Promise 对象，返回的是 Promise 成功的值。如果失败的，则会抛出异常，需要通过 try... catch 捕捉。

如果是其他值，则直接将此值作为 await 的返回值。

// 例子1
async function fn() {
  return 1;
}
const result = fn();
console.log(result); // Promise { 1 }
result.then((value) => {
  console.log(value); // 1
});

// 例子2
async function fn() {
  throw "异常";
}
const result = fn();
console.log(result); // Promise { <rejected> '异常' }
result
  .then((value) => {
    console.log(value);
  })
  .catch((reason) => {
    console.error(reason); // 异常
  });

// 例子3
async function fn() {
  return new Promise((resolve, reject) => {
    reject("error");
  });
}
const result = fn(); // 失败的Promise
result.catch((reason) => {
  console.log(reason); // error
});

await 会阻塞函数的运行，需要等后边的异步代码执行完毕才会向下执行。

回调函数#

回调函数（Callback）分为同步回调函数，异步回调函数。

什么函数才是回调函数?

  • 你定义的
  • 你没有直接调用
  • 但最终它执行了（在特定条件或时刻）

常见的回调函数:

  • DOM 事件函数
  • 定时器函数
  • AJAX 回调函数
  • 生命周期回调函数

回调函数的特性导致其层层嵌套，形成回调地狱。

涉及面试题：什么是回调函数？回调函数有什么缺点？如何解决回调地狱问题？

回调函数（Callback）分为同步回调（数组遍历）和异步回调。

使用回调函数容易导致回调地狱问题。回调地狱的根本问题就是：

  1. 嵌套函数存在耦合性，一旦有所改动，就会牵一发而动全身
  2. 嵌套函数一多，就很难处理错误。

当然，回调函数还存在着别的几个缺点，比如不能使用 try catch 捕获错误，不能直接 return。

for (var i = 0; i < 3; i++) {
  setTimeout(function () {
    console.log(i);
  }, 0);
}
// 输出 3 3 3

for (var i = 0; i < 3; i++) {
  (function (i) {
    setTimeout(function () {
      console.log(i);
    }, 0);
  })(i);
}
// 输出 0 1 2

面试题 简述同步和异步的区别，如何避免回调地狱？

同步：方法调用一旦开始，调用者必须等到方法调用返回后，才能继续后续的行为

异步：方法调用一旦开始，方法调用就会立刻返回，调用者就可以继续后续的操作，而异步方法通常会在另外一个线程中，整个过程畅通无阻。

避免回调地狱：

  • Promise
  • async / await
  • generator
  • 事件发布/监听模式

Generstor#

状态机，异步生成器。

function* gen(){
    yield ...

}
    
gen.next();

Generstor 是可迭代的（for.. of）。

// 使用计时器模拟依次请求 a数据 b数据
function* gen(){

        it.next();

    yield setTimeout(() => {
        console.log("b数据请求成功")
    }, 1000);
}

// 开始请求
const it = gen()
it.next();

ES this#

JS 的作用域是静态的，但是函数的调用却是动态的，由于函数可以在不同的运行环境内执行，因此 JS 在函数体内定义了this关键字来获取当前的运行环境。this 最终代表的是函数的执行者，使用者，调用者。它是一个指针型变量。

1、如何判断 this 指向#

  • 函数是否在new中调用，如果是的话，this绑定的是新创建的对象
  • 函数是否通过call、apply 显示绑定，如果是，则this绑定的是执行的对象
  • 函数是否在某个上下文中调用，如果有，则this绑定在这个上下文对象上
  • 如果都不是，默认绑定window

1.1 默认绑定#

this 默认指向 window，在无法应用其他规则的时候使用

function foo() {
  console.log(this);
}
foo(); //window

var obj = {
  do: function () {
    foo();
  },
};
// foo是直接使用不带任何修饰的函数引用进行调用，因此只能使用默认绑定规则

obj.do(); // window

1.2 隐式绑定#

当函数引用有上下文对象的时候（obj），隐式绑定规则会把函数中的 this 绑定到这个上下文对象上。

function foo() {
  console.log(this.a);
}
var obj = { a: 2, foo: foo };

obj.foo(); // 2
// 当foo调用的时候，它的落脚点确实是指向的obj对象，当函数引用有上下文对象的时候（obj），隐式绑定规则会把函数中的this绑定到这个上下文对象上。

隐式绑定可能会出现隐式丢失问题。被隐式绑定的函数，丢失了绑定对象

function foo() {
  console.log(this.a);
}
var obj = {
  a: 2,
  foo: foo,
};
var fn1 = obj.foo;
var a = "hello";
fn1(); // hello
// 虽然fn1是obj.foo的一个引用，但是实际上它的引用是foo函数本身，因此fn1其实是一个不带任何修饰的函数调用，属于默认绑定

每个函数对象都有 call 和 apply 方法，本质是设置函数体内 this 对象的值。

2、call、apply、bind#

call、apply、bind 是 Function 对象自带的方法。

这三个函数的存在意义是改变函数执行时的上下文，再具体一点就是改变函数运行时的 this 指向。

2.1 call#

call() 方法在使用一个指定的 this 值和若干个指定的参数值的前提下调用某个函数或方法。当调用一个函数时，可以赋值一个不同的 this 对象。this 引用当前对象，即 call 方法的第一个参数。

通过 call 方法，你可以在一个对象上借用另一个对象上的方法

// 语法
fun.call( thisArg[, arg1..])
// 参数列表
// - thisArg 指定函数运行时的 this值
//   - 不传，或者传 null，undefined， 函数中的this指向window对象
//   - 传递另一个函数的函数名，函数中的this指向这个函数的引用，并不一定是该函数执行时真正的this值
//   - 值为原始值(数字，字符串，布尔值)的this会指向该原始值的自动包装对象，如 String Number Boolean
//   - 传递一个对象，函数中的this指向这个对象
// - arg 函数所需要的参数，逗号间隔
function a() {
  //输出函数a中的this对象
  console.log(this);
}
//定义函数b
function b() {}

var obj = { name: "我是obj" }; //定义对象obj
a.call(); //window
a.call(null); //window
a.call(undefined); //window
a.call(1); //Number
a.call(""); //String
a.call(true); //Boolean
a.call(b); // function b(){}
a.call(obj); //Object

2.2 apply#

apply() 语法与 call() 方法的语法几乎完全相同，唯一的区别在于，apply 的第二个参数必须是一个包含多个参数的数组（或类数组对象）。

2.3 bind#

bind 函数会创建一个新函数（称为绑定函数）

  • 传参和 call 或 apply 类似
  • 不会执行对应的函数，call 或 apply 会自动执行对应的函数
  • 返回对函数的引用
  • 语法 fun.bind(thisArg[, arg1[, arg2[, ...]]])。

闭包#

1、什么是闭包#

闭包是一个包含被引用局部变量的 closure 对象，存在嵌套的内部函数中。

2、怎么产生闭包 & 生命周期#

必须满足三个条件

  • 函数嵌套函数
  • 内部函数调用外部函数的局部变量（引入全局变量是不会产生闭包的）
  • 调用外部函数

生命周期

  • 内部函数定义执行完成就产生
  • 内部函数成为垃圾对象就销毁

3、闭包的作用与缺点#

作用

  • 延长了局部变量的存活时间
  • 让函数外部操作函数内部的数据

缺点

  • 用太多可能会导致内存泄露（因为占用内部空间不释放）

4、闭包的应用#

  • 迭代器
  • React 组件高阶函数（为了复用函数，给内部函数传参）
  • Vue 原理中存在（双向数据绑定原理）

面向对象和面向过程是两种不同的编程思想和方法。
面向过程是以过程为中心，就像流水线一样设计程序的思想。面向对象是以对象为中心的编程思想。更符合人类的思维习惯。

面向对象的三个基本特性: 封装、继承、多态。
对象（Object） 可以是一个变量，一个函数，一种数据结构。在 JS 中，一切皆为对象。JS 允许自定义对象，从狭义的概念来分析，对象是最基本的数据类型，是复合型的结构、引用型的数据，它是无序数据集合。对象中每个成员被称为属性。对象是无序的键值对，数组是有序的键值对，函数是一种可执行的特殊对象。

函数#

函数是可以反复使用的并且命名的代码块。本质上也是一种属于对象类型的数据。与其他数据类型不同的是，函数的值是可执行的代码。分为系统函数和自定义函数。

函数是数据，函数名是这个数据的标识。需要访问函数的数据去掉圆括号就好。一个函数可以有多个标识。

1、函数的作用#

  • 解决代码的冗余问题，实现代码的复用。
  • 封装代码，让函数内部的代码对外部不可见。
  • 可以让整个代码项目通过函数模块化。

2、新建函数#

函数名定义的规则遵循标识符的规则。函数的声明函数名可以重复，一旦重名，前面的覆盖后面的。

2.1 声明函数 常#

function name(args) {
  // 函数体
  return statement; // 返回语句
}

// name -- 函数名
// args -- 参数列表（可选）

var 和 function 都是声明语句，它们声明的变量和函数都在 JS 预编译时被解析，也被称为变量提升和函数提升。

ES5 中，不允许函数书写在非函数的代码块中（比如 if 语句中）。

2.2 函数表达式 常#

// 赋值调用（被称为函数表达式，这种方法只能提升函数名，不能提升函数体）
// 调用需在创建之后
let fun = function () {
  // 函数体
};

函数可以不书写名字，这种被称为匿名函数。

执行方法：

// 自调用
(function () {
  // 函数体
})();

!(function () {
  // 函数体
})();

2.3 构造函数式 了解#

类如 const x = new XXXX() 的形式，被称为构造函数。

const fun = new Function("执行字符串");
// 这个定义函数的方法几乎不用（会执行两次）

2.4 箭头函数 ES6 常#

  • 内部没有this ，箭头函数与包围它的代码共享一个this。
  • 不能用于构造函数（不能被new 实例化）。
  • 内部没有没有argument对象，可用rest参数代替。
  • 箭头函数不能使用yield命令，即不能用作Generator函数。
// 1、函数有参数，函数体只有一句返回值
function fn1(n) {
  return n + 1;
}

const fn1 = (n) => n + 1;

// 2、函数没有参数，函数体只有一句返回值
function fn2() {
  return "hello world";
}
const fn2 = () => "hello world";

// 3、函数参数随意，函数体不仅仅是一句返回值
function fn3() {}
const fn3 = () => {};

// 用箭头函数和reduce数组求和
const arrSum = (arr) => arr.reduce((p, n) => p + n);

3、使用函数#

函数需要调用才会执行。

4、参数#

JS 数据传递只有值传递，没有引用传递。

4.1 形参与实参#

形参 形式上的参数，声明函数时候提供的参数。

实参 在调用时，实际使用的参数。

4.2 默认参数#

// ES6之前
function f(name, age) {
  age ? age : (age = 18);
}
// ES6
function f(name, age = 18) {}

4.3 剩余参数#

// 可以把不确定的参数表示成数组
function f(a, ...b) {
  return b;
}
f(1, 2, 3, 4); // [2,3,4]

4.4 argument 对象#

二者在传值的时候是一一对应的。二者的个数可以不相等，形参多的时候，多出来的自动被赋值 undefined，实参多的时候，多出来的放在**arguments**对象中。该对象在函数内部才有，实际上是一个类数组 array-like 的对象，可以访问这个对象获得传递给函数的每一个实参。函数中的形参可写可不写，即使不写形参也可以使用 arguments 对象获得，使用形参比较方便。arguments 对象的 length 属性用来确定传递进来多少个实参。可以通过 length 属性的值的不同（根据传入的参数个数的不同），进行不同的操作。arguments对象中保存着一个叫callee的属性，这个属性是一个指针，它指向拥有这个arguments对象的函数。

  • 使用 arguments 对象的 length 属性可以获取函数的实参个数。
  • argument 对象只能在函数内可见,因此 arguments.legth 也只能在函数体内使用。
  • 使用函数对象的 length 属性可以获取函数的形参个数,该属性为只读属性,在函数体内,体外都可以使用

callee 是 arguments 对象的属性，它引用当前 argument 对象所在的函数，使用该属性可以在函数体内调用自身。

// 检测函数的形参和实参是否一致
arguments.length != arguments.callee.length;

5、返回值#

return 没有指定值时函数返回的是undefined，函数在执行完该语句以后立即停止执行并退出函数。

返回值是一个函数的函数称为高阶函数。

错误与异常处理#

ECMA-262 规范了7种错误类型。其中Error是基类，其他6种错误类型是子类，都继承了基类。基类的主要作用是自定义错误对象。

1、错误类型#

类型 说明
Error 基类
name 可以读写异常类型（TypeError）
message 可以读写详细的错误信息。
EvalError 不正确使用eval()方法时抛出
SyntaxError 语法错误
ReferenceError 读取不存在的变量时抛出
TypeError 值的类型发生错误的时候抛出
RangeError 数字超出合法范围
URIError URI编码和解码错误

2、try…catch#

try{
    // 调试代码块
} catch (e) {
    // 捕获异常，并进行异常处理代码块
} finally {
    
}

一般情况下 try...catch 就够用了。

按顺序执行 try 子句中的代码，如果没有异常发生，将会忽略 catch 跳转到 finally 子句继续执行。

无法捕获逻辑错误。

3、throw#

抛出异常。

try {
    let num = 1 / 0;
    if (num = Infinity) {
        // 抛出错误
        throw new Error("除数不能为0");
    }
} catch (exception) {
    console.log(exception.message);
}

抛出错误之后，后续代码不会执行。

&& 与

  • 第一步：计算第一个操作数
  • 第二步：如果第一个操作数转换成 false，那么就会结束运算，直接第一个操作数
  • 第三步：如果第一个操作数返回 true，则计算第二个操作数的值
  • 第四步：第二个操作数如果返回 true 则逻辑与返回第二个操作数，否则返回第一个操作数

|| 或

  • 第一步：计算第一个操作数的值。
  • 第二步：检测第一个操作数的值。如果左侧的表达式的值可以转换为 true，那么就会结束运算。直接返回第一个操作数的值
  • 第三步：如果第一个操作数可以转换为 false，则计算第二个操作数的值。并返回第二个操作数的值

! 非

false || ""; // f || f 返回 ""
