二、HTTP 协议 ==常考==

Hyper Text Transfer Protocol 超文本传输协议（通信协议规范）

客户端和服务器端通信传输的内容称为报文。

请求报文 响应报文

HTTP + SSL = HTTPS

  • General 通用头部
  • Response Headers 响应头部
  • Request Headers 请求头部

Response 响应的具体数据

Initiator 初始化内容

Cookies 携带的cookie

2、报文

HTTP应用程序之间发送的数据块。在客户端、服务器、代理之间流动。分为请求报文和响应报文。

GET 请求报文

请求报文首行

请求方式 请求地址 请求参数(查询字符串) 协议版本

  • 请求方式 GET
  • 协议版本 HTTP/1.1 兼容性 HTTP/2 最新

请求报文头部

  • 请求的主机地址 Host
  • 连接方式 Connection 保持长连接 keep-alive
  • Upgrade-Insecure-Requests 允许使用https访问
  • User-Agent 用户代理（浏览器信息）
  • Accept 可以接收的类型
  • Sec-Fetch-Size -Mode -User -Dest
  • Accept-Encoding 客户端可以接受的压缩格式
  • Accept-Language 可以支持的语言

请求报文空行 一个回车或者空格

3、请求方式

GET 查 用于请求指定的页面信息，并返回实体

POST 增 向指定资源提交数据，进行处理 一般用于登陆注册

PUT 改 更改服务器数据

DELETE 删

OPTIONS 跨域请求

4、MIME 类型

  • text/plain 纯文本类型
  • text/html
  • text/css
  • application/javascript

5、响应状态码 ==常考==

1xx 信息响应

  • 100 请求继续
  • 101 正在切换协议

2xx 请求成功

  • 200 请求成功
  • 204 请求成功 页面主体不发生修改 put delete 一般返回

3xx 重定向

  • 301 永久重定向
  • 302 临时重定向
  • 304 读取缓存（协商缓存）

4xx 客户端错误

  • 400 响应报文有问题需要修改
  • 403 服务器拒绝访问
  • 404 服务器找不到请求的资源，服务端不想响应

5xx 服务端错误

  • 500 服务器执行请求发生错误
  • 503 服务器超负荷或者停机维护中 无法请求处理

7、从输入URL到网页最终渲染 ==常考==

（1）DNS解析

  • 输入的是域名，解析成IP地址
  • 四个缓存（浏览器DNS缓存，计算机DNS缓存，路由器DNS缓存，网络运营商DNS缓存），一个递归查询

（2）TCP连接（三次握手）

  • 浏览器 我准备好了
  • 服务器 我收到了
  • 浏览器 我收到确认信息了，准备发
  • ==确认客户端、服务端双方都有正常的收发能力==（为什么要三次握手呢？）
  • 保持长连接说的就是这里。

（3）发送请求 客户端发请求给服务器

（4）返回响应 服务器返回响应

（5）渲染页面

  • 构建DOM树（html - DOM树）
  • 样式计算（样式树）
  • 布局
  • 分层
  • 图层绘制
  • 栅格化
  • 合成和显示

（6）断开连接（四次挥手）

  • 客户端 我请求完了
  • 服务器 我收到了
  • 服务器 我响应完了
  • 客户端 我收到了
