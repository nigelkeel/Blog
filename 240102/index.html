前端代码规范

发表于 2020-12-24 更新于 2021-02-01 分类于 2401：前端工程化

前端代码规范#

JS

  1. 使用单引号
  2. 代码换号
  3. : 后加空格，函数名和 () 前后加空格
  4. 废弃 var 用 let 和 const
  5. 去除未使用的变量和函数
  6. 如有多个属性引用至同一个对象：
    // 错误写法
this.$route.query.a
this.$route.query.b
// 正确写法
let query = this.$route 
query.a
query.b
  7. 不用分号
  8. 四个空格缩进
  9. 使用小驼峰命名法，并且变量都有明确的用途 
    let getComputedStyle
let setProjectName
let deleteProject
  10. 代码注释

CSS：

  1. 尽量在三层嵌套以内
  2. 不使用 *
  3. 注重公共样式提取
  4. 命名用-拼接，应明确功能用途

HTML：

  1. 组件尽量遵守柯里化，例如一个确认框组件，尽可能拆散：
    （1. 头部组件
    - 标题
    - close

    （2. 中间内容

    （3. 底部组件
    - 按钮组件

    （4. 遮罩组件
    …

  2. class 名使用 - 拼接

  3. Tab / 4格缩进

  4. 尽量各个组件维护各个组件的 JS/CSS/HTML
